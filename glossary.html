単語集

Java Silverで出る用語を、意味・コード上の見え方・試験での注意点まで整理します。

この単語集の読み方

Java Silverでは、用語の和訳よりも「その用語が選択肢でどう使われるか」が重要です。たとえば「ポリモーフィズム」を説明できても、A a = new B(); で呼べるメソッドと実行されるメソッドを分けられなければ点にはなりません。

このページでは、用語を 意味、コード上の見え方、試験での注意 の3点で整理します。

型・変数・リテラル

用語意味コード上の見え方試験での注意
プリミティブ型値そのものを扱う型int, double, boolean, charnull 不可。boolean は数値と非互換。
参照型オブジェクトへの参照を扱う型String, 配列, List, 自作クラスnull 可。== は基本的に参照比較。
リテラルソースコードに直接書く値10, 10L, "abc", null整数は基本 int、小数は基本 double。
識別子変数名・クラス名・メソッド名sample, _value, $data数字開始不可。予約語不可。var は予約語ではないが型名としては特殊。
ローカル変数メソッドやブロック内の変数void m(){ int x; }初期値なし。使用前に確定代入が必要。
フィールドクラスに属する変数class A { int x; }型ごとのデフォルト値が入る。
型推論初期化式から型を決める仕組みvar x = 10;var x = null;、フィールドの var は不可。
確定代入使用前に必ず代入済みと判断されることif の全分岐で代入するコンパイラ視点で全経路を満たす必要がある。

文字列・配列・コレクション

用語意味コード上の見え方試験での注意
不変作成後に中身を変えられない性質Stringreplace や substring は新しい文字列を返す。
可変同じオブジェクトの中身を変えられる性質StringBuilder参照コピー後に変更すると、同じ実体を見ている変数にも影響する。
文字列プール文字列リテラルを共有する領域String s = "A";new String("A") は別インスタンス。
配列固定長の連続要素int[] a = new int[3];長さは length。要素には初期値がある。
多次元配列配列の要素が配列である構造int[][] a各行の長さは揃っていなくてもよい。
ArrayListサイズ変更可能なリスト実装new ArrayList<String>()remove(int) と remove(Object) の違いに注意。
Listリストのインタフェース型List<String> list宣言型が List でも実体により変更可否が変わる。
ジェネリクス扱う要素型を指定する仕組みList<Integer>プリミティブ型は直接指定不可。List<int> は不可。

演算子・制御構造

用語意味コード上の見え方試験での注意
単項演算子1つの値に対する演算子++a, a--, !flag前置・後置の戻り値の違いが出る。
二項演算子2つの値に対する演算子a + b, a == b型昇格と評価順を分けて見る。
短絡評価右辺を評価しないことがある論理演算&&, ||右辺の ++ が実行されるかを確認する。
非短絡評価booleanでも右辺まで評価する演算&, |数値ビット演算だけでなくboolean演算にも使える。
三項演算子条件で値を選ぶ演算子cond ? a : b式なので値を持つ。型決定にも注意。
フォールスルーswitch で次のcaseへ流れることbreak なしswitch文で起こる。switch式では基本的に別物。
到達不能文絶対に実行されない文return; の直後の文コンパイルエラーになる場合がある。
ラベルbreak/continueの移動先指定outer:ネストループでどこを抜けるか確認する。

クラス・メソッド・インスタンス

用語意味コード上の見え方試験での注意
クラスオブジェクトの設計図class A {}トップレベルクラスに使える修飾子は限定される。
インスタンスクラスから生成された実体new A()参照を経由して扱う。
コンストラクタインスタンス生成時の初期化処理A(){}戻り値型を書かない。継承されない。
デフォルトコンストラクタ明示コンストラクタがない場合に自動追加される無引数コンストラクタソースには見えない1つでもコンストラクタを書くと自動追加されない。
シグネチャメソッド名と引数リストtest(int)戻り値はシグネチャに含まれない。
オーバーロード同名で引数が異なるメソッド定義test(int), test(String)戻り値だけ変えても不可。
staticインスタンスではなくクラスに属することstatic int countstaticメソッドからインスタンスメンバへ直接アクセス不可。
ガベージコレクション不要になったインスタンスを回収する仕組み参照がなくなったインスタンス対象になるタイミングと実際に回収されるタイミングは別。

継承・interface・ポリモーフィズム

用語意味コード上の見え方試験での注意
継承既存クラスを拡張して新しいクラスを作る仕組みclass B extends Aコンストラクタとprivateメンバは引き継いで使えるわけではない。
スーパークラス継承元のクラスA暗黙の super() が入ることがある。
サブクラス継承先のクラスB extends Aoverride時のアクセス修飾子・戻り値・例外に制約がある。
オーバーライド親クラスのメソッドを子クラスで再定義すること@Overrideアクセスを狭くできない。static/privateは通常のoverride対象ではない。
ポリモーフィズム同じ型の変数で異なる実体を扱う性質A a = new B();呼べるメソッドは宣言型、実行されるoverrideメソッドは実体。
アップキャストサブクラス型をスーパークラス型として扱うことA a = new B();暗黙に可能。
ダウンキャストスーパークラス型の参照をサブクラス型へ変換することB b = (B) a;実体が合わないと ClassCastException。
interface扱い方を定める型interface X抽象メソッドは暗黙に public abstract。
defaultメソッドinterface内に持てる実装付きインスタンスメソッドdefault void m(){}衝突時の解決、Objectメソッドとの関係に注意。

例外・Java 17

用語意味コード上の見え方試験での注意
チェック例外コンパイラが処理を要求する例外IOException, Exceptioncatchまたはthrowsが必要。
非チェック例外処理が強制されない例外RuntimeException 系NPEやClassCastExceptionなど。
throw例外を実際に投げる文throw new Exception();その地点で通常処理は止まる。
throwsメソッドが例外を投げる可能性の宣言void m() throws IOException呼び出し元に処理責任が移る。
try-with-resourcesリソースを自動closeする構文try (var r = ...)closeは宣言と逆順。finallyより先。
record値を保持するための簡潔なクラス構文record Point(int x, int y){}アクセサは x()。フィールドはprivate final。
sealed継承・実装できる型を制限する仕組みsealed class A permits B許可されたサブクラス側にも final / sealed / non-sealed が必要。
switch式値を返せるswitchvar x = switch(n) { ... };yield、網羅性、breakとの違いに注意。
