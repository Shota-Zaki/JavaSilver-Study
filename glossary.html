実行環境・ファイル

6語
JDK

Java開発キット。`javac` など開発用ツールを含む。

注意コンパイルは `javac`、実行は `java`。JREだけでは開発ツールが足りない。
JVM

classファイルを実行する仮想マシン。

注意実行時例外は、コンパイル後にJVM上で動かしたときに発生する。
ソースファイル

Javaコードを書いた `.java` ファイル。

注意publicなトップレベルクラスがある場合、ファイル名は `クラス名.java`。
classファイル

コンパイル後に作られるバイトコード。

注意1つの `.java` から複数の `.class` が作られることがある。
package

クラスを分類し、名前空間を作る宣言。

注意コメントを除いてソースの先頭。`import` より前に書く。
import

完全修飾名を書かずにクラスを使うための宣言。

注意`java.lang` と同一パッケージ内はimport不要。サブパッケージは含まれない。

型・変数

6語
primitive型

値そのものを扱う型。boolean, byte, short, int, long, char, float, double。

注意`null` は入らない。booleanと数値は互換性なし。
参照型

オブジェクトへの参照を持つ型。String、配列、クラス型など。

注意`null` 可。`==` は原則として参照の同一性比較。
リテラル

コードに直接書いた値。例: `10`, `10L`, `"abc"`。

注意整数は基本int、小数は基本double。longはL、floatはF。
var

ローカル変数の型推論。右辺から型を決める。

注意フィールド、引数、戻り値、catch引数、配列要素型には使えない。
スコープ

変数を参照できる範囲。

注意if/for/tryブロック内で宣言したローカル変数は外で使えない。
初期化

変数に最初の値を入れること。

注意ローカル変数は使う前に必ず代入が必要。フィールドは型ごとの初期値を持つ。

クラス・オブジェクト

8語
class

オブジェクトの設計図。

注意トップレベルclassにprivate/protectedは使えない。
instance

`new` で生成された実体。

注意変数にはインスタンスそのものではなく参照が入る。
constructor

インスタンス生成時の初期化処理。

注意戻り値型を書かない。継承されない。先頭で `this()` / `super()`。
this

現在のインスタンスを指す。

注意staticメソッド内では使えない。
super

スーパークラス側を指す。

注意`super()` はコンストラクタ先頭でのみ明示呼び出し可能。
static

インスタンスではなくクラスに属する。

注意staticメソッドから非staticメンバへ直接アクセス不可。
final

再代入・継承・オーバーライドを禁止する。

注意final参照変数でも、参照先オブジェクトの状態変更は別問題。
abstract

未実装メソッドを持てる。インスタンス化不可。

注意abstract methodは本文なし。具象サブクラスで実装が必要。

継承・インタフェース

7語
extends

クラス継承、またはインタフェース継承。

注意クラスの多重継承は不可。インタフェースは複数継承可。
implements

クラスがインタフェースを実装する。

注意抽象メソッド未実装ならクラスをabstractにする。
override

親のインスタンスメソッドを子で再定義。

注意シグネチャ同一。戻り値は同じか共変。アクセスは狭くできない。
overload

同名メソッドを引数違いで複数定義。

注意戻り値だけ違うのは不可。
polymorphism

親型変数で子インスタンスを扱う性質。

注意呼べるメンバは変数の型、実行されるオーバーライドメソッドは実体の型で決まる。
cast

型変換。参照型ではダウンキャストが問われやすい。

注意コンパイル可能でも実行時にClassCastExceptionになる場合がある。
sealed

継承/実装できる型を制限する。

注意許可されたサブクラスはfinal, sealed, non-sealedのいずれかが必要。

例外・判定

6語
コンパイルエラー

実行前にコンパイラが検出する誤り。

注意型不一致、未初期化ローカル変数、到達不能コード、構文ミスなど。
実行時例外

コンパイル後、実行中に発生する例外。

注意NullPointerException、ArrayIndexOutOfBoundsException、ClassCastExceptionなど。
checked例外

コンパイラが処理を要求する例外。

注意catchするかthrowsで宣言する。IOExceptionなど。
unchecked例外

RuntimeException系。コンパイラは処理を強制しない。

注意試験では「コンパイルは通るが実行時に落ちる」判定で出る。
try-catch-finally

例外処理の構文。

注意finallyは例外の有無に関係なく原則実行される。
try-with-resources

AutoCloseableなリソースを自動で閉じる構文。

注意closeは宣言と逆順に呼ばれる。
