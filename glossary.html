Glossary

Java Silver単語集

頻出語を「意味」と「注意」に分けています。暗記より、問題でどう問われるかを優先して読んでください。

カテゴリ

基礎型文字列配列OOP継承例外

JDK

基礎

意味：Javaプログラムの開発に使う一式。javacなどを含む。

注意：JREやJVMと混同しない。開発にはJDKが必要。

JRE

基礎

意味：Javaプログラムの実行環境。

注意：現在の学習ではJDK/JVMの役割差を押さえれば十分。

JVM

基礎

意味：classファイルを実行する仮想マシン。

注意：Javaはソースを直接実行するのではなく、基本的にコンパイル後のclassを実行する。

javac

基礎

意味：Javaソースをclassファイルへコンパイルするコマンド。

注意：コンパイルエラーはここで発生し、実行以前に止まる。

java

基礎

意味：classファイルをJVMで実行するコマンド。

注意：実行時例外は基本的にこちらの段階で発生する。

package

基礎

意味：クラスを分類し、名前空間やアクセス制御に関係する宣言。

注意：ソース先頭、importより前。前に書けるのはコメント程度。

import

基礎

意味：他パッケージのクラス名を短く書くための宣言。

注意：java.langは暗黙import。サブパッケージはワイルドカードで含まれない。

mainメソッド

基礎

意味：プログラムの開始点。public static void main(String[] args)など。

注意：String... argsも可。戻り値、static、引数型が崩れると実行開始点にならない。

プリミティブ型

型

意味：値そのものを保持する基本型。byte/short/int/long/float/double/char/boolean。

注意：nullは入らない。booleanと数値に互換性はない。

参照型

型

意味：オブジェクトへの参照を保持する型。String、配列、ArrayListなど。

注意：==は基本的に参照の同一性比較。中身比較はequals。

リテラル

型

意味：コード中に直接書く値。10, 10L, 3.14, "abc", nullなど。

注意：整数は基本int、浮動小数点は基本double。

数値リテラルの_

型

意味：数値を読みやすくする区切り。1_000など。

注意：先頭末尾、小数点の隣、接尾辞の直前などは不可。

キャスト

型

意味：型を明示的に変換する記法。

注意：コンパイルは通っても実行時にClassCastExceptionになる場合がある。

型昇格

型

意味：演算時に小さい型がint以上へ広がる仕組み。

注意：byte+byteの結果は基本int。代入先がbyteなら注意。

var

型

意味：ローカル変数の型推論。

注意：フィールド、引数、戻り値、catchパラメータなどには使えない。null単独も不可。

String

文字列

意味：不変の文字列クラス。変更に見える操作は新しい文字列を作る。

注意：==とequals、substring、replace、trim、indexOfが頻出。

StringBuilder

文字列

意味：可変の文字列バッファ。append/delete/insert/reverseなど。

注意：equalsをオーバーライドしていないため、中身比較に注意。

intern

文字列

意味：文字列プール上の同値文字列への参照を返す。

注意：new String("x")と"x"は通常別参照。intern後は比較結果が変わる。

配列

配列

意味：固定長の連続要素。lengthフィールドを持つ。

注意：length()メソッドではない。範囲外はArrayIndexOutOfBoundsException。

ArrayList

配列

意味：可変長リスト。add/remove/set/get/sizeなど。

注意：配列とはAPIが違う。intなどプリミティブは直接型引数にできない。

List.of

配列

意味：変更不可リストを作るファクトリメソッド。

注意：add/remove/set不可。null要素不可。

static

OOP

意味：インスタンスではなくクラス側に属するメンバ。

注意：staticメソッドから非staticメンバへ直接アクセス不可。

this

OOP

意味：現在のインスタンスを指す参照。

注意：static文脈では使えない。コンストラクタ呼び出しthis(...)は先頭。

super

OOP

意味：スーパークラス側を参照するキーワード。

注意：super(...)もコンストラクタ先頭。省略時はsuper()が入る。

コンストラクタ

OOP

意味：インスタンス生成時に呼ばれる初期化処理。

注意：戻り値型を書かない。クラス名と同じ。継承されない。

オーバーロード

OOP

意味：同名メソッドを引数違いで複数定義する。

注意：戻り値だけ違っても不可。解決はコンパイル時の引数型で決まる。

オーバーライド

OOP

意味：継承先で同じシグネチャのメソッドを再定義する。

注意：アクセスは狭くできない。戻り値は同じか共変戻り値。

ポリモーフィズム

OOP

意味：親型の変数で子インスタンスを扱う仕組み。

注意：メソッドは実体の型、フィールドは変数の型で決まる点が罠。

interface

継承

意味：実装クラスに扱い方を定める型。

注意：抽象メソッドは暗黙public abstract。フィールドはpublic static final。

defaultメソッド

継承

意味：インタフェースに実装を持たせるメソッド。

注意：Object由来メソッドとの関係、実装クラスでの競合解決に注意。

abstract

継承

意味：未実装を含み得るクラスやメソッド。

注意：abstractクラスはnewできない。抽象メソッドは本体なし。

final

OOP

意味：変数・メソッド・クラスに再代入不可/override不可/継承不可を与える。

注意：参照型finalは参照の再代入不可で、オブジェクト内部変更不可ではない。

sealed

継承

意味：継承・実装できる型をpermitsで制限する。

注意：サブクラスはfinal/sealed/non-sealedのいずれかが必要。

instanceof

OOP

意味：実行時に型互換性を判定する演算子。

注意：パターン変数はtrue側のスコープで使える。nullはfalse。

チェック例外

例外

意味：コンパイラが処理を要求する例外。Exception直下やIOExceptionなど。

注意：catchするかthrows宣言が必要。RuntimeException系は別。

非チェック例外

例外

意味：RuntimeException系。コンパイラは処理を強制しない。

注意：NullPointerExceptionなどは実行時に出る。

throw

例外

意味：例外インスタンスを投げる文。

注意：throw new XxxException(); の形。throwsと混同しない。

throws

例外

意味：メソッドが投げ得る例外を宣言する。

注意：チェック例外では特に重要。override時に広げられない。

finally

例外

意味：例外発生有無に関わらず実行されるブロック。

注意：System.exitなど特殊ケースを除き、returnが絡む問題で頻出。

try-with-resources

例外

意味：AutoCloseableリソースを自動closeするtry。

注意：close順は宣言の逆順。finallyより先にcloseされる。
