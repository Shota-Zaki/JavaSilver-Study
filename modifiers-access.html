修飾子・アクセス制御

public/protected/default/private、static/final/abstract/sealed、トップレベルクラスの制限を整理します。

アクセス修飾子

アクセス修飾子は「どこから見えるか」を決めます。継承しているか、同じパッケージか、同じクラスかを分けて判定します。

public

どこからでもアクセス可能です。

protected

同じパッケージ、または別パッケージのサブクラスからアクセス可能です。

デフォルト

修飾子なし。package-private。同じパッケージ内からだけアクセス可能です。

private

同じトップレベルクラス内からだけアクセス可能です。サブクラスだから見える、ではありません。

protected を「同じパッケージのサブクラスだけ」と覚えるのは誤りです。同じパッケージなら継承関係がなくてもアクセスできます。

トップレベルクラスに使える修飾子

トップレベルクラスに使えるアクセス修飾子は public かデフォルトです。private や protected はトップレベルクラスには使えません。

public class A { }
class B { }

// private class C { }    // NG。トップレベルでは不可
// protected class D { }  // NG。トップレベルでは不可

その他の修飾子

修飾子は対象によって使える・使えないが変わります。選択肢問題では「その修飾子がその場所で使えるか」だけで落とされます。

static

トップレベルクラスには使えません。ネストしたクラスやメンバに使います。

abstract

クラスやメソッドに使用。abstractメソッドは本体を持ちません。

final

クラスなら継承不可、メソッドならオーバーライド不可、変数なら再代入不可。

sealed / non-sealed

sealed階層で使用。non-sealedはsealedな親を持つ直接サブタイプで使います。

override時のアクセス

オーバーライドでは、親メソッドよりアクセス範囲を狭くできません。interfaceの抽象メソッドはpublicなので、実装メソッドもpublicが必要です。

interface A {
    void test(); // public abstract
}

class B implements A {
    public void test() { } // OK
    // void test() { }     // NG。publicより狭い
}
