コンパイルエラーと実行時例外

コンパイル可否、実行時例外、正常出力を分離して判定するためのページです。

失敗段階を分ける

Java Silverの選択肢では、「コンパイルエラー」「実行時例外」「正常出力」が混ざります。ここを段階で分けないと、実行結果を追ってからコンパイルエラーに気づく無駄が発生します。

  1. 文法・宣言・修飾子が正しいか。
  2. 型変換、メソッド呼び出し、アクセス修飾子が正しいか。
  3. チェック例外が処理されているか。
  4. ここまで通って初めて、実行時の値・例外・出力順を追う。
出力候補がそれっぽくても、コンパイルできないなら出力はありません。試験ではこの順番を守るだけで失点が減ります。

コンパイルエラーの典型

型不一致

int に long、float に double、String に char など。

未初期化ローカル変数

ローカル変数はデフォルト値を持ちません。使用前の初期化が必須です。

static文脈

staticメソッドからインスタンスメンバへ直接アクセスできません。

アクセス不可

private、package-private、protectedの条件を満たさないアクセス。

オーバーライド違反

アクセスを狭める、戻り値が不適切、throwsが広すぎるなど。

到達不能catch

親例外catchを先に書いて、子例外catchが到達不能になる。

実行時例外の典型

コンパイルが通っても、実行時の値によって例外が発生します。例外が発生したら、その行以降のtry内処理は実行されず、対応するcatchへ移ります。

String s = null;
// s.length();               // NullPointerException

int[] a = new int[0];
// a[0] = 10;                // ArrayIndexOutOfBoundsException

// Integer.parseInt("A");    // NumberFormatException

Object obj = "abc";
// Integer n = (Integer) obj; // ClassCastException

正常実行の追跡

コンパイル可否と例外可能性を潰したら、最後に出力順を追います。追跡対象は、代入、インクリメント、短絡評価、メソッドの戻り値、副作用、参照の付け替えです。

代入

右辺評価後に左辺へ代入。複合代入は型変換の扱いに注意。

短絡評価

&& と || は右辺が実行されないことがあります。

参照代入

オブジェクトのコピーではなく、参照のコピーです。

戻り値

String系メソッドなど、戻り値を受け取らないと変化が残らないものがあります。
