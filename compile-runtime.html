コンパイルエラーと実行時例外

「いつ失敗するのか」を分けると、出力問題の誤答が減ります。

最初に覚える結論

Java Silverの出力問題は、必ず コンパイルエラー → 実行時例外 → 正常出力 の順で判定します。

1コンパイルエラー

実行前に止まる。出力も例外発生行も存在しない。

2実行時例外

コンパイル後、その行に到達して初めて発生する。

3正常出力

コンパイルも実行時例外も通過した場合だけ追う。

コンパイルエラーになる代表例

分類例理由

型不一致int a = 10L;longをintへ暗黙代入できない。
未初期化ローカル変数int a; System.out.println(a);ローカル変数は自動初期化されない。
アクセス不可privateメンバを別クラスから直接参照アクセス修飾子違反。
チェック例外未処理throw new IOException();catchまたはthrowsが必要。
catch順序catch(Exception) の後に catch(IOException)後続catchが到達不能。
static文脈staticメソッドから非staticフィールドを直接参照thisが存在しない。

例題：未初期化はコンパイルエラー

public class Main {
  public static void main(String[] args) {
    int x;
    System.out.println(x);
  }
}

ローカル変数 x は初期化されていないため、実行前にコンパイルエラーです。

例題：nullは実行時例外

public class Main {
  public static void main(String[] args) {
    String s = null;
    System.out.println(s.length());
  }
}

型としてはStringなのでコンパイルできます。ただし実行時に s.length() へ到達した時点でNullPointerExceptionです。

試験での実用手順

  1. 全コードを見て、文法・型・アクセス・チェック例外を確認する。
  2. コンパイルできる場合だけ、実行経路を上から追う。
  3. 例外が発生したら、対応するcatchがあるかを見る。
  4. catchされたらその後のfinallyや後続処理へ進む。catchされなければ実行時例外として終了。
