Compile or Runtime

コンパイルエラーと実行時例外

「コンパイルできるか」を最初に判定します。実行時例外や出力は、その後でしか発生しません。

判定の優先順位

1構文

Javaとして書けているか。

2型

代入・引数・戻り値が合うか。

3実行時

null、範囲外、キャスト失敗など。

4出力

最後に表示内容を追う。

コンパイルエラーになりやすいもの

カテゴリ例
構文;不足、括弧不一致、try/finally/catch順序不正。
型intへlong代入、floatへdouble代入、booleanと数値の混在。
変数ローカル変数未初期化、スコープ外参照、存在しない変数。
メソッド戻り値不足、引数不一致、staticから非static直接参照。
継承override条件違反、抽象メソッド未実装、sealed違反。
例外チェック例外未処理、catch順序不正。

実行時例外になりやすいもの

例外典型例
NullPointerExceptionnull参照でメソッド/フィールドへアクセス。
ArrayIndexOutOfBoundsException配列の範囲外アクセス。
StringIndexOutOfBoundsExceptioncharAt/substringの範囲外。
IndexOutOfBoundsExceptionArrayListの範囲外。
ClassCastException実体型と合わないキャスト。
NumberFormatExceptionInteger.parseInt("abc")。
ArithmeticException整数の0除算。

チェック例外はコンパイル段階で見る

void test() {
  throw new java.io.IOException(); // 未処理ならコンパイルエラー
}

void ok() throws java.io.IOException {
  throw new java.io.IOException();
}
