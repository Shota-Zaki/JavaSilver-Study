コンパイルエラーと実行時例外

問題を解くときの最重要フローである「コンパイル可否 → 実行時例外 → 出力」を固定します。

絶対にこの順で判定する

Java Silverの出力問題は、最初に出力を計算し始めるとミスが増えます。必ず次の順にしてください。

  1. コンパイルエラーがないか: 型、アクセス、宣言位置、override、例外処理。
  2. 起動できるか: main形式、ファイル名、javaコマンド。
  3. 実行時例外が起きるか: null、配列範囲外、キャスト失敗、数値変換。
  4. 出力を追う: 評価順、参照、メソッド動的束縛。
出力が選択肢にあっても、コンパイルエラーがあれば出力は存在しません。

出力計算に入る条件

出力計算は、コンパイル・起動・実行時例外の可能性を潰してからです。出力を追う場合も、式の評価順・参照・動的束縛のどれを見ているか分けます。

式の評価順

++、--、短絡評価、代入式。

参照

a = b 後にどのインスタンスを見ているか。

動的束縛

オーバーライドメソッドは実体で決まる。

例外処理

try/catch/finally、close順。

判定サンプル

class A { void test() { System.out.println("A"); } }
        class B extends A { void test() { System.out.println("B"); } }

        public class Main {
            public static void main(String[] args) {
                A a = new B();
                a.test();
            }
        }
  1. コンパイル可。A に test がある。
  2. 実行時例外なし。
  3. オーバーライドメソッドなので実体 B の test が実行。
  4. 出力は B。

ミニ判定

Q1. Object o = "abc"; System.out.println(o.length()); は？

コンパイルエラーです。実体はStringでも、宣言型 Object に length() はありません。

Q2. String s = null; System.out.println(s); は？

null と表示されます。println に参照を渡しているだけで、s のメソッドを呼んでいません。

Q3. String s = null; s.equals("x") は？

実行時に NullPointerException です。null参照に対してメソッドを呼んでいます。
