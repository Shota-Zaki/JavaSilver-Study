頻出メソッド一覧

String、StringBuilder、List、ラッパークラスなど、出題されやすいメソッドを用途別に整理します。

String

メソッド戻り値注意
length()int配列は length フィールド。Stringは length() メソッド。
charAt(index)char添字は0始まり。範囲外は実行時例外。
substring(begin, end)Stringbeginは含む、endは含まない。元のStringは変わらない。
indexOf(str)int見つからない場合は -1。
replace(old, new)String戻り値を受けないと変更結果は失われる。
equals(obj)boolean内容比較。nullに対して呼ぶと例外。
equalsIgnoreCase(str)boolean大文字小文字を無視。
trim() / strip()String空白除去。試験では戻り値を受けているかを見る。

StringBuilder

メソッド戻り値注意
append(x)StringBuilder自分自身を変更して返す。
insert(index, x)StringBuilder指定位置に挿入。範囲外は実行時例外。
delete(start, end)StringBuilderendは含まない。
reverse()StringBuilder自分自身を反転。
toString()StringStringへ変換。
equals()boolean内容比較ではない。Object由来の参照比較。

ArrayList / List

メソッド意味注意
add(e)末尾追加戻り値はboolean。
add(index, e)指定位置に挿入その位置以降が後ろへずれる。index == size はOK。
set(index, e)置換既存要素が必要。index == size はNG。
remove(index)位置で削除戻り値は削除した要素。
remove(obj)値で削除戻り値はboolean。Integerリストでは overload に注意。
size()要素数配列は length。
contains(obj)含まれるか内部でequalsを使う。
List.of(...)固定的なList作成要素変更不可。null不可。

ラッパークラス・Object・System

対象メソッド/機能注意
IntegerparseInt(String)戻り値は int。不正文字列はNumberFormatException。
IntegervalueOf(...)戻り値は Integer。
ObjecttoString()出力時に呼ばれやすい。null参照では呼べない。
Objectequals(Object)未オーバーライドなら参照比較。
SystemSystem.out.printlnSystem と out の大文字小文字に注意。

戻り値を捨てると結果が変わるメソッド

String s = " Java ";
s.trim();
System.out.println(s); // " Java " のまま

StringBuilder sb = new StringBuilder("A");
sb.append("B");
System.out.println(sb); // AB

String は不変なので戻り値を受ける。StringBuilder は自分自身を変更する。この違いは頻出です。
