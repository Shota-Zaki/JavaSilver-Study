継承で引き継がれるもの

対象扱い注意
フィールド/メソッドアクセス可能なものは利用できるprivateはサブクラスから直接使えない。
コンストラクタ引き継がれないサブクラスのコンストラクタからsuper(...)で呼ぶ。
staticメンバ継承したように見える実体はクラス所属。overrideではなく隠蔽。
finalメソッド継承はされるがoverride不可finalクラスは継承不可。

extends / implements / abstract

形意味必要なこと
class B extends Aクラス継承Aのコンストラクタ呼び出しが必要。
class C implements Iinterface実装抽象メソッドをすべて実装する。
interface B extends Ainterface継承抽象メソッドの契約を引き継ぐ。
abstract class C implements I抽象クラスによる実装未実装メソッドを残せる。具象サブクラスが実装。

interfaceの暗黙修飾

メンバ暗黙の扱い注意
フィールドpublic static final初期化必須。インスタンスフィールドではない。
抽象メソッドpublic abstractprivate/protected不可。実装時はpublic必須。
defaultメソッドpublic本体あり。実装クラスが継承できる。
staticメソッドpublicインタフェース名で呼ぶ。実装クラスへ継承されるわけではない。
privateメソッドJava 9以降可default/static内の共通処理用。

オーバーロードとオーバーライド

項目オーバーロードオーバーライド
関係同じクラス内でも継承関係でも可継承関係が必要
引数変える同じ
戻り値判定材料ではない同じか共変戻り値
アクセス判定材料ではない親より狭くできない
例外関係薄いチェック例外は親より広げられない
決定時期コンパイル時実行時に実体型で決まる

ポリモーフィズム

class A { void test() { System.out.println("A"); } }
class B extends A { @Override void test() { System.out.println("B"); } void onlyB() {} }

A a = new B();
a.test();  // B。overrideメソッドは実体型で決まる
a.onlyB(); // コンパイルエラー。宣言型AにonlyBがない

キャストとinstanceof

場面コンパイル時実行時
A a = new B();OK。上位型へ代入実体はB。
B b = (B) a;AとBに継承関係があればコンパイル可実体がBでなければClassCastException。
if (a instanceof B b)型確認と変数宣言trueブロック内でbをBとして使える。
