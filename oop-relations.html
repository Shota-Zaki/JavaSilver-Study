継承・interface・override

extends、implements、abstract、defaultメソッド、override、interfaceの基本を整理します。

継承の基本

サブクラスのインスタンスは、スーパークラス部分とサブクラス部分を持ちます。サブクラスに書かれていないメンバでも、継承されたものは使えます。ただしコンストラクタとprivateメンバは引き継ぎません。

class Parent {
    String name;
}
class Child extends Parent {
    void hello() {
        System.out.println(name); // Parentのフィールドを利用
    }
}

コンストラクタは継承されない

スーパークラスに引数ありコンストラクタしかない場合、サブクラスのコンストラクタは明示的に super(...) を呼ぶ必要があります。

class A {
    A(String name) { }
}
class B extends A {
    B() {
        super("B");
    }
}

overrideの条件

条件説明
メソッド名・引数同じシグネチャ
戻り値型同じか、より具体的な参照型
アクセス修飾子同じか、より広い
例外チェック例外は同じか、より狭い
staticインスタンスメソッドとstaticメソッドを相互にoverrideしない
class A {
    protected Number test() { return 1; }
}
class B extends A {
    @Override
    public Integer test() { return 1; } // OK: アクセス広い、戻り値はサブクラス型
}

interfaceの基本

interfaceの抽象メソッドは暗黙的に public abstract です。フィールドは暗黙的に public static final です。

interface Worker {
    void work(); // public abstract扱い
    int MAX = 10; // public static final扱い
}

実装クラスで work を実装する場合、public を付ける必要があります。アクセスを狭くできないからです。

default/static/privateメソッド

interfaceにはdefaultメソッド、staticメソッド、privateメソッドを定義できます。defaultは実装クラスに継承される実装付きメソッドです。privateはinterface内の補助用です。

interface A {
    default void test() {
        log();
    }
    private void log() {
        System.out.println("log");
    }
}

abstractクラスとinterface実装

抽象クラスは、interfaceの抽象メソッドをすべて実装しなくても構いません。具象クラスになるまでに実装すればよいからです。

interface A { void test(); }
abstract class B implements A { } // OK
class C extends B {
    public void test() { }
}

ミニ演習

Q. interfaceのメソッドを実装するとき、publicを省略できる？

できません。interfaceの抽象メソッドはpublicなので、実装側でアクセスを狭くするとコンパイルエラーです。

Q. abstract classはinterfaceメソッド未実装でもよい？

よいです。ただし、そのクラス自体はabstractである必要があります。

overrideとoverloadの見分け

同名でも引数が違えばoverloadです。overrideしたつもりで引数型が違うと、親メソッドは残ったままになります。@Override があるとコンパイラが検出してくれます。

class A { void test(Object o) {} }
class B extends A {
    // @Override を付けるとエラー。これはoverload。
    void test(String s) {}
}

interface変数で呼べるもの

interface型の変数から呼べるのは、そのinterfaceに宣言されたメソッドです。実体クラスに独自メソッドがあっても、宣言型に存在しなければ呼べません。

interface A { void test(); }
class B implements A {
    public void test() {}
    void onlyB() {}
}
A a = new B();
a.test();
// a.onlyB(); // NG
