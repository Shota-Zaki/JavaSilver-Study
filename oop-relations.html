継承・interface・override

extends、implements、abstract、interface、default/privateメソッド、override、sealedをまとめて整理します。

extendsとimplements

extends はクラス継承またはインタフェース継承、implements はクラスがインタフェースを実装するときに使います。

関係書き方複数可否
クラスがクラスを継承class B extends A1つだけ
クラスがインタフェースを実装class C implements I, J複数可
インタフェースがインタフェースを継承interface J extends I複数可

クラスの多重継承はできませんが、インタフェースは複数実装できます。

継承で引き継がないもの

サブクラスはスーパークラスのメンバを利用できますが、何でも引き継ぐわけではありません。

  • コンストラクタ: 継承されません。ただしサブクラスのコンストラクタから super(...) で呼びます。
  • privateメンバ: サブクラスから直接アクセスできません。
  • 初期化の責任: スーパークラス部分はスーパークラスのコンストラクタで初期化されます。
class A {
            private int x;
            A(int x) { this.x = x; }
        }
        class B extends A {
            B() { super(10); }
        }

abstractクラスと抽象メソッド

abstract メソッドは本体を持たないメソッドです。抽象メソッドを持つクラスは abstract クラスでなければなりません。

abstract class Employee {
            abstract void work();
        }

        class Engineer extends Employee {
            @Override
            void work() { System.out.println("code"); }
        }

具象クラスは、継承した抽象メソッドをすべて実装する必要があります。実装しないなら、そのクラスも abstract にします。

interfaceのメンバ

インタフェースの抽象メソッドは暗黙に public abstract です。フィールドは暗黙に public static final です。

interface Worker {
            int MAX = 10;        // public static final
            void work();         // public abstract
            default void rest() {}
            static void info() {}
            private void helper() {}
        }

default メソッドと private メソッドには本体が必要です。抽象メソッドに本体は書けません。

インタフェースの抽象メソッドはpublicです。実装クラスでアクセス修飾子を省略するとデフォルトアクセスになり、公開範囲を狭めるためコンパイルエラーです。

オーバーライドの条件

オーバーライドは、スーパークラスまたはインタフェースのメソッドをサブクラス側で再定義することです。

条件内容
シグネチャ同じメソッド名・同じ引数リスト
戻り値同じ型、または共変戻り値
アクセス狭くできない
例外チェック例外は広くできない
staticインスタンスメソッドとは相互にoverride不可
class A {
            protected Number test() throws java.io.IOException { return 1; }
        }
        class B extends A {
            @Override
            public Integer test() { return 1; } // OK。アクセスは広い、戻り値はサブクラス、例外は減る
        }

defaultメソッドの衝突

複数のインタフェースから同じシグネチャのdefaultメソッドを継承すると、クラス側で明示的に解決する必要があります。

interface A { default void test() { System.out.println("A"); } }
        interface B { default void test() { System.out.println("B"); } }

        class C implements A, B {
            @Override
            public void test() {
                A.super.test();
            }
        }

クラスのメソッドとインタフェースのdefaultメソッドが競合する場合は、クラス側が優先されます。

sealed / non-sealed / permits

sealed は、継承・実装できる相手を制限する仕組みです。許可された直接サブクラスは、final、sealed、non-sealed のいずれかで方針を明示します。

sealed class Animal permits Dog, Cat {}
        final class Dog extends Animal {}
        non-sealed class Cat extends Animal {}

permits にないクラスは直接継承できません。また、許可されたサブクラス側も継承方針を明示しないとコンパイルエラーです。

ミニ判定

Q1. インタフェースのメソッドを実装するとき、void work() でよい？

通常は不可です。インタフェースの抽象メソッドはpublicなので、実装側も public void work() が必要です。

Q2. コンストラクタは継承される？

されません。サブクラスのコンストラクタから super(...) で呼び出すだけです。
