OOP関係整理

継承、interface、オーバーロード、オーバーライドを関係として理解します。

関係の全体像

概念何を決めるかいつ決まるか典型ミス
継承クラス間の親子関係。差分を追加する。コンパイル時の型階層コンストラクタも引き継がれると誤解する。
interface実装扱い方の契約を満たす。コンパイル時の型チェック実装メソッドをpublicにし忘れる。
オーバーロード同名メソッドの引数違い。コンパイル時に宣言型と引数型で決定戻り値だけ違うものを過大評価する。
オーバーライド親のインスタンスメソッドを再定義。実行時に実体型で決定フィールドも同じように実行時解決されると誤解する。
ポリモーフィズム親型・interface型で子の実体を扱う。宣言型で見える範囲、実体型で実行宣言型にないメソッドを呼ぼうとする。

overload と override の比較

項目オーバーロードオーバーライド
名前同じ同じ

戻り値判定材料にならない同じか共変戻り値
アクセス自由だが同一シグネチャ衝突に注意親より狭くできない

宣言型と実体型

class A { void hello(){ System.out.println("A"); } }
class B extends A { void hello(){ System.out.println("B"); } void bye(){} }
A a = new B();
a.hello(); // B: overrideは実体型
// a.bye(); // NG: 宣言型Aにbyeがない

interfaceとoverload/overrideの接点

interface Worker { void work(); }
class Engineer implements Worker {
    public void work() { }       // override的に契約を実装
    public void work(int hour) { } // overload。Workerの契約を満たすのはwork()だけ
}

interfaceを実装するには、指定されたシグネチャを実装する必要があります。似た名前のオーバーロードを追加しても、契約を満たしたことにはなりません。
