細かい挙動・境界ルール

小さい差で正解が変わる境界ルールを集約します。

戻り値を使う/使わない

呼び出し戻り値注意
String.substring()新しいString元のStringは変わらない。代入しないと結果は捨てられる。
StringBuilder.append()同じStringBuilder参照自身を変更する。戻り値も自分。
ArrayList.add(E)boolean追加成功でtrue。
ArrayList.set(i,e)置き換え前の要素サイズは増えない。
System.out.println()void式として値を返さない。

境界値

"abc".charAt(2);      // 'c'
// "abc".charAt(3);   // StringIndexOutOfBoundsException
"abc".substring(1, 3); // "bc"。endは含まない
// "abc".substring(3, 1); // 例外

length / length() / size()

対象書き方理由
配列array.lengthフィールド。括弧なし。
Stringstr.length()メソッド。括弧あり。
Listlist.size()メソッド。括弧あり。

finallyとreturn

static int test() {
    try {
        return 1;
    } finally {
        System.out.println("finally");
    }
}
// return前にfinallyが実行される。

finally内でさらにreturnすると戻り値を上書きするため危険です。試験では実行順の確認に使われます。
