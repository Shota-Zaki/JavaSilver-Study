細かい仕様メモ（教材版）

1行単位で落とされやすい仕様を、コード例と判定順で整理します。

このページの使い方

このページは、単元別の記事では拾いにくい「1行だけで落とされる仕様」を集めた教材版です。Java Silverでは、大きな概念を理解していても、length と length()、println(null) と null.toString() のような小さい差で失点します。

読み方は単純です。各項目について、コンパイルできるか、実行時例外になるか、正常なら何が出力されるかの順に確認してください。説明を読んで「知っている」と思った項目でも、コード例を見て即答できなければ試験では弱点です。

実行引数・mainメソッドまわり

String[] args は引数なしでも null ではない

コマンドライン引数を渡さずに実行しても、args には長さ0の配列が渡されます。したがって args.length は 0 です。args == null ではありません。

public class Main {
    public static void main(String[] args) {
        System.out.println(args.length);
    }
}

引数なしで実行すると 0 が表示されます。ただし args[0] にアクセスすると、長さ0の配列に対する要素外アクセスなので ArrayIndexOutOfBoundsException です。

mainメソッドとして認められる形

基本形は public static void main(String[] args) です。String... args も可変長引数なので認められます。引数名は args でなくても構いません。

宣言判定理由
public static void main(String[] args)OK標準形
public static void main(String... data)OKString[] と同等に扱える
static public void main(String[] args)OK修飾子の順序は本質ではない
public void main(String[] args)実行不可static ではない
public static int main(String[] args)実行不可戻り値が void ではない

長さ・サイズ・文字数のAPI差

Java Silverでは、似た名前のAPIを混ぜた選択肢が頻出です。これは暗記ではなく、「対象が配列か、Stringか、Listか」を先に決めれば切れます。

対象使うものメソッドか例
配列lengthフィールドarr.length
Stringlength()メソッドstr.length()
StringBuilderlength()メソッドsb.length()
List / ArrayListsize()メソッドlist.size()

arr.length()、str.length、list.length は全部コンパイルエラーです。見た目で流すと落ちます。

null の扱い

println(null)

System.out.println(s) の s が null なら、文字列 null として表示されます。メソッド呼び出しではありません。

null.toString()

null 参照に対してインスタンスメソッドを呼ぶので NullPointerException です。

"x" + null

文字列連結では null は "null" として連結されます。"xnull" になります。

null instanceof X

false です。例外にはなりません。パターン変数も有効になりません。

String s = null;
System.out.println(s);        // null
System.out.println("A" + s);  // Anull
System.out.println(s.length()); // NullPointerException

初期値・確定代入・スコープ

フィールドには型ごとのデフォルト値があります。一方、ローカル変数にはデフォルト値がありません。宣言しただけで使うとコンパイルエラーです。

場所初期値試験での見方
インスタンスフィールドあるint は0、参照型は null、boolean は false
staticフィールドあるクラスに属するだけで、初期値ルールはフィールドと同じ
ローカル変数ない使用前に確実に代入されていないとコンパイルエラー
配列要素あるnew int[3] の各要素は0

条件分岐があるローカル変数

int x;
if (flag) {
    x = 10;
}
System.out.println(x); // flagがfalseなら未代入なのでコンパイルエラー

「実行時にflagがtrueかもしれない」では足りません。コンパイラが全経路で代入済みと判断できる必要があります。

数値・文字・booleanの小さい罠

char は整数系

char c = 'A'; は文字ですが、演算すると数値として扱われます。'A' + 1 は int 型の 66 です。

boolean は数値非互換

true == 1、boolean b = 0; は不可です。他言語経験者ほど引っかかります。

整数は基本 int

10 は int、10L は long です。小数 10.0 は基本 double です。

byte / short は演算で int

byte b = 1; b = b + 1; はコンパイルエラー。b += 1; は複合代入なので可。

コード/状況結果理由

継承・interface・修飾子の細かい仕様

コンストラクタは継承されない

サブクラスのインスタンス生成時にスーパークラスのコンストラクタは呼ばれますが、コンストラクタ自体をメンバとして引き継ぐわけではありません。

private メンバはサブクラスから使えない

サブクラスのインスタンス内にスーパークラス部分はありますが、private には直接アクセスできません。

interfaceの抽象メソッドは public

実装クラスでアクセス修飾子を省略するとdefaultアクセスになり、publicより狭いためコンパイルエラーです。

defaultメソッドとObjectメソッド

interfaceのdefaultメソッドで toString など Object 由来のpublicメソッドを上書きする形は不可です。

staticメソッドはoverrideではない

同名staticメソッドは隠蔽です。インスタンスの実体ではなく、参照型・クラス名に基づきます。

フィールドはoverrideされない

同名フィールドがあっても、多態的に解決されません。参照変数の型でアクセス先が決まります。

例外処理の細かい仕様

仕様要点ひっかけ

最終チェックリスト

  • args は引数なしでも長さ0配列。
  • 配列は length、Stringは length()、Listは size()。
  • ローカル変数に初期値はない。
  • null に対するメソッド呼び出しは NullPointerException。
  • String は不変、StringBuilder は可変。
  • StringBuilder.equals() は内容比較ではない。
  • byte / short / char の演算結果は基本 int。
  • boolean と数値は非互換。
  • コンストラクタは継承されない。
  • interface実装メソッドは public が必要。
  • フィールドはoverrideされない。
  • staticメソッドはoverrideされない。
  • catchは狭い例外から広い例外へ。
  • try-with-resourcesのcloseは逆順。
