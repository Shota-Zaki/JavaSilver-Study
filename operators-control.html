演算子・分岐・ループ

評価順、型、短絡評価、switch、break/continueを、実行結果問題で使える形に整理します。

演算子は優先順位より先に型を見る

演算子問題は、計算順序だけでなく型も見ます。boolean に <= は使えませんし、String が混ざれば + は文字列結合になります。

int a = 3;
int b = a += 5;
System.out.println(a + b); // 16

// boolean x = true, y = false;
// System.out.println(x <= y); // NG

算術演算子

+、-、*、/、%。整数除算は小数部を切り捨てます。

関係演算子

<、<=、>、>= は数値比較。booleanには使えません。

等価演算子

==、!=。参照型なら参照比較です。

論理演算子

&&、|| は短絡評価。&、| は両辺を評価します。

++ / -- の読み方

前置は変更後の値を使い、後置は現在の値を使ってから変更します。複雑な式では、左から「その場所で使われる値」と「式の後に残る値」を分けて追います。

int a = 10;
int b = a++ + a + a-- - a-- + ++a;
System.out.println(b); // 32

この種の問題は暗算で処理しようとしない方が安全です。各項を表にして、使う値と更新後の値を分けます。

a++

式では元の値を使い、その後aを1増やします。

++a

先にaを1増やし、式では増えた後の値を使います。

a--

式では元の値を使い、その後aを1減らします。

--a

先にaを1減らし、式では減った後の値を使います。

短絡評価と副作用

&& と || は、左辺だけで結果が決まる場合に右辺を評価しません。右辺に ++ やメソッド呼び出しがあると、値が変わるかどうかに影響します。

int a = 10;
int b = 10;

if (10 < a && 10 < ++b) {
    a++;
}
System.out.println(a + b); // 20。右辺は評価されない

& や | をbooleanに使うと、短絡せず両辺を評価します。見た目が似ているので要注意です。

if / switch

if の条件式はbooleanでなければなりません。switch は使える型とcaseラベルの条件が問われます。

int n = 2;
String result = switch (n) {
    case 1 -> "one";
    case 2 -> "two";
    default -> "other";
};
System.out.println(result); // two

switch文

処理文として使います。従来形式では break 忘れによるフォールスルーに注意。

switch式

値を返します。-> や yield が絡みます。

caseラベル

定数式が基本です。変数や実行時に決まる値は使えません。

default

必須ではありませんが、switch式では網羅性に関わります。

ループ・break・continue・ラベル

break はループやswitchを抜け、continue は次の反復へ進みます。ラベル付きなら外側のループを対象にできますが、ラベルを付けられる位置と使える文脈は限定されます。

outer:
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (j == 1) continue outer;
        System.out.println(i + ":" + j);
    }
}
ラベルは「好きな場所へジャンプする機能」と雑に覚えると危険です。Javaのラベル付きbreak/continueは、対象にできる文が制限されます。
