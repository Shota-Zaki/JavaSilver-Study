演算子・制御構造

演算子の評価順、短絡評価、switch文/式を整理します。

インクリメント・デクリメント

int a = 5;
int x = a++; // x=5, a=6
int y = ++a; // a=7, y=7

後置は「式に使う値を先に返し、その後で変数を更新」。前置は「更新してから式に使う」。同じ式に何度も同じ変数が出る問題は、左から値と副作用を表にして追うのが安全です。

&& と &

演算子短絡評価右辺評価
&&あり左辺がfalseなら右辺は評価しない。
&なしboolean同士なら必ず右辺も評価する。
||あり左辺がtrueなら右辺は評価しない。
|なしboolean同士なら必ず右辺も評価する。

switch文とswitch式

項目switch文switch式
値を返すか返さない返す。代入やreturnに使える。
末尾セミコロン不要式なので代入文全体の末尾に必要。
yield使わないブロック形式で値を返す。
網羅性必須ではない値を返すなら原則必要。defaultで補うことが多い。

yieldの返り値

int result = switch (n) {
    case 1 -> 10;
    case 2 -> {
        int tmp = n * 10;
        yield tmp; // このcaseブロックの値になる
    }
    default -> 0;
};
System.out.println(result);

yield はswitch式のcaseブロックから値を返します。メソッドから抜ける return とは別物です。

switch(null)

String s = null;
switch (s) {          // Java Silver範囲ではNullPointerExceptionに注意
    case "A" -> System.out.println("A");
    default -> System.out.println("default");
}
