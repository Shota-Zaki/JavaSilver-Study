演算子・分岐・ループ

評価順、インクリメント、短絡評価、条件演算子、switch、ループ制御を、出力問題で使える粒度で整理します。

演算子問題の基本姿勢

演算子問題では、式を見た瞬間に左から雑に計算すると負けます。先に確認するのは次の4つです。

  1. コンパイルできる型か: booleanに <= を使っていないか、数値型へ代入できるか。
  2. 短絡評価か: && と || で右辺が実行されるか。
  3. 前置/後置: ++a と a++ の値の使われ方。
  4. 代入を含む式か: a += 5 の結果がそのまま式の値になる。

代入演算子と複合代入

代入式自体にも値があります。int b = a += 5; は、まず a に加算代入し、その結果を b に代入します。

int a = 3;
        int b = a += 5;
        System.out.println(a + b); // 16

a += 5 後の a は8です。b も8になります。

前置・後置インクリメント

前置は「変えてから値として使う」、後置は「今の値を使ってから変える」です。

int a = 10;
        int x = ++a; // a=11, x=11
        int y = a++; // y=11, a=12

複雑な式では、各項が式に渡す値を順にメモすると崩れません。

int a = 10;
        int b = a++ + a + a-- - a-- + ++a;
        // 渡される値: 10 + 11 + 11 - 10 + 10
        System.out.println(b); // 32
副作用のある式は実務では避けるべきですが、試験では出ます。出力問題として割り切って、値の渡し方だけ追ってください。

短絡評価と非短絡評価

&& と || は短絡評価します。左辺だけで結果が決まる場合、右辺は評価されません。& と | はboolean同士にも使えますが、短絡しません。

int a = 10;
        int b = 10;
        if (10 < a && 10 < ++b) {
            a++;
        }
        System.out.println(a + b); // 20。右辺は評価されない
int a = 10;
        int b = 10;
        if (10 < a & 10 < ++b) {
            a++;
        }
        System.out.println(a + b); // 21。右辺は評価される

関係演算子とboolean

<、<=、>、>= は数値比較に使います。boolean 同士には使えません。

boolean a = true;
        boolean b = false;
        System.out.println(a == b); // OK
        // System.out.println(a <= b); // NG

== と != はbooleanにも使えます。

switch文とswitch式

Java 17では、従来のswitch文とswitch式の両方が出題対象になります。switch式は値を返すため、すべての経路で値が必要です。

int n = 2;
        String s = switch (n) {
            case 1 -> "one";
            case 2 -> "two";
            default -> "other";
        };

従来の case: 形式では、break がないと次のcaseへ流れます。-> 形式は基本的にフォールスルーしません。

int n = 1;
        switch (n) {
            case 1:
                System.out.print("A");
            case 2:
                System.out.print("B");
                break;
            default:
                System.out.print("C");
        }
        // AB

ラベル付きbreak/continue

ラベルは文につけられます。試験では二重ループから抜ける問題で使われます。break label; はラベルの付いた文そのものを抜けます。continue label; はラベル付きループの次の反復へ進みます。

outer:
        for (int i = 0; i < 3; i++) {
            for (int j = 0; j < 3; j++) {
                if (j == 1) break outer;
                System.out.print(i + ":" + j + " ");
            }
        }
        // 0:0

continue の対象はループでなければなりません。ラベルなら何でも continue できるわけではありません。

ミニ判定

Q1. false && ++x > 0 で x は増える？

増えません。&& は左辺がfalseなら右辺を評価しません。

Q2. false & ++x > 0 で x は増える？

増えます。& はboolean演算でも短絡しません。

Q3. boolean に <= は使える？

使えません。== または != だけです。
