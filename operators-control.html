演算子問題の基本方針

演算子問題では、暗算で一気に答えようとすると崩れます。副作用を持つ演算子を見つけたら、変数表を書いて左から評価します。ただし、優先順位と短絡評価は先に確認します。

確認見るもの
優先順位括弧、単項演算子、算術、比較、論理、代入
副作用++, --, += など
短絡評価&&, || で右辺が評価されるか
型比較できる型か、booleanが必要な場所か

前置・後置インクリメント

前置は増やしてから値として使います。後置は現在値を使ってから増やします。

int a = 10;
int b = ++a; // a=11, b=11
int c = a++; // c=11, その後 a=12
System.out.println(a + "," + b + "," + c); // 12,11,11

短絡評価と非短絡評価

&& と || は短絡評価します。& と | はbooleanにも使えますが、右辺も評価します。

int a = 10;
int b = 10;
if (a < 10 && ++b > 10) { }
System.out.println(b); // 10: 右辺は評価されない

if (a < 10 & ++b > 10) { }
System.out.println(b); // 11: 右辺も評価される

booleanに使えない比較

<、<=、>、>= は数値などの大小比較に使います。boolean同士に <= は使えません。

boolean a = true;
boolean b = false;
System.out.println(a == b); // OK
// System.out.println(a <= b); // NG

三項演算子

三項演算子は 条件 ? 式1 : 式2 です。条件部分はbooleanでなければいけません。式1と式2の型の調整も発生します。

int score = 80;
String result = score >= 60 ? "OK" : "NG";
System.out.println(result);

switchの基本

switchは、対象型、caseラベル、breakの有無、defaultの位置を確認します。breakがなければフォールスルーします。

int n = 2;
switch (n) {
    case 1: System.out.print("A");
    case 2: System.out.print("B");
    default: System.out.print("C");
}
// BC

ミニ演習

Q. if (10) は？

コンパイルエラーです。if条件はbooleanでなければいけません。

Q. && の左辺がfalseのとき右辺の ++ は実行される？

実行されません。短絡評価です。

代入演算子も値を返す

a += 5 のような代入式は、代入後の値を式全体の値として返します。複合代入と他の代入が連なる問題では、この性質を使います。

int a = 3;
int b = a += 5;
System.out.println(a + b); // 16

switchのcase定数

caseラベルには、switch対象型と互換性のあるコンパイル時定数が必要です。変数を置けばよいわけではありません。

final int A = 1;
int b = 2;
switch (b) {
    case A: break; // OK
    // case b: break; // NG: 変数bは定数ではない
}
