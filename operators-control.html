演算子・分岐・ループ

評価順、++/--、短絡評価、switch式、yield、while、do-while、break/continueをまとめます。

++ / -- の前置と後置

前置は値を変更してから式に使います。後置は元の値を式に使ってから変更します。式の評価が終わったあとも、副作用として変数の値は変わっています。

int i = 1;
System.out.println(++i); // 2。先に増える
System.out.println(i++); // 2。表示後に増える
System.out.println(i);   // 3

条件式の中にある後置インクリメントも、条件がtrueでもfalseでも評価された時点で副作用は残ります。

int i = 0;
while (i++ < 3) {
    System.out.print(i);
}
System.out.println("/" + i);
// 123/4

短絡評価と非短絡評価

<code>&&</code> と <code>||</code> は短絡評価です。左辺だけで結果が確定する場合、右辺は評価されません。<code>&</code> と <code>|</code> はbooleanにも使えますが、右辺も必ず評価します。

int a = 0;
if (false && ++a > 0) {
}
System.out.println(a); // 0

if (false & ++a > 0) {
}
System.out.println(a); // 1

switch文とswitch式

switch文は処理を実行する構文です。switch式は値を返す構文です。Java Silver 17では、switch式の <code>yield</code> が重要です。

int n = 2;
String result = switch (n) {
    case 1 -> "one";
    case 2 -> "two";
    default -> "other";
};
System.out.println(result); // two

switch式でブロックを使う場合、ブロック内で返す値を <code>yield</code> で指定します。yieldはswitch式の値を返すためのキーワードで、メソッドから抜けるreturnとは違います。

int n = 2;
String result = switch (n) {
    case 1 -> "one";
    case 2 -> {
        System.out.println("case 2");
        yield "two";
    }
    default -> "other";
};

yield

switch式の結果値を返します。メソッド全体から戻るわけではありません。

return

メソッドから戻ります。switch式のブロック内で値を返す用途には通常使いません。

switch式末尾のセミコロン

代入文の一部なので }; が必要です。

網羅性

switch式は値を返すため、すべてのケースを網羅する必要があります。通常はdefaultが必要です。
switch対象がnullの場合、通常は実行時にNullPointerExceptionです。コンパイルエラーではありません。

while と do-while

whileは最初に条件判定します。条件が最初からfalseなら本体は0回です。do-whileは本体を実行してから条件判定するため、条件がfalseでも最低1回は実行されます。

int a = 0;
while (a > 0) {
    System.out.println("while");
}

int b = 0;
do {
    System.out.println("do");
} while (b > 0);

do-whileの最後にはセミコロンが必要です。ここは問題文の記号1つで正解が変わります。

do {
    System.out.println("A");
} while (false); // この ; が必要

for、break、continue

通常のfor文は、初期化 → 条件判定 → 本体 → 更新式 → 条件判定、の順に進みます。breakとcontinueでは、更新式が実行されるかが違います。

for (int i = 0; i < 3; i++) {
    if (i == 1) break;
    System.out.print(i);
}
// 0
// breakすると、その回の残り処理も更新式も実行しない
for (int i = 0; i < 3; i++) {
    if (i == 1) continue;
    System.out.print(i);
}
// 02
// continueすると、forの更新式へ進む

while文でcontinueする場合は注意が必要です。continueすると下の更新処理を飛ばすため、更新前にcontinueすると無限ループになりやすいです。
