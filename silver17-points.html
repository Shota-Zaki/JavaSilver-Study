Java 17論点

record、sealed、switch式、instanceofパターンマッチングなど、Java Silverで新しめの論点を整理します。

record

recordは、データを保持するクラスを簡潔に書く仕組みです。コンポーネントに対応するprivate finalフィールド、アクセサ、コンストラクタ、equals/hashCode/toStringが生成されます。

record Item(String name, int price) {}

Item item = new Item("apple", 100);
System.out.println(item.name());  // apple
System.out.println(item.price()); // 100

アクセサは getName() ではなく、コンポーネント名と同じ name() です。

recordの制限

recordは暗黙的にfinalです。別クラスをextendsできません。ただしinterfaceの実装はできます。追加のインスタンスフィールドは定義できません。

interface Printable { void print(); }
record User(String name) implements Printable {
    public void print() { System.out.println(name); }
    // int age; // NG: 追加インスタンスフィールド不可
}

sealed class / interface

sealedは継承・実装できる相手を制限します。許可されたサブクラスは、final、sealed、non-sealed のいずれかを明示します。

sealed class Animal permits Dog, Cat { }
final class Dog extends Animal { }
non-sealed class Cat extends Animal { }

switch式

switch式は値を返します。case -> の形式ではフォールスルーしません。ブロックで値を返す場合は yield を使います。

int n = 2;
String result = switch (n) {
    case 1 -> "one";
    case 2 -> "two";
    default -> "other";
};
System.out.println(result); // two

instanceof パターンマッチング

obj instanceof String s のように書くと、trueと判定された範囲内で s をStringとして使えます。

Object obj = "abc";
if (obj instanceof String s) {
    System.out.println(s.length());
}

ミニ演習

Q. recordのアクセサは getName()？

違います。コンポーネント名と同じ name() です。

Q. sealedクラスを継承したクラスは何か修飾が必要？

必要です。final、sealed、non-sealed のいずれかを付けます。

recordのコンパクトコンストラクタ

recordでは、コンパクトコンストラクタを使って検証や正規化を書けます。引数リストを書かない点が通常コンストラクタと違います。

record User(String name) {
    User {
        if (name == null) throw new IllegalArgumentException();
        name = name.trim();
    }
}

switch式とswitch文を混同しない

switch式は値を返すため、すべての経路で値を生成する必要があります。switch文は従来どおり処理を書く構文です。

int n = 1;
String s = switch (n) {
    case 1 -> "one";
    default -> "other";
};
