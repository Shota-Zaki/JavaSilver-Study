Java 17 Silver新しめ論点

record、sealed、switch式、instanceofパターンなど、Java 17 Silverで混ざりやすい論点を整理します。

Silver 17で特に崩れやすい論点

Java 17 Silverでは、従来の基本文法に加えて、record、sealed、switch式、instanceofのパターンマッチングなどが混ざります。新機能そのものより、「既存ルールと組み合わさったとき」に落とされます。

record：データ保持用クラスを短く書く構文

public record Person(String name, int age) {}

recordでは、private finalなフィールド相当、正準コンストラクタ、name()、age()、equals、hashCode、toStringが主に用意されます。

ひっかけ：アクセサ名は getName() ではなく name()。recordは暗黙的にfinalなので、通常クラスで継承できません。

sealed：継承できる相手を制限する

public sealed class Animal permits Dog, Cat {}
public final class Dog extends Animal {}
public non-sealed class Cat extends Animal {}
  • permits に書かれたクラスだけが継承できる。
  • sealedを継承したクラスは、final / sealed / non-sealed のいずれかが必要。
  • non-sealed は「ここから先は制限を外す」という意味。

instanceof パターンマッチング

Object obj = "java";
if (obj instanceof String s) {
    System.out.println(s.length());
}

obj instanceof String s は、objがStringとして扱える場合だけ、変数 s をそのブロック内で使えます。条件がfalseになりうる場所では使えません。

switch式：値を返せるswitch

int result = switch (rank) {
    case 1 -> 100;
    case 2 -> 80;
    default -> 0;
};
  • switch式は値を返すので、全分岐で値を返す必要がある。
  • -> 形式では基本的にフォールスルーしない。
  • ブロックで複数文を書く場合は yield で値を返す。
String text = switch (n) {
    case 1 -> "one";
    default -> {
        String s = "other";
        yield s;
    }
};

private interface method

interfaceには、defaultメソッドやstaticメソッドから使う補助用としてprivateメソッドを定義できます。ただし、外部から呼ぶためのものではありません。

interface A {
    default void run() {
        helper();
    }
    private void helper() {
        System.out.println("help");
    }
}

privateメソッドは実装クラスから直接呼べません。interface内の処理を共通化するための部品です。
