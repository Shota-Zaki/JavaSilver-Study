試験ひっかけ集

よくある誤答パターンを、原因別にまとめます。

文字・記号の罠

罠例対策
== と =条件式内の代入boolean以外への代入ならコンパイルエラーになりやすい。
; の有無do-while末尾、switch式末尾文か式かを確認。
String[] と String...mainや可変長引数どちらも配列として扱える。
0 と O数値と英字識別子かリテラルかを確認。

思い込みの罠

  • Stringのメソッドは元を変えると思い込む。
  • フィールドもoverrideされると思い込む。
  • List.ofが変更可能だと思い込む。
  • コンストラクタを継承すると考える。
  • 例外が発生してもtryの残りが続くと考える。

選択肢問題の読み方

選択肢の日本語は、絶対表現に注意します。「必ず」「すべて」「一切」「のみ」は反例を探します。Java Silverでは、正しい仕様を知らなくても反例で消せる選択肢があります。
