論点落とし穴
package/importpackage宣言はコメントを除いて先頭。importが先にあるとコンパイルエラー。
main引数起動クラス名はargsに入らない。args[0]はコマンドで渡した1個目。
publicクラス名publicトップレベルクラスがある場合、ファイル名と一致が必要。
varvarはローカル変数の型推論。フィールド、引数、戻り値型には使えない。
var = nullvar b = null; は型を推論できないため不可。
整数リテラル0始まりは8進数。0827のように8を含むと不可。
アンダースコア数値リテラルの先頭・末尾・小数点前後・接尾辞直前は不可。
float1.0はdouble。floatには1.0Fが必要。
byte演算byte/short/charは演算時にintへ昇格する。
複合代入b += 1 は暗黙キャストを含むが、b = b + 1 は不可。
booleanbooleanと数値に互換性はない。if(1)は不可。
String不変concat/replace/toUpperCaseなどは戻り値を受けないと元は変わらない。
StringBuilderappend/delete/reverseなどは自身を変更する。Stringと逆。
==参照型の==は同一参照。内容比較ではない。
interninternは文字列プール上の参照を返す。new Stringとは別参照。
null printlnprintln(null参照)はnull表示。ただしnull参照のメソッド呼び出しはNPE。
配列長配列のlengthはフィールド。Stringのlength()はメソッド。
配列0要素new int[0]は作れる。a[0]に触った時点で例外。
ArrayList removeremove(int)とremove(Object)のオーバーロードに注意。
List.of変更不可。add/remove/setで実行時例外。
拡張forループ変数に代入しても元の要素は差し替わらない。
短絡評価&&は左falseで右を評価しない。||は左trueで右を評価しない。
& と |booleanでも&と|は使えるが短絡しない。右辺も評価される。
後置++式には元の値を渡し、その後増える。
switch文breakがなければフォールスルー。
switch式値を返す。ブロックではyieldが必要。
ラベルラベルは文に付ける。変数宣言だけをbreak先のように扱えない。
コンストラクタ戻り値型を書いたらコンストラクタではなくメソッド。
this()コンストラクタ内のthis()/super()は最初の文でなければならない。
super()明示しなくても先頭にsuper()が暗黙挿入される。親に無引数がないと注意。
overload戻り値型だけ違う同名同引数は不可。
varargs可変長引数は引数リストの最後に1つだけ。
staticstaticメソッドからインスタンスメンバを直接参照できない。
privateprivateメンバは継承関係でも直接アクセス不可。
protected同一パッケージ、または継承関係でアクセス可能。別パッケージでは条件がやや厳しい。
field hidingフィールドはoverrideされない。宣言型で解決される。
method overrideインスタンスメソッドは実体で動的束縛される。
interface methodinterfaceの抽象メソッドはpublic abstract扱い。実装側もpublic。
default methodinterfaceのdefaultは実装を持てる。Object由来メソッドの扱いに注意。
abstract class抽象メソッドを持つクラスはabstract必須。abstractクラスは未実装でもよい。
sealed許可サブクラスだけが継承可能。サブクラス側もfinal/sealed/non-sealedが必要。
recordアクセサはgetXではなくx()。recordは暗黙的にfinal。
宣言型呼べるメソッドは宣言型で決まる。実体にだけあるメソッドは呼べない。
castキャストでコンパイルは通っても、実体が合わなければClassCastException。
instanceofnull instanceof 型 はfalse。例外にはならない。
checked例外catchまたはthrowsしないとコンパイルエラー。
RuntimeException非チェック例外。処理しなくてもコンパイル可。
catch順子例外から親例外へ。親が先だと子catchは到達不能。
finally例外の有無に関わらず基本的に実行される。
try-with-resourcescloseは宣言の逆順。finallyより先にcloseされる。

特に優先して潰すべき10個

  1. コンパイル可否を先に見る。 出力を追うのは最後。
  2. 宣言型で呼べるかを見る。 実体にメソッドがあっても宣言型になければ不可。
  3. byte/short/charの演算結果はint。 Silverの定番。
  4. Stringは不変、StringBuilderは可変。 戻り値を受けたかを見る。
  5. nullは表示とメソッド呼び出しで違う。 printlnだけなら表示、呼び出しならNPE。
  6. catchは子から親。 到達不能catchはコンパイルエラー。
  7. interface実装はpublic。 書き忘れが多い。
  8. recordアクセサは部品名と同じ。 getXではない。
  9. sealedサブクラス側の修飾子。 permitsだけ見ても足りない。
  10. List.ofは変更不可。 コンパイルではなく実行時例外。
