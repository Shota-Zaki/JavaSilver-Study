Java Silverで選択肢を誤らせる典型パターンを、論点別に整理します。

コンパイル前に出力を追わせる罠

一番多い罠は、見た目は出力問題なのに、実際はコンパイルエラーの問題です。

罠見る点
ローカル変数未初期化全経路で代入済みか
varの禁止場所フィールド、引数、戻り値で使っていないか
booleanと数値if(1) になっていないか
override違反アクセス、戻り値、throws

実行時例外を見落とす罠

コンパイルできても、該当行に到達した時点で例外が出る場合があります。後続の出力を入れてしまうと誤答です。

罠典型
nullメソッド呼び出しs.length()
範囲外アクセスargs[0], array[0]
不正キャスト親型変数から違う子型へ
数値変換失敗parseInt

意味を逆に覚えやすい罠

  • == は参照型では同一性。内容比較は原則 equals。
  • String のメソッドは元を変えない。StringBuilder は元を変える。
  • List.of は変更不可。コンパイルではなく実行時に落ちる。
  • protected は同じパッケージでも使える。別パッケージの継承アクセスは条件付き。
  • finally は例外発生の有無に関係なく通る。
