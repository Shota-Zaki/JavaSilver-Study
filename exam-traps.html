Java Silver Study

Java Silver Reference

ひっかけ集

Java Silverで選択肢を誤らせる典型パターンを、短く確認します。

型とリテラル

  • float f = 1.0; は不可。1.0 はdouble。
  • int i = 10L; は不可。longからintへ暗黙変換不可。
  • byte b = b + 1; は不可。演算結果はint。
  • b += 1; は可。複合代入には暗黙キャストがある。
  • boolean と数値は互換性なし。

Stringと参照

  • new String("x") == "x" はfalse。
  • intern() はプール上の参照を返す。
  • String.concat の戻り値を捨てると元は変わらない。
  • println(null参照) は文字列 null 表示。ただし null参照.メソッド() はNPE。

継承・interface

  • コンストラクタは継承されない。
  • privateメンバはサブクラスから直接アクセス不可。
  • interfaceの抽象メソッド実装は public。
  • フィールドはoverrideされない。
  • 宣言型にないメソッドは、実体にあっても呼べない。

例外

  • try内で例外が出たら、その後のtry内処理は実行されない。
  • finallyはcatch後にも実行される。
  • catchは子から親へ。
  • try-with-resourcesはcloseが逆順。
  • RuntimeExceptionは非チェック例外。

出力問題

  • 短絡評価で右辺が実行されないことがある。
  • 後置 a++ は元の値を式に渡してから増やす。
  • switch文の break 忘れはフォールスルー。
  • 拡張forのループ変数代入では元配列・元リストは差し替わらない。
