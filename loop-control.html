論点ルール注意
後置++式では元の値を使い、その後増えるi++ < 3 は比較後にiが増える。
前置++先に増やした値を使う++i < 3。
&& / ||短絡評価あり左だけで結果が決まると右辺は実行しない。
& / |booleanでも右辺を評価副作用の有無が変わる。

switch文とswitch式

項目switch文switch式
目的処理を分岐する文値を返す式
代入直接は値にならないint n = switch(x) { ... }; が可能
case ->使用可使用可
yield通常不要ブロックから値を返すとき必要
末尾セミコロンswitch文自体には不要代入式なら最後に ; が必要
網羅性必須ではない値を返すため原則defaultなどで網羅が必要
nullswitch対象がnullならNullPointerException同じ。case nullは通常範囲外。

yieldの返り値

int result = switch (score) {
    case 1 -> 10;
    case 2 -> {
        int tmp = score * 10;
        yield tmp;      // このブロックが返す値
    }
    default -> 0;
};
System.out.println(result);

ループの実行順

構文実行順抜ける基準
while条件 → 本体 → 条件最初からfalseなら0回。
do-while本体 → 条件 → 本体最後に ; が必要。falseでも1回は本体実行。
for初期化 → 条件 → 本体 → 更新 → 条件breakなら更新式を実行せず抜ける。continueなら更新式へ進む。
拡張for各要素を順に取り出すインデックスは直接扱わない。

break / continue / ラベル

命令通常ラベル付き
break一番内側のループ/switchを抜ける指定ラベルのブロックを抜ける。
continue一番内側のループの次回判定へ指定ラベルのループの次回判定へ。ブロックラベルには不可。
finallybreak/continue/return前にも実行例外・制御移動より優先して挟まる。

do-whileと++の例

int i = 0;
do {
    System.out.print(i);
} while (i++ < 2);
System.out.println("/" + i);
// 出力: 012/3
// 条件判定で i++ が実行され、falseになった判定でも増加は残る。
