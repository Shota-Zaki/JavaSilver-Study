ループ・do-while・break/continue

ループは条件判定の位置、更新式のタイミング、break/continue後の動作で決まります。

while / do-while / forの違い

構文条件判定最低実行回数注意
while本体前0回条件が最初からfalseなら本体なし。
do-while本体後1回最後の;が必要。
for本体前0回更新式は本体後。continue後にも更新式へ進む。

do-while と i++

int i = 0;
do {
    System.out.print(i);
} while (i++ < 3);
System.out.println("/" + i);
// 出力: 0123/4

条件式の i++ は、比較に使った後でiを増やします。条件がfalseになった最後の評価でも副作用は残ります。

break と continue

for (int i = 0; i < 5; i++) {
    if (i == 2) break;
    System.out.print(i);
}
// 01。breakするとforの更新式には進まない。

for (int i = 0; i < 5; i++) {
    if (i == 2) continue;
    System.out.print(i);
}
// 0134。continue後はforの更新式へ進む。

whileでcontinueを使う注意

int i = 0;
while (i < 3) {
    if (i == 1) continue; // i++に到達せず無限ループ
    i++;
}

whileでは更新処理を自分で書くため、continueで更新前に戻ると無限ループになりやすいです。

ラベル付きbreak/continue

outer:
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (j == 1) break outer; // 外側ループごと抜ける
    }
}
