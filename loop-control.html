ループ・do-while・break/continue

ループは「どの時点で条件判定するか」「++がいつ反映されるか」「抜けたあとに何が実行されるか」で正誤が決まります。

ループ問題の判定順

ループ問題は、雰囲気で追うとほぼ間違えます。次の順番で機械的に追ってください。

  1. 初期化：変数の初期値、配列・Listの中身、参照先を確定する。
  2. 条件判定の位置：while / for は本体前、do-while は本体後。
  3. 本体実行：出力、代入、++ / --、break / continue を上から処理する。
  4. 更新処理：for の第3式は、本体が正常終了した後に実行される。continue でも実行される。
  5. 終了後：条件がfalse、break、return、例外のどれで抜けたかを分ける。
結論
「条件を見る前に本体を実行するのは do-while だけ」です。ここを固定すると、初回実行の有無を落としにくくなります。

while文：条件がfalseなら本体は1回も実行しない

int i = 0;
while (i > 0) {
  System.out.println(i);
  i--;
}
System.out.println("end");

// 出力
// end

while は本体に入る前に条件を評価します。最初から条件がfalseなら、ループ内は1回も実行されません。

int i = 0;
while (i++ < 3) {
  System.out.print(i + " ");
}
System.out.println("/ after=" + i);

// 出力
// 1 2 3 / after=4

i++ < 3 は、比較には増える前の値を使い、その後で i が増えます。最後に 3 < 3 がfalseになった時点でも、後置インクリメント自体は実行済みなので i は4になります。

do-while文：本体を実行してから条件を見る

int i = 0;
do {
  System.out.println(i);
  i++;
} while (i < 0);
System.out.println("end");

// 出力
// 0
// end

do-while は条件判定が後ろにあるため、条件が最初からfalseでも本体は必ず1回実行されます。

試験での注意
do { ... } while (条件); の最後の ; は必須です。while 文や for 文のブロック終端とは違います。
int i = 3;
do {
  System.out.print(i + " ");
} while (i++ < 3);
System.out.println("/ after=" + i);

// 出力
// 3 / after=4

条件式の i++ < 3 は、比較に3を使ってfalseになります。しかし i++ は評価済みなので、抜けた後の i は4です。条件がfalseでも、条件式内の副作用は残る点が重要です。

for文：初期化 → 条件 → 本体 → 更新 → 条件

for (int i = 0; i < 3; i++) {
  System.out.print(i + " ");
}

// 出力
// 0 1 2

for は次の順序で動きます。

  1. 初期化：int i = 0
  2. 条件：i < 3
  3. 本体
  4. 更新：i++
  5. 再び条件
for (int i = 0; i++ < 3;) {
  System.out.print(i + " ");
}

// 出力
// 1 2 3

条件式に i++ がある場合、条件判定のたびに i が増えます。本体で見える i は、条件式評価後の値です。

for文の更新式とcontinue：continueしても更新式は実行される

for (int i = 0; i < 5; i++) {
  if (i == 2) {
    continue;
  }
  System.out.print(i + " ");
}

// 出力
// 0 1 3 4

continue は現在の反復の残りをスキップします。ただし for 文では、その後に第3式の更新処理が実行され、次の条件判定へ進みます。

int i = 0;
while (i < 5) {
  if (i == 2) {
    i++;
    continue;
  }
  System.out.print(i + " ");
  i++;
}

// 出力
// 0 1 3 4

while には自動の更新式がありません。continue 前に i++ を忘れると、i == 2 のまま無限ループになります。

break：すぐにループを抜ける。更新式は実行されない

for (int i = 0; i < 5; i++) {
  if (i == 2) {
    break;
  }
  System.out.print(i + " ");
}
System.out.println("end");

// 出力
// 0 1 end

break はループを即終了します。break が実行された反復では、for の更新式 i++ は実行されません。

int i = 0;
for (; i < 5; i++) {
  if (i == 2) break;
}
System.out.println(i);

// 出力
// 2

i == 2 で break したため、その回の更新式は動きません。抜けた後の i は2です。

continue：ループを抜けない。次の反復へ進む

for (int i = 0; i < 4; i++) {
  System.out.print("A");
  if (i % 2 == 0) continue;
  System.out.print("B");
}

// 出力
// AAABA B ではない
// 実際: AAAB

空白を入れずに追うと誤読しやすいので、反復ごとに分けます。

i処理出力
0Aを出力。偶数なのでcontinueA
1Aを出力。continueしないのでBも出力AB
2Aを出力。偶数なのでcontinueA
3Aを出力。Bも出力AB

合計は AABAAB です。途中でcontinueしてもループ自体は終わりません。

ラベル付きbreak / continue：どのループに効くか

outer:
for (int i = 0; i < 3; i++) {
  for (int j = 0; j < 3; j++) {
    if (i == 1 && j == 1) {
      break outer;
    }
    System.out.print(i + ":" + j + " ");
  }
}

// 出力
// 0:0 0:1 0:2 1:0

break outer; は外側のラベル付きループ全体を抜けます。内側ループだけを抜ける普通の break とは違います。

outer:
for (int i = 0; i < 3; i++) {
  for (int j = 0; j < 3; j++) {
    if (j == 1) {
      continue outer;
    }
    System.out.print(i + ":" + j + " ");
  }
}

// 出力
// 0:0 1:0 2:0

continue outer; は外側ループの次の反復へ進みます。内側の残り処理は実行されません。

ループを抜けた後、ループ内の残りは実行されるか

原因ループ内の残りfor更新式ループ後の文
条件がfalse実行しない条件falseになった後は実行しない実行する
break実行しない実行しない実行する
continue実行しないfor では実行するまだ抜けない
return実行しない実行しない同じメソッド内の後続文は実行しない
例外実行しない実行しないcatchされなければ実行しない
判断基準
ループを抜ける原因が break なのか、条件falseなのか、return なのかを分けます。特に break と continue は「抜ける」と「次へ進む」で全く違います。

do-while内の++：本体 → 条件式の順に副作用が起きる

int i = 0;
do {
  System.out.print(i + " ");
} while (++i < 3);
System.out.println("/ after=" + i);

// 出力
// 0 1 2 / after=3

++i は先に増やしてから比較します。各反復は次のように進みます。

本体で出力条件式継続?
0++i で1、1 < 3継続
1++i で2、2 < 3継続
2++i で3、3 < 3終了
int i = 0;
do {
  System.out.print(i + " ");
} while (i++ < 3);
System.out.println("/ after=" + i);

// 出力
// 0 1 2 3 / after=4

i++ は比較に増える前の値を使うため、3 < 3 でfalseになる時点でも本体ではすでに3を出力済みです。そして条件式評価後に i は4になります。

空文ループ：セミコロン1つで本体が空になる

int i = 0;
while (i++ < 3);
System.out.println(i);

// 出力
// 4

while (...); の ; は空文です。つまりループ本体は何もしません。条件式だけが繰り返し評価されます。

int i = 0;
while (i++ < 3)
  System.out.println(i);

// 出力
// 1
// 2
// 3

見た目が似ていても、while (条件); と while (条件) 文 は別物です。試験ではセミコロンの有無を必ず見ます。

拡張for文：取り出し変数に代入しても配列要素は変わらない

int[] nums = {1, 2, 3};
for (int n : nums) {
  n *= 10;
}
System.out.println(nums[0]);

// 出力
// 1

拡張for文の変数 n は、要素のコピーを受け取るローカル変数です。n に代入しても配列の中身は変わりません。

StringBuilder[] array = {
  new StringBuilder("A"),
  new StringBuilder("B")
};
for (StringBuilder sb : array) {
  sb.append("X");
}
System.out.println(array[0]);

// 出力
// AX

参照型の場合、取り出し変数には参照のコピーが入ります。参照先のオブジェクトを変更する操作なら、元の配列から見える中身も変わります。ただし、sb = new StringBuilder("Z") のように変数自体を別参照へ向けても、配列要素は変わりません。

ループ中にListを変更する問題

ArrayList<String> list = new ArrayList<>();
list.add("A");
list.add("B");
list.add("C");

for (String s : list) {
  if (s.equals("B")) {
    list.remove(s);
  }
}

// 状況によって ConcurrentModificationException

拡張for文は内部的にイテレータを使います。反復中にList本体を直接変更すると、実行時に ConcurrentModificationException が発生する可能性があります。

Java Silverでは、問題の選択肢にこの例外名が出ないこともあります。その場合は「実行時に例外がスローされる」を選びます。コンパイルエラーではありません。

ミニ演習：do-whileと++

int i = 0;
do {
  System.out.print(i);
} while (i++ < 2);
System.out.println("/" + i);
解答

出力は 012/3 です。

  1. 本体で0を出力。条件は 0 < 2 でtrue。その後 i は1。
  2. 本体で1を出力。条件は 1 < 2 でtrue。その後 i は2。
  3. 本体で2を出力。条件は 2 < 2 でfalse。その後 i は3。

ミニ演習：break後に更新式は実行されるか

int i = 0;
for (; i < 5; i++) {
  if (i == 3) break;
}
System.out.println(i);
解答

出力は 3 です。i == 3 の反復で break したので、その回の更新式 i++ は実行されません。

ミニ演習：continueとfor更新式

for (int i = 0; i < 4; i++) {
  if (i == 1) continue;
  System.out.print(i);
}
解答

出力は 023 です。i == 1 では出力を飛ばしますが、for の更新式は実行されるため次は i == 2 になります。

ループ系の試験チェックリスト

  • while は本体前に条件を見る。falseなら0回。
  • do-while は本体後に条件を見る。最低1回。
  • do-while の最後の ; は必須。
  • for の更新式は、本体正常終了後と continue 後に実行される。
  • break 後は更新式を実行しない。
  • continue はループを抜けない。次の反復へ進む。
  • i++ は値を使ってから増える。++i は増えてから値を使う。
  • 条件式がfalseでも、条件式内の i++ などの副作用は実行済み。
  • while(...); は空文ループ。
  • 拡張for文の取り出し変数への再代入は、元の配列・List要素を置き換えない。
