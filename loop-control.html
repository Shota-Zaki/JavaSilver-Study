ループ制御

while、do-while、for、拡張for、break、continue、ラベルの使い分けを整理します。

ループ問題の見る順番

  1. 初期値を表に書く。
  2. 条件判定が先か後かを確認する。do-while は最低1回実行。
  3. 更新式がいつ実行されるか確認する。
  4. break / continue でどこへ移動するか確認する。
  5. ラベルが付いている場合、対象がブロックまたは文として正しいか見る。

whileとdo-while

int a = 10;
while (a < 10) {
    System.out.println("while");
}

do {
    System.out.println("do");
} while (a < 10);

while は条件が先なので0回の可能性があります。do-while は本体を実行してから条件を見るので最低1回実行されます。

for文の実行順序

通常のfor文は「初期化 → 条件 → 本体 → 更新 → 条件 → ...」です。

for (int i = 0; i < 3; i++) {
    System.out.print(i);
}
// 012

continueと更新式

for文で continue すると、本体の残りを飛ばして更新式へ進みます。while文では条件判定へ戻ります。

for (int i = 0; i < 3; i++) {
    if (i == 1) continue;
    System.out.print(i);
}
// 02

ラベル付きbreak/continue

ラベルは任意の文に付けられますが、continue の飛び先にできるのはループです。break はラベル付きブロックから抜ける用途にも使えます。

outer:
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (j == 1) break outer;
    }
}
continue label; のlabelが単なるブロックについている場合はコンパイルエラーです。

無限ループと到達不能

while(true) の後ろは到達不能になる場合があります。ただし、ループ内に条件付きbreakがあると到達可能と判断されることがあります。

while (true) {
    break;
}
System.out.println("reachable"); // OK

ミニ演習

Q. do-whileの末尾にセミコロンは必要？

必要です。do { } while (condition); の形です。

Q. ラベルはif文にも付けられる？

付けられます。ただし continue の対象にはできません。

関連: 判定フロー / コンパイル・実行時例外 / ひっかけ集

for文の各部は省略できる

for文は初期化式、条件式、更新式を省略できます。条件式を省略するとtrue扱いです。無限ループかどうか、breakで抜けるかを見ます。

for (;;) {
    break;
}
System.out.println("end"); // 到達可能

ネストループのbreak/continue

ラベルなしの break は最も内側のループだけを抜けます。外側まで抜けるにはラベル付きbreakを使います。

for (int i = 0; i < 2; i++) {
    for (int j = 0; j < 2; j++) {
        break; // jのループだけ抜ける
    }
    System.out.print(i);
}
// 01
