数値・リテラル・型昇格

整数・小数リテラル、進数表記、アンダースコア、byte/short/charのint昇格、定数式、複合代入を詳しく整理します。

数値問題は値より先に式の型を決める

数値・型昇格の問題は、見た目の値だけで判断すると高確率で間違えます。毎回、次の順番で処理してください。

  1. 各リテラルの型を決める。整数は基本 int、小数は基本 double。
  2. 演算があるかを見る。byte/short/char は演算で原則 int に昇格。
  3. 定数式か変数式かを見る。範囲内の定数式だけ特別に小さい型へ代入できる場合がある。
  4. 代入先の型へ暗黙変換できるかを見る。
  5. 複合代入なら、暗黙キャストを含むことを考慮する。
重要: byte b = 1 + 2; と byte b = a + 2; は同じではありません。前者は定数式、後者は変数を含む式です。

数値リテラルの基本型

まず、ソースコードに直接書いた数値が何型として扱われるかを決めます。

リテラル型例注意
整数リテラルint10範囲を超える整数は L が必要。
L/l付き整数long10L小文字 l は 1 と紛らわしい。
小数リテラルdouble1.0float へ直接代入不可。
F/f付き小数float1.0Ffloat へ入れるなら基本これ。
D/d付き小数double1.0D明示しなくても小数は基本 double。
 
int a = 10;       // 10 は int
long b = 10;      // int から long へ拡大変換
long c = 10L;     // 10L は long

// float d = 1.0; // NG: 1.0 は double
float e = 1.0F;   // OK
double f = 1.0F;  // OK: float から double へ拡大変換

整数リテラルの進数表記

同じ整数でも、先頭の書き方で基数が変わります。特に8進数の 0 始まりは見落としやすいです。

表記例10進での値注意
10進数1717通常の表記。
2進数0b10001170b または 0B。
8進数02117先頭が 0。8 と 9 は使えない。
16進数0x11170x または 0X。
 
int a = 17;
int b = 0b10001;
int c = 021;
int d = 0x11;
System.out.println(a + b + c + d); // 68

// int e = 089; // NG: 8進数に 8/9 は使えない

アンダースコアのルール

数値リテラルの途中には _ を置けます。読みやすくするための記号で、値には影響しません。ただし置ける場所に制限があります。

コード可否理由
int a = 123_456;OK数字の間。
int b = 1__2__3;OK連続しても数字の間なら可。
int c = _123;NGリテラルの先頭。
int d = 123_;NGリテラルの末尾。
float e = 3_.14F;NG小数点の直前。
float f = 3._14F;NG小数点の直後。
long g = 100_L;NG接尾辞 L の直前。
int h = 0x_FF;NG基数接頭辞 0x の直後。
int i = 0xF_F;OK数字の間。

拡大変換と縮小変換

代入では、右辺の型から左辺の型へ安全に変換できるかを見ます。広い型へは暗黙変換できますが、狭い型へは原則キャストが必要です。

int i = 10;
long l = i;       // OK: int から long
float f = l;      // OK: long から float
// int x = l;     // NG: long から int
int y = (int) l;  // OK: 明示キャスト

ただし、浮動小数では「範囲」と「精度」は別です。long から float への変換は許されますが、精度が失われる可能性はあります。試験では主にコンパイル可否を見ます。

byte・short・charは演算でintに昇格する

ここが最重要です。byte、short、char は、算術演算に入ると基本的に int に昇格します。

byte a = 1;
byte b = 2;
// byte c = a + b; // NG: a + b は int
int d = a + b;     // OK
byte e = (byte)(a + b); // OK: 明示キャスト
char c = 'A';
// char next = c + 1; // NG: c + 1 は int
char next = (char)(c + 1); // OK
覚え方: 小さい整数型の演算結果は、だいたい int になる。代入先が byte/short/char なら警戒。

二項数値昇格の優先順位

左右のオペランドの型が違う場合、広い型に寄せられます。大まかな優先順位は次の通りです。

含まれる型演算結果の型例
double が含まれるdouble1 + 2.0 は double
float が含まれるfloat1L + 2.0F は float
long が含まれるlong1 + 2L は long
それ以外の整数型intbyte + short は int
 
int a = 10;
long b = 20L;
// int c = a + b;  // NG: a + b は long
long d = a + b;    // OK

float f = 1.0F;
double g = 2.0;
// float h = f + g; // NG: f + g は double
double i = f + g;   // OK

数値演算は「昇格→演算→代入」の3段階で解く

数値問題では、最終値だけを暗算すると型のエラーを見落とします。各オペランドをどの型へ昇格させ、どの型で演算し、その結果をどこへ代入するかを分けます。

  1. 単項演算なら単項数値昇格を確認する。byte・short・charは多くの場合intになる。
  2. 二項演算ならdouble → float → long → intの優先順で共通型を決める。
  3. 演算結果の型から、代入先への変換が自動で許可されるかを確認する。
  4. 定数式、複合代入、明示的キャストという例外規則があるかを最後に確認する。
byte a = 10;
      byte b = 20;
      // byte c = a + b;  // NG: aとbはintへ昇格
      int c = a + b;      // OK
      
      a += b;             // OK: 概念的に a = (byte)(a + b)
複合代入

次のコードはコンパイルできますか。

byte b = 127;
      b += 1;
      System.out.println(b); // -128
複合代入後：コンパイルできます。複合代入には代入先型への暗黙的な変換が含まれます。値が範囲外なら切り詰めが起こります。
char演算

char c = 'A'; c++;は可能ですか。

演算型の違い：可能です。++は変数へ戻す変換を含みます。一方、c = c + 1は右辺がintなので明示的キャストが必要です。

定数式なら小さい型へ代入できる場合がある

右辺がコンパイル時に値を確定できる定数式で、代入先の範囲内なら、byte、short、char へ代入できます。

byte a = 10;        // OK
byte b = 10 + 20;   // OK: 定数式で範囲内
// byte c = 100 + 100; // NG: 200 は byte の範囲外

short s = 1000 + 2000; // OK
char ch = 65;          // OK

定数式の例外を知らないと、「整数リテラルはintだから全部NG」と誤判定します。逆に、変数を含む式までOKにしてしまうのも誤りです。

final変数とコンパイル時定数

final が付いていて、宣言時に定数式で初期化されている場合、コンパイル時定数として扱われることがあります。

final int x = 10;
byte a = x;       // OK: x はコンパイル時定数で範囲内
byte b = x + 20;  // OK

int y = 10;
// byte c = y;    // NG: y は変数
// byte d = y + 20; // NG
final int z;
z = 10;
// byte e = z; // NGになり得る: 宣言時初期化のコンパイル時定数ではない
試験用判断: final int x = 10; のように宣言と同時に定数で初期化されているかを確認します。

定数式・変数・finalを区別する

finalを付ければ必ず定数式になるわけではありません。コンパイル時に値が決まり、許可された構成だけで作られた式であることが必要です。

final int A = 100;
      byte x = A;              // OK
      
      final int B;
      B = 100;
      // byte y = B;           // NGになり得る: 宣言時初期化の定数変数ではない
      
      int n = 100;
      final int C = n;
      // byte z = C;           // NG: 初期化式が定数式ではない
定数式の判定：値の大きさだけでなく、右辺がコンパイル時定数式として成立しているかを確認します。

複合代入は暗黙キャストを含む

+=、-=、*=、/= などの複合代入は、左辺の型へのキャストを含むように動きます。

byte b = 1;
// b = b + 1; // NG: b + 1 は int
b += 1;       // OK: b = (byte)(b + 1) 相当

ただし、これは安全という意味ではありません。値が範囲を超えれば、オーバーフローした結果になります。

byte b = 127;
b += 1;
System.out.println(b); // -128

オーバーフロー: コンパイル可否と値を分ける

整数型は範囲を超えると循環します。キャストや複合代入では、コンパイルできるが値が予想外になることがあります。

byte b = (byte)128;
System.out.println(b); // -128

int max = Integer.MAX_VALUE;
System.out.println(max + 1); // -2147483648

Java Silverでは「コンパイルエラー」なのか「実行できるが値が変わる」のかを分けることが重要です。

オーバーフローとキャスト位置

整数演算は、演算に使われる型の範囲で実行されます。後から広い型へ代入しても、先に発生したオーバーフローは戻りません。

int a = 1_000_000;
      long x = a * a;          // int同士で演算後にlongへ代入
      long y = (long) a * a;   // 片方を先にlongへ変換して演算
      
      System.out.println(x);   // オーバーフローした値
      System.out.println(y);   // 1000000000000
オーバーフローを防ぐ位置：式全体ではなく、演算前のオペランドを広い型へ変換します。
キャストの順序

(long)(a * a)と(long)a * aは同じですか。

更新演算の扱い：同じではありません。前者はint演算後に変換し、後者はlong演算へ昇格させます。

整数除算と小数除算

割り算は、オペランドの型で結果が変わります。整数同士なら小数部は切り捨てです。

System.out.println(5 / 2);     // 2
System.out.println(5 / 2.0);   // 2.5
System.out.println(5.0 / 2);   // 2.5

int a = 5;
int b = 2;
double c = a / b;
System.out.println(c);         // 2.0

double c = a / b; は、先に int / int が計算されて 2 になり、その後 double に代入されます。2.5 にはなりません。

余り演算子 %

% は割った余りを返します。出力追跡問題でよく使われます。

System.out.println(10 % 3); // 1
System.out.println(10 % 5); // 0
System.out.println(3 % 10); // 3

符号付きの余りも出題される可能性はありますが、Silverではまず正の整数の基本を確実に処理できれば十分です。

整数除算・剰余・浮動小数点

式結果の考え方注意点
5 / 2整数同士なので2代入先がdoubleでも演算後は変わらない
5 / 2.0double演算で2.5片方が浮動小数点なら昇格
-5 % 2-1剰余の符号は被除数側
1.0 / 0.0正の無限大浮動小数点の0除算は整数と異なる
0.0 / 0.0NaNNaNは自身との==もfalse

charの演算と出力

char は文字ですが、演算では数値として扱われ、結果は原則 int です。

char c = 'A';
System.out.println(c);     // A
System.out.println(c + 1); // 66
System.out.println((char)(c + 1)); // B

println に char 単体を渡すと文字として出ますが、c + 1 は int なので数値として出ます。

String結合が混ざる数値問題

+ は数値加算にも文字列結合にも使われます。左から評価され、途中で String が出ると以後は文字列結合になります。

System.out.println(1 + 2 + "A"); // 3A
System.out.println("A" + 1 + 2); // A12
System.out.println("A" + (1 + 2)); // A3

これは数値型昇格そのものではありませんが、数値問題の出力選択肢で頻出です。

キャストの優先順位

キャストはどこにかかっているかを正確に見ます。

byte a = 10;
byte b = 20;

byte c = (byte)(a + b); // OK: 合計結果を byte にキャスト
// byte d = (byte)a + b; // NG: (byte)a は byte だが、+ b で int になる

キャストを見つけたら「式全体にかかっているのか、一部だけか」を括弧で確認します。

出力追跡の例: 型を表にして追う

byte b = 10;
int i = 20;
long l = 30L;
System.out.println(b + i + l);
処理型値
b + iint30
30 + llong60
printlnlong値の表示60

出力には L は付きません。L はソースコード上のリテラル表記です。

昇格と代入で結果が変わる例

1. byte b = 128;

コンパイルエラー。byte の範囲は -128〜127 です。

2. short s = 10 + 20;

OK。リテラルだけの定数式で、結果30は short の範囲内です。

3. short a = 10, b = 20; short c = a + b;

コンパイルエラー。a + b は int です。

4. float f = 10.0;

コンパイルエラー。10.0 は double です。10.0F にします。

5. int x = 2 * 3L;

コンパイルエラー。3L があるので演算結果は long です。

6. int x = 08;

コンパイルエラー。先頭 0 は8進数で、8進数に 8 は使えません。

昇格・定数式・複合代入を見分ける

Q1. byte b = 1 + 2; は？

OKです。リテラルだけの定数式で、結果3は byte の範囲内です。

Q2. byte a = 1; byte b = 2; byte c = a + b; は？

コンパイルエラーです。a + b は int になります。

Q3. byte b = 127; b += 1; System.out.println(b); は？

-128 と表示されます。b += 1 はコンパイルできますが、byte の範囲を超えてオーバーフローします。

Q4. double d = 5 / 2; の値は？

2.0 です。先に int / int で 2 になり、その後 double に代入されます。

Q5. System.out.println('A' + 1); は？

66 と表示されます。'A' は演算で int に昇格します。

数値式を読み終える前の確認

  • 整数リテラルは基本 int、小数リテラルは基本 double と判断できる。
  • L、F、D 接尾辞の意味を説明できる。
  • 2進数・8進数・16進数表記を見分けられる。
  • 数値リテラルの _ を置ける場所・置けない場所を判断できる。
  • byte/short/char の演算結果が原則 int になると判断できる。
  • 定数式と変数式を区別できる。
  • 複合代入が暗黙キャストを含むことを説明できる。
  • 整数除算と小数除算の出力差を追跡できる。
