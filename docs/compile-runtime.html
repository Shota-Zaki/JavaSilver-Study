コンパイルエラーと実行時例外

コード問題で最初に切るべき「コンパイルできるか」「実行中に例外が出るか」「正常出力か」を整理します。

3分類を混ぜない

Java Silverの選択肢は、しばしば「コンパイルエラー」「実行時例外」「出力結果」を混ぜてきます。ここを混同すると、解説を読んでも同じ間違いを繰り返します。

分類発生タイミング例
コンパイルエラー実行前。classファイルを作れない型不一致、未初期化ローカル変数、override違反、catch到達不能
実行時例外コンパイル後、実行中NullPointerException、ArrayIndexOutOfBoundsException、ClassCastException
正常出力コンパイルも実行も成功printlnの結果、または何も表示されない

判定を4段階に分ける

段階確認内容代表的な失敗
1. コンパイル構文、型、アクセス、到達可能性、例外検査コンパイルエラー
2. 起動・読み込みクラス名、クラスパス、main、依存クラス起動失敗・リンク時エラー
3. 初期化static初期化、クラス初期化ExceptionInInitializerErrorなど
4. 実行null、添字、キャスト、算術、明示的throw実行時例外または正常終了

試験の多くは1と4の区別ですが、コマンド問題では2、static初期化を含む問題では3も分けると精度が上がります。

nullフィールド

参照型フィールドを初期化せず、メソッドでfield.length()を呼ぶとどうなりますか。

発生フェーズ：フィールド自体はnullで初期化されるためコンパイルは通り、実行時にNullPointerExceptionです。
未初期化ローカル

ローカル参照変数を宣言だけしてlocal.length()とするとどうなりますか。

コンパイル時検査：使用前の確実な代入がないためコンパイルエラーです。

コンパイルエラーの典型

int a;
System.out.println(a); // 未初期化ローカル変数

byte b = 1;
b = b + 1;             // b + 1 はint

class A { private void test() {} }
class B extends A { @Override void test() {} } // privateはoverride対象外

コンパイルエラーは、JVMが実行する前に発生します。したがって「途中まで表示される」は原則ありません。

実行時例外の典型

String s = null;
System.out.println(s.length()); // NullPointerException

int[] a = new int[0];
a[0] = 10;                     // ArrayIndexOutOfBoundsException

Object obj = "abc";
Integer n = (Integer) obj;      // ClassCastException

実行時例外は、該当行に到達して初めて起きます。到達しない行の例外は発生しません。

到達不能コードとcatch順序

catchは上から順に評価されます。スーパークラスの例外を先にcatchすると、サブクラスのcatchは到達不能になりコンパイルエラーです。

try {
    throw new java.io.IOException();
} catch (Exception e) {
} catch (java.io.IOException e) { // NG: 先にExceptionで捕まる
}

何も表示されないケース

「何も表示されない」は、コンパイルエラーや例外とは別です。正常に実行されたが println に到達しない、または空文字を出力するなどのケースです。

String s = "";
System.out.print(s); // 正常終了。ただし見た目は何も出ない

失敗するフェーズを分類する

Q. String s = null; System.out.println(s); は例外？

例外ではありません。println(Object) はnullを文字列 null として表示します。s.length() のようにインスタンスメソッドを呼ぶと例外です。

Q. コンパイルエラーのコードは、途中のprintlnだけ実行される？

されません。そもそも実行用のclassファイルを作れないので、出力はありません。

関連: 判定フロー / コンパイル・実行時例外 / ひっかけ集

「途中まで出る」かどうかの境界

コンパイルエラーなら何も実行されません。実行時例外なら、例外発生前の出力は残ります。この違いが選択肢を切る鍵です。

// コンパイルエラー: 何も表示されない
int a;
System.out.println("A");
System.out.println(a);

// 実行時例外: Aは表示される
System.out.println("A");
String s = null;
s.length();
System.out.println("B");

出力がどこまで行われるかを追う

System.out.print("A");
      int[] values = new int[0];
      System.out.print(values[0]);
      System.out.print("B");
      // Aを表示後、ArrayIndexOutOfBoundsException。Bは表示されない

例外発生前に完了した副作用は残ります。式の途中で例外が起きる場合は、左からどこまで評価済みかを確認します。

catchされない例外

try内で例外が発生しても、型がどのcatchにも適合しない場合はfinallyが実行されますか。

伝播までの処理：finallyがあれば実行され、その後例外が呼び出し元へ伝播します。

例外がcatchされる場合とされない場合

発生した例外型がcatchの型に代入可能ならcatchされます。代入可能でなければ、そこで呼び出し元へ伝播します。

try {
    Object o = "abc";
    Integer i = (Integer) o;
} catch (NullPointerException e) {
    System.out.println("NPE");
}
// ClassCastExceptionはcatchされない。異常終了。
