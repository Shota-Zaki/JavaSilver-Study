switch式

switch文との違い、yield、網羅性、case定数、結果型、副作用を、式が値を生成する経路に沿って追います。

最初に「文」か「式」かを判定する

switchを見つけても、すぐcaseを追ってはいけません。switch全体が処理の分岐なのか、値を生成する式なのかで、必要な終端とコンパイル条件が変わります。

switch文

switch (code) {
    case 1 -> System.out.println("one");
    default -> System.out.println("other");
}

値を返しません。代入式の右辺には置けません。

switch式

String label = switch (code) {
    case 1 -> "one";
    default -> "other";
};

式全体が値を生成します。閉じ波括弧の後のセミコロンも代入文の一部です。

矢印ラベル->はswitch式専用ではなく、switch文でも使えます。「矢印があるから式」と決めつけず、switch全体がどこに置かれているかを確認します。

1つのcaseが値へ到達する経路を追う

矢印の右側が単一式なら、その式の値がswitch式の値になります。複数の文を実行するブロックでは、yieldで値を渡します。

int score = 75;
String rank = switch (score / 10) {
    case 10, 9 -> "A";
    case 8 -> "B";
    case 7 -> {
        System.out.println("border");
        yield "C";
    }
    default -> "D";
};

yieldはメソッドを終了するreturnではありません。現在のswitch式へ値を返し、そのswitch式の評価を終えます。

String value = switch (n) {
    case 1 -> {
        // return "one"; // このメソッド自体を終了するため用途が違う
        yield "one";
    }
    default -> throw new IllegalArgumentException();
};

throwの分岐は正常値を生成しませんが、必ず例外で終了するため、値を返す経路として不足とは扱われません。

矢印形式とコロン形式では流れが異なる

形式次のcaseへ進むかswitch式で値を返す方法
case 1 ->進まない右辺の式、ブロック内のyield、throw
case 1:明示的に終端しなければ進む最終的にyieldへ到達
int value = switch (n) {
    case 1:
        System.out.println("one");
        yield 10;
    case 2:
    case 3:
        yield 20;
    default:
        yield 0;
};

コロン形式のswitch式でbreak 10;は使いません。Java 17では値の返却にyield 10;を使います。

switch式はすべての入力を処理できなければならない

式は値を必要とするため、どの入力でも値または例外へ到達する必要があります。整数や文字列では通常defaultが必要です。

int result = switch (n) {
    case 1 -> 10;
    case 2 -> 20;
    // defaultがなく、他の整数で値が決まらないためエラー
};

enumの全定数をcaseで覆うなど、コンパイラが網羅的だと判断できる場合は、明示的なdefaultを省略できます。

enum Size { SMALL, LARGE }

int price = switch (size) {
    case SMALL -> 100;
    case LARGE -> 200;
};

「今ある定数を全部書いたように見えるか」ではなく、セレクタ型に対してswitch式が網羅的かを判断します。

セレクタ型とcaseラベルを別々に検査する

検査対象代表的に使用できるもの使用できない代表例
switchのセレクタbyte、short、char、intとラッパー、String、enumlong、float、double、boolean
caseラベルセレクタ型に適合する定数式、enum定数、文字列リテラル実行時に決まる変数、重複値
final int A = 1;
int b = 2;

switch (n) {
    case A -> System.out.println("A"); // 定数式なので可
    // case b -> ...                  // bは定数式ではない
}

finalが付いているだけでは定数変数になるとは限りません。初期値がコンパイル時定数式であることまで確認します。

分岐ごとの型からswitch式全体の型が決まる

var value = switch (n) {
    case 1 -> 10;
    default -> 2.5;
};

各分岐の値がintとdoubleなので、数値昇格を考慮した式全体の型はdoubleです。varは右辺のswitch式からその型を推論します。

Number number = switch (n) {
    case 1 -> Integer.valueOf(10);
    default -> Double.valueOf(2.5);
};

代入先がある場合は、その文脈に適合するかも確認します。caseを選ぶ処理と、式全体の型を決める処理は別です。

nullとプレビュー構文を混ぜない

通常のJava 17のswitchで、セレクタがnullなら実行時にNullPointerExceptionが発生します。

String text = null;
String result = switch (text) {
    case "A" -> "alpha";
    default -> "other";
}; // 実行時にNullPointerException

Java 17にはswitchのパターンマッチングも存在しますが、これはプレビュー機能です。通常のJava 17問題として、型パターンのcaseやcase nullを無条件に使用できると判断してはいけません。プレビュー有効化の明示がない問題では、正式機能のswitch式として読みます。

出力問題ではセレクタを一度だけ評価する

int x = 1;
int result = switch (x++) {
    case 1 -> x + 10;
    default -> 0;
};
System.out.println(x + ":" + result);

セレクタ式x++を評価した時点で、照合に使う値は1、変数xは2です。その後case 1が選ばれ、右辺のx + 10は12になります。出力は2:12です。caseを追う前にセレクタ式の副作用を確定します。

switch式の読解順

  1. switch全体が文か式かを確認する。
  2. セレクタ式を一度評価し、副作用後の値を記録する。
  3. セレクタ型がswitchに使用可能か確認する。
  4. caseラベルが定数で、重複していないか確認する。
  5. 選ばれるcaseを決める。
  6. 矢印なら右辺、ブロックならyield、コロンなら到達するyieldまで追う。
  7. 全分岐の網羅性と、生成される値の型を確認する。
