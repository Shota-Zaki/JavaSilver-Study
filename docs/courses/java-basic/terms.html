Java Basic Terms

Java基礎 用語と定義

Java基礎レッスンで使う用語を、コード上の役割とJavaの規則に結び付けて確認するための参照ページです。

基礎解説トップへ戻る準備から読む

この用語集の読み方

1. わからない用語を確認する

本文で意味が分からない言葉だけ、この一覧で確認します。

2. コード例に戻る

定義を読んだら、元のレッスンへ戻って使われ方を確認します。

3. 似た言葉を分ける

JDKとJVM、引数と戻り値など、役割が違う言葉を混同しないようにします。

4. 確認問題で使う

用語を説明するだけでなく、コンパイル可否や出力の判断に使います。

レッスン別の定義

各レッスンで使う主な用語です。詳しい意味とコード例は本文で確認できます。

準備

JDK
Javaを開発するための道具一式。javacとjavaを含む。
エディタ
ソースコードを書くための道具。
ターミナル
コマンドを入力して操作する画面。
作業フォルダ
Javaファイルを置き、コマンドを実行する基準の場所。

Lesson 1

ソースコード
人間が書く.javaのプログラム。
バイトコード
javacが作る.classの中身で、JVMが実行する中間命令です。
機械語
CPUが直接実行できる命令。
JVM
Javaの.classを実行する仮想マシン。

Lesson 2

OS
WindowsやmacOSなどの基本ソフト。
JRE
Javaを実行する環境を指す言葉です。学習ではJDKを用意すれば、実行に必要な部品もそろいます。
PATH
コマンド名だけで実行ファイルを探すための設定。
クラスパス
Javaがクラスを探す場所。PATHとは別物。

Lesson 3

コンパイル
ソースコードを検査し、.classへ変換すること。
実行
JVMが.classを読み込んで処理を動かすこと。
classファイル
コンパイル後に生成される実行対象ファイル。

Lesson 4

mainメソッド
Javaアプリケーションの起動入口。
public
外部からアクセス可能にする修飾子。
static
インスタンス生成なしでクラス側から扱える修飾子。
void
戻り値がないことを表す戻り値型。

Lesson 5

変数
値を保持する名前付きの場所。
値
計算や表示の対象になるデータ。
型
値の扱い方を決める分類。
リテラル
コードに直接書いた値。

Lesson 6

キャスト
明示的な型変換。
拡大変換
より広い範囲の型への変換。
縮小変換
より狭い範囲の型への変換。
数値昇格
演算時にbyteやshortなどがint以上へ変わること。

Lesson 7

String
文字列を扱うクラス。
文字列プール
文字列リテラルを再利用する領域。
不変
一度作った中身が変わらない性質。
null
参照先がないことを表す特別な値。

Lesson 8

条件式
trueまたはfalseになる式。
boolean
true/falseだけを扱う型。
短絡評価
左側だけで結果が決まる場合に右側を評価しない仕組み。

Lesson 9

ループ
処理を繰り返す構造。
break
繰り返しやswitchを抜ける命令。
continue
今回の繰り返しの残りを飛ばして次へ進む命令。

Lesson 10

配列
同じ型の値を複数並べる固定長の入れ物。
要素
配列の中身1つ。
添字
要素の位置を表す番号。0から始まる。

Lesson 11

メソッド
処理をまとめた部品。
引数
メソッドへ渡す情報。
戻り値
メソッドが呼び出し元へ返す結果。
値渡し
値のコピーを渡す方式。

Lesson 12

クラス
オブジェクトを作る設計図。
インスタンス
クラスからnewで作られた実体。
メンバ
クラス内に定義するフィールドやメソッドなど。
フィールド
オブジェクトやクラスが持つ変数。

Lesson 13

クラスメンバ
staticが付いた、クラス側に属するメンバ。
インスタンスメンバ
newされた個体ごとに属するメンバ。
static文脈
特定のインスタンスがない状態のコード。

Lesson 14

コンストラクタ
new時に実行される初期化処理。
デフォルトコンストラクタ
コンストラクタ未定義時に自動追加される引数なしコンストラクタ。
this
現在のインスタンスを指す参照。
super
親クラス側を指すためのキーワード。

Lesson 15

継承
既存クラスを元に新しいクラスを作る仕組み。
スーパークラス
継承元の親クラス。
サブクラス
継承して作る子クラス。
ポリモーフィズム
同じ型で扱い、実体に応じて動作が変わる仕組み。

Lesson 16

interface
実装すべき扱い方を定める型。
implements
クラスがinterfaceを実装することを示すキーワード。
抽象メソッド
本体を持たず、実装先で処理を書くメソッド。
defaultメソッド
interface内に処理本体を持てるメソッド。

Lesson 17

例外
実行中の異常を表すオブジェクト。
try
例外が発生しうる処理を書くブロック。
catch
発生した例外を受け取って処理するブロック。
finally
例外有無に関係なく最後に実行したい処理を書くブロック。

Lesson 18

コンパイルエラー
実行前にコード検査で止まる失敗。
実行時例外
起動後の処理中に発生する失敗。
スタックトレース
実行時例外がどこから発生したかを示す履歴。

Lesson 19

トップレベルクラス
他のクラスの外側に直接定義するクラス。
パッケージ
クラスを分類し、名前の衝突を避ける仕組み。
完全修飾クラス名
パッケージ名を含めたクラス名。例: ex.Sample。

Lesson 20

評価順
式の中で何が先に処理されるか。
制御フロー
ifやforにより処理が進む順番。
スコープ
変数名などを使える範囲。

Lesson 21

ビット
0か1を表す最小単位。
バイト
8ビットのまとまりです。Javaのbyte型も8ビットです。
2進数
0と1で値を書く表記。
16進数
0〜9とA〜Fで値を書く表記。

Lesson 22

文字コード
文字と番号を対応させる仕組み。
Unicode
世界中の文字に番号を割り当てるための文字集合です。
char
16ビットのUTF-16コード単位を1つ持つプリミティブ型です。
文字リテラル
単一引用符で書くchar値です。

Lesson 23

スタック
学習上は、主にメソッド呼び出しやローカル変数を追うための作業領域として考えます。
ヒープ
学習上は、newされたオブジェクトや配列が置かれる領域として考えます。
ガベージコレクション
不要になったオブジェクトをJVMが回収する仕組み。

Lesson 24

javac
Javaソースをコンパイルするコマンド。
java
コンパイル済みクラスを実行するコマンド。
-d
classファイルの出力先を指定するjavacオプション。
-cp
クラスを探す場所を指定するオプション。
