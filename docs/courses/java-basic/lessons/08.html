Java基礎 Lesson 8

if文と条件式

if文の構文、boolean条件、比較演算、論理演算、短絡評価の規則を説明します。

このレッスンのゴール

条件式がbooleanでなければならない理由と、`&&`・`||`が右辺を評価しない条件を説明できること。

基本用語の定義

Javaのif文は数値を真偽値として扱いません。丸かっこ内の式は必ず`boolean`へ評価される必要があります。

条件式とはtrue または false になる式。
booleanとは真偽値だけを扱う型。true/false。
比較演算子とは>、==など、比較結果をbooleanで返す演算子。
短絡評価とは&&や||で右側の評価が省略されること。
ブロックとは{ }で囲んだ処理のまとまり。

if文と条件式の成立条件

項目定義・規則具体例
条件式評価結果がbooleanになる式。age >= 18
if文条件がtrueのときに文またはブロックを実行します。if (condition) { ... }
else直前の対応可能なifに結び付きます。波かっこで対応関係を明示できます。
比較演算子<、<=、>、>=、==、!=。結果型はbooleanです。
短絡AND&&は左辺がfalseなら右辺を評価しません。危険な処理を右辺で避けられます。
短絡OR||は左辺がtrueなら右辺を評価しません。右辺の副作用が実行されない場合があります。

ifの基本

if (条件式) {
    条件がtrueのときの処理
} else {
    条件がfalseのときの処理
}

条件式に書けるのはtrueまたはfalseになる式。if (1)やif (0)はJavaでは使えない。

比較演算子

演算子意味例
==等しいa == b
!=等しくないa != b
<小さいa < b
>大きいa > b
<=以下a <= b
>=以上a >= b
参照型の==

Stringなどの参照型で==を使うと、内容ではなく参照先が同じかを見る。

短絡評価

&&は左がfalseなら全体がfalseなので右を評価しない。||は左がtrueなら全体がtrueなので右を評価しない。

右側が実行されない例

int x = 0;
if (x > 0 && ++x > 0) {
    System.out.println("A");
}
System.out.println(x); // 0

&や|をbooleanに使うと短絡しない。右側も評価される。

代入と比較の混同

これはコンパイルできる

boolean flag = false;
if (flag = true) {
    System.out.println("run");
}

flag = trueは代入式で、結果としてtrueになるためifに書ける。比較のつもりならflag == trueだが、通常はif (flag)で十分。

条件式と短絡評価の最小例

`Main.java`として保存する完全なコード

public class Main {
    public static void main(String[] args) {
        String text = null;
        if (text != null && text.length() > 0) {
            System.out.println(text);
        } else {
            System.out.println("empty");
        }
    }
}

左辺text != nullがfalseなので、右辺text.length() > 0は評価されません。そのため、このコードではNullPointerExceptionは発生しません。

関連する用語との違い

混同しやすいもの違い見分け方
= / ===は代入、==は比較。条件式内で代入していないか見る。
&& / &&&は短絡評価あり。&は右辺も評価する。右辺に副作用があると差が出る。
数値 / booleanJavaでは数値を真偽値として使えない。if (x)はxがbooleanでなければ不可。

条件分岐を評価する規則

  1. 条件式全体がboolean型か確認する。
  2. 演算子の優先順位に従って式をグループ化する。
  3. オペランドを左から評価する。
  4. &&または||なら短絡条件を適用する。
  5. 最終結果がtrueの分岐だけを実行する。

確認用コード

int a = 10;
int b = 0;
if (a < 5 && ++b > 0) {
    a++;
}
System.out.println(a + ":" + b);
見る点このコードでの判断結論
左辺a < 5はfalse&&なので右辺を評価しない
++b実行されないbは0のまま
出力aは10、bは010:0

ifが受け取るのはboolean

Javaのif文では、条件式の結果が必ずbooleanでなければなりません。0や1を自動的にfalseやtrueとして扱うことはありません。

int n = 1;
// if (n) { }      // コンパイルエラー
if (n == 1) {      // OK。比較結果はboolean
    System.out.println("ok");
}

もう1つ重要なのが短絡評価です。&&は左がfalseなら右を評価しません。||は左がtrueなら右を評価しません。これは単なる高速化ではなく、例外を避けるためにも使われます。

String s = null;
if (s != null && s.length() > 0) {
    System.out.println(s);
}

上のコードは、左側のs != nullがfalseなので、右側のs.length()は実行されません。逆に&を使うと右側まで評価され、NullPointerExceptionになります。

演算子右辺評価主な使いどころ
&&必要なときだけ評価通常の条件分岐
||必要なときだけ評価どちらか満たせばよい条件
& / |基本的に両辺を評価booleanでは意図がある場合だけ使う
条件式で見る順番

まず型がbooleanになるか判定し、その後に左から評価順序を追います。見た目の自然言語ではなく、Javaの評価順序で読むと整理しやすいです。

定義と規則の整理

ifと条件式の誤解

つまずき正しい考え方
Javaでも0や1を真偽値にできると思うJavaのif条件はbooleanだけ。数値とは互換性がない。
=と==を混同する=は代入、==は比較です。boolean変数への代入式もboolean値になるため、条件式の中でflag = trueと書くと比較ではなく代入が実行されます。
elseの対応先を感覚で読むelseは直前の未対応のifに結びつく。

確認できれば次へ

  • if条件に置ける型を答えられる
  • 代入式が値を持つことを説明できる
  • dangling elseを説明できる
理解の基準

成立するコードと成立しないコード

成立する例：boolean変数で判定する

boolean passed = true;
if (passed) {
    System.out.println("pass");
}

成立しない例：intを条件式にしている

int score = 80;
if (score) {
    System.out.println("pass");
}

成立する形：比較式でbooleanを作る

int score = 80;
if (score >= 60) {
    System.out.println("pass");
}
確認ポイント

Javaのif条件はbooleanだけです。数値をそのまま条件にはできません。

定義と規則を確認する

用語の確認

条件式に必要な型は何ですか。

説明を見る

Javaのif、while、forの条件式はbooleanです。数値の0/1を真偽値として扱いません。

コードの確認

次のコードはコンパイルできますか。

int n = 1;
if (n) {
    System.out.println("A");
}
説明を見る

コンパイルできません。ifの条件式はbooleanが必要です。n == 1 のように比較式にします。

仕組みを説明する

if (a = true) がコンパイルできる場合がある理由を説明してください。

説明を見る

aがboolean変数なら、代入式a = true自体の値もbooleanです。そのため条件式として成立します。ただし比較ではなく代入です。

基本ルールのまとめ

  • ifの条件式がbooleanになるか確認できる。
  • =と==を区別できる。
  • &&と||の短絡評価で右辺が実行されるか判断できる。

コードに規則を当てはめる

次のコードの出力を答える。

確認用コード

int x = 0;
if (true || ++x > 0) {
    System.out.println(x);
}
説明を見る

出力は0。

||は左辺がtrueなら右辺を評価しない。したがって++xは実行されず、xは0のまま。
