Java基礎 Lesson 1

コンピュータと機械語

ソースコード、バイトコード、機械語を区別し、Javaが実行されるまでの変換を説明します。

このレッスンのゴール

Javaのソースコードがclassファイルへ変換され、JVM上で実行される関係を説明できること。

基本用語の定義

「コンピュータが直接読む命令」と「Javaが生成するバイトコード」は同じものではありません。

ソースコードとは.java に書く人間向けのプログラム。
機械語とはCPUが直接実行できる命令。Java文法そのものではない。
バイトコードとはjavac が作る .class の中身。JVMが実行する。
JVMとはJava Virtual Machine。バイトコードを実行する仮想的な実行環境。

ソースコード・バイトコード・機械語の関係

項目定義・規則具体例
ソースコードJavaの文法で人が記述したテキスト。.javaファイルに保存します。
コンパイルソースコードを検査し、Javaバイトコードへ変換する処理。javacが担当します。
バイトコードJVMが解釈・実行する命令形式。特定CPUの機械語そのものではありません。.classファイルに格納されます。
JVMclassファイルを読み、実行環境に合わせて命令を処理する仮想マシン。OSとCPUに対応したJVMが必要です。
機械語CPUが直接実行する命令。CPUの種類に依存します。JVMの実装が最終的な実行を仲介します。
移植性同じclassファイルを、互換性のあるJVMがある環境で実行できる性質。JVMなしで実行できるという意味ではありません。

機械語を難しく考えすぎない

機械語は、足し算する、値を比較する、メモリから値を読む、条件によって次に進む場所を変える、という単純な命令の集合だと読みればよい。Javaのif、for、メソッド呼び出しも、最終的にはこうした単純な処理の組み合わせとして実行される。

人間が見るもの機械側で起きること学習上の意味
int a = 10;値を入れる領域を用意し、10を保持する変数は「名前付きの場所」として考える
if (a > 5)値を比較し、条件により進む先を変える分岐は「条件判定＋ジャンプ」と見る
method()別の処理単位へ移動し、終わったら戻るメソッドは処理のまとまり

Javaの場合の実行イメージ

人間が書くソースコード

public class Main {
    public static void main(String[] args) {
        System.out.println("Hello");
    }
}

このコードは、いきなりCPUで実行されるのではない。まずjavac Main.javaでMain.classのようなバイトコードへ変換される。その後、java MainでJVMがクラスを読み込み、mainメソッドから処理を開始する。

  1. .javaを書く。
  2. javacが文法や型を検査し、.classを作る。
  3. javaコマンドがJVMを起動する。
  4. JVMがmainを探して実行する。

悪い理解と修正

誤解なぜ成立しないか正しい見方
CPUがJavaの文法を直接読むCPUはpublicやclassをJava文法として理解しないJavaコードはJVMが実行できる形へ変換される
コンパイルできれば必ず正しく動くコンパイルは主に文法・型の検査であり、実行中の値までは保証しない実行時例外は実行して初めて起きる
エラーは全部同じコンパイルエラーと実行時例外は発生タイミングが違うまず「いつ失敗したか」を見る

変換される対象をコードで確認する

`Main.java`として保存する完全なコード

public class Main {
    public static void main(String[] args) {
        int left = 2;
        int right = 3;
        System.out.println(left + right);
    }
}

変数名や+はソースコードの表現です。javacはこの構造を検査してバイトコードを生成し、JVMが実行します。

関連する用語との違い

混同しやすいもの違い見分け方
ソースコード / 機械語ソースコードは人間が読む文章。機械語はCPUが実行する命令。.javaを直接CPUが読むわけではない。
コンパイル時 / 実行時コンパイル時は文法・型の検査。実行時は実際の値を使って処理が進む。「いつ失敗したか」を先に見る。
文法上正しい / 意図通り動く文法上正しくても、ロジックが間違っていれば意図通りにはならない。出力結果や変数の変化を追う。

Javaプログラムが実行される順序

  1. 人が.javaファイルへソースコードを書く。
  2. javacが構文、型、アクセスなどを検査する。
  3. 検査に通れば.classファイルが生成される。
  4. javaがJVMを起動し、対象クラスを読み込む。
  5. JVMがバイトコードを実行環境に合わせて処理する。

確認用コード

int a = 3;
int b = a + 2;
System.out.println(b);
見る点このコードでの判断結論
文法int変数に整数を代入しているコンパイル可能
実行時の値aは3、bは5出力は5
失敗の種類文法エラーでも例外でもない正常終了

Javaコードが実行されるまで

JavaのソースファイルをCPUが直接読むわけではありません。javacがソースコードをバイトコードへ変換し、JVMがそのバイトコードを解釈・最適化しながら実行します。最終的にCPUが処理するのは機械語の命令です。

段階見るものここで起きること
人間が書くMain.javaJavaの文法で処理を書く。CPUが直接読むものではない。
変換するMain.classjavacが文法・型を検査し、バイトコードを作る。
実行するJVM上の処理JVMがバイトコードを解釈・最適化し、最終的にCPUが実行できる命令として処理します。

重要なのは、機械語やCPUの細部を暗記することではありません。
Javaコードを読むときに「今考えているのはコンパイル前か、実行中か」を分けることです。
この分離ができると、エラー原因の切り分けが速くなります。

int x = 10;
int y = x + 5;
System.out.println(y);

この3行を見るときは、まずJava文法として成立するかを判定し、次に実行時にxとyがどう変化するかを追います。いきなり「画面に何が出るか」だけを考えると、途中の判断を飛ばす癖がつきます。

ここで整理する誤解

「Javaを覚える＝命令を暗記だけする」ではありません。実際には、書いたコードがどの段階で検査され、どの段階で値が動くのかを分けて読めることが基礎力です。

定義と規則の整理

コンピュータの話で混同しやすい点

つまずき正しい考え方
プログラムを「そのままCPUが読んでいる」と読むCPUが直接扱うのは機械語。Javaコードは変換と実行基盤を経て動く。
高級言語と機械語を対立だけで覚える高級言語は人間が読み書きしやすい表現。最終的には機械が処理できる形になる。
Javaだけ特別にOSなしで動くと思うJVMもOS上で動くプログラム。JavaはJVMによりOS差を吸収しやすいだけ。

確認できれば次へ

  • 機械語とは何かを一文で説明できる
  • JavaソースコードがCPUに直接読まれるわけではない理由を説明できる
  • OSとJVMの位置関係を説明できる
理解の基準

成立するコードと成立しないコード

成立する例：Javaソースをclassファイルへ変換して実行する

// Main.java
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello");
    }
}

// terminal
javac Main.java
java Main

成立しない例：ソースファイルをclassファイルとして実行している

// Main.javaだけがあり、Main.classがない状態
java Main

成立する形：javacでバイトコードを作ってからjavaで実行する

javac Main.java
java Main
確認ポイント

Javaではソースコードを直接CPUが読むわけではありません。.classファイルとJVMの役割を分けます。

定義と規則を確認する

用語の確認

ソースコード、バイトコード、機械語の違いを説明できますか。

説明を見る

ソースコードは人間が書く文字列、バイトコードはjavac後の.class、機械語はCPUが直接実行する命令です。JavaはJVMがバイトコードを実行します。

コードの確認

次の実行は正しいですか。

javac Main.java
java Main.class
説明を見る

コンパイルは正しいですが、実行が誤りです。javaは.classファイル名ではなくクラス名を指定するため、java Mainです。

仕組みを説明する

なぜJavaは.javaをそのまま実行するのではなく、通常は.classにしてから実行しますか。

説明を見る

Javaの基本的な実行手順は、ソースを javac でバイトコードへ変換し、そのclassファイルをJVMで動かす流れです。Java 11以降は単一のソースファイルを java Main.java で簡易実行できますが、このコースではコンパイルと実行を分けて理解するため、javac と java を別々に使います。

基本ルールのまとめ

  • 人が書くソースコードと、JVMが実行するバイトコードを分けられる。
  • コンパイルと実行が別の段階だと説明できる。
  • コンパイル成功と、意図どおりに動くことは別だと理解している。

コードに規則を当てはめる

次の説明は正しいか。「JavaのSystem.out.printlnはCPUが直接理解して画面へ表示している」。

説明を見る

誤り。CPUがJava文法を直接理解しているわけではない。

人間が書いたJavaコードはバイトコードへ変換され、JVMが実行する。最終的にはOSやCPUの機能を使うが、Java文法をCPUがそのまま読んでいるわけではない。
