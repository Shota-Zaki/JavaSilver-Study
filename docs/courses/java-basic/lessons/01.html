Java基礎 Lesson 1

Javaのコードが動くまで

Javaのコードは、そのままCPUに渡されるわけではありません。ソースコード、classファイル、機械語の順で役割を分けて見ます。

ここまでできればOK

ソースコードから実行までの流れを、自分の言葉で説明できるようになります。

まず押さえる言葉

「コンピュータが直接読む命令」と「Javaが生成するバイトコード」は同じものではありません。

ソースコードとは.java に書く人間向けのプログラム。
機械語とはCPUが直接実行できる命令。Java文法そのものではない。
バイトコードとはjavac が作る .class の中身。JVMが実行する。
JVMとはJava Virtual Machine。バイトコードを実行する仮想的な実行環境。

ソースコード・バイトコード・機械語の関係

項目意味・使い方具体例
ソースコードJavaの文法で人が記述したテキスト。.javaファイルに保存します。
コンパイルソースコードを検査し、Javaバイトコードへ変換する処理。javacが担当します。
バイトコードJVMが解釈・実行する命令形式。特定CPUの機械語そのものではありません。.classファイルに格納されます。
JVMclassファイルを読み、実行環境に合わせて命令を処理する仮想マシン。OSとCPUに対応したJVMが必要です。
機械語CPUが直接実行する命令。CPUの種類に依存します。JVMの実装が最終的な実行を仲介します。
移植性同じclassファイルを、互換性のあるJVMがある環境で実行できる性質。JVMなしで実行できるという意味ではありません。

機械語を難しく考えすぎない

機械語は、足し算する、値を比較する、メモリから値を読む、条件によって次に進む場所を変える、という単純な命令の集合だと読みればよい。Javaのif、for、メソッド呼び出しも、最終的にはこうした単純な処理の組み合わせとして実行される。

人間が見るもの機械側で起きること学習上の意味
int a = 10;値を入れる領域を用意し、10を保持する変数は「名前付きの場所」として考える
if (a > 5)値を比較し、条件により進む先を変える分岐は「条件確認＋ジャンプ」と見る
method()別の処理単位へ移動し、終わったら戻るメソッドは処理のまとまり

Javaの場合の実行イメージ

人間が書くソースコード

public class Main {
    public static void main(String[] args) {
        System.out.println("Hello");
    }
}

このコードは、いきなりCPUで実行されるのではない。まずjavac Main.javaでMain.classのようなバイトコードへ変換される。その後、java MainでJVMがクラスを読み込み、mainメソッドから処理を開始する。

  1. .javaを書く。
  2. javacが文法や型を検査し、.classを作る。
  3. javaコマンドがJVMを起動する。
  4. JVMがmainを探して実行する。

よくある勘違い

誤解なぜ成立しないか正しい見方
CPUがJavaの文法を直接読むCPUはpublicやclassをJava文法として理解しないJavaコードはJVMが実行できる形へ変換される
コンパイルできれば必ず正しく動くコンパイルは主に文法・型の検査であり、実行中の値までは保証しない実行時例外は実行して初めて起きる
エラーは全部同じコンパイルエラーと実行時例外は発生タイミングが違うまず「いつ失敗したか」を見る

変換される対象をコードで確認する

`Main.java`として保存する完全なコード

public class Main {
    public static void main(String[] args) {
        int left = 2;
        int right = 3;
        System.out.println(left + right);
    }
}

変数名や+はソースコードの表現です。javacはこの構造を検査してバイトコードを生成し、JVMが実行します。

似た言葉との違い

混同しやすいもの違い見分け方
ソースコード / 機械語ソースコードは人間が読む文章。機械語はCPUが実行する命令。.javaを直接CPUが読むわけではない。
コンパイル時 / 実行時コンパイル時は文法・型の検査。実行時は実際の値を使って処理が進む。「いつ失敗したか」を先に見る。
文法上正しい / 意図通り動く文法上正しくても、ロジックが間違っていれば意図通りにはならない。出力結果や変数の変化を追う。

Javaプログラムが実行される順序

  1. 人が.javaファイルへソースコードを書く。
  2. javacが構文、型、アクセスなどを検査する。
  3. 検査に通れば.classファイルが生成される。
  4. javaがJVMを起動し、対象クラスを読み込む。
  5. JVMがバイトコードを実行環境に合わせて処理する。

確認用コード

int a = 3;
int b = a + 2;
System.out.println(b);
見る点このコードでの判断結論
文法int変数に整数を代入しているコンパイル可能
実行時の値aは3、bは5出力は5
失敗の種類文法エラーでも例外でもない正常終了

ここまでの要点

  • 人が書くソースコードと、JVMが実行するバイトコードを分けられる。
  • コンパイルと実行が別の段階だと説明できる。
  • コンパイル成功と、意図どおりに動くことは別だと理解している。
