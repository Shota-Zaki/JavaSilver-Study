Java基礎 Lesson 13

static

staticメンバをクラスに属するメンバとして定義し、インスタンスメンバとのアクセス規則を説明します。

このレッスンのゴール

staticフィールドの共有範囲と、staticメソッドから直接参照できるメンバを説明できること。

基本用語の定義

`static`が付いたメンバは特定のインスタンスではなくクラスに関連付けられます。

staticとはインスタンスではなくクラスに属することを示す修飾子。
クラスメンバとはstaticなフィールドやメソッド。クラス名から使える。
インスタンスメンバとはnewされた各インスタンスに属するフィールドやメソッド。
static文脈とはstaticメソッドやstatic初期化ブロックの中。インスタンスなしで実行される。
共有とはstaticフィールドが全インスタンスで同じ1つの値を持つこと。

staticメンバとインスタンスメンバの規則

項目定義・規則具体例
staticフィールドクラスに1つ用意され、同じクラスの全インスタンスから共有されます。static int count;
インスタンスフィールド各インスタンスが個別に持つ状態。String name;
staticメソッドインスタンスを用意せず呼び出せるメソッド。ClassName.method()
static文脈thisやsuperが存在しない実行文脈。インスタンスメンバを名前だけで直接参照できません。
クラス初期化staticフィールド初期化子とstatic初期化ブロックをソース順に実行します。通常、最初の能動的使用時に行われます。
推奨呼び出しstaticメンバはクラス名で修飾すると所属が明確です。Counter.count

staticフィールド

class Counter {
    static int total;
    int value;
}

Counter a = new Counter();
Counter b = new Counter();
a.value = 1;
b.value = 2;
Counter.total = 100;

valueはインスタンスごとに別。totalはクラス側に1つ。

staticメソッド

staticメソッドはインスタンスなしで呼べる。その代わり、インスタンスフィールドやインスタンスメソッドを直接使えない。

成立しない例

class Sample {
    int value = 10;

    static void print() {
        System.out.println(value); // コンパイルエラー
    }
}

修正例

class Sample {
    int value = 10;

    static void print() {
        Sample s = new Sample();
        System.out.println(s.value);
    }
}

呼び出し方

staticメンバーはクラス名で呼ぶのが基本。インスタンス変数からも呼べる場合があるが、読み手を混乱させるため避けるべき。

Math.max(3, 5);
Integer.parseInt("123");
System.out.println("OK");

static初期化の考え方

staticフィールドはクラスが使われるタイミングで準備される。インスタンス生成とは別軸で考える。
初心者段階では、まず「staticはクラス側」「非staticはインスタンス側」と切るだけでよい。

共有されるstaticフィールド

`Main.java`として保存する完全なコード

class Counter {
    static int total;
    int personal;

    void increment() {
        total++;
        personal++;
    }
}

public class Main {
    public static void main(String[] args) {
        Counter a = new Counter();
        Counter b = new Counter();
        a.increment();
        b.increment();
        System.out.println(Counter.total);
        System.out.println(a.personal);
    }
}

totalは2つのインスタンスから共有されるため2、a.personalはa固有の値なので1です。

関連する用語との違い

混同しやすいもの違い見分け方
クラス側 / インスタンス側staticはクラス側。非staticはインスタンス側。static修飾子の有無を見る。
共有 / 個別staticフィールドは共有。インスタンスフィールドは個別。newを複数回したときに値が分かれるか。
直接参照 / インスタンス経由staticから非staticを直接参照できない。対象オブジェクトが指定されているか見る。

staticアクセスを判断する規則

  1. 対象メンバにstaticが付いているか確認する。
  2. staticメンバならクラス単位、付いていなければインスタンス単位で状態を考える。
  3. 現在のコードがstatic文脈か確認する。
  4. static文脈からインスタンスメンバを使う場合は、対象インスタンスの参照を用意する。
  5. static初期化の順序は、同じクラス内では原則としてソース上の記述順に追う。

確認用コード

class Counter {
    static int total;
    int count;
}

Counter a = new Counter();
Counter b = new Counter();
a.total++;
a.count++;
System.out.println(b.total + ":" + b.count);
見る点このコードでの判断結論
totalstaticフィールドaとbで共有
countインスタンスフィールド各インスタンスで別
出力b.totalは1、b.countは01:0

staticはクラスに属する

staticが付いたメンバは、特定のインスタンスではなくクラスに属します。インスタンスメンバを使うには、どのインスタンスを対象にするかを示す必要があります。

class Counter {
    int instanceCount = 0;
    static int classCount = 0;

    void add() {
        instanceCount++;
        classCount++;
    }
}

instanceCountはインスタンスごとに別々です。classCountはクラス側に1つです。複数のCounterインスタンスを作っても、classCountは共有されます。

場所使えるもの理由
インスタンスメソッドインスタンスメンバ、staticメンバ対象インスタンスがあるため
staticメソッドstaticメンバ特定のインスタンスがないため
main基本はstatic側main自体がstaticだから
class Sample {
    int value = 10;

    static void print() {
        // System.out.println(value); // コンパイルエラー
        Sample s = new Sample();
        System.out.println(s.value); // OK
    }
}
staticの判断基準

「オブジェクトごとに違う状態か？」を先に考えています。違うならインスタンス側、全体で1つならstatic側です。

定義と規則の整理

staticで混同しやすい点

つまずき正しい考え方
staticを「速い」という意味で覚えるstaticはインスタンスに属さずクラス側に属するという意味で読む。
staticメソッドからインスタンスフィールドを直接使うインスタンスが特定できないため直接参照できない。
staticフィールドをインスタンスごとの値と思うstaticフィールドはクラス側で共有される。

確認できれば次へ

  • staticメンバとインスタンスメンバの違いを説明できる
  • mainがstaticである理由を説明できる
  • static文脈で直接使えないものを答えられる
理解の基準

成立するコードと成立しないコード

成立する例：インスタンス文脈からインスタンス変数を使う

class Counter {
    int count;
    void up() { count++; }
}

成立しない例：staticメソッドからインスタンス変数を直接使っている

class Counter {
    int count;
    static void up() { count++; }
}

成立する形：staticメソッドで使う変数もstaticにする

class Counter {
    static int count;
    static void up() { count++; }
}
確認ポイント

staticメソッドからインスタンスフィールドへ直接アクセスすることはできません。

定義と規則を確認する

用語の確認

staticメンバとインスタンスメンバを説明できますか。

説明を見る

staticメンバはクラスに属します。インスタンスメンバはnewされた各インスタンスに属します。

コードの確認

次のコードはコンパイルできますか。

class Sample {
    int value = 10;
    static void test() {
        System.out.println(value);
    }
}
説明を見る

コンパイルできません。staticメソッドからインスタンスフィールドvalueを直接参照できません。

仕組みを説明する

mainがstaticである理由を説明してください。

説明を見る

javaコマンドはインスタンスを作る前に入口を呼び出します。そのためmainはクラスから直接呼べるstaticである必要があります。

基本ルールのまとめ

  • staticメンバとインスタンスメンバを区別できる。
  • staticメソッドからインスタンスフィールドを直接使えない理由を説明できる。
  • 複数のインスタンスで共有される値を判断できる。

コードに規則を当てはめる

次のコードがコンパイルエラーになる理由を説明する。

確認用コード

public class Main {
    int value = 10;
    static void print() {
        System.out.println(value);
    }
}
説明を見る

printはstaticメソッドで、インスタンスなしで呼ばれる。

valueはインスタンスフィールドなので、どのインスタンスのvalueか決まらない。Main m = new Main();のように対象を作ってm.valueとする必要がある。
