Java基礎 Lesson 11

メソッドと引数

メソッド宣言、仮引数、実引数、戻り値、シグネチャ、値渡しの規則を説明します。

このレッスンのゴール

メソッドの宣言と呼び出しを分け、引数として何がコピーされるかを説明できること。

基本用語の定義

メソッドは名前付きの処理単位です。呼び出し時には実引数の値が評価され、仮引数へコピーされます。

static int add(int a, int b) {
    return a + b;
}

int result = add(3, 4);
メソッドとは名前を付けた処理のまとまり。呼び出されると本体の処理が実行される。
引数とはメソッドへ渡す情報。呼び出し側の値を、定義側の仮引数で受け取る。
戻り値とはメソッドが呼び出し元へ返す結果。voidなら戻り値はない。
シグネチャとはメソッド名と引数リストの組み合わせ。オーバーロード判定で重要。
値渡しとは変数の中身のコピーを渡すこと。参照型でも参照のコピーが渡る。

メソッド宣言と呼び出しの規則

項目定義・規則具体例
戻り値型呼び出し元へ返す値の型。値を返さない場合はvoidです。int、String、voidなど。
メソッド名呼び出しで指定する識別子。add、printなど。
仮引数宣言側で受け取るローカル変数。int left
実引数呼び出し側で渡す式。add(2, 3)の2と3。
シグネチャメソッド名と仮引数型の並び。戻り値型だけを変えてもオーバーロードできません。
値渡し実引数の値を仮引数へコピーする方式。参照型では参照の値がコピーされます。

メソッドの部品

戻り値ありのメソッド

public static int add(int a, int b) {
    return a + b;
}
部分意味
publicアクセス範囲
staticクラスから呼べる
int戻り値の型
addメソッド名
int a, int b引数
return値を返してメソッドを終える

値渡しを正しく理解する

プリミティブ型を渡すと、値のコピーが渡る。メソッド内で引数を書き換えても、呼び出し元の変数は変わらない。

static void change(int x) {
    x = 99;
}

public static void main(String[] args) {
    int n = 10;
    change(n);
    System.out.println(n); // 10
}

参照型でも値渡し

参照型を渡すと、参照のコピーが渡る。同じオブジェクトを指しているので中身の変更は見える。ただし、引数変数そのものに別オブジェクトを代入しても呼び出し元は変わらない。

中身を変更する例

static void change(int[] x) {
    x[0] = 99;
}

public static void main(String[] args) {
    int[] a = {10};
    change(a);
    System.out.println(a[0]); // 99
}

参照変数を付け替えるだけの例

static void replace(int[] x) {
    x = new int[] {99};
}

public static void main(String[] args) {
    int[] a = {10};
    replace(a);
    System.out.println(a[0]); // 10
}

戻り値の注意

戻り値型がvoidでないメソッドは、値を返す必要がある。条件分岐がある場合は、すべての経路で戻り値があるかを見る。

static int sign(int n) {
    if (n > 0) {
        return 1;
    }
    // n <= 0 のとき return がないのでコンパイルエラー
}

宣言・呼び出し・戻り値の最小例

`Main.java`として保存する完全なコード

public class Main {
    static int add(int left, int right) {
        return left + right;
    }

    public static void main(String[] args) {
        int result = add(2, 3);
        System.out.println(result);
    }
}

add(2, 3)を呼ぶ前に実引数を左から評価し、値2と3を仮引数leftとrightへコピーします。

関連する用語との違い

混同しやすいもの違い見分け方
実引数 / 仮引数呼び出し側に書く値が実引数。受け取る側の変数が仮引数。呼び出し行とメソッド定義を対応させる。
オブジェクト変更 / 参照の付け替え参照先の中身変更は見える。仮引数への再代入は呼び出し元変数を変えない。p.name=...とp=new ...を区別。
戻り値 / 出力戻り値は呼び出し元へ返す値。出力は画面に表示する処理。returnとprintlnは別物。

メソッド呼び出しを処理する規則

  1. 呼び出し対象のメソッドを名前と引数型から決める。
  2. 実引数を左から評価する。
  3. 評価した値を仮引数へコピーする。
  4. メソッド本体を実行する。
  5. returnがあれば式を評価して呼び出し元へ値を返す。

確認用コード

static void change(int x) {
    x = 20;
}

public static void main(String[] args) {
    int a = 10;
    change(a);
    System.out.println(a);
}
見る点このコードでの判断結論
渡されるものaの値10のコピーxは別変数
メソッド内x = 20aは変わらない
出力aは1010

Javaの引数はすべて値渡し

メソッドへ渡されるのは、変数そのものではなく、その変数に入っている値のコピーです。参照型の場合も、コピーされるのは参照値です。

static void change(int x) {
    x = 20;
}

public static void main(String[] args) {
    int n = 10;
    change(n);
    System.out.println(n); // 10
}

Javaは値渡しです。change(n)では、nそのものが移動するのではなく、nに入っている値10がxへコピーされます。メソッド内でxを変更しても、呼び出し側のnは変わりません。

参照型でも「参照の値」がコピーされます。
オブジェクトの中身を変更すれば呼び出し側からも見えますが、仮引数そのものを別オブジェクトへ代入しても、呼び出し側の変数は差し替わりません。

static void edit(int[] data) {
    data[0] = 99;          // 配列の中身を変更
    data = new int[]{1};   // 仮引数dataの参照を差し替えるだけ
}

public static void main(String[] args) {
    int[] nums = {10, 20};
    edit(nums);
    System.out.println(nums[0]); // 99
}
ここで整理する誤解

「参照型は参照渡し」は不正確です。Javaは常に値渡しです。参照型の場合は、オブジェクトへの参照という値がコピーされます。

定義と規則の整理

メソッドで混同する点

つまずき正しい考え方
引数と戻り値を同じものとして扱う引数は呼び出し側から渡す値。戻り値はメソッドから返す値。
仮引数を書き換えると呼び出し元の変数も必ず変わると思うJavaは値渡し。参照型でも「参照値」がコピーされる。
voidでも値を返せると思うvoidは戻り値なし。return;は可能だが値は返せない。

確認できれば次へ

  • 仮引数と実引数を区別できる
  • 戻り値型とreturn文の関係を説明できる
  • Javaの値渡しを一文で説明できる
理解の基準

成立するコードと成立しないコード

成立する例：戻り値を使って出力する

static int add(int a, int b) {
    return a + b;
}
System.out.println(add(1, 2));

成立しない例：戻り値intなのに値を返していない

static int add(int a, int b) {
    System.out.println(a + b);
}

成立する形：intの値をreturnする

static int add(int a, int b) {
    return a + b;
}
確認ポイント

戻り値型がvoidでないメソッドは、対応する型の値をreturnする必要があります。

定義と規則を確認する

用語の確認

メソッド、引数、戻り値を説明できますか。

説明を見る

メソッドは処理のまとまり、引数は呼び出し時に渡す値、戻り値はメソッドから返す値です。

コードの確認

次のコードはコンパイルできますか。

static int add(int a, int b) {
    a + b;
}
説明を見る

コンパイルできません。戻り値型がintなのでreturnでint値を返す必要があります。

仕組みを説明する

voidメソッドでreturn;を書ける理由を説明してください。

説明を見る

voidは値を返さないメソッドです。return; はメソッドをそこで終了するだけで、値を返しません。

基本ルールのまとめ

  • 実引数と仮引数を対応させられる。
  • 戻り値型とreturnする値の型を確認できる。
  • 参照型でも参照値のコピーが渡ると説明できる。

コードに規則を当てはめる

次の出力を答える。

確認用コード

static int add(int x) {
    x = x + 5;
    return x;
}

public static void main(String[] args) {
    int n = 10;
    add(n);
    System.out.println(n);
}
説明を見る

出力は10。

add(n)は15を返すが、戻り値を変数に代入していない。n自体は変わらない。
