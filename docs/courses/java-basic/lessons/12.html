Java基礎 Lesson 12

クラスとオブジェクト

クラス、オブジェクト、インスタンス、参照、フィールド、メソッド、thisを定義します。

このレッスンのゴール

クラス定義と生成されたインスタンスを区別し、複数の参照が同じオブジェクトを指す状態を説明できること。

基本用語の定義

クラスは型と構造の定義です。`new`で生成される個別の実体がオブジェクトまたはインスタンスです。

クラスとはオブジェクトを作るための設計図。フィールドやメソッドを定義する。
オブジェクトとはプログラム上で扱う実体。Javaでは多くの場合、newで作る。
インスタンスとはクラスから実際に作られたオブジェクト。
メンバとはクラスの中に直接定義される要素。主にフィールド、メソッド、コンストラクタ。
フィールドとはインスタンスやクラスが保持するデータ。
メソッドとはクラスの中に定義する処理。

クラス・インスタンス・参照の関係

項目定義・規則具体例
クラスフィールド、メソッド、コンストラクタなどをまとめた型の定義。class User { ... }
オブジェクト実行時に生成されるデータと振る舞いの実体。new User()で生成します。
インスタンスあるクラスから生成されたオブジェクトを、そのクラスとの関係で呼ぶ語。Userクラスのインスタンス。
参照オブジェクトへ到達するための値。参照型変数が保持します。
フィールドオブジェクトまたはクラスの状態を保持する変数。インスタンスフィールドは各オブジェクトが持ちます。
this現在処理しているインスタンスを表す参照。インスタンスメソッドやコンストラクタで使えます。

クラスの基本構造

class Item {
    String name;      // フィールド
    int price;        // フィールド

    void printInfo() { // メソッド
        System.out.println(name + ":" + price);
    }
}

nameとpriceは各インスタンスが持つ状態。printInfoはその状態を使って処理するメソッド。

インスタンスは別々に状態を持つ

Item a = new Item();
Item b = new Item();

a.name = "apple";
b.name = "banana";

System.out.println(a.name); // apple
System.out.println(b.name); // banana

aとbは別々のインスタンスを指しているため、フィールドの値も別々に持てる。

参照を代入すると同じ実体を見る

Item a = new Item();
Item b = new Item();
a.name = "apple";

b = a;
b.name = "orange";

System.out.println(a.name); // orange

b = aで、bはaと同じインスタンスを指す。中身をコピーしているわけではない。

フィールドとローカル変数

種類宣言場所初期化寿命のイメージ
フィールドクラス直下デフォルト値ありインスタンスが生きている間
ローカル変数メソッド内自動初期化なしメソッド実行中
混同しない

同じintでも、フィールドなのかローカル変数なのかで初期化ルールが変わる。

2つの参照が同じインスタンスを指す例

`Main.java`として保存する完全なコード

class User {
    String name;
}

public class Main {
    public static void main(String[] args) {
        User first = new User();
        first.name = "A";

        User second = first;
        second.name = "B";

        System.out.println(first.name);
    }
}

second = firstでコピーされるのは参照です。新しいUserは生成されないため、firstとsecondは同じインスタンスを指し、出力はBです。

関連する用語との違い

混同しやすいもの違い見分け方
クラス / インスタンスクラスは定義。インスタンスは実体。newされているかを見る。
フィールド / ローカル変数フィールドはオブジェクトの状態。ローカル変数はメソッド内の一時変数。宣言場所を見る。
同じ値 / 同じインスタンス同じ値を持っていても別インスタンスの場合がある。==は参照の同一性を見る。

オブジェクトの状態を追う規則

  1. newが実行された回数を確認する。
  2. 各参照型変数がどのオブジェクトを指すか記録する。
  3. 参照の代入では、オブジェクトではなく参照値がコピーされる。
  4. フィールド代入がどのオブジェクトに対して行われるか確認する。
  5. 参照がnullなら、フィールドやインスタンスメソッドへアクセスできない。

    int price;

Item a = new Item();
Item b = new Item();
a.price = 100;
b.price = 200;
a = b;
System.out.println(a.price);

new2回実行別々のItemインスタンス
a = baがbと同じ参照になる最初のインスタンスへの参照を失う
出力aはb側を見る200

変数とインスタンスを分けて考える

クラスはインスタンスを作るための定義です。コードを読むときは、クラス名だけでなく、newで作られた各インスタンスと、それを参照する変数を分けます。

class Item {
    int price;
}

Item a = new Item();
Item b = new Item();
a.price = 100;
b.price = 200;
a = b;
System.out.println(a.price); // 200

a = b の時点で、a は b と同じインスタンスを参照します。最初に a が参照していたインスタンスは、ほかに参照がなければガベージコレクションの対象になり得ます。

読む対象見ること例
new新しいインスタンスが作られるnew Item()
フィールド代入どのインスタンスの状態を変えるかa.price = 100
参照代入変数の参照先が変わるa = b
メソッド呼び出しどのインスタンスに対して呼ぶかa.print()

thisは「今動いているインスタンス自身」を指します。フィールドとローカル変数の名前が同じとき、this.priceならフィールド、priceだけなら近いスコープの変数を指す、という区別に使います。

クラス名だけで状態を判断しない

クラス名だけを見ても、インスタンスの状態は分かりません。インスタンスが何個作られ、変数がどのインスタンスを参照しているかを図にすると一気に読みやすくなります。

定義と規則の整理

クラスとオブジェクトで混ざる点

つまずき正しい考え方
クラス自体が動いていると思う通常はクラスを元にインスタンスを作り、そのインスタンスのメンバを使う。
参照型変数にオブジェクト本体が入っていると思う参照型変数にはオブジェクトへの参照が入る。
フィールドとローカル変数を同じ扱いにするフィールドはメンバ。ローカル変数はメソッド内などの一時的な変数。

確認できれば次へ

  • クラス・オブジェクト・インスタンスを区別できる
  • 参照コピーで何がコピーされるか説明できる
  • メンバの代表例を挙げられる
理解の基準

成立するコードと成立しないコード

成立する例：クラス・インスタンス・参照

Item a = new Item();
Item b = new Item();
a.price = 100;
b.price = 200;

成立しない例

Item a;
a.price = 100;

成立する形

Item a = new Item();
a.price = 100;
確認ポイント

参照型変数を宣言しただけではインスタンスは作られていません。newが必要です。

定義と規則を確認する

用語の確認

クラス、オブジェクト、インスタンス、メンバを説明できますか。

説明を見る

クラスは設計図、インスタンスはnewで作られた実体、オブジェクトは扱う対象、メンバはクラスに定義するフィールドやメソッドなどです。

コードの確認

次のコードで出力は何ですか。

class Item { int price; }
Item a = new Item();
Item b = a;
b.price = 100;
System.out.println(a.price);
説明を見る

100です。aとbは同じインスタンスを参照しています。

仕組みを説明する

参照型変数を代入してもオブジェクト本体がコピーされない理由を説明してください。

説明を見る

参照型変数に入っているのはオブジェクト本体ではなく参照です。代入でコピーされるのは参照であり、本体ではありません。

基本ルールのまとめ

  • クラスとインスタンスの違いを説明できる。
  • newの回数から作られるインスタンス数を判断できる。
  • 参照変数の代入で同じインスタンスを共有する場合を説明できる。

コードに規則を当てはめる

次の出力を答える。

確認用コード

class Box { int value; }

Box x = new Box();
Box y = x;
y.value = 7;
System.out.println(x.value);
説明を見る

出力は7。

y = xにより、xとyは同じBoxインスタンスを参照する。y.valueの変更は、その同じインスタンスのフィールド変更。
