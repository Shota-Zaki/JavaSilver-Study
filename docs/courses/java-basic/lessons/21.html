ビット・バイト・2進数

ビットとバイト、2進・8進・16進リテラル、符号付き整数、ビット演算、シフトの規則を説明します。

このレッスンのゴール

整数の表記と値を区別し、ビット演算後の型と結果を説明できること。

基本用語の定義

2進数や16進数は値の表記方法です。同じ整数値を異なる基数で書いても、変数に格納された後は同じ数値として扱われます。

ビットとは0または1を表す最小単位。
バイトとは通常8ビットのまとまり。
2進数とは0と1だけで数を表す方法。Javaでは0bで書ける。
8進数とは0〜7で数を表す方法。Javaでは先頭0に注意。
16進数とは0〜9とA〜Fで数を表す方法。Javaでは0xで書ける。

整数表記とビット演算の基本規則

項目定義・規則具体例
ビット0または1の二値を表す最小単位。8ビットで256通りの組み合わせ。
バイト8ビットのまとまり。Javaのbyte型は符号付き8ビット整数です。範囲は-128から127。
2進リテラル0bまたは0Bで始めます。0b1010は10。
8進リテラル先頭を0にします。012は10。
16進リテラル0xまたは0Xで始めます。0xAは10。
ビット演算&、|、^、~、<<、>>、>>>。byteやshortは演算時に通常intへ昇格します。

bitとbyte

単位意味例
bit0または1の1桁スイッチのON/OFFのような情報
byteJavaのbyte型では8bitファイルサイズ、メモリサイズ
KB/MB/GBbyteを大きくした単位ストレージ容量など

2進数の読み方

2進数は右から1, 2, 4, 8, 16...の重みを持つ。1になっている桁の重みを足す。

16進数は0〜9に加えてA〜Fを使う。Javaでは0xを付けて書く。

Javaの整数リテラル

int a = 10;      // 10進数
int b = 0b1010;  // 2進数。10
int c = 012;     // 8進数。10
int d = 0xA;     // 16進数。10
8進数に注意

012は10進数の12ではなく、8進数表記。初心者はここで見た目に騙されやすい。

同じ値を異なる基数で表す

`Main.java`として保存する完全なコード

public class Main {
    public static void main(String[] args) {
        int binary = 0b1010;
        int octal = 012;
        int decimal = 10;
        int hexadecimal = 0xA;

        System.out.println(binary == octal);
        System.out.println(decimal == hexadecimal);
        System.out.println(binary << 1);
    }
}

4つの変数にはすべて整数値10が入ります。10 << 1はビット列を左へ1桁移動し、結果は20です。

関連する用語との違い

混同しやすいもの違い見分け方
表記 / 値0b1010は表記が2進数なだけで、値は10。10進数に直して考える。
byte型 / 1バイトJavaのbyteは-128〜127の整数型。単に8ビット幅というだけではない。符号付き範囲を見る。
ビット数 / 扱える範囲ビット数が増えるほど表現できる値の種類が増える。型ごとの範囲と結びつける。

ビット式を評価する規則

  1. リテラルの基数を確認して整数値へ読み替える。
  2. サフィックスを含めてリテラルの型を決める。
  3. byte・short・charの演算ではintへの数値昇格を適用する。
  4. ビット演算またはシフトを適用する。
  5. 代入先が狭い型なら、縮小変換の必要性を確認する。

確認用コード

byte b = 0b10000000;
見る点このコードでの判断結論
2進数の値0b10000000は12810進数に直す
byte範囲-128〜127128は範囲外
結論代入不可コンパイルエラー

表記と値を分ける

2進数、8進数、10進数、16進数は、同じ値を異なる書き方で表したものです。接頭辞を確認して値を求めた後、代入先の型の範囲を確認します。

表現例読み方
2進数0b10102の重みで読む。10進数では10。
8進数012先頭0に注意。10進数では10。
16進数0x0A0〜9とA〜Fで読む。10進数では10。
int a = 0b1010; // 10
int b = 012;    // 10
int c = 0x0A;   // 10

見た目が違っても、変数に入る値としては同じ10です。表記方法と値そのものを分けると判定しやすくなります。特に先頭に0が付く整数は8進数として扱われるため、意図せず違う値になることがあります。

ここで整理する誤解

「2進数だから別の型になる」わけではありません。整数リテラルの表記が違うだけで、基本的にはintの整数値として扱われます。

定義と規則の整理

ビット・バイトで混同する点

つまずき正しい考え方
bitとbyteを同じ単位と思う1 byteは通常8 bit。Javaのbyte型は-128〜127。
2進数表記ならbyte型になると思う0b...は表記方法であって、リテラルの基本型は通常int。
符号付き範囲を忘れるJavaの整数型は基本的に符号付き。byteの上限は127。

確認できれば次へ

  • bitとbyteの違いを説明できる
  • 0b10000000がbyteにそのまま入らない理由を説明できる
  • byte/short/int/longの範囲感を説明できる
理解の基準

成立するコードと成立しないコード

成立する例：ビット・バイト・2進数

int n = 0b1010;
System.out.println(n); // 10

成立しない例

byte b = 0b10000000;

成立する形

byte b = (byte) 0b10000000;
確認ポイント

2進数表記でも整数リテラルの基本はintです。byte範囲外ならキャストが必要です。

定義と規則を確認する

用語の確認

ビット、バイト、2進数を説明できますか。

説明を見る

ビットは0/1の最小単位、バイトは通常8ビット、2進数は0と1で値を表す表記です。

コードの確認

次のリテラルは10進数でいくつですか。

0b1010
説明を見る

10です。8 + 2 = 10です。0bは2進数リテラルを表します。

仕組みを説明する

byte型の範囲を超える2進数リテラルをbyteへ直接代入できない理由を説明してください。

説明を見る

整数リテラルは基本的にintとして扱われ、byteの範囲に収まる場合だけ直接代入できます。範囲外ならキャストが必要です。

基本ルールのまとめ

  • Javaのbyteが8ビットだと説明できる。
  • 0b、0、0xから基数を判断できる。
  • リテラルの値が型の範囲に収まるか確認できる。

コードに規則を当てはめる

0b1010は10進数でいくつか。またint x = 0b1010;はコンパイルできるか。

説明を見る

0b1010は10進数で10。8 + 2 = 10。

int x = 0b1010;はコンパイルできる。整数リテラルとしてintに代入できる値。
