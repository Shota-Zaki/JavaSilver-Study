Java基礎 Lesson 9

繰り返し処理

forとwhileは、条件が成り立つ間だけ処理を繰り返します。初期化、条件、更新、本体の順番を固定して追います。

ここまでできればOK

ループの回数と、break・continue後の流れを正確に追えるようになります。

まず押さえる言葉

繰り返し文は「何回回るか」ではなく、初期化、条件、本体、更新がどの順で実行されるかで整理します。

ループとは条件を満たす間、同じ処理を繰り返す構文。
初期化式とはfor文の最初に一度だけ実行される部分。
更新式とはfor文の各周回後に実行される部分。
breakとはループやswitchを抜ける文。
continueとは現在の周回の残りを飛ばして次の周回へ進む文。

繰り返し文の実行順

項目意味・使い方具体例
for文初期化、条件、本体、更新の順で繰り返します。for (int i=0; i<3; i++)
while文本体の前に条件を評価します。最初からfalseなら本体は0回です。
do-while文本体の後に条件を評価します。本体は最低1回実行されます。
break最も内側のループまたはswitchを終了します。ラベル付きbreakでは指定文を終了できます。
continue現在の反復の残りを飛ばし、次の反復へ進みます。forでは更新式へ進みます。
スコープforの初期化部で宣言した変数は、そのfor文の範囲で有効です。ループ終了後には参照できません。

for文の読み方

0, 1, 2が表示される

for (int i = 0; i < 3; i++) {
    System.out.println(i);
}
  1. int i = 0を最初に1回だけ実行する。
  2. i < 3を確認する。
  3. trueなら本体を実行する。
  4. i++を実行する。
  5. 条件確認へ戻る。

whileとdo-while

構文条件確認のタイミング最低実行回数
while本体の前0回
do-while本体の後1回
int x = 10;
while (x < 5) {
    System.out.println("while");
}

do {
    System.out.println("do");
} while (x < 5);

breakとcontinue

breakはループ自体を終了する。continueは現在の反復をそこで終え、次の反復へ進む。for文ではcontinue後に更新式へ進む。

0, 1, 3が表示される

for (int i = 0; i < 5; i++) {
    if (i == 2) {
        continue;
    }
    if (i == 4) {
        break;
    }
    System.out.println(i);
}

到達不能コード

break や continue の直後に同じブロック内の文を書くと、その文へ到達できないためコンパイルエラーになります。

成立しない例

while (true) {
    break;
    System.out.println("never"); // 到達不能
}

for文の実行順序を確認する

`Main.java`として保存する完全なコード

public class Main {
    public static void main(String[] args) {
        for (int i = 0; i < 5; i++) {
            if (i == 2) {
                continue;
            }
            if (i == 4) {
                break;
            }
            System.out.print(i);
        }
    }
}

i == 2では出力を飛ばして更新式へ進み、i == 4ではループを終了します。出力は013です。

似た言葉との違い

混同しやすいもの違い見分け方
while / do-whilewhileは先に判定。do-whileは後で判定。条件が最初からfalseのときに差が出る。
break / continuebreakは終了。continueは次へ。ループを抜けるか、次回へ進むか。
更新式 / 本体forの更新式は本体実行後。出力してから増えるのか、増えてから出力かを追う。

ループを追う順番

  1. 初期化部を1回だけ実行する。
  2. 条件式を評価し、falseなら終了する。
  3. 本体を上から実行する。
  4. continue、break、例外、returnの有無を確認する。
  5. 通常のfor文では更新式を実行してから次の条件評価へ戻る。

確認用コード

for (int i = 0; i < 3; i++) {
    System.out.println(i);
}
見る点このコードでの判断結論
1回目i=0、条件true、出力0、更新で1継続
2回目i=1、条件true、出力1、更新で2継続
3回目i=2、条件true、出力2、更新で3次で終了

ここまでの要点

  • for文の実行順を説明できる。
  • whileとdo-whileの違いを説明できる。
  • breakとcontinueが次にどこへ進むか判断できる。
