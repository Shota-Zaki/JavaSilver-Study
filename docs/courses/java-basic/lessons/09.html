Java基礎 Lesson 9

for・while

for、while、do-whileの構文と、条件評価・更新・break・continueの制御規則を説明します。

このレッスンのゴール

各ループで条件式と本体が評価される順序を説明し、終了条件を判断できること。

基本用語の定義

繰り返し文は「何回回るか」ではなく、初期化、条件、本体、更新がどの順で実行されるかで定義します。

ループとは条件を満たす間、同じ処理を繰り返す構文。
初期化式とはfor文の最初に一度だけ実行される部分。
更新式とはfor文の各周回後に実行される部分。
breakとはループやswitchを抜ける文。
continueとは現在の周回の残りを飛ばして次の周回へ進む文。

繰り返し文の実行規則

項目定義・規則具体例
for文初期化、条件、本体、更新の順で繰り返します。for (int i=0; i<3; i++)
while文本体の前に条件を評価します。最初からfalseなら本体は0回です。
do-while文本体の後に条件を評価します。本体は最低1回実行されます。
break最も内側のループまたはswitchを終了します。ラベル付きbreakでは指定文を終了できます。
continue現在の反復の残りを飛ばし、次の反復へ進みます。forでは更新式へ進みます。
スコープforの初期化部で宣言した変数は、そのfor文の範囲で有効です。ループ終了後には参照できません。

for文の読み方

0, 1, 2が表示される

for (int i = 0; i < 3; i++) {
    System.out.println(i);
}
  1. int i = 0を最初に1回だけ実行する。
  2. i < 3を判定する。
  3. trueなら本体を実行する。
  4. i++を実行する。
  5. 条件判定へ戻る。

whileとdo-while

構文条件判定のタイミング最低実行回数
while本体の前0回
do-while本体の後1回
int x = 10;
while (x < 5) {
    System.out.println("while");
}

do {
    System.out.println("do");
} while (x < 5);

breakとcontinue

breakはループ自体を終了する。continueは現在の反復をそこで終え、次の反復へ進む。for文ではcontinue後に更新式へ進む。

0, 1, 3が表示される

for (int i = 0; i < 5; i++) {
    if (i == 2) {
        continue;
    }
    if (i == 4) {
        break;
    }
    System.out.println(i);
}

到達不能コード

break や continue の直後に同じブロック内の文を書くと、その文へ到達できないためコンパイルエラーになります。

成立しない例

while (true) {
    break;
    System.out.println("never"); // 到達不能
}

for文の実行順序を確認する

`Main.java`として保存する完全なコード

public class Main {
    public static void main(String[] args) {
        for (int i = 0; i < 5; i++) {
            if (i == 2) {
                continue;
            }
            if (i == 4) {
                break;
            }
            System.out.print(i);
        }
    }
}

i == 2では出力を飛ばして更新式へ進み、i == 4ではループを終了します。出力は013です。

関連する用語との違い

混同しやすいもの違い見分け方
while / do-whilewhileは先に判定。do-whileは後で判定。条件が最初からfalseのときに差が出る。
break / continuebreakは終了。continueは次へ。ループを抜けるか、次回へ進むか。
更新式 / 本体forの更新式は本体実行後。出力してから増えるのか、増えてから出力かを追う。

ループを追跡する規則

  1. 初期化部を1回だけ実行する。
  2. 条件式を評価し、falseなら終了する。
  3. 本体を上から実行する。
  4. continue、break、例外、returnの有無を確認する。
  5. 通常のfor文では更新式を実行してから次の条件評価へ戻る。

確認用コード

for (int i = 0; i < 3; i++) {
    System.out.println(i);
}
見る点このコードでの判断結論
1回目i=0、条件true、出力0、更新で1継続
2回目i=1、条件true、出力1、更新で2継続
3回目i=2、条件true、出力2、更新で3次で終了

ループは順番を書き出して追う

繰り返し回数を感覚で決めると、1回多い・少ないという誤りが起きます。for文では、初期化、条件判定、本体、更新の順番を固定して追います。

for (int i = 0; i < 3; i++) {
    System.out.println(i);
}
順番処理この例
1初期化int i = 0。最初に1回だけ。
2条件判定i < 3。trueなら本体へ。
3本体println(i)を実行。
4更新i++。その後、条件判定へ戻る。

whileは条件判定が先、do-whileは本体が先です。つまりdo-whileは条件が最初からfalseでも最低1回は実行されます。

int i = 10;
do {
    System.out.println(i);
} while (i < 3); // 10が1回表示される
ここで整理する誤解

ループ回数は見た目で決めず、初期値から条件がfalseになるまでを追います。初期値、条件、更新の3点がそろって初めて回数が決まります。

定義と規則の整理

ループで読み間違える点

つまずき正しい考え方
for文を左から1回ずつ読む初期化→条件→本体→更新→条件…の順。更新式は本体の後。
continueで更新式も飛ぶと思う通常のfor文ではcontinue後に更新式へ進む。
do-whileをwhileと同じだと思うdo-whileは条件判定前に本体が最低1回実行される。

確認できれば次へ

  • for文の実行順を説明できる
  • breakとcontinueの違いを説明できる
  • whileとdo-whileの違いを説明できる
理解の基準

成立するコードと成立しないコード

成立する例：ループの更新

for (int i = 0; i < 3; i++) {
    System.out.println(i);
}

成立しない例

int i = 0;
while (i < 3) {
    System.out.println(i);
}

成立する形

int i = 0;
while (i < 3) {
    System.out.println(i);
    i++;
}
確認ポイント

whileでは更新処理を書き忘れると無限ループになります。

定義と規則を確認する

用語の確認

for文の初期化、条件、更新、本体の実行順を説明できますか。

説明を見る

初期化 → 条件判定 → 本体 → 更新 → 条件判定、の順です。更新式は本体の後に動きます。

コードの確認

次のコードは何回表示しますか。

for (int i = 0; i < 3; i++) {
    System.out.print(i);
}
説明を見る

3回です。表示は012です。iが0,1,2のとき本体が実行され、3で条件がfalseになります。

仕組みを説明する

拡張for文で要素変数を書き換えても配列本体が変わらない場合がある理由を説明してください。

説明を見る

拡張for文の変数は要素値または参照のコピーを受け取ります。プリミティブ値を書き換えても配列要素そのものは書き換わりません。

基本ルールのまとめ

  • for文の実行順を説明できる。
  • whileとdo-whileの違いを説明できる。
  • breakとcontinueが次にどこへ進むか判断できる。

コードに規則を当てはめる

次の出力を答える。

確認用コード

for (int i = 0; i < 5; i++) {
    if (i == 2) continue;
    if (i == 4) break;
    System.out.print(i);
}
説明を見る

出力は013。

i=2ではcontinueにより出力を飛ばす。i=4ではbreakによりループを終了する。
