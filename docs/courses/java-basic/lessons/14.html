Java基礎 Lesson 14

コンストラクタ

コンストラクタをインスタンス初期化のための特別な宣言として定義し、this・superの呼び出し規則を説明します。

このレッスンのゴール

コンストラクタとメソッドを区別し、デフォルトコンストラクタが生成される条件を説明できること。

基本用語の定義

コンストラクタは戻り値型を持たず、クラス名と同じ名前を使います。`new`による生成時に初期化処理として呼び出されます。

コンストラクタとはインスタンス生成時に実行される初期化処理。クラス名と同じ名前で戻り値型を書かない。
デフォルトコンストラクタとはコンストラクタを1つも書かない場合に、コンパイラが自動追加する引数なしコンストラクタ。
thisとは現在のインスタンス自身を指す参照。
this()とは同じクラス内の別コンストラクタを呼ぶ構文。
super()とはスーパークラスのコンストラクタを呼ぶ構文。

コンストラクタの宣言と連鎖の規則

項目定義・規則具体例
名前コンストラクタ名はクラス名と完全に一致します。大文字・小文字も一致させます。
戻り値型コンストラクタには戻り値型を書きません。void ClassName()はメソッドです。
オーバーロード引数型の並びが異なる複数のコンストラクタを宣言できます。User()とUser(String)。
デフォルトコンストラクタコンストラクタを1つも宣言しない場合にコンパイラが補います。1つでも宣言すると自動追加されません。
this()同じクラスの別コンストラクタを呼び出します。コンストラクタ本体の先頭文に限ります。
super()直接の親クラスのコンストラクタを呼び出します。明示しない場合は原則として引数なしsuper()が補われます。

基本形

class Item {
    String name;

    Item(String name) {
        this.name = name;
    }
}

Item item = new Item("apple");

new Item("apple")のタイミングでコンストラクタが呼ばれ、フィールドnameへ値が入る。

メソッドではない

項目コンストラクタメソッド
名前クラス名と同じ自由に付ける
戻り値型書かないvoidやintなどを書く
呼び出しnew時に呼ばれる明示的に呼ぶ
目的初期化処理の実行

戻り値型を書くとメソッドになる

class Sample {
    void Sample() { } // これはコンストラクタではなくメソッド
}

デフォルトコンストラクタ

コンストラクタを1つも書かない場合だけ、引数なしコンストラクタが自動で用意される。

自動生成される

class A { }
new A(); // OK

1つ書いたので自動生成されない

class B {
    B(String name) { }
}
new B(); // コンパイルエラー

this()とsuper()

this()は同じクラスの別コンストラクタを呼ぶ。super()は親クラスのコンストラクタを呼ぶ。どちらもコンストラクタの先頭にしか書けない。

class Item {
    String name;
    int price;

    Item() {
        this("unknown", 0);
    }

    Item(String name, int price) {
        this.name = name;
        this.price = price;
    }
}

コンストラクタの連鎖を確認する

`Main.java`として保存する完全なコード

class User {
    String name;

    User() {
        this("unknown");
    }

    User(String name) {
        this.name = name;
    }
}

public class Main {
    public static void main(String[] args) {
        User user = new User();
        System.out.println(user.name);
    }
}

new User()は引数なしコンストラクタを呼び、最初のthis("unknown")から引数付きコンストラクタへ処理が移ります。

関連する用語との違い

混同しやすいもの違い見分け方
コンストラクタ / メソッドコンストラクタは戻り値型なし。メソッドは戻り値型がある。void ClassName()はコンストラクタにならない。
this() / super()this()は同じクラスの別コンストラクタ、super()は親コンストラクタ。最初の文にしか書けない。
デフォルトコンストラクタ / 自作コンストラクタコンストラクタを書かない場合だけ、引数なしコンストラクタが自動追加される。1つでも書いたら自動追加されない。

インスタンス初期化を追う規則

  1. newで選択されるコンストラクタを引数型から決める。
  2. コンストラクタ先頭のthis()またはsuper()を確認する。
  3. 親クラス側の初期化を先に完了する。
  4. インスタンスフィールド初期化子と初期化ブロックを記述順に処理する。
  5. 最後に選択されたコンストラクタ本体の残りを実行する。

確認用コード

class Item {
    String name;

    void Item(String name) {
        this.name = name;
    }
}

Item item = new Item("pen");
見る点このコードでの判断結論
void Item戻り値型voidがあるコンストラクタではなくメソッド
new Item("pen")String引数コンストラクタを探す存在しない
結論一致するコンストラクタなしコンパイルエラー

newから初期化の順番を追う

コンストラクタは、新しく作られたインスタンスへ初期値を設定するための特別な処理です。戻り値型は書かず、クラス名と同じ名前にします。

class User {
    String name;

    User(String name) {
        this.name = name;
    }
}

ここでのthis.nameはフィールド、右辺のnameは引数です。コンストラクタは、外から受け取った値をインスタンスの状態へセットする役割を持ちます。

状況何が起きるか注意
コンストラクタを1つも書かない引数なしのデフォルトコンストラクタが用意される自動で作られる
引数ありコンストラクタを書くデフォルトコンストラクタは自動生成されないnew User()は不可
this(...)を書く同じクラスの別コンストラクタを呼ぶ先頭行にしか書けない
super(...)を書く親クラスのコンストラクタを呼ぶ先頭行にしか書けない
class Item {
    String name;

    Item() {
        this("unknown");
    }

    Item(String name) {
        this.name = name;
    }
}
よくあるミス

コンストラクタにvoidを書くと、コンストラクタではなく通常メソッドになります。見た目が似ていても、生成時には呼ばれません。

定義と規則の整理

コンストラクタで混同しやすい点

つまずき正しい考え方
コンストラクタをメソッドだと思うコンストラクタはインスタンス生成時の初期化処理。戻り値型を書かない。
デフォルトコンストラクタが常にあると思う引数ありでも自作コンストラクタを書くと、引数なしのデフォルトコンストラクタは自動追加されない。
this()やsuper()をどこにでも書けると思うコンストラクタ呼び出しはコンストラクタの先頭行でなければならない。

確認できれば次へ

  • コンストラクタとメソッドの違いを説明できる
  • デフォルトコンストラクタが追加される条件を説明できる
  • this()とsuper()の先頭行ルールを説明できる
理解の基準

成立するコードと成立しないコード

成立する例：引数なしコンストラクタ

class User {
    User() {
        System.out.println("new");
    }
}

成立しない例：戻り値型があるためコンストラクタではなくメソッドになる

class User {
    void User(String name) {
        System.out.println(name);
    }
}

成立する形：戻り値型を書かずクラス名と同じ名前にする

class User {
    User(String name) {
        System.out.println(name);
    }
}
確認ポイント

コンストラクタには戻り値型を書きません。voidを書くと同名メソッドになります。

定義と規則を確認する

用語の確認

コンストラクタとメソッドの違いを説明できますか。

説明を見る

コンストラクタはインスタンス生成時の初期化処理で、クラス名と同じ名前で戻り値型を書きません。メソッドは通常の処理です。

コードの確認

次のSampleはコンストラクタを定義していますか。

class Sample {
    void Sample() {
        System.out.println("A");
    }
}
説明を見る

定義していません。voidがあるため、これはSampleという名前のメソッドです。コンストラクタではありません。

仕組みを説明する

this() または super() をコンストラクタの先頭行に置く必要がある理由を説明してください。

説明を見る

インスタンス初期化では、どのコンストラクタを先に使うかを最初に確定する必要があります。そのためthis()やsuper()はコンストラクタ本体の先頭にしか置けません。

基本ルールのまとめ

  • コンストラクタと同名メソッドを区別できる。
  • デフォルトコンストラクタが追加される条件を説明できる。
  • this()とsuper()による呼び出し順を追える。

コードに規則を当てはめる

次のコードはなぜコンパイルエラーになるか。

確認用コード

class User {
    User(String name) { }
}

public class Main {
    public static void main(String[] args) {
        User u = new User();
    }
}
説明を見る

User(String name)を自分で定義しているため、引数なしコンストラクタは自動生成されない。

new User()に対応する引数なしコンストラクタが存在しないため、コンパイルエラーになる。
