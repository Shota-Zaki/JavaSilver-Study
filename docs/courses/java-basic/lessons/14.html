Java基礎 Lesson 14

コンストラクタ

newすると、フィールドの準備とコンストラクタの処理が順番に行われます。thisとsuperの呼び出しも含めて追います。

ここまでできればOK

コンストラクタの宣言と呼び出し順をコードから説明できるようになります。

まず押さえる言葉

コンストラクタは戻り値型を持たず、クラス名と同じ名前を使います。`new`による生成時に初期化処理として呼び出されます。

コンストラクタとはインスタンス生成時に実行される初期化処理。クラス名と同じ名前で戻り値型を書かない。
デフォルトコンストラクタとはコンストラクタを1つも書かない場合に、コンパイラが自動追加する引数なしコンストラクタ。
thisとは現在のインスタンス自身を指す参照。
this()とは同じクラス内の別コンストラクタを呼ぶ構文。
super()とはスーパークラスのコンストラクタを呼ぶ構文。

コンストラクタの宣言と呼び出し

項目意味・使い方具体例
名前コンストラクタ名はクラス名と完全に一致します。大文字・小文字も一致させます。
戻り値型コンストラクタには戻り値型を書きません。void ClassName()はメソッドです。
オーバーロード引数型の並びが異なる複数のコンストラクタを宣言できます。User()とUser(String)。
デフォルトコンストラクタコンストラクタを1つも宣言しない場合にコンパイラが補います。1つでも宣言すると自動追加されません。
this()同じクラスの別コンストラクタを呼び出します。コンストラクタ本体の先頭文に限ります。
super()直接の親クラスのコンストラクタを呼び出します。明示しない場合は原則として引数なしsuper()が補われます。

基本形

class Item {
    String name;

    Item(String name) {
        this.name = name;
    }
}

Item item = new Item("apple");

new Item("apple")のタイミングでコンストラクタが呼ばれ、フィールドnameへ値が入る。

メソッドではない

項目コンストラクタメソッド
名前クラス名と同じ自由に付ける
戻り値型書かないvoidやintなどを書く
呼び出しnew時に呼ばれる明示的に呼ぶ
目的初期化処理の実行

戻り値型を書くとメソッドになる

class Sample {
    void Sample() { } // これはコンストラクタではなくメソッド
}

デフォルトコンストラクタ

コンストラクタを1つも書かない場合だけ、引数なしコンストラクタが自動で用意される。

自動生成される

class A { }
new A(); // OK

1つ書いたので自動生成されない

class B {
    B(String name) { }
}
new B(); // コンパイルエラー

this()とsuper()

this()は同じクラスの別コンストラクタを呼ぶ。super()は親クラスのコンストラクタを呼ぶ。どちらもコンストラクタの先頭にしか書けない。

class Item {
    String name;
    int price;

    Item() {
        this("unknown", 0);
    }

    Item(String name, int price) {
        this.name = name;
        this.price = price;
    }
}

コンストラクタの連鎖を確認する

`Main.java`として保存する完全なコード

class User {
    String name;

    User() {
        this("unknown");
    }

    User(String name) {
        this.name = name;
    }
}

public class Main {
    public static void main(String[] args) {
        User user = new User();
        System.out.println(user.name);
    }
}

new User()は引数なしコンストラクタを呼び、最初のthis("unknown")から引数付きコンストラクタへ処理が移ります。

似た言葉との違い

混同しやすいもの違い見分け方
コンストラクタ / メソッドコンストラクタは戻り値型なし。メソッドは戻り値型がある。void ClassName()はコンストラクタにならない。
this() / super()this()は同じクラスの別コンストラクタ、super()は親コンストラクタ。最初の文にしか書けない。
デフォルトコンストラクタ / 自作コンストラクタコンストラクタを書かない場合だけ、引数なしコンストラクタが自動追加される。1つでも書いたら自動追加されない。

インスタンス初期化の確認順

  1. newで選択されるコンストラクタを引数型から決める。
  2. コンストラクタ先頭のthis()またはsuper()を確認する。
  3. 親クラス側の初期化を先に完了する。
  4. インスタンスフィールド初期化子と初期化ブロックを記述順に処理する。
  5. 最後に選択されたコンストラクタ本体の残りを実行する。

確認用コード

class Item {
    String name;

    void Item(String name) {
        this.name = name;
    }
}

Item item = new Item("pen");
見る点このコードでの判断結論
void Item戻り値型voidがあるコンストラクタではなくメソッド
new Item("pen")String引数コンストラクタを探す存在しない
結論一致するコンストラクタなしコンパイルエラー

ここまでの要点

  • コンストラクタと同名メソッドを区別できる。
  • デフォルトコンストラクタが追加される条件を説明できる。
  • this()とsuper()による呼び出し順を追える。
