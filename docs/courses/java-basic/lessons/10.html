Java基礎 Lesson 10

配列

配列を固定長のオブジェクトとして定義し、生成、初期値、添字、length、多次元配列の規則を説明します。

このレッスンのゴール

配列変数と配列オブジェクトを区別し、有効な添字範囲と要素の初期値を説明できること。

基本用語の定義

配列型の変数が保持するのは配列オブジェクトへの参照です。配列の長さは生成時に決まり、後から変更できません。

配列とは同じ型の値を番号付きで並べる固定長の入れ物。
要素とは配列の中の1つ1つの値。
添字とは要素の位置番号。Javaでは0から始まる。
lengthとは配列の要素数を持つフィールド。メソッドではない。
参照コピーとは配列本体ではなく、配列を指す参照がコピーされること。

配列の生成と要素アクセスの規則

項目定義・規則具体例
配列型同じ要素型の値を添字で扱う参照型。int[]、String[]
生成new 要素型[長さ]で配列オブジェクトを作ります。new int[3]
初期値生成時に全要素へ型ごとのデフォルト値が入ります。intは0、参照型はnullです。
添字0から始まる要素位置。長さ3なら0、1、2が有効です。
length配列の要素数を表すfinalフィールド。メソッドではなくarray.lengthです。
多次元配列配列を要素として持つ配列。各行の長さは同じでなくてもかまいません。int[][]はint[]の配列です。

配列の作り方

int[] a = new int[3];
a[0] = 10;
a[1] = 20;
a[2] = 30;
式意味
int[] aintを入れる配列への参照変数
new int[3]長さ3のint配列を作る
a[0]0番目の要素
a.length配列の長さ。メソッドではなくフィールド

デフォルト値

配列をnewすると、各要素は型ごとのデフォルト値で初期化される。

要素型初期値
数値型0または0.0
booleanfalse
char\u0000
参照型null
String[] names = new String[2];
System.out.println(names[0]); // null

参照のコピー

配列変数を別の変数へ代入しても、配列の中身がコピーされるわけではない。同じ配列を指す参照がコピーされる。

int[] a = {1, 2, 3};
int[] b = a;
b[0] = 99;
System.out.println(a[0]); // 99

多次元配列

Javaの多次元配列は「配列の配列」。各行の長さを揃えなくてもよい。

int[][] table = new int[2][];
table[0] = new int[3];
table[1] = new int[1];

配列の生成・代入・参照

`Main.java`として保存する完全なコード

public class Main {
    public static void main(String[] args) {
        int[] numbers = new int[3];
        numbers[0] = 10;
        numbers[1] = 20;

        for (int i = 0; i < numbers.length; i++) {
            System.out.println(numbers[i]);
        }
    }
}

numbers[2]には明示的に代入していないため、intの初期値0が入っています。numbers[3]へアクセスすると実行時にArrayIndexOutOfBoundsExceptionが発生します。

関連する用語との違い

混同しやすいもの違い見分け方
配列変数 / 配列要素配列変数は参照。要素は中に入っている値または参照。aとa[0]を分ける。
長さ / 最後の添字長さが3なら最後の添字は2。length - 1が最後。
配列コピー / 参照コピー代入は参照コピー。中身の複製ではない。b = a後に片方を変更して判定。

配列アクセスを判断する規則

  1. 配列変数がnullでないか確認する。
  2. 配列オブジェクトのlengthを確認する。
  3. 添字式を評価する。
  4. 添字が0以上かつlength未満か確認する。
  5. 有効範囲外なら、そのアクセス式でArrayIndexOutOfBoundsExceptionが発生する。

確認用コード

int[] a = new int[3];
int[] b = a;
b[0] = 10;
System.out.println(a[0]);
見る点このコードでの判断結論
a長さ3の配列を参照要素初期値は0
b = a同じ配列を参照配列本体はコピーされない
b[0] = 10同じ配列の0番を変更a[0]も10

配列変数が持つのは参照

配列はオブジェクトです。配列変数を別の変数へ代入しても要素が複製されるわけではなく、同じ配列への参照がコピーされます。

int[] a = {1, 2, 3};
int[] b = a;
b[0] = 99;
System.out.println(a[0]); // 99

b = aは配列の中身を複製しているのではありません。参照をコピーしています。したがって、aとbは同じ配列オブジェクトを見ています。

コード意味注意
new int[3]要素数3のint配列を作る初期値は0
a.length配列の長さメソッドではないので()不要
a[0]先頭要素添字は0から始まる
a[3]4番目の要素長さ3なら実行時例外

配列の長さは作った後に変えられません。要素を増やしたいなら、新しい配列を作るか、後で学ぶArrayListのようなコレクションを使います。

配列で最初に疑うべき箇所

添字が範囲内か、参照をコピーして同じ配列を見ていないか、初期値を勘違いしていないか。この3つを先に見ると分かりやすいです。

定義と規則の整理

配列で混同しやすい点

つまずき正しい考え方
配列の長さを後から変えられると思う配列の長さは生成時に固定。増減したいならArrayListなどを使う。
length()と書く配列はフィールドのlength。Stringはメソッドのlength()。
多次元配列は必ず長方形だと思うJavaの多次元配列は配列の配列。行ごとに長さが違ってよい。

確認できれば次へ

  • 配列の添字範囲を説明できる
  • 配列の初期値を型別に答えられる
  • 配列とStringのlengthの違いを説明できる
理解の基準

成立するコードと成立しないコード

成立する例：配列の添字

int[] nums = {10, 20, 30};
System.out.println(nums[0]);
System.out.println(nums[2]);

成立しない例

int[] nums = {10, 20, 30};
System.out.println(nums[3]);

成立する形

int[] nums = {10, 20, 30};
System.out.println(nums[nums.length - 1]);
確認ポイント

配列の添字は0から始まります。最後の要素はlengthではなくlength - 1です。

定義と規則を確認する

用語の確認

配列のlengthと添字の範囲を説明できますか。

説明を見る

lengthは要素数です。添字は0からlength - 1までです。length番目の添字は存在しません。

コードの確認

次のコードはどうなりますか。

int[] a = new int[3];
System.out.println(a[3]);
説明を見る

コンパイルはできますが、実行時にArrayIndexOutOfBoundsExceptionです。有効な添字は0,1,2です。

仕組みを説明する

int[] a = b; のように参照をコピーすると、配列本体は増えますか。

説明を見る

増えません。aとbが同じ配列オブジェクトを参照します。片方から要素を書き換えると同じ本体が変わります。

基本ルールのまとめ

  • 配列の添字が0からlength - 1までだと判断できる。
  • 配列要素の初期値を型ごとに説明できる。
  • 配列変数の代入では参照がコピーされると理解している。

コードに規則を当てはめる

次のコードの出力を答える。

確認用コード

String[] names = new String[2];
System.out.println(names[0]);
説明を見る

出力はnull。

Stringは参照型なので、配列要素の初期値はnullになる。ローカル変数names自体はnew String[2]で初期化済み。
