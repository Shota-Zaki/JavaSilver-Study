文字コードとUnicode

Unicodeコードポイント、UTF-16コード単位、char、String、ソースエンコーディングを区別して説明します。

このレッスンのゴール

charが常に人間の見た目の1文字を表すわけではない理由と、補助文字の扱いを説明できること。

基本用語の定義

Javaの`char`は16ビットのUTF-16コード単位を1つ保持します。Unicodeの1コードポイントがchar 1個に収まるとは限りません。

文字コードとは文字と数値を対応させるルール。
Unicodeとは多くの文字を統一的に扱うための文字集合。
charとは1文字を表すプリミティブ型。シングルクォートで書く。
文字リテラルとは'A' のように1文字を直接書いたもの。
エスケープシーケンスとは や のように特殊文字を表す書き方。

Unicode・char・Stringの定義

項目定義・規則具体例
コードポイントUnicodeで文字や記号へ割り当てられた番号。U+0041はA。
UTF-16コード単位UTF-16で使う16ビット単位。Javaのcharが1単位を保持します。
char0から65535までの符号なし16ビット値。BMP内の多くの文字はchar 1個で表せます。
サロゲートペア補助文字を2つのcharで表す組み合わせ。一部の絵文字などはString.length()が2になります。
Stringcharの並びとして扱われる不変オブジェクト。コードポイント数とlengthが一致しない場合があります。
ソースエンコーディング.javaファイルのバイト列を文字へ変換する方式。保存時とjavac -encodingを一致させます。

charとStringの違い

表記型意味
'A'charUTF-16コード単位を1つ
"A"String文字列オブジェクト
'AB'不可charリテラルにはUTF-16コード単位を1つだけ書ける
""String長さ0の文字列

charは数値的な性質も持つ

charは文字を扱うが、内部的には文字に対応する数値を持つ。'A'は数値として扱うと65。

char c = 'A';
int n = c;
System.out.println(n); // 65

Unicodeエスケープ

Javaでは\uXXXXの形でUnicodeエスケープを書ける。基礎段階では「文字にも数値的な対応がある」と理解できれば十分。

char c = '\u0041';
System.out.println(c); // A

文字列連結での挙動

char同士を足すと数値計算になる場合がある。Stringが混ざると文字列連結になる。

System.out.println('A' + 1);      // 66
System.out.println("A" + 1);      // A1
System.out.println('A' + "B");    // AB

char数とコードポイント数の違い

`Main.java`として保存する完全なコード

public class Main {
    public static void main(String[] args) {
        String text = "A😀";
        System.out.println(text.length());
        System.out.println(text.codePointCount(0, text.length()));
        System.out.println((int) 'A');
    }
}

Aはchar 1個、絵文字はサロゲートペアのchar 2個です。そのためlengthは3、コードポイント数は2です。

関連する用語との違い

混同しやすいもの違い見分け方
'A' / "A"前者はchar、後者はString。引用符の種類を見る。
文字 / 文字列charはUTF-16コード単位1つです。Stringは0文字以上の並びです。Stringは空文字""も持てる。
文字としての数字 / 数値'1'は文字、1は数値。クォートされているかを見る。

文字データを判断する規則

  1. 対象がchar、String、コードポイントのどれかを区別する。
  2. charはUTF-16コード単位1個として扱う。
  3. 補助文字ではサロゲートペアになる可能性を考える。
  4. 文字数を数える目的に応じてlength()かコードポイントAPIかを選ぶ。
  5. ソースファイルの文字化けでは、保存エンコーディングとコンパイラ設定を確認する。

確認用コード

char c = 'A';
int n = c;
System.out.println(n);

charとStringは別の型

charは16ビットのUTF-16コード単位を1つ保持するプリミティブ型です。英数字や多くの基本的な文字はchar 1つで表せますが、補助文字の中には2つのcharが必要なものもあります。Stringは0文字以上の文字列を扱う参照型です。

型例本質
char'A'UTF-16コード単位を1つ持つプリミティブ型です。数値的な性質もあります。
String"A"文字列オブジェクトへの参照。
Unicode'A'文字を番号で表す仕組み。
char c = 'A';
String s = "A";
System.out.println(c + 1); // 66
System.out.println(s + 1); // A1

charは数値的に扱われる場面があり、Stringは文字列連結になります。この差を「見た目が似ているから同じ」と処理すると、出力結果を間違えます。

ここで整理する誤解

シングルクォートとダブルクォートは見た目の好みではありません。'A'はchar、"A"はStringです。

定義と規則の整理

文字コードで詰まる点

つまずき正しい考え方
文字と数値が完全に別物だと思うJavaのcharはUnicodeのコード単位として数値的にも扱われる。
文字化けを文法エラーだと思う文字コード不一致はソースの読み取り設定の問題。-encoding UTF-8が必要なことがある。
1文字=1バイトと思うUnicodeやUTF-8では文字によりバイト数が異なる。日本語は特に注意。

確認できれば次へ

  • charとUnicodeの関係を説明できる
  • 文字コード不一致の代表的な症状を説明できる
  • javacのencoding指定を説明できる
理解の基準

成立するコードと成立しないコード

成立する例：UTF-8のソースをUTF-8としてコンパイルする

// Main.java
public class Main {
    public static void main(String[] args) {
        System.out.println("こんにちは");
    }
}

// terminal
javac -encoding UTF-8 Main.java
java Main

成立しない例：保存時の文字コードとコンパイル時の文字コードがずれている

// Main.javaはUTF-8で保存
javac -encoding Shift_JIS Main.java

成立する形：保存時とjavacのencodingをそろえる

javac -encoding UTF-8 Main.java
java Main
確認ポイント

文字コード不一致は文法ではなく読み取りの問題です。日本語コメントや文字列で出やすいです。

定義と規則を確認する

用語の確認

文字コード、Unicode、エンコーディングを説明できますか。

説明を見る

Unicodeは文字に番号を割り当てる体系、エンコーディングはその番号をバイト列にする方式です。文字コード問題は保存形式と読み込み指定を分けます。

コードの確認

UTF-8で保存したソースをWindowsでコンパイルして文字化けエラーが出る場合、まず何を指定しますか。

javac Sample.java
説明を見る

javac -encoding UTF-8 Sample.java のように、ソースの文字コードを明示します。

仕組みを説明する

charが1文字を常に完全に表せると言い切れない理由を説明してください。

説明を見る

charは16ビットのUTF-16コード単位です。サロゲートペアが必要な文字はchar 1つで完全な1文字として扱えません。

基本ルールのまとめ

  • シングルクォートとダブルクォートから型を判断できる。
  • charがUTF-16コード単位1つを持つ型だと説明できる。
  • ソースの保存形式とjavacのencoding指定をそろえられる。

コードに規則を当てはめる

次のうち型が正しいものを説明する。

確認用コード

char a = 'A';
String b = "A";
char c = "A";
String d = 'A';
説明を見る

正しいのはchar a = 'A';とString b = "A";。

char c = "A";はダブルクォートなのでString。String d = 'A';はシングルクォートなのでchar。どちらも型が合わない。
