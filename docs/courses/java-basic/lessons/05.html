Java基礎 Lesson 5

変数・値・型

変数、値、型、宣言、代入、初期化、スコープをJavaの用語として定義します。

このレッスンのゴール

変数宣言を構成する要素と、ローカル変数・フィールドの初期化規則を説明できること。

基本用語の定義

変数は値そのものではありません。型で定められた値または参照を保持する、名前付きの記憶場所です。

変数とは値や参照を保持する名前付きの領域。
値とは計算や比較の対象になるデータそのもの。例: 10、true。
型とは値の種類と扱い方を決めるルール。例: int、String。
プリミティブ型とは数値や真偽値など、値そのものを扱う基本型。
参照型とはオブジェクトの場所を指す参照を扱う型。

変数宣言と初期化の基本規則

項目定義・規則具体例
宣言変数の型と名前を定めること。int count;
代入右辺の値を評価し、左辺の変数へ格納すること。count = 10;
初期化変数へ最初の値を与えること。int count = 10;
ローカル変数メソッド、コンストラクタ、ブロック内で宣言する変数。使用前に必ず値が代入済みである必要があります。
フィールドクラス本体に宣言する変数。インスタンス生成時またはクラス初期化時に型ごとの初期値が入ります。
スコープその変数名をコード上で参照できる範囲。ブロックを出ると、そのローカル変数名は使えません。

プリミティブ型と参照型

分類代表例変数に入るものnull
プリミティブ型int, double, char, boolean値そのもの入らない
参照型String, 配列, 自作クラスオブジェクトを指す参照入る
int n = 10;        // nには数値10が入る
String s = "A";    // sにはStringオブジェクトへの参照が入る
String t = null;   // tは何も参照していない

ローカル変数とフィールドの初期値

メソッド内で宣言したローカル変数は、自動初期化されない。使う前に必ず代入が必要。クラスのフィールドは、型ごとのデフォルト値で初期化される。

差が出る例

public class Main {
    static int field;

    public static void main(String[] args) {
        int local;
        System.out.println(field); // 0
        System.out.println(local); // コンパイルエラー
    }
}

型は「扱い方」を決める

型は単なるラベルではない。その変数に何を代入できるか、どんな演算ができるか、どのメソッドを呼べるかを決める。

コード結果理由
int x = 10;OKintに整数値を代入できる
int x = null;エラープリミティブ型にnullは入らない
String s = null;OK参照型はnullを持てる
boolean b = 1;エラーJavaのbooleanは数値ではない

規則に反する例と修正

成立しない例：初期化前に使っている

public class Main {
    public static void main(String[] args) {
        int count;
        count++;
        System.out.println(count);
    }
}

修正例

public class Main {
    public static void main(String[] args) {
        int count = 0;
        count++;
        System.out.println(count);
    }
}

宣言・初期化・再代入の最小例

`Main.java`として保存する完全なコード

public class Main {
    static int field;

    public static void main(String[] args) {
        int local = 10;
        local = 20;
        System.out.println(field);
        System.out.println(local);
    }
}

fieldはintの初期値0を持ちます。localは宣言時に10で初期化され、その後20へ再代入されます。

関連する用語との違い

混同しやすいもの違い見分け方
ローカル変数 / フィールドローカル変数は自動初期化されない。フィールドは初期値を持つ。宣言場所を見る。
値 / 参照intは値、Stringや配列は参照。型がプリミティブか参照型かを見る。
代入 / コピー代入は変数の中身をコピーする。参照型なら参照がコピーされる。オブジェクト自体が複製されるとは限らない。

変数へ値を入れるときの規則

  1. 宣言された型を確認する。
  2. 右辺の式を評価し、その結果の型を決める。
  3. 右辺の型が左辺へ代入可能か確認する。
  4. ローカル変数は読み出す前に必ず初期化する。
  5. 参照型では、変数と参照先のオブジェクトを分けて考える。

確認用コード

public class Main {
    static int field;

    public static void main(String[] args) {
        int local;
        System.out.println(field);
        System.out.println(local);
    }
}
見る点このコードでの判断結論
fieldstaticフィールドデフォルト値0で初期化済み
localローカル変数初期化されていない
結論local使用で失敗コンパイルエラー

変数には型に応じたものが入る

変数を単なる「箱」として覚えるだけでは、参照型の動きを説明できません。プリミティブ型の変数には値が入り、参照型の変数にはオブジェクトへたどるための参照が入ります。

見る対象見ること判断
宣言int n; / String s;変数名と型が決まる。まだ使える値があるとは限らない。
初期化n = 10; / s = "Java";その変数を読んでよい状態になる。
再代入n = 20; / s = "Java";変数が保持する値、または参照が差し替わる。
使用System.out.println(n);ここまでに必ず初期化されている必要がある。

特にローカル変数は、宣言しただけでは使えません。フィールドには型ごとの初期値がありますが、ローカル変数には自動初期化がありません。
この差を曖昧にしたまま進むと、後のメソッド、コンストラクタ、クラスの単元でも混乱します。

public class Sample {
    int field; // フィールド。intの初期値0が入る

    public void test() {
        int local; // ローカル変数。まだ使えない
        // System.out.println(local); // コンパイルエラー
        System.out.println(field);    // 0
    }
}
ここで整理する誤解

「変数を宣言したら何か値が入っている」は雑すぎます。フィールドなのかローカル変数なのか、プリミティブ型なのか参照型なのかで話が変わります。

定義と規則の整理

変数・値・型で混ざる点

つまずき正しい考え方
変数そのものが値だと思う変数は値を入れる入れ物または参照を保持する名前。値そのものではない。
型を「見た目」だけで判断するリテラルには基本型がある。整数は基本的にint、小数は基本的にdouble。
ローカル変数の初期値を期待するローカル変数は自動初期化されない。使う前に明示的な代入が必要。

確認できれば次へ

  • 値・型・変数の違いを説明できる
  • ローカル変数とフィールドの初期値の違いを説明できる
  • リテラルの基本型を答えられる
理解の基準

成立するコードと成立しないコード

成立する例：変数・値・型

int age = 20;
String name = "Java";

成立しない例

int age = "20";
String name = 100;

成立する形

int age = 20;
String name = "100";
確認ポイント

変数には型があります。代入できる値は、その型と互換性がある必要があります。

定義と規則を確認する

用語の確認

変数、値、型、リテラルを分けて説明できますか。

説明を見る

変数は値を入れる名前、値は実データ、型は扱い方のルール、リテラルはコード上に直接書いた値です。

コードの確認

次のコードはコンパイルできますか。

int a;
System.out.println(a);
説明を見る

コンパイルできません。ローカル変数は使う前に初期化が必要です。フィールドの自動初期値とは分けます。

仕組みを説明する

int a = 10; のうち、型・変数名・値はどれですか。

説明を見る

intが型、aが変数名、10が値です。10は整数リテラルでもあります。

基本ルールのまとめ

  • プリミティブ型と参照型の違いを説明できる。
  • ローカル変数は使用前に代入が必要だと判断できる。
  • フィールドには型ごとの初期値が入ると理解している。

コードに規則を当てはめる

次のコードはコンパイルできるか。できない場合は理由を説明する。

確認用コード

public class Main {
    public static void main(String[] args) {
        int x;
        String s = null;
        System.out.println(s);
        System.out.println(x);
    }
}
説明を見る

コンパイルできない。

sは参照型なのでnullを代入できます。コンパイルエラーの原因はxです。ローカル変数xは初期化されないまま使われているため、コンパイルエラーになる。
