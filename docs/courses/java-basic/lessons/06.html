数値型と型変換

整数型と浮動小数点型の範囲、リテラルの型、数値昇格、キャストの規則を説明します。

このレッスンのゴール

数値式の型がどのように決まり、暗黙変換または明示的キャストが必要になる条件を説明できること。

基本用語の定義

数値の見た目だけでは代入可否を判断できません。リテラルと式には型があり、演算時にも型変換が行われます。

リテラルとはコードに直接書いた値。例: 10、10L、"abc"。
型変換とはある型の値を別の型として扱うこと。
キャストとは明示的に型変換を指定すること。例: (int) 3.5。
オーバーフローとは型の範囲を超えて値が回り込むこと。
数値昇格とは演算時に小さい数値型がintなどへ広がること。

数値型・リテラル・型変換の規則

項目定義・規則具体例
整数型byte、short、int、long。符号付き整数を表します。整数リテラルの基本型はintです。
浮動小数点型floatとdouble。小数を近似値として表します。小数リテラルの基本型はdoubleです。
拡大変換表現範囲の狭い型から広い型への変換。intからlongは通常キャスト不要です。
縮小変換表現範囲の広い型から狭い型への変換。longからintには通常キャストが必要です。
二項数値昇格算術演算の前にオペランドの型をそろえる規則。byte + byteの結果型はintです。
複合代入演算後の値を左辺型へ暗黙に変換して再代入します。b += 1はb = (byte)(b + 1)に近い動作です。

代表的な数値型

型大まかな役割注意点
byte小さい整数計算時はintになりやすい
short小さい整数リテラル代入と計算結果代入を分ける
int整数の基本整数リテラルの基本型
long大きい整数Lを付けることがある
float単精度小数Fを付けないと小数リテラルは基本double
double小数の基本小数リテラルの基本型

リテラル代入と計算結果代入は同じ扱いにならない

byte b = 10;は通る。10がbyteの範囲内で、定数として明らかだから。しかしbyte c = b + 1;は通らない。b + 1の計算結果はintとして扱われるから。

成立しない例

byte b = 10;
byte c = b + 1; // コンパイルエラー

修正例

byte b = 10;
byte c = (byte) (b + 1); // 明示的にキャスト

複合代入は暗黙のキャストを含む

+=などの複合代入は、単純な+とは挙動が違う。x += 1は、計算後に左辺の型へ戻す処理を含む。

通常の加算

byte x = 1;
x = x + 1; // コンパイルエラー

複合代入

byte x = 1;
x += 1; // OK

小数リテラルの注意点

小数リテラルは基本的にdouble。float f = 1.0;は、doubleをfloatへ入れようとしているためコンパイルエラーになる。

float f1 = 1.0;   // コンパイルエラー
float f2 = 1.0F;  // OK
double d = 1.0;   // OK

数値式の型を確認する

`Main.java`として保存する完全なコード

public class Main {
    public static void main(String[] args) {
        byte left = 10;
        byte right = 20;
        int sum = left + right;
        long total = sum;
        double average = total / 2.0;
        System.out.println(average);
    }
}

left + rightでは両方のbyteがintへ数値昇格するため、結果型はintです。2.0はdoubleなので、除算結果もdoubleになります。

関連する用語との違い

混同しやすいもの違い見分け方
代入可能 / キャスト必要狭い型へ広い型を入れるなら基本キャストが必要。左辺と右辺の型を見る。
整数 / 小数10は整数、10.0は小数。小数はdouble扱いが基本。
リテラル / 変数定数リテラルは範囲内なら特別に代入できる場合がある。変数計算は別。byte b = 10;とbyte b = a + 1;を区別する。

数値式の型を決める順序

  1. 各リテラルと変数の型を確認する。
  2. 単項演算・二項演算で行われる数値昇格を適用する。
  3. 式全体の結果型を決める。
  4. 代入先の型へ暗黙変換できるか確認する。
  5. 縮小変換が必要なら、値の欠落や桁あふれを理解したうえでキャストする。

確認用コード

byte a = 10;
byte b = 20;
byte c = a + b;
見る点このコードでの判断結論
a + bbyte同士の計算intへ昇格
左辺byte cintをそのまま入れられない
結論暗黙変換不可コンパイルエラー

右辺の型を先に確認する

数値が範囲内に見えても、式の型が代入先と合わなければコンパイルできません。リテラルだけの定数式と、変数を含む計算式も同じ扱いではありません。

byte a = 10;       // OK。10はbyteの範囲内
byte b = 20;       // OK
byte c = 10 + 20;  // OK。定数式として30と判断できる
// byte d = a + b; // コンパイルエラー。byte同士の計算結果はint

a + bは、値としては30になりそうでも、式の型はintです。byteやshortは、計算時にintへ昇格する、と覚える必要があります。これは暗記ではなく、Javaが小さい整数型の計算を安全に扱うための基本ルールです。

コード見るポイント結果
short s = 100;整数リテラルが範囲内OK
short s = 100000;shortの範囲外コンパイルエラー
int i = 10L;longをintに入れようとしているコンパイルエラー
float f = 1.0;小数リテラルは基本doubleコンパイルエラー
float f = 1.0F;FでfloatリテラルにしているOK
計算結果の値だけで判断しない

右辺の計算結果が小さく見えても、そのまま代入できるとは限りません。整数リテラルだけの代入か、変数を含む式かを分け、式全体の型を確認します。

定義と規則の整理

型変換で混同しやすい点

つまずき正しい考え方
小さい型同士の演算結果も小さい型だと思うbyteやshortは演算時に基本的にintへ昇格する。
複合代入を単純代入と同じだと思うa += bには暗黙キャストが入る場合がある。a = a + bと完全同一ではない。
charを文字専用だと思うcharは数値的にも扱われる。ただしshortと自由に互換ではない。

確認できれば次へ

  • 拡大変換と縮小変換を区別できる
  • byte + byte の結果型を説明できる
  • +=でコンパイルが通る理由を説明できる
理解の基準

成立するコードと成立しないコード

成立する例：数値型と型変換

int a = 10;
long b = a;

成立しない例

long a = 10L;
int b = a;

成立する形

long a = 10L;
int b = (int) a;
確認ポイント

広い型から狭い型へ入れるときは、値が収まるかではなく型としてキャストが必要です。

定義と規則を確認する

用語の確認

拡大変換、縮小変換、数値昇格を説明できますか。

説明を見る

拡大変換は安全な方向の型変換、縮小変換は範囲が狭くなる変換、数値昇格は演算時にint以上などへ引き上げられる仕組みです。

コードの確認

次のコードはコンパイルできますか。

byte a = 1;
byte b = 2;
byte c = a + b;
説明を見る

コンパイルできません。byte同士の演算結果はintになります。byteへ戻すにはキャストが必要です。

仕組みを説明する

byte b = 127; は通るのに byte b = 128; が通らない理由を説明してください。

説明を見る

byteの範囲は-128〜127です。整数リテラルが範囲内なら代入できますが、128は範囲外です。

基本ルールのまとめ

  • 整数リテラルは基本int、小数リテラルは基本doubleだと説明できる。
  • byteやshortを含む計算結果がintになる場合を判断できる。
  • 拡大変換、縮小変換、キャストを使い分けられる。

コードに規則を当てはめる

次のうち、なぜコンパイルエラーになるかを説明する。

確認用コード

short s = 10;
s = s + 1;
説明を見る

s + 1の結果はshortではなくintとして扱われる。

intをshort変数へそのまま代入できないため、コンパイルエラーになる。s = (short)(s + 1);のような明示的キャストが必要。
