Java基礎 準備

Javaを動かす準備をする

JDKを入れ、コマンドで確認し、最初のプログラムを動かします。最初は道具の名前より、何をどの順番で使うかを押さえます。

ここまでできればOK

自分のPCでJavaファイルをコンパイルし、実行結果を表示できるようになります。

まず押さえる言葉

環境構築では、何をインストールするかだけでなく、どの道具がどの処理を担当するかを分けて理解します。

JDKとはJavaを開発するための道具一式。javac や java が含まれる。
エディタとはソースコードを書くための道具。VS Codeやメモ帳など。
ターミナルとは文字でコマンドを入力して、コンパイルや実行を行う画面。
作業フォルダとはJavaファイルを置いて作業する場所。コマンド実行時の現在地と一致させる。

開発に必要な道具とファイルの基本

項目意味・使い方具体例
JDKJavaプログラムの開発と実行に必要なツールをまとめた配布物。javac、java、標準ライブラリなどを含みます。
javacJavaコンパイラ。ソースファイルを読み、classファイルを生成します。javac Hello.java
javaJavaランチャー。JVMを起動し、指定されたクラスのmainメソッドを呼び出します。java Hello
ソースファイル人が記述するJavaのテキストファイル。拡張子は.javaです。Hello.java
classファイルコンパイル後のバイトコードを格納するファイル。Hello.class
PATHOSがコマンドを探すフォルダの一覧。Java固有の機能ではありません。JDKのbinを検索対象にします。

最小のJavaプログラム

`Hello.java`として保存する完全なコード

public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello, Java");
    }
}

public class Helloの名前とファイル名Hello.javaを一致させます。コンパイル後はjava Helloとクラス名を指定して実行します。

準備から実行までの流れ

  1. java -versionでランチャーが見つかることを確認する。
  2. javac -versionでコンパイラが見つかることを確認する。
  3. ソースファイル名とpublicクラス名を一致させる。
  4. javacには.javaファイルを渡す。
  5. javaには通常、拡張子を付けないクラス名を渡す。

インストール後の確認

JDKを入れたら、PowerShellまたはターミナルで次を実行します。

java -version
javac -version

両方でバージョンが表示されれば、最低限の準備はできています。

状態意味対応
java -versionだけ通る実行環境だけ見えていて、コンパイラが見えていない可能性JDKを入れたか、PATHが通っているか見る
javacが認識されないJDK未インストール、またはPATH設定の問題JDKを入れ直すか、インストーラーのPATH設定を見る
古いバージョンが出る別のJavaが先に見つかっている複数JDKが入っていないか見る

作業フォルダを作る

最初はパスで詰まらないように、英数字だけの短いフォルダ名にします。

mkdir C:\java-work
cd C:\java-work

日本語や空白を含むフォルダ名でも動くことはありますが、最初の段階では余計な原因を減らす方が分かりやすいです。

最初に動かすコード

C:\java-workにMain.javaを作り、次を保存します。

public class Main {
    public static void main(String[] args) {
        System.out.println("Hello Java");
    }
}

保存したら、同じフォルダで次を実行します。

javac Main.java
java Main

Hello Javaと表示されれば成功です。

ここで覚えること

javacはファイル名を指定します。javaはクラス名を指定します。java Main.classではありません。

JDK / JVM / javac / java の役割

用語役割大まかに言うと
JDKJava開発に必要な道具一式開発キット
JVM.classを実行する仮想マシンJavaを動かす実行基盤
javac.javaを.classへ変換するコンパイラ
javaJVMを起動し、指定したクラスのmainを実行する実行コマンド

この違いを曖昧にしたまま進むと、エラーが出たときに原因を切り分けられません。

やらない方がいいこと

非公式サイトからJDKを入れる

開発環境は公式配布元から入れる。余計なインストーラーを踏む必要はありません。

複数のJDKを同時に入れる

最初から複数入れると、どのjavaやjavacを使っているのか分からなくなります。

エディタの実行ボタンだけに頼る

ボタン実行は便利ですが、裏で何が起きているか見えにくいです。最初はjavacとjavaを一度は手で実行してください。

ここまでの要点

  • java と javac の役割を分けて確認できる。
  • ソースファイルをコンパイルしてから実行できる。
  • ターミナルの現在地とファイルの保存場所を確認できる。
