Java基礎 準備

始める準備

Javaを作成して実行するために必要な道具と、各コマンドの役割を定義します。

このページのゴール

JDK、javac、java、ソースファイル、classファイルの役割を区別し、最小のプログラムを実行できること。

基本用語の定義

環境構築では、何をインストールするかだけでなく、どの道具がどの処理を担当するかを分けて理解します。

JDKとはJavaを開発するための道具一式。javac や java が含まれる。
エディタとはソースコードを書くための道具。VS Codeやメモ帳など。
ターミナルとは文字でコマンドを入力して、コンパイルや実行を行う画面。
作業フォルダとはJavaファイルを置いて作業する場所。コマンド実行時の現在地と一致させる。

開発に必要な道具とファイルの定義

項目定義・規則具体例

最小のJavaプログラム

`Hello.java`として保存する完全なコード

public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello, Java");
    }
}

public class Helloの名前とファイル名Hello.javaを一致させます。コンパイル後はjava Helloとクラス名を指定して実行します。

準備から実行までの規則

  1. java -versionでランチャーが見つかることを確認する。
  2. javac -versionでコンパイラが見つかることを確認する。
  3. ソースファイル名とpublicクラス名を一致させる。
  4. javacには.javaファイルを渡す。
  5. javaには通常、拡張子を付けないクラス名を渡す。

インストール後の判定

JDKを入れたら、PowerShellまたはターミナルで次を実行します。

java -version
javac -version

両方でバージョンが表示されれば、最低限の準備はできています。

状態意味対応
java -versionだけ通る実行環境だけ見えていて、コンパイラが見えていない可能性JDKを入れたか、PATHが通っているか見る
javacが認識されないJDK未インストール、またはPATH設定の問題JDKを入れ直すか、インストーラーのPATH設定を見る
古いバージョンが出る別のJavaが先に見つかっている複数JDKが入っていないか見る

作業フォルダを作る

最初はパスで詰まらないように、英数字だけの短いフォルダ名にします。

mkdir C:\java-work
cd C:\java-work

日本語や空白を含むフォルダ名でも動くことはありますが、最初の段階では余計な原因を減らす方が合理的です。

最初に動かすコード

C:\java-workにMain.javaを作り、次を保存します。

public class Main {
    public static void main(String[] args) {
        System.out.println("Hello Java");
    }
}

保存したら、同じフォルダで次を実行します。

javac Main.java
java Main

Hello Javaと表示されれば成功です。

ここで覚えること

javacはファイル名を指定します。javaはクラス名を指定します。java Main.classではありません。

JDK / JVM / javac / java の役割

用語役割大まかに言うと
JDKJava開発に必要な道具一式開発キット
JVM.classを実行する仮想マシンJavaを動かす実行基盤
javac.javaを.classへ変換するコンパイラ
javaJVMを起動し、指定したクラスのmainを実行する実行コマンド

この違いを曖昧にしたまま進むと、エラーが出たときに原因を切り分けられません。

やらない方がいいこと

非公式サイトからJDKを入れる

開発環境は公式配布元から入れる。余計なインストーラーを踏む必要はありません。

複数のJDKを同時に入れる

最初から複数入れると、どのjavaやjavacを使っているのか分からなくなります。

エディタの実行ボタンだけに頼る

ボタン実行は便利ですが、裏で何が起きているか見えにくいです。最初はjavacとjavaを一度は手で実行してください。

定義と規則の整理

環境構築で止まる典型

つまずき正しい考え方
java と javac を同じものとして扱うjavaは実行、javacはコンパイル。片方だけ通る状態は普通に起きる。
エディタの実行ボタンだけで進める裏でどのフォルダ・どのJDK・どのクラス名で実行しているか見えない。最初は手打ちで判定する。
作業フォルダを意識しないターミナルの現在地とファイルの置き場所がずれると、文法が正しくても動かない。

確認できれば次へ

  • java -version と javac -version の違いを説明できる
  • javac Main.java と java Main の指定対象を説明できる
  • 作業フォルダを変えると何が変わるか説明できる
理解の基準

成立するコードと成立しないコード

成立する例：JDKの判定

java -version
javac -version

成立しない例：classファイルをjavacへ渡している

javac Sample.class

成立する形：ソースをコンパイルしてから実行する

javac Sample.java
java Sample
確認ポイント

Javaを動かす準備では、まず実行用のjavaとコンパイル用のjavacが使えるかを分けて判定します。

基本ルールのまとめ

  • java と javac の役割を分けて確認できる。
  • ソースファイルをコンパイルしてから実行できる。
  • ターミナルの現在地とファイルの保存場所を確認できる。

定義と規則を確認する

用語の確認

JDK、JVM、javac、javaの役割をそれぞれ説明できますか。

説明を見る

JDKは開発キット、JVMはclassファイルを実行する仕組み、javacはコンパイル、javaは実行です。最初にこの4つを分けます。

コードの確認

次のコマンドのうち、環境判定として不足しているものはどれですか。

java -version
説明を見る

javaだけでは実行環境の判定に寄ります。開発ではjavacも必要なので、javac -versionも判定します。

仕組みを説明する

javaは使えるのにjavacが使えない場合、何が起きていますか。

説明を見る

JRE相当の実行環境だけが見えている、またはJDKのbinへのPATHが通っていない可能性があります。コンパイルできる状態ではありません。

コードに規則を当てはめる

次のうち、Javaを始める準備として本当に必要なものはどれか。

  • JDK
  • エディタ
  • 作業フォルダ
  • 有料IDE
  • Webサーバー
説明を見る

最初に必要なのは、JDK、エディタ、作業フォルダです。有料IDEやWebサーバーは不要です。

Javaの基礎学習では、.javaを書き、javacでコンパイルし、javaで実行できれば十分です。
