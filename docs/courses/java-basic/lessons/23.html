Java基礎 Lesson 23

スタック・ヒープ・参照

参照型のコードは、変数とオブジェクトを別々に描くと分かりやすくなります。スタックとヒープはそのための学習用モデルとして使います。

ここまでできればOK

参照の共有、null代入、GC対象になる条件を追えるようになります。

まず押さえる言葉

スタックとヒープはJVMの実行時データ領域を理解するためのモデルです。特定JVMの物理配置や最適化を断定せず、仕様上の役割を中心に扱います。

メモリとは実行中の値やオブジェクトを保持する領域。
スタックとは学習上は、主にメソッド呼び出しやローカル変数を追うための領域として扱う。
ヒープとは学習上は、主にnewされたオブジェクトを追うための領域として扱う。
nullとは参照型変数がどのオブジェクトも指していない状態。
ガベージコレクションとは参照されなくなったオブジェクトをJVMが回収する仕組み。

JVMの実行時データ領域と参照

項目意味・使い方具体例
ヒープすべてのスレッドで共有され、オブジェクトや配列が割り当てられる実行時領域。ガベージコレクションの対象です。
Java仮想マシンスタックスレッドごとに作られ、メソッド呼び出しごとのフレームを保持します。別スレッドとは共有しません。
スタックフレーム1回のメソッド呼び出しに対応するデータ領域。ローカル変数や演算途中の値を扱います。
参照オブジェクトへ到達するための値。複数の変数が同じオブジェクトを参照できます。
到達可能性実行中のルートから参照をたどってオブジェクトへ到達できる状態。到達不能になったオブジェクトは回収候補です。
ガベージコレクション到達不能なオブジェクトが使う領域を自動的に回収する仕組み。実行時期は保証されず、System.gc()も要求にすぎません。

スタックとヒープの役割

領域何を置くと読むか学習上の使い道
スタック学習上は、メソッド呼び出しやローカル変数など今どのメソッドを実行中か、ローカル変数が何を持つかを見る
ヒープ学習上は、newされたオブジェクトや配列参照型変数が何を指しているかを見る

参照型の図解を言葉で書く

Item a = new Item();
Item b = a;

このとき、ヒープ上にItemオブジェクトが1つ作られる。スタック上のaとbは、その同じオブジェクトを指す参照を持つ。

b = new Item();

ここでbだけが新しいItemを指すようになる。aが指す先は変わらない。

プリミティブ型は値のコピー

int a = 10;
int b = a;
b = 20;
System.out.println(a); // 10

intでは値がコピーされる。bを変えてもaは変わらない。参照型の「同じオブジェクトを見ている」動きとは別です。

ガベージコレクションの入口

どこからも参照されなくなったオブジェクトは、将来的にガベージコレクションの対象になる。いつ回収されるかをプログラマが正確に決めるわけではない。

Object a = new Object();
a = null; // さきほどのObjectは参照されなくなる

参照の共有と到達可能性

`Main.java`として保存する完全なコード

class Box {
    int value;
}

public class Main {
    public static void main(String[] args) {
        Box first = new Box();
        first.value = 10;

        Box second = first;
        first = null;

        System.out.println(second.value);
    }
}

first = null後もsecondからBoxへ到達できるため、そのオブジェクトは回収候補になりません。出力は10です。

似た言葉との違い

混同しやすいもの違い見分け方
変数 / オブジェクト参照型変数は参照を持つ。オブジェクト本体は別にある。newされたものと変数を分ける。
参照あり / 参照なしどこかの変数などから到達できるかどうか。代入やnullで参照が外れたか見る。
GC対象 / 即回収対象になっても即座に回収されるとは限らない。「対象」と「実行タイミング」を分ける。

参照とオブジェクトを追う順番

  1. メソッド呼び出しごとにローカル変数の範囲を分ける。
  2. newで生成されるオブジェクトを別に記録する。
  3. 各変数がどのオブジェクトを参照しているか線で結ぶ。
  4. 代入やnull代入で参照の線がどう変わるか追う。
  5. どのルートからも到達できないオブジェクトだけを回収候補として考える。

確認用コード

Object a = new Object();
Object b = new Object();
Object c = a;
a = null;
b = null;
見る点このコードでの判断結論
1つ目のObjectcがまだ参照しているGC対象ではない
2つ目のObjectbがnullになり参照なしGC対象
結論対象は2つ目だけただし回収タイミングは未定

ここまでの要点

  • 変数とオブジェクト本体を分けて考えられる。
  • 参照型変数の代入で参照がコピーされると説明できる。
  • GC対象になることと、直ちに回収されることを区別できる。
