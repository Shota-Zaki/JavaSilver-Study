Stringの基本

Stringを文字列オブジェクトとして定義し、不変性、参照比較、内容比較、連結の規則を説明します。

このレッスンのゴール

Stringメソッドが元のオブジェクトを書き換えないことと、`==`と`equals`の役割を説明できること。

基本用語の定義

String変数は文字列そのものを内包するのではなく、Stringオブジェクトへの参照を保持します。

Stringとは文字列を表す参照型。内容は変更できない。
文字列リテラルとは"abc" のようにコードへ直接書く文字列。
文字列プールとは同じ文字列リテラルを再利用するための領域。
不変とはインスタンス生成後に内容を変更できないこと。Stringは不変。
equalsとはオブジェクトの内容比較に使うメソッド。

Stringオブジェクトの基本規則

項目定義・規則具体例
文字列リテラル二重引用符で記述するStringの値。"Java"
不変性Stringオブジェクトの内容は生成後に変更されません。toUpperCase()などは結果のStringを返します。
文字列プール同じ文字列リテラルを共有できる領域・仕組み。リテラル同士の参照が同じになる場合があります。
==2つの参照が同じオブジェクトを指すか比較します。文字列内容の比較には使いません。
equalsStringでは文字列の内容を比較します。a.equals(b)
連結一方がStringになると+は文字列連結として処理されます。左結合なので評価の途中結果を追います。

Stringは不変

replace、substring、toUpperCaseなどは、元のStringを書き換えない。戻り値を受け取らないと、結果は捨てられる。

戻り値を受け取らない例

String s = "java";
s.toUpperCase();
System.out.println(s); // java

戻り値を受け取る

String s = "java";
s = s.toUpperCase();
System.out.println(s); // JAVA

==とequals

==は、2つの変数が同じオブジェクトを参照しているかを比較します。equalsは文字列の内容を比較します。文字列リテラル同士で==がtrueになる場合があっても、内容比較にはequalsを使います。

String a = new String("java");
String b = "java";

System.out.println(a == b);      // false
System.out.println(a.equals(b)); // true

文字列連結

+にStringが混ざると文字列連結になる。左から評価されるため、数値計算と文字列連結が混ざると結果が変わる。

System.out.println(1 + 2 + "A"); // 3A
System.out.println("A" + 1 + 2); // A12

nullとの組み合わせ

String変数がnullのとき、その変数からメソッドを呼ぶとNullPointerExceptionになる。内容が空の文字列""とは別物。

値意味メソッド呼び出し
null何も参照していないs.length()で実行時例外
""長さ0の文字列オブジェクトs.length()は0
" "空白1文字の文字列s.length()は1

不変性と比較を確認する

`Main.java`として保存する完全なコード

public class Main {
    public static void main(String[] args) {
        String original = "java";
        String upper = original.toUpperCase();
        String copy = new String("java");

        System.out.println(original);
        System.out.println(upper);
        System.out.println(original == copy);
        System.out.println(original.equals(copy));
    }
}

toUpperCase()の戻り値をupperが参照し、originalは引き続き"java"を表します。copyは内容が同じでも別のオブジェクトです。

関連する用語との違い

混同しやすいもの違い見分け方
== / equals==は同じ参照か、equalsは内容が同じか。文字列内容を見たいならequals。
文字列リテラル / new Stringリテラルは文字列プールが絡む。newは新規オブジェクト。参照比較で差が出る。
変更 / 新規作成Stringは変更されず、新しい結果が作られる。返り値を代入しているか見る。

String式を判断する規則

  1. 変数が参照しているStringオブジェクトを区別する。
  2. メソッドの戻り値を受け取っているか確認する。
  3. ==なら参照、equalsなら内容を比較する。
  4. +を左から評価し、どの時点でString連結になるか確認する。
  5. 参照がnullの場合、インスタンスメソッド呼び出しでNullPointerExceptionが発生する。

確認用コード

String s = "Java";
s.concat("Java");
System.out.println(s);
見る点このコードでの判断結論
初期値sは"Java"を参照OK
concat新しい文字列を返すが代入していないsは変わらない
出力sの参照先は元のままJava

Stringは元の文字列を書き換えない

Stringは不変のクラスです。concatやsubstringなどを呼び出すと、元の文字列を変更するのではなく、結果となる別のStringを返します。

String s = "abc";
s.substring(1);
System.out.println(s); // abc

このコードでbcが表示されると思ったなら、まだStringを「書き換わる箱」として見ています。substringは新しい文字列を返すだけです。戻り値を受け取らない限り、変数sの参照先は変わりません。

String s = "abc";
s = s.substring(1);
System.out.println(s); // bc
比較何を見ているか使う場面
a == b同じオブジェクトを参照しているか参照そのものを判定したいとき
a.equals(b)文字列の内容が同じか通常の文字列比較
"x".equals(a)定数側から比較aがnullの可能性があるとき
ここで整理する誤解

==がtrueになる文字列の例を見て「文字列は==で比較できる」と結論づけるのは間違いです。たまたま同じ参照を見ているだけの場合があります。

定義と規則の整理

Stringで詰まる典型

つまずき正しい考え方
Stringが変更されると思うStringは不変。replaceやsubstringは新しい文字列を返す。
==で文字列内容比較をする==は参照比較。内容比較は通常equals。
インデックスの終端を含むと思うsubstring(begin,end)のendは含まない。

確認できれば次へ

  • Stringが不変である意味を説明できる
  • ==とequalsの違いを説明できる
  • substringの第2引数が含まれないことを説明できる
理解の基準

成立するコードと成立しないコード

成立する例：戻り値を別変数で受け取る

String s = "Java";
String upper = s.toUpperCase();
System.out.println(upper);

成立しない例：戻り値を捨てている

String s = "Java";
s.toUpperCase();
System.out.println(s);

成立する形：戻り値を元の変数へ代入する

String s = "Java";
s = s.toUpperCase();
System.out.println(s);
確認ポイント

Stringのメソッドは元の文字列を変更せず、新しい結果を返すものが多いです。

定義と規則を確認する

用語の確認

Stringが不変であるとはどういう意味ですか。

説明を見る

Stringオブジェクトの中身は変更されません。replaceやsubstringは新しいStringを返します。

コードの確認

次の出力は何ですか。

String s = "abc";
s.replace("a", "x");
System.out.println(s);
説明を見る

abcです。replaceの戻り値を受け取っていないため、変数sの参照先は変わりません。

仕組みを説明する

== と equals の違いをStringで説明できる状態にします。

説明を見る

==は参照先が同じかを比べ、equalsは文字列の内容を比べます。new Stringで作ったオブジェクトと文字列リテラルは、内容が同じでも同じ参照とは限りません。

基本ルールのまとめ

  • Stringが参照型であり、不変だと説明できる。
  • 内容比較にはequalsを使う理由を説明できる。
  • 文字列操作メソッドの戻り値を受け取る必要があると判断できる。

コードに規則を当てはめる

次の出力を答え、理由を説明する。

確認用コード

String a = "ab";
String b = new String("ab");
System.out.println(a == b);
System.out.println(a.equals(b));
説明を見る

1行目はfalse。aとbは同じ内容だが、参照先が同じとは限らない。new Stringで別オブジェクトを作っている。

2行目はtrue。equalsは文字列内容を比較するため。
