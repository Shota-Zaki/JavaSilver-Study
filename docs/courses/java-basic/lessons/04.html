Java基礎 Lesson 4

mainメソッド

mainメソッドを、JVMがプログラムの開始位置として探すための宣言規則として説明します。

このレッスンのゴール

起動用mainメソッドの各要素と、受け付けられる宣言形を説明できること。

基本用語の定義

mainは特別なキーワードではなく、JVMのランチャーが規定の形で探すstaticメソッドです。

mainメソッドとはJavaアプリケーションを起動したときに最初に呼ばれる入口。
publicとは外部からアクセスできることを示すアクセス修飾子。
staticとはインスタンスを作らなくてもクラス側から使えること。
voidとは戻り値がないことを表す戻り値型。
String[] argsとはコマンドライン引数を受け取るString配列。

起動用mainメソッドの宣言規則

項目定義・規則具体例
publicランチャーから呼び出せるアクセス範囲にする修飾子。省略すると起動用mainとして扱えません。
staticインスタンスを生成せずクラスに対して呼び出せることを表します。JVMはMainオブジェクトを作らずに呼び出します。
void呼び出し元へ値を返さない戻り値型。int mainは起動用mainではありません。
mainランチャーが探すメソッド名。大文字・小文字を区別します。
String[]コマンドライン引数を文字列の配列として受け取る型。変数名はargs以外でもかまいません。
可変長引数String... argsもString[] argsと同等の引数型として使えます。起動用mainとして認識されます。

起動用mainメソッドの最小形

`Main.java`として保存する完全なコード

public class Main {
    public static void main(String[] args) {
        System.out.println("arguments: " + args.length);
    }
}

java Main A Bと実行すると、argsは{"A", "B"}に相当する2要素の配列を参照します。

mainを分解する

部分意味なぜ必要か
public外部から呼び出せるJVMがクラス外から呼び出すため
staticインスタンスなしで呼べるプログラム開始前にオブジェクトを作らなくてよいため
void戻り値なし入口メソッドとして戻り値を返さない
main決められた名前JVMがこの名前を探す
String[] argsコマンドライン引数起動時に渡した文字列を受け取る

argsの読み方

argsは配列。java Main apple bananaと実行すると、args[0]が"apple"、args[1]が"banana"になる。Main自体はargsには入らない。

java Main apple で実行すると apple

public class Main {
    public static void main(String[] args) {
        System.out.println(args[0]);
    }
}
引数なしの注意点

引数を渡さずにargs[0]を読むと、配列の範囲外アクセスになり実行時例外が起きる。コンパイルエラーではない。

許される形と紛らわしい形

形入口として認識されるか補足
public static void main(String[] args)される基本形
public static void main(String... args)される可変長引数も配列として扱える
static public void main(String[] x)される修飾子の順序や引数名は本質ではない
public void main(String[] args)されないstaticがない
public static int main(String[] args)されない戻り値型がvoidではない

staticがあることの影響

mainはstaticなので、同じクラスのインスタンスフィールドを直接読めません。インスタンス側の値を使いたいなら、オブジェクトを作ってから参照する。

成立しない例

public class Main {
    int value = 10;

    public static void main(String[] args) {
        System.out.println(value); // コンパイルエラー
    }
}

修正例

public class Main {
    int value = 10;

    public static void main(String[] args) {
        Main m = new Main();
        System.out.println(m.value);
    }
}

関連する用語との違い

混同しやすいもの違い見分け方
mainメソッド / 普通のメソッドmainは入口としてJVMから呼ばれる。普通のメソッドは自分のコードから呼ぶ。メソッド名と引数の形を見る。
static / インスタンスstaticはクラス側。インスタンスフィールドはオブジェクト側。mainから直接使えるかで判断する。
String[] args / 出力argsは入力側。System.out.printlnは出力側。コマンドから入るのがargs、画面に出るのが出力。

mainを判定する規則

  1. メソッド名が小文字のmainか確認する。
  2. 戻り値型がvoidか確認する。
  3. publicとstaticが付いているか確認する。
  4. 引数型がString[]またはString...か確認する。
  5. メソッド本体ではstatic文脈のため、インスタンスメンバを直接参照できないことを確認する。

確認用コード

public class Main {
    int value = 10;

    public static void main(String[] args) {
        System.out.println(value);
    }
}
見る点このコードでの判断結論
mainの形入口としての形はあるここはOK
valueインスタンスフィールドstaticなmainから直接参照不可
結論インスタンス生成がないコンパイルエラー

mainはJVMが探す起動入口

mainは単なる決まり文句ではなく、javaコマンドで起動したときにJVMが探す入口です。名前、戻り値型、修飾子、引数の形が条件に合っている必要があります。

要素意味ずれるとどうなるか
publicJVMから呼び出せる入口として公開する入口として認識されない
staticインスタンス生成なしで呼べるJVMが最初に呼び出せない
void戻り値を返さない入口の形式と合わない
String[] argsコマンドライン引数を受け取る実行時引数を受け取れない
public static void main(String[] args) {
    System.out.println(args.length);
}

argsは「実行時に外から渡された文字列の配列」です。引数を渡さずに実行しても、args自体がnullになるわけではありません。要素数0の配列として扱われます。

java Main one two
// args[0] は "one"
// args[1] は "two"

ここでjava Mainという実行コマンドのMainはクラス名であり、argsには入りません。これも混同しやすいので、早い段階で切り分けます。

ここで整理する誤解

mainは「よく見る形」だから正しいのではありません。JVMが入口として探す条件を満たしているから実行されます。

定義と規則の整理

mainメソッドで混同しやすい点

つまずき正しい考え方
mainなら何でも起動できると思う起動対象は原則 public static void main(String[] args) または同等形。
String args[] と String[] args を別物と思うどちらも配列引数として有効。ただし読みやすさではString[] argsが標準的。
引数なしならargsがnullだと思う引数なしでもargsは長さ0の配列。args[0]は範囲外例外。

確認できれば次へ

  • mainメソッドに必要な4要素を言える
  • String... と String[] の関係を説明できる
  • 起動引数なしのargs.lengthを答えられる
理解の基準

成立するコードと成立しないコード

成立する例：mainメソッドの可変長引数

public static void main(String... args) {
    System.out.println("start");
}

成立しない例：staticがないため起動用mainにならない

public void main(String[] args) {
    System.out.println("start");
}

成立する形：public static void mainにする

public static void main(String[] args) {
    System.out.println("start");
}
確認ポイント

mainはJVMから呼び出される入口です。staticがないと通常の起動入口として扱えません。

定義と規則を確認する

用語の確認

mainメソッドは何のためにありますか。

説明を見る

javaコマンドでプログラムを開始するときの入口です。mainがあるクラスを指定して実行します。

コードの確認

次のmainは入口として使えますか。

public static void main(String args) {
    System.out.println("A");
}
説明を見る

使えません。引数はString[] argsまたはString... argsである必要があります。String args単体は入口のmainになりません。

仕組みを説明する

mainがあるだけで何でも直接は使えない理由を説明してください。

説明を見る

javaコマンドが探すのは決まった形のpublic static void main(String[] args)です。
名前がmainでも、戻り値・static・引数の形がずれると入口になりません。

基本ルールのまとめ

  • 起動用mainの基本形を説明できる。
  • argsは起動時に渡された文字列の配列だと理解している。
  • staticなmainからインスタンスメンバを直接使えない理由を説明できる。

コードに規則を当てはめる

次のコードでnameを表示したい。何を直すべきか。

確認用コード

public class Main {
    String name = "Java";

    public static void main(String[] args) {
        System.out.println(name);
    }
}
説明を見る

mainはstaticなので、nameを直接参照できない。

Main m = new Main();でインスタンスを作り、System.out.println(m.name);のように参照する。
