Java基礎 Lesson 4

mainメソッド

Javaアプリケーションは、決まった形のmainメソッドから始まります。宣言を部品ごとに分けて読みます。

ここまでできればOK

起動できるmainメソッドと、形が違うメソッドを見分けられるようになります。

まず押さえる言葉

mainは特別なキーワードではなく、JVMのランチャーが規定の形で探すstaticメソッドです。

mainメソッドとはJavaアプリケーションを起動したときに最初に呼ばれる入口。
publicとは外部からアクセスできることを示すアクセス修飾子。
staticとはインスタンスを作らなくてもクラス側から使えること。
voidとは戻り値がないことを表す戻り値型。
String[] argsとはコマンドライン引数を受け取るString配列。

mainメソッドの決まり

項目意味・使い方具体例
publicランチャーから呼び出せるアクセス範囲にする修飾子。省略すると起動用mainとして扱えません。
staticインスタンスを生成せずクラスに対して呼び出せることを表します。JVMはMainオブジェクトを作らずに呼び出します。
void呼び出し元へ値を返さない戻り値型。int mainは起動用mainではありません。
mainランチャーが探すメソッド名。大文字・小文字を区別します。
String[]コマンドライン引数を文字列の配列として受け取る型。変数名はargs以外でもかまいません。
可変長引数String... argsもString[] argsと同等の引数型として使えます。起動用mainとして認識されます。

起動用mainメソッドの最小形

`Main.java`として保存する完全なコード

public class Main {
    public static void main(String[] args) {
        System.out.println("arguments: " + args.length);
    }
}

java Main A Bと実行すると、argsは{"A", "B"}に相当する2要素の配列を参照します。

mainを分解する

部分意味なぜ必要か
public外部から呼び出せるJVMがクラス外から呼び出すため
staticインスタンスなしで呼べるプログラム開始前にオブジェクトを作らなくてよいため
void戻り値なし入口メソッドとして戻り値を返さない
main決められた名前JVMがこの名前を探す
String[] argsコマンドライン引数起動時に渡した文字列を受け取る

argsの読み方

argsは配列。java Main apple bananaと実行すると、args[0]が"apple"、args[1]が"banana"になる。Main自体はargsには入らない。

java Main apple で実行すると apple

public class Main {
    public static void main(String[] args) {
        System.out.println(args[0]);
    }
}
引数なしの注意点

引数を渡さずにargs[0]を読むと、配列の範囲外アクセスになり実行時例外が起きる。コンパイルエラーではない。

許される形と紛らわしい形

形入口として認識されるか補足
public static void main(String[] args)される基本形
public static void main(String... args)される可変長引数も配列として扱える
static public void main(String[] x)される修飾子の順序や引数名は本質ではない
public void main(String[] args)されないstaticがない
public static int main(String[] args)されない戻り値型がvoidではない

staticがあることの影響

mainはstaticなので、同じクラスのインスタンスフィールドを直接読めません。インスタンス側の値を使いたいなら、オブジェクトを作ってから参照する。

成立しない例

public class Main {
    int value = 10;

    public static void main(String[] args) {
        System.out.println(value); // コンパイルエラー
    }
}

修正例

public class Main {
    int value = 10;

    public static void main(String[] args) {
        Main m = new Main();
        System.out.println(m.value);
    }
}

似た言葉との違い

混同しやすいもの違い見分け方
mainメソッド / 普通のメソッドmainは入口としてJVMから呼ばれる。普通のメソッドは自分のコードから呼ぶ。メソッド名と引数の形を見る。
static / インスタンスstaticはクラス側。インスタンスフィールドはオブジェクト側。mainから直接使えるかで判断する。
String[] args / 出力argsは入力側。System.out.printlnは出力側。コマンドから入るのがargs、画面に出るのが出力。

mainメソッドの確認順

  1. メソッド名が小文字のmainか確認する。
  2. 戻り値型がvoidか確認する。
  3. publicとstaticが付いているか確認する。
  4. 引数型がString[]またはString...か確認する。
  5. メソッド本体ではstatic文脈のため、インスタンスメンバを直接参照できないことを確認する。

確認用コード

public class Main {
    int value = 10;

    public static void main(String[] args) {
        System.out.println(value);
    }
}
見る点このコードでの判断結論
mainの形入口としての形はあるここはOK
valueインスタンスフィールドstaticなmainから直接参照不可
結論インスタンス生成がないコンパイルエラー

ここまでの要点

  • 起動用mainの基本形を説明できる。
  • argsは起動時に渡された文字列の配列だと理解している。
  • staticなmainからインスタンスメンバを直接使えない理由を説明できる。
