Java基礎 Lesson 2

OS・JDK・JVM

OS、JDK、JVM、javaコマンド、PATHの責務を分けて説明します。

このレッスンのゴール

Javaの開発環境と実行環境を構成する要素を区別し、問題が起きた層を判断できること。

基本用語の定義

OS、JDK、JVMは包含関係ではなく、それぞれ異なる責務を持つ要素です。

OSとはWindowsやmacOSなど、アプリとハードウェアの間を管理する基本ソフト。
JDKとはJavaの開発キット。コンパイル用のjavacと実行用のjavaを含む。
JREとはJavaを実行するための環境を指す言葉です。現在の学習ではJDKを入れれば、実行に必要な部品もそろいます。
PATHとはコマンド名だけでプログラムを呼び出すための検索先設定。

OS・JDK・JVMが担当する範囲

項目定義・規則具体例
OSファイル、プロセス、メモリ、入出力装置などを管理する基本ソフトウェア。Windows、Linux、macOSなど。
JDKJavaの開発ツールと実行に必要な機能を含む配布物。コンパイラ、ランチャー、標準APIを含みます。
JVMclassファイルを実行する抽象的な計算機。OS・CPUごとに対応した実装があります。
javaコマンドJVMを起動するランチャー。指定されたクラスを検索してmainメソッドを呼び出します。
javacコマンドソースコードをコンパイルするツール。JDKに含まれます。
PATH入力されたコマンド名から実行ファイルを探すためのOS側の設定。PATHが誤っているとjavaやjavacを見つけられません。

JDKは「開発者用の工具箱」

Javaを書くなら基本的にJDKを入れる。JDKには、ソースコードをコンパイルするjavac、プログラムを起動するjava、標準ライブラリなどが含まれる。

インストール判定の例

javac -version
java -version

この2つのコマンドが動くなら、少なくともJavaをコンパイル・実行する入口は用意できている。

JVMは「Java用の実行環境」

JVMは.classファイルを読み込み、バイトコードを実行する。クラスを読み込む、メモリを確保する、例外を発生させる、不要になったオブジェクトを回収する、といった処理にも関わる。

重要

JVMがあるから、同じJavaコードを複数のOS上で動かしやすい。ただしOS差が完全に消えるわけではない。ファイルパスや文字コードなど、環境差が出る部分もある。

エラー切り分けの考え方

状況疑う場所例
javacが見つからないJDKのインストールまたはPATHjavac : 用語が認識されません
javacで失敗するソースコードの文法・型・アクセス; expected、cannot find symbol
java実行後に失敗する実行中の値・main・クラスパスNullPointerException、Could not find or load main class

成立しない例

JDKとJVMを同じものとして扱うと、次のような判断ミスが起きる。

  • javacがないのに、コードの文法ミスだと思い込む。
  • NullPointerExceptionなのに、コンパイル設定の問題だと思い込む。
  • java Main.classのように、実行コマンドの指定を間違える。

環境の状態をコマンドで確認する

ターミナルで実行する確認コマンド

java -version
javac -version

javaだけが見つかる状態と、javaとjavacの両方が見つかる状態は区別します。開発にはコンパイラを含むJDKが必要です。

関連する用語との違い

混同しやすいもの違い見分け方
JDK / JREJDKは開発用の道具を含みます。JREは実行環境を指す言葉で、学習ではJDKを用意すれば実行に必要な部品もそろいます。javacが必要ならJDK。
javac / javajavacはコンパイル、javaは実行。ファイル名を渡すか、クラス名を渡すかも違う。
環境エラー / コードエラーコマンドが見つからないのは環境。文法・型の指摘はコード。エラーメッセージの先頭を見る。

環境エラーを切り分ける規則

  1. コマンド自体が見つからない場合はPATHまたはインストール先を確認する。
  2. javaは見つかるがjavacが見つからない場合はJDKのbinを確認する。
  3. コンパイルエラーはJVM起動前の問題として扱う。
  4. クラスが見つからない場合はクラスパスと現在のフォルダを確認する。
  5. 実行中の例外は、JVM起動後にコードが実行された段階の問題として扱う。

確認用コード

javac -version
java -version
見る点このコードでの判断結論
javacコンパイラの判定動かなければJDK/PATHを疑う
java実行環境の判定動かなければPATHまたはインストールを疑う
両方OKコードをコンパイル・実行する準備がある次はファイル名とクラス名を見る

役割を分けると原因を探しやすい

OSはパソコン全体を管理し、JDKはJavaの開発に必要な道具を提供し、JVMはコンパイル済みのクラスを実行します。似た名前ですが、同じものではありません。

部品主な役割詰まったときに見る場面
OSファイル、プロセス、画面、入出力などを管理するパス、権限、コマンド実行、ファイル位置
JDKJava開発に必要な道具一式。javacも含むjavacが見つからない、コンパイルできない
JVMコンパイル済みのバイトコードを実行するjava実行時のクラス読み込みや例外

たとえばjavacが見つからないなら、コードの文法以前にJDKやPATHの問題です。一方、java Mainで例外が出るなら、JVMが実行を始めた後の問題です。この区別は、初心者が無駄に時間を溶かしやすい箇所です。

javac Main.java  // JDKのコンパイラを使う
java Main        // JVMでMainクラスを実行する

この2つのコマンドは似ていますが、担当が違います。javacは「作る」、javaは「動かす」です。動かす前に作れていないなら、実行結果を考えても意味がありません。

ここで整理する誤解

JDKとJVMを同じものとして扱うと、原因調査が大まかになります。開発道具の問題なのか、実行環境の問題なのか、コードの問題なのかを切り分けると判定しやすくなります。

定義と規則の整理

JDK・JVM周りの詰まり方

つまずき正しい考え方
JDKとJVMを同じ意味で使うJDKは開発道具一式。JVMはclassファイルを実行する仮想マシン。
JREを入れれば開発できると思うJREは実行環境。コンパイルするには通常JDKが必要。
PATHを「Javaの機能」と思うPATHはOSの設定。コマンド名だけで実行ファイルを探すための仕組み。

確認できれば次へ

  • JDKに含まれる代表的なコマンドを言える
  • JVMが実行する対象が.javaではなく.classだと説明できる
  • PATHが通っていないと何が起きるか説明できる
理解の基準

成立するコードと成立しないコード

成立する例：JDKが見えている状態

java -version
javac -version

javac Main.java
java Main

成立しない例：実行環境だけを見て開発環境もあると判断している

java -version

// javacが見えない状態でコンパイルしようとしている
javac Main.java

成立する形：javacも使えるJDKのbinをPATHへ通す

// 例: Windows PowerShell
$env:Path += ";C:\path\to\jdk\bin"  # 実際のJDKの場所に置き換える

javac Main.java
java Main
確認ポイント

JDKとJVMを混同すると、環境エラーの原因を見誤ります。

定義と規則を確認する

用語の確認

OS、JDK、JVMの関係を説明できますか。

説明を見る

OSはPC全体の基本ソフト、JDKはJava開発道具一式、JVMはJavaプログラムを実行する仮想マシンです。JDKの中に実行・コンパイル用の道具があります。

コードの確認

次の状態でJava開発は十分ですか。

java -version は表示される
javac -version はエラー
説明を見る

不十分です。実行環境は見えていますが、コンパイラが使えていません。JDKまたはPATHを判定します。

仕組みを説明する

PATHが通っていないと、なぜコマンドが見つからないのですか。

説明を見る

ターミナルはPATHに登録されたフォルダから実行ファイルを探します。javac.exeがある場所がPATHにないと、インストール済みでもコマンドとして見つかりません。

基本ルールのまとめ

  • OS、JDK、JVMの役割を分けて説明できる。
  • javacが見つからない場合は、コードより先にJDKやPATHを確認できる。
  • PATHとクラスパスを混同しない。

コードに規則を当てはめる

PowerShellでjavacが認識されず、javaは認識される。このとき最初に疑うべきものは何か。

説明を見る

最初に疑うのはJDKのインストール状態、またはPATH設定。

javaだけ使える状態でも、javacが使えないならコンパイルできない。コードの文法を直す前に環境を判定する。
