Java基礎 Lesson 24

ターミナル・javac・java

ターミナル、シェル、カレントディレクトリ、javac、java、クラスパス、出力先の規則を説明します。

このレッスンのゴール

コンパイル時のファイル指定と、実行時のクラス名・クラスパス指定を区別できること。

基本用語の定義

`javac`はソースファイルを処理し、`java`はJVMを起動してクラスを実行します。両者の引数と検索対象は異なります。

javacとはJavaソースファイルをコンパイルするコマンド。対象は基本的に.javaファイル。
javaとはコンパイル済みクラスやソースファイルモードを実行するコマンド。通常はクラス名を指定する。
-dとはコンパイル後の.classの出力先を指定するオプション。
-cpとは実行時やコンパイル時にクラスを探す場所を指定するオプション。
クラスパスとはJVMやjavacがクラスを探す検索場所。

コマンドと検索範囲の定義

項目定義・規則具体例
ターミナル文字でコマンドを入力・表示するための画面やアプリケーション。Windows Terminalなど。
シェル入力されたコマンドを解釈してプログラムを起動するソフトウェア。PowerShell、bashなど。
カレントディレクトリ相対パスの基準になる現在のフォルダ。pwdやcdで確認・変更します。
javac.javaソースをコンパイルするコマンド。-dはclassファイルの出力ルートを指定します。
javaJVMを起動してクラスまたはソースファイルを実行するコマンド。通常のクラス実行では拡張子を付けません。
クラスパスJVMやコンパイラがクラスを探すルートの集合。-cpまたは--class-pathで指定します。

コンパイルと実行のコマンド対応

カレントディレクトリに`Main.java`がある場合

javac -d out Main.java
java -cp out Main

# Java 11以降のソースファイルモード
java Main.java

通常のクラス実行ではjavaへMain.classを渡さず、クラスパス上のバイナリ名Mainを渡します。packageがある場合は完全修飾名を使います。

基本の作業手順

  1. 作業用フォルダを作る。
  2. Main.javaを保存する。
  3. ターミナルでそのフォルダへ移動する。
  4. javac Main.javaを実行する。
  5. Main.classができたか判定する。
  6. java Mainを実行する。

Windows PowerShellの例

cd C:\java-work
dir
javac Main.java
java Main

よくあるエラーの読み方

エラーの雰囲気主な原因
javacが認識されないJDK未インストール、またはPATH設定の問題
file not found今いるフォルダとファイルの場所が違う
class Main is public, should be declared...publicクラス名とファイル名が不一致
Could not find or load main class実行する場所、クラス名、パッケージ指定のどれかが違う
main method not foundmainメソッドの形が入口として認識されない

ファイル名と実行名を分ける

javac Main.java  // ファイル名を指定
java Main        // クラス名を指定。拡張子は付けない
大まかに覚えない

javacはファイルを対象にする。javaはクラスを対象にする。この違いが分かれば、かなりのエラーを自分で切り分けられる。

コードで確認する最小例

わざとjava Main.classを実行し、どんなエラーになるか判定する。その後java Mainで直す。エラー文を読む練習が目的。

関連する用語との違い

混同しやすいもの違い見分け方
cd / dircdは移動、dirは一覧表示。現在地を変えるのか、見るのか。
javac / java前者はコンパイル、後者は実行。ファイル名かクラス名か。
エラーメッセージ / 出力エラーは失敗の説明。出力はプログラムが表示した結果。誰が出した文字かを区別する。

コマンドエラーを調べる規則

  1. 現在のフォルダと入力ファイルの場所を確認する。
  2. javacにはソースファイルのパスを指定する。
  3. -dで生成されたclassファイルのルートを確認する。
  4. java -cpにはパッケージ階層より上のルートを指定する。
  5. javaには拡張子なしの完全修飾クラス名を指定する。

ターミナル操作を大量に覚える必要はありません。まず現在地を確認し、ソースファイルをコンパイルし、生成されたクラスを実行する手順を固定します。

ここで整理する誤解

ターミナルで詰まる原因の多くはJava文法ではなく、場所・名前・コマンドの指定ミスです。コードを疑う前に、実行環境の前提を見ると分かりやすいです。

定義と規則の整理

コマンド実行で最後に詰まる点

つまずき正しい考え方
-dをjavaコマンドに付ける-dはjavac側でclassファイルの出力先を指定するオプション。
package付きクラスを単純名で実行するpackageがあるなら完全修飾クラス名で実行する。例：java -cp out ex.Main。
クラスパスをソースの場所だと思う実行時のクラスパスはclassファイルを探す場所。ソースファイルの場所ではない。

確認できれば次へ

  • -dと-cpの役割を説明できる
  • package付きクラスの実行コマンドを書ける
  • javacとjavaの注意点を3つ挙げられる
理解の基準

成立するコードと成立しないコード

成立する例：カレントディレクトリでコンパイル・実行する

cd src
javac ex/Main.java
java ex.Main

成立しない例：javaコマンドにjavac用オプションを渡している

java -d out ex.Main
java out/ex/Main

成立する形：javacは-d、javaは-cpを使う

javac -d out src/ex/Main.java
java -cp out ex.Main
確認ポイント

-dはjavac側、-cpはjava側で使う代表的な指定です。package付きクラスは完全修飾名で起動します。

定義と規則を確認する

用語の確認

-d、-cp、packageの関係を説明できますか。

説明を見る

-dはclassファイルの出力先、-cpは実行時にclassを探す場所、packageは完全修飾クラス名とディレクトリ構造に関わります。

コードの確認

package ex; のMainをoutへコンパイルした後、正しい実行はどれですか。

java -cp out ex.Main
説明を見る

正しいです。package付きクラスは完全修飾クラス名で実行します。java -cp out Mainではありません。

仕組みを説明する

クラスパスがソースファイルの場所を指定しない理由を説明してください。

説明を見る

java実行時に必要なのはコンパイル済みのclassを探す場所です。ソースファイルの場所ではなく、class出力先を指定します。

基本ルールのまとめ

  • cdとdirで現在地とファイルを確認できる。
  • javacにはソースファイル、javaにはクラス名を指定できる。
  • package付きクラスを-dと-cpを使って実行できる。

コードに規則を当てはめる

次の状況で最初に見ることは何か。「javac Main.javaを実行したら、ファイルが見つからないと言われた」。

説明を見る

最初に見るのは、ターミナルの現在地とファイルの保存場所が一致しているか。

dirでMain.javaが表示されるか判定する。表示されないなら、cdで正しいフォルダへ移動するか、ファイルの保存場所を直す。
