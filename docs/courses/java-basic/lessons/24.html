Java基礎 Lesson 24

ターミナル・javac・java

ターミナルでJavaを動かすときは、現在のフォルダ、ファイル名、クラス名を順に確認します。

ここまでできればOK

javac、java、-d、-cpを用途に応じて使い分けられるようになります。

まず押さえる言葉

`javac`はソースファイルを処理し、`java`はJVMを起動してクラスを実行します。両者の引数と検索対象は異なります。

javacとはJavaソースファイルをコンパイルするコマンド。対象は基本的に.javaファイル。
javaとはコンパイル済みクラスやソースファイルモードを実行するコマンド。通常はクラス名を指定する。
-dとはコンパイル後の.classの出力先を指定するオプション。
-cpとは実行時やコンパイル時にクラスを探す場所を指定するオプション。
クラスパスとはJVMやjavacがクラスを探す検索場所。

コマンドと検索範囲の基本

項目意味・使い方具体例
ターミナル文字でコマンドを入力・表示するための画面やアプリケーション。Windows Terminalなど。
シェル入力されたコマンドを解釈してプログラムを起動するソフトウェア。PowerShell、bashなど。
カレントディレクトリ相対パスの基準になる現在のフォルダ。pwdやcdで確認・変更します。
javac.javaソースをコンパイルするコマンド。-dはclassファイルの出力ルートを指定します。
javaJVMを起動してクラスまたはソースファイルを実行するコマンド。通常のクラス実行では拡張子を付けません。
クラスパスJVMやコンパイラがクラスを探すルートの集合。-cpまたは--class-pathで指定します。

コンパイルと実行のコマンド対応

カレントディレクトリに`Main.java`がある場合

javac -d out Main.java
java -cp out Main

# Java 11以降のソースファイルモード
java Main.java

通常のクラス実行ではjavaへMain.classを渡さず、クラスパス上のバイナリ名Mainを渡します。packageがある場合は完全修飾名を使います。

基本の作業手順

  1. 作業用フォルダを作る。
  2. Main.javaを保存する。
  3. ターミナルでそのフォルダへ移動する。
  4. javac Main.javaを実行する。
  5. Main.classができたか確認する。
  6. java Mainを実行する。

Windows PowerShellの例

cd C:\java-work
dir
javac Main.java
java Main

よくあるエラーの読み方

エラーの雰囲気主な原因
javacが認識されないJDK未インストール、またはPATH設定の問題
file not found今いるフォルダとファイルの場所が違う
class Main is public, should be declared...publicクラス名とファイル名が不一致
Could not find or load main class実行する場所、クラス名、パッケージ指定のどれかが違う
main method not foundmainメソッドの形が入口として認識されない

ファイル名と実行名を分ける

javac Main.java  // ファイル名を指定
java Main        // クラス名を指定。拡張子は付けない
大まかに覚えない

javacはファイルを対象にする。javaはクラスを対象にする。この違いが分かれば、かなりのエラーを自分で切り分けられる。

コードで確認する最小例

わざとjava Main.classを実行し、どんなエラーになるか確認する。その後java Mainで直す。エラー文を読む練習が目的。

似た言葉との違い

混同しやすいもの違い見分け方
cd / dircdは移動、dirは一覧表示。現在地を変えるのか、見るのか。
javac / java前者はコンパイル、後者は実行。ファイル名かクラス名か。
エラーメッセージ / 出力エラーは失敗の説明。出力はプログラムが表示した結果。誰が出した文字かを区別する。

コマンドエラーの確認順

  1. 現在のフォルダと入力ファイルの場所を確認する。
  2. javacにはソースファイルのパスを指定する。
  3. -dで生成されたclassファイルのルートを確認する。
  4. java -cpにはパッケージ階層より上のルートを指定する。
  5. javaには拡張子なしの完全修飾クラス名を指定する。

確認用コード

cd C:\java-work
dir
javac Main.java
java Main
見る点このコードでの判断結論
cd作業フォルダへ移動保存場所と一致させる
javacMain.javaをコンパイルMain.class生成
javaMainクラスを実行出力または実行時例外が出る

ここまでの要点

  • cdとdirで現在地とファイルを確認できる。
  • javacにはソースファイル、javaにはクラス名を指定できる。
  • package付きクラスを-dと-cpを使って実行できる。
