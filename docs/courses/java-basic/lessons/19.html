Java基礎 Lesson 19

ファイル名・クラス名・package

Javaでは、ファイル名、publicクラス名、package名、実行時のクラス名が関係します。小さな構成から確認します。

ここまでできればOK

package付きクラスを正しい名前でコンパイル・実行できるようになります。

まず押さえる言葉

1つの`.java`ファイルはコンパイル単位です。package宣言、import宣言、トップレベル型宣言を規定の順で記述します。

トップレベルクラスとは他のクラスの中ではなく、ファイル直下に定義されたクラス。
public classとは外部から利用できる公開クラス。1つのファイルに最大1つ。
ファイル名一致とはpublicトップレベルクラスがある場合、ファイル名をそのクラス名に合わせるルール。
パッケージとはクラスを分類し、名前の衝突を避ける仕組み。
完全修飾名とはパッケージ名を含めたクラス名。例: ex.Main。

コンパイル単位とトップレベル型

項目意味・使い方具体例
コンパイル単位通常は1つの.javaファイルに対応するソースコード単位。package、import、型宣言を含みます。
package宣言型が属するパッケージを指定します。コメント等を除き、importや型宣言より前に置きます。
import宣言別パッケージの型名を単純名で使えるようにします。package宣言の後、型宣言の前に置きます。
トップレベル型他の型の内部ではなく、コンパイル単位の直下に宣言する型。class、interfaceなど。
public型とファイル名publicトップレベル型の名前とファイルの基本名を一致させます。public class MainはMain.java。
完全修飾名パッケージ名と型名を連結した名前。sample.app.Main

トップレベルクラス

他のクラスの内側ではなく、ファイル直下に書かれたクラスをトップレベルクラスと呼ぶ。

どちらもトップレベルクラス

class A { }
class B { }

publicクラスとファイル名

ファイル内容ファイル名結果
public class Main { }Main.javaOK
public class Main { }Sample.javaコンパイルエラー
class Main { }Sample.javaOK
public class A { } public class B { }どちらでも不可publicトップレベルクラスが複数あるためエラー

実行するクラス名

javaコマンドで指定するのは、ファイル名ではなくクラス名。mainメソッドを持つクラスが複数あれば、それぞれ実行できる。

A.javaでなくても、publicがなければ保存名の自由度がある

class A {
    public static void main(String[] args) {
        System.out.println("A");
    }
}
class B {
    public static void main(String[] args) {
        System.out.println("B");
    }
}
javac Sample.java
java A
java B

パッケージが絡む場合

package ex; のような宣言があるクラスは、実行時に java ex.Main のような完全修飾クラス名で指定します。javac -d を使うと、package名に対応した出力先へclassファイルを配置できます。まずはpublicクラス名とソースファイル名を一致させる規則を確実に押さえます。

package付きクラスの最小構成

`src/sample/app/Main.java`として保存する完全なコード

package sample.app;

public class Main {
    public static void main(String[] args) {
        System.out.println("package");
    }
}

javac -d out src/sample/app/Main.javaで出力先のパッケージ構造を作り、java -cp out sample.app.Mainで完全修飾名を指定して実行します。

似た言葉との違い

混同しやすいもの違い見分け方
ファイル名 / publicクラス名publicトップレベルクラスがあるなら一致が必要。public class XならX.java。
public / 非publicpublicトップレベルクラスは1ファイル1つまで。非publicは複数可。修飾子を見る。
コンパイル対象 / 実行対象コンパイルはファイル、実行はクラス。javac A.java、java A。

ファイル名と実行名の確認順

  1. package宣言からパッケージ名を確認する。
  2. publicトップレベル型の名前と.javaファイル名を一致させる。
  3. コンパイル時の出力ルートを-dで決める。
  4. 実行時のクラスパスにはパッケージ階層の上のルートを指定する。
  5. javaコマンドにはファイルパスではなく完全修飾クラス名を渡す。

確認用コード

// A.java
public class A { }
class B {
    public static void main(String[] args) {
        System.out.println("B");
    }
}
見る点このコードでの判断結論
ファイル名A.javapublic class A と一致
クラスB非publicトップレベルクラス同じファイルに書ける
実行Bにmainがあるjava Bで実行可能

ここまでの要点

  • publicなトップレベルクラスとファイル名の規則を説明できる。
  • 1ファイルに複数のトップレベルクラスを書ける条件を説明できる。
  • パッケージ付きクラスを完全修飾クラス名で実行できる。
