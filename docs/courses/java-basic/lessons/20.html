Java基礎 Lesson 20

コードの読み方

コードを見たら、いきなり出力だけを予想しません。コンパイル可否、実行順、値と参照の変化の順で追います。

ここまでできればOK

短いコードを表やメモに分け、評価順を正確に追えるようになります。

まず押さえる言葉

ソースコードの見た目の並びだけではなく、演算子の結び付きとオペランドの評価順を分けて考えます。

評価順とは式やメソッド呼び出しが処理される順番。
制御フローとはif、for、return、例外などで処理が進む道筋。
スコープとは変数や名前が有効な範囲。
ライフタイムとは変数やオブジェクトが使える期間。
副作用とは値を返すだけでなく、変数変更や出力など状態を変えること。

式・文・ブロック・評価の基本

項目意味・使い方具体例
式評価すると値になるコードのまとまり。a + b、method()、x = 10。
文実行の単位となる構文。式文、if文、return文など。
ブロック波かっこで囲まれた文の並び。独自のスコープを作ります。
優先順位どの演算子が強く結び付くかを決める規則。1 + 2 * 3は1 + (2 * 3)。
評価順オペランドを実際に評価する順序。Javaでは演算子のオペランドを原則として左から評価します。
副作用値の計算以外に状態を変える動作。代入、インクリメント、メソッド呼び出しなど。

読解の基本手順

  1. publicクラス名・ファイル名・mainの形を見る。
  2. 宣言されている変数の型を見る。
  3. コンパイルエラーになる箇所がないか確認する。
  4. newで作られるインスタンスや配列を図にする。
  5. 値が変わる行だけを追う。
  6. メソッド呼び出しでどこへ移動するか確認する。
  7. 最後に出力順を確認する。

小さい例で読む

class Item {
    int price;
}

public class Main {
    public static void main(String[] args) {
        Item a = new Item();
        Item b = a;
        b.price = 100;
        System.out.println(a.price);
    }
}

読み方は、aが新しいItemを指す、b = aで同じItemを指す、b.priceを書き換える、同じ実体をaから読んで100が出る、という順。

読み違いを防ぐための確認

  • いきなり出力だけを予想する。
  • コンパイルエラー判定を飛ばす。
  • 宣言型と実体型を混同する。
  • Stringや配列の参照を図にせず頭だけで追う。
  • ++や+=の副作用を読み飛ばす。

優先順位と評価順を分けて確認する

`Main.java`として保存する完全なコード

public class Main {
    static int value(String label, int n) {
        System.out.print(label);
        return n;
    }

    public static void main(String[] args) {
        int result = value("A", 1) + value("B", 2) * value("C", 3);
        System.out.println("=" + result);
    }
}

演算の結び付きは1 + (2 * 3)ですが、メソッド呼び出しは左からA、B、Cの順に評価されます。出力はABC=7です。

似た言葉との違い

混同しやすいもの違い見分け方
定義順 / 実行順上に書いてあるから先に実行されるとは限らない。mainから呼び出しを追う。
型 / 実体変数の型とnewされた型が異なる場合がある。A a = new B()を分解する。
宣言 / 代入宣言だけでは値が入らない場合がある。初期化済みか見る。

式と文を読む順番

  1. 構文として成立するか、型が適合するかを先に確認する。
  2. 演算子の優先順位と結合規則で式の構造を決める。
  3. オペランドを左から評価し、各副作用をその時点で反映する。
  4. if・ループ・returnなどの制御文で次に実行する文を決める。
  5. ブロックを出た後は、そのブロック内で宣言したローカル変数を参照できない。

確認用コード

class Item {
    int price;
    Item(int price) { this.price = price; }
    void change(int price) { this.price = price; }
}
Item item = new Item(100);
item.change(200);
System.out.println(item.price);
見る点このコードでの判断結論
new Item(100)コンストラクタ実行price=100
change(200)フィールドを200に更新price=200
出力item.price200

ここまでの要点

  • コンパイル可否を最初に確認できる。
  • mainや呼び出し元から実行順を追える。
  • 値と参照の変化を表やメモにして確認できる。
