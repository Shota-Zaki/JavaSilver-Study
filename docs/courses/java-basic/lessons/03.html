ソースコードから実行まで

保存、コンパイル、クラスの読み込み、mainメソッドの実行を別の段階として説明します。

このレッスンのゴール

`.java`から`.class`が生成され、JVMがプログラムを実行するまでの各段階を区別できること。

基本用語の定義

コンパイルと実行は別の処理です。エラーが発生した段階によって、原因の種類も異なります。

コンパイルとは.java を検査し、.class に変換すること。
実行とはjava コマンドでJVMを起動し、mainから処理を始めること。
classファイルとはjavac によって作られるバイトコードのファイル。
エントリポイントとはプログラムの開始地点。Javaでは基本的にmainメソッド。

コンパイルと実行の境界

項目定義・規則具体例
保存ソースコードを文字データとして.javaファイルへ書き込むこと。保存しただけではclassファイルは生成されません。
コンパイルソースコードの構文・型などを検査してバイトコードを生成すること。成功すると通常は.classが生成されます。
クラスの読み込みJVMが実行に必要なクラス定義を探して読み込むこと。クラスパスが検索範囲になります。
初期化staticフィールドやstatic初期化子を規則に従って実行すること。mainより前に対象クラスの初期化が行われます。
実行mainメソッドから文を順に評価すること。実行時例外はこの段階で発生します。
ソースファイルモードJava 11以降でjava Main.javaと実行する方式。内部でソースをコンパイルして実行しますが、javacの通常手順とは区別します。

1. ソースコードを書く

ソースコードは人間が読むためのテキスト。拡張子は.java。publicなトップレベルクラスがあるなら、ファイル名はそのクラス名と一致させる必要がある。

Main.java

public class Main {
    public static void main(String[] args) {
        System.out.println("OK");
    }
}

2. javacでコンパイルする

javacは、文法、型、アクセス、未初期化ローカル変数、checked例外の扱いなどを検査する。通れば.classファイルが作られる。

コード結果理由
int x = 10;通る宣言と初期化が成立している
int x = "10";通らないintにStringは代入できない
System.out.println(x);だけを書く状況次第で通らないxが宣言・初期化されていなければエラー

3. javaで実行する

javaコマンドでは、ファイル名ではなくクラス名を指定する。Main.classではなくMainと書く。

正しい実行例

java Main

コマンド指定を誤った例

java Main.class

失敗の読み分け

失敗する段階見方例
保存前ファイル名や保存場所の問題違うフォルダに保存している
コンパイル時文法・型・アクセスの問題cannot find symbol
実行時値や実行環境の問題ArrayIndexOutOfBoundsException
順番を飛ばさない

実行時の出力を考える前に、まずコンパイルできるかを判定する。ここを飛ばすと、存在しない出力結果を追いかけることになる。

通常のコンパイルと実行

`Main.java`として保存する完全なコード

public class Main {
    public static void main(String[] args) {
        System.out.println("start");
        int result = 6 * 7;
        System.out.println(result);
    }
}

通常はjavac Main.javaでコンパイルし、java Mainで実行します。java Main.classとは書きません。

関連する用語との違い

混同しやすいもの違い見分け方
ファイル名 / クラス名コンパイル時はファイル名、実行時はクラス名を意識する。javac Main.java、java Main。
.java / .class.javaはソース、.classはコンパイル結果。手で編集するのは基本的に.java。
コンパイルエラー / 実行時例外前者は.class生成前、後者は実行中。出力を問う前にコンパイル可否を見る。

ソースから実行へ進む規則

  1. ファイルを保存し、コンパイラが読む内容を確定する。
  2. 構文・型・アクセス・初期化の規則に違反していないかコンパイル時に検査する。
  3. classファイルが生成された後に、JVMが対象クラスを検索する。
  4. クラスの初期化後、mainメソッドから実行を開始する。
  5. 実行時例外が発生した場合は、その式以降の通常処理は進まない。

確認用コード

// Main.java
public class Main {
    public static void main(String[] args) {
        System.out.println("OK");
    }
}
見る点このコードでの判断結論
ファイル名Main.javapublic class Main と一致
コンパイルjavac Main.javaMain.classができる
実行java MainOKが表示される

保存・コンパイル・実行を混ぜない

Javaプログラムは、ファイルへ保存しただけでは動きません。ソースを保存し、javacでコンパイルし、生成されたクラスをjavaコマンドで実行します。

順番操作失敗するとどうなるか
1Main.javaを保存する保存場所やファイル名を間違えるとコマンドから見つからない
2javac Main.java文法・型・ファイル名などの問題でコンパイルエラー
3java Mainクラス名指定ミス、実行時例外、出力結果の不一致が起きる

特に重要なのは、javacに渡すのはファイル名、javaに渡すのはクラス名だという点です。ここを混同すると、コードが正しくても実行できません。

// ファイル名: Main.java
public class Main {
    public static void main(String[] args) {
        System.out.println("OK");
    }
}
javac Main.java
java Main

出力結果を考えるのは、コンパイルが成功した後です。コンパイルエラーが出るコードに対して、実行結果を考えても意味がありません。この順序を守るだけで読解精度は上がります。

ここで整理する誤解

javac Mainやjava Main.javaのような混同は、かなり基本的ですが実務でも最初に詰まりやすい箇所です。ファイルを渡す場面とクラス名を渡す場面を明確に分けると判定しやすくなります。

定義と規則の整理

ソースから実行までの誤解

つまずき正しい考え方
.javaを実行していると思う通常はjavacで.classを作り、javaでクラスを起動する。
コンパイル成功と正しい動作を同一視するコンパイル成功は文法・型などの最低条件を満たしただけ。実行時例外や論理ミスは残る。
エラーの段階を見ないコンパイルエラー、実行時例外、出力違いは原因が違う。まず段階を分ける。

確認できれば次へ

  • .java、.class、JVMの流れを説明できる
  • コンパイルエラーと実行時例外を区別できる
  • 出力結果を追う前の確認順を説明できる
理解の基準

成立するコードと成立しないコード

成立する例：ソースから実行まで

public class Main {
    public static void main(String[] args) {
        System.out.println("OK");
    }
}

// terminal
javac Main.java
java Main

成立しない例：class拡張子を付けて実行している

javac Main.java
java Main.class

成立する形：javaには完全修飾クラス名を渡す

javac Main.java
java Main
確認ポイント

通常は.javaをjavacでコンパイルし、クラス名をjavaに渡して実行します。

定義と規則を確認する

用語の確認

コンパイルと実行の違いを説明できますか。

説明を見る

コンパイルは.javaを.classに変換する作業、実行は.classをJVMで動かす作業です。エラーもコンパイル時と実行時で分けます。

コードの確認

次の順番は正しいですか。

java Main
javac Main.java
説明を見る

順番が逆です。通常は先にjavac Main.javaでclassを作り、その後java Mainで実行します。

仕組みを説明する

コンパイルエラーが出たコードは、なぜ実行結果を考えなくてよいですか。

説明を見る

classファイルを正しく作れないため、実行段階には進みません。実行結果を考える前に、まずコンパイル可否を確認します。

基本ルールのまとめ

  • 保存、コンパイル、実行の順番を説明できる。
  • javac Main.java と java Main を使い分けられる。
  • コンパイルエラーがあるコードは実行されないと判断できる。

コードに規則を当てはめる

確認用コード

    public static void main(String[] args) {

    }

説明を見る
