Java基礎 Lesson 3

保存・コンパイル・実行

Javaプログラムは、保存しただけでは動きません。ファイルを保存し、javacでclassファイルを作り、javaで実行する流れを確認します。

ここまでできればOK

javacとjavaに何を指定するかを間違えずに使えるようになります。

まず押さえる言葉

コンパイルと実行は別の処理です。エラーが発生した段階によって、原因の種類も異なります。

コンパイルとは.java を検査し、.class に変換すること。
実行とはjava コマンドでJVMを起動し、mainから処理を始めること。
classファイルとはjavac によって作られるバイトコードのファイル。
エントリポイントとはプログラムの開始地点。Javaでは基本的にmainメソッド。

コンパイルと実行の境界

項目意味・使い方具体例
保存ソースコードを文字データとして.javaファイルへ書き込むこと。保存しただけではclassファイルは生成されません。
コンパイルソースコードの構文・型などを検査してバイトコードを生成すること。成功すると通常は.classが生成されます。
クラスの読み込みJVMが実行に必要なクラス定義を探して読み込むこと。クラスパスが検索範囲になります。
初期化staticフィールドやstatic初期化子を規則に従って実行すること。mainより前に対象クラスの初期化が行われます。
実行mainメソッドから文を順に評価すること。実行時例外はこの段階で発生します。
ソースファイルモードJava 11以降でjava Main.javaと実行する方式。内部でソースをコンパイルして実行しますが、javacの通常手順とは区別します。

1. ソースコードを書く

ソースコードは人間が読むためのテキスト。拡張子は.java。publicなトップレベルクラスがあるなら、ファイル名はそのクラス名と一致させる必要がある。

Main.java

public class Main {
    public static void main(String[] args) {
        System.out.println("OK");
    }
}

2. javacでコンパイルする

javacは、文法、型、アクセス、未初期化ローカル変数、checked例外の扱いなどを検査する。通れば.classファイルが作られる。

コード結果理由
int x = 10;通る宣言と初期化が成立している
int x = "10";通らないintにStringは代入できない
System.out.println(x);だけを書く状況次第で通らないxが宣言・初期化されていなければエラー

3. javaで実行する

javaコマンドでは、ファイル名ではなくクラス名を指定する。Main.classではなくMainと書く。

正しい実行例

java Main

コマンド指定を誤った例

java Main.class

失敗の読み分け

失敗する段階見方例
保存前ファイル名や保存場所の問題違うフォルダに保存している
コンパイル時文法・型・アクセスの問題cannot find symbol
実行時値や実行環境の問題ArrayIndexOutOfBoundsException
順番を飛ばさない

実行時の出力を考える前に、まずコンパイルできるかを確認する。ここを飛ばすと、存在しない出力結果を追いかけることになる。

通常のコンパイルと実行

`Main.java`として保存する完全なコード

public class Main {
    public static void main(String[] args) {
        System.out.println("start");
        int result = 6 * 7;
        System.out.println(result);
    }
}

通常はjavac Main.javaでコンパイルし、java Mainで実行します。java Main.classとは書きません。

似た言葉との違い

混同しやすいもの違い見分け方
ファイル名 / クラス名コンパイル時はファイル名、実行時はクラス名を意識する。javac Main.java、java Main。
.java / .class.javaはソース、.classはコンパイル結果。手で編集するのは基本的に.java。
コンパイルエラー / 実行時例外前者は.class生成前、後者は実行中。出力を問う前にコンパイル可否を見る。

ソースから実行までの確認順

  1. ファイルを保存し、コンパイラが読む内容を確定する。
  2. 構文・型・アクセス・初期化の規則に違反していないかコンパイル時に検査する。
  3. classファイルが生成された後に、JVMが対象クラスを検索する。
  4. クラスの初期化後、mainメソッドから実行を開始する。
  5. 実行時例外が発生した場合は、その式以降の通常処理は進まない。

確認用コード

// Main.java
public class Main {
    public static void main(String[] args) {
        System.out.println("OK");
    }
}
見る点このコードでの判断結論
ファイル名Main.javapublic class Main と一致
コンパイルjavac Main.javaMain.classができる
実行java MainOKが表示される

ここまでの要点

  • 保存、コンパイル、実行の順番を説明できる。
  • javac Main.java と java Main を使い分けられる。
  • コンパイルエラーがあるコードは実行されないと判断できる。
