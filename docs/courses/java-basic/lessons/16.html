interfaceの入口

interfaceを操作の契約として定義し、implements、抽象メソッド、定数、defaultメソッドの規則を説明します。

このレッスンのゴール

interfaceに宣言できる主要なメンバと、実装クラスが満たすべきアクセス規則を説明できること。

基本用語の定義

interfaceはインスタンスの具体的な作り方ではなく、その型として提供する操作や定数を宣言します。

interfaceとはクラスが実装すべき扱い方を定める型。
implementsとはクラスがinterfaceを実装することを示すキーワード。
抽象メソッドとは本体を持たず、実装クラス側で処理を書くメソッド。
defaultメソッドとはinterface内に書ける実装付きメソッド。
実装とは決められたメソッドの具体的な処理を書くこと。

interfaceのメンバと実装規則

項目定義・規則具体例
interface宣言クラスとは別の参照型を定義します。interface Printable { ... }
implementsクラスがinterfaceの契約を実装することを表します。クラスは複数のinterfaceを実装できます。
抽象メソッド本体を持たないインスタンスメソッド。通常は暗黙にpublic abstractです。実装メソッドはpublicにします。
フィールドinterfaceのフィールドは暗黙にpublic static finalです。宣言時に初期化が必要です。
defaultメソッドinterfaceが本体を持つインスタンスメソッド。実装クラスで継承またはオーバーライドできます。
staticメソッドinterface自体に属するメソッド。InterfaceName.method()で呼びます。

基本形

interface Worker {
    void work();
}

class Engineer implements Worker {
    @Override
    public void work() {
        System.out.println("coding");
    }
}

Worker型として扱うならwork()を呼べる、というルールを作っている。

interfaceのメソッドはpublic扱い

interfaceの抽象メソッドは暗黙にpublic abstract。そのため、実装クラス側ではpublicで実装する必要がある。アクセスを弱くするとコンパイルエラー。

成立しない例

interface Worker {
    void work();
}

class Engineer implements Worker {
    void work() { } // コンパイルエラー：publicでない
}

修正例

class Engineer implements Worker {
    public void work() { } // OK
}

interface型で受けるメリット

Worker w = new Engineer();
w.work();

変数の型をWorkerにすると、実体がEngineerでも、呼べるのはWorkerとして見えるメソッドになる。これは継承の親型変数と同じ読み方。

defaultメソッド

defaultメソッドは、interface内に実装を持てるメソッド。実装クラスが必ずしも書かなくてよい共通処理を置ける。

interface Greet {
    default void hello() {
        System.out.println("hello");
    }
}

class User implements Greet { }
new User().hello();

フィールドは定数扱い

interfaceに書いたフィールドは暗黙にpublic static final。つまり共有される定数であり、あとから代入し直せません。

interfaceを実装する最小例

`Main.java`として保存する完全なコード

interface Printable {
    void print();

    default void printHeader() {
        System.out.println("header");
    }
}

class Report implements Printable {
    @Override
    public void print() {
        System.out.println("report");
    }
}

public class Main {
    public static void main(String[] args) {
        Printable value = new Report();
        value.printHeader();
        value.print();
    }
}

interface内のvoid print();はpublicな抽象メソッドです。そのため、実装側のprint()もpublicで宣言します。

関連する用語との違い

混同しやすいもの違い見分け方
extends / implementsクラス継承はextends、インタフェース実装はimplements。右側がクラスかインタフェースかを見る。
抽象メソッド / defaultメソッド抽象メソッドは実装がない。defaultメソッドは実装を持てる。本体{}があるか、defaultかを見る。
interfaceのメソッド / 実装クラスのメソッドinterface側はpublic扱い。実装時もpublicにする必要がある。アクセス修飾子を弱くしていないか見る。

interface実装を判断する規則

  1. interfaceが要求する抽象メソッドを確認する。
  2. 実装クラスがabstractでなければ、必要なメソッドをすべて実装する。
  3. 実装メソッドのアクセス修飾子をpublicにする。
  4. defaultメソッドの競合がある場合は、実装クラス側でオーバーライドして解決する。
  5. interface型の変数から呼べるのは、そのinterfaceに宣言されたメンバを基本とする。

確認用コード

interface Worker {
    void work();
}
class Engineer implements Worker {
    void work() {
        System.out.println("build");
    }
}
見る点このコードでの判断結論
interface側workは暗黙にpublic外から呼べる約束
実装側void work()はデフォルトアクセスpublicより弱い
結論アクセス権を弱くしているコンパイルエラー

interfaceは外から見える操作をそろえる

interfaceは、異なるクラスを同じ型として扱うための約束を定義します。実装クラスは、interfaceが要求する抽象メソッドをpublicで実装します。

interface Printer {
    void print(String text);
}

class ConsolePrinter implements Printer {
    public void print(String text) {
        System.out.println(text);
    }
}

Printer p = new ConsolePrinter();
p.print("hello");

Printer pとして受けているので、呼べるのはPrinterに定義された操作です。実体はConsolePrinterですが、変数の型はPrinterです。この考え方がポリモーフィズムの入口です。

interface内の要素扱い注意
抽象メソッド基本的にpublic abstract実装時もpublicが必要
フィールドpublic static final扱いインスタンスごとの状態ではない
defaultメソッド実装を持てる実装クラスで上書き可能
staticメソッドinterface自身に属する実装クラスのインスタンスから呼ぶものではない
interface Sample {
    int VALUE = 10; // public static final 扱い
    void test();    // public abstract 扱い
}
interfaceで最初に見ること

「何を実装しているか」より先に、「そのinterface型として何を呼べるか」を見ると分かりやすいです。型としての役割を見ないと、実装クラスの詳細に引っ張られます。

定義と規則の整理

interfaceで混同する点

つまずき正しい考え方
interfaceをインスタンス化できると思うinterface型の変数は作れるが、interface自体はnewできない。実装クラスのインスタンスを扱う。
interfaceのメソッドはprivateだと思う抽象メソッドは暗黙にpublic abstract。実装側もpublicが必要。
extendsとimplementsを混同するクラスを継承するのがextends、interfaceを実装するのがimplements。

確認できれば次へ

  • interfaceを「扱い方の約束」として説明できる
  • implementsの意味を説明できる
  • defaultメソッドと抽象メソッドを区別できる
理解の基準

成立するコードと成立しないコード

成立する例：interfaceとimplements

interface Printable {
    void print();
}
class Report implements Printable {
    public void print() { System.out.println("R"); }
}

成立しない例

interface Printable {
    void print();
}
class Report implements Printable {
    void print() { System.out.println("R"); }
}

成立する形

class Report implements Printable {
    public void print() { System.out.println("R"); }
}
確認ポイント

interfaceの抽象メソッドはpublic扱いです。実装時にアクセス範囲を狭くできません。

定義と規則を確認する

用語の確認

interface、implements、抽象メソッド、defaultメソッドを説明できますか。

説明を見る

interfaceは扱い方を定める型です。
implementsはクラスがinterfaceを実装する指定、抽象メソッドは実装を持たないメソッド、defaultメソッドはinterface内に持てる実装付きメソッドです。

コードの確認

次のinterface定義はコンパイルできますか。

interface A {
    void test() {
        System.out.println("A");
    }
}
説明を見る

コンパイルできません。interfaceの通常メソッドに本体を持たせるならdefault、static、privateなどの条件が必要です。

仕組みを説明する

interfaceのフィールドが実質的に定数になる理由を説明してください。

説明を見る

interfaceのフィールドは暗黙にpublic static finalです。インスタンスごとの状態ではなく、公開された定数として扱われます。

基本ルールのまとめ

  • interfaceとclassの役割の違いを説明できる。
  • 実装メソッドをpublicにする理由を説明できる。
  • defaultメソッドと抽象メソッドを区別できる。

コードに規則を当てはめる

次のコードをコンパイルできるようにするには何を直すか。

確認用コード

interface A {
    void test();
}
class B implements A {
    void test() { }
}
説明を見る

Bのtestをpublicにする。

インタフェースのtestは暗黙にpublic。実装時にアクセス修飾子を弱くできないため、public void test() { }と書く。
