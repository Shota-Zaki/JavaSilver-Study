継承の入口

継承、サブクラス、オーバーライド、宣言型、実行時型、動的ディスパッチを定義します。

このレッスンのゴール

親型の変数が子クラスのインスタンスを参照するとき、呼べるメンバと実行されるメソッドを説明できること。

基本用語の定義

継承は既存クラスを基に新しいクラスを定義する仕組みです。Javaのクラス継承は直接の親クラスを1つだけ持ちます。

継承とは既存クラスを元にして、新しいクラスを定義する仕組み。
スーパークラスとは継承元の親クラス。
サブクラスとは継承して作られる子クラス。
オーバーライドとはサブクラスで、親クラスのメソッドを同じシグネチャで再定義すること。
ポリモーフィズムとは同じ型の変数で、実体に応じて異なる動作をさせる仕組み。

クラス継承とオーバーライドの規則

項目定義・規則具体例
extends直接の親クラスを指定します。class Dog extends Animal
サブクラス親クラスを継承して定義されたクラス。子クラスとも呼びます。
オーバーライド継承したインスタンスメソッドを同じシグネチャで再定義します。アクセス範囲を狭くできません。
宣言型変数宣言に書かれた型。コンパイル時に参照できるメンバの範囲を決めます。
実行時型実際に参照しているオブジェクトのクラス。オーバーライドされたインスタンスメソッドの実装を決めます。
継承されないものコンストラクタは継承されません。privateメンバは子クラスから直接アクセスできません。親のprivate状態自体は親部分に存在します。

継承の基本

class Animal {
    void speak() {
        System.out.println("...");
    }
}

class Dog extends Animal {
    @Override
    void speak() {
        System.out.println("bow");
    }
}

親型の変数で子を参照する

Animal a = new Dog();
a.speak(); // bow

aの宣言型はAnimalだが、実体はDog。speakはオーバーライドされているため、実行時にはDog側が呼ばれる。

呼べるかどうかは宣言型で決まる

実体がDogでも呼べない

class Dog extends Animal {
    void run() { }
}

Animal a = new Dog();
a.run(); // コンパイルエラー

Animal型の変数からは、Animalとして見えるメソッドしか呼べない。実体が何かは、コンパイル時の呼び出し可否には使われない。

フィールドはオーバーライドされない

class A {
    String name = "A";
}
class B extends A {
    String name = "B";
}

A x = new B();
System.out.println(x.name); // A

メソッドとフィールドは同じ仕組みで動かない。フィールドアクセスは変数の宣言型で決まる。

継承で引き継がないもの

  • コンストラクタは引き継がない。
  • privateメンバーは子クラスから直接アクセスできない。
  • 親に引数なしコンストラクタがない場合、子は明示的にsuper(引数)を書く必要がある。

宣言型と実行時型の違い

`Main.java`として保存する完全なコード

class Animal {
    void speak() {
        System.out.println("animal");
    }
}

class Dog extends Animal {
    @Override
    void speak() {
        System.out.println("dog");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal animal = new Dog();
        animal.speak();
    }
}

変数の宣言型はAnimal、実行時型はDogです。speak()はオーバーライドされたDog側が実行されます。

関連する用語との違い

混同しやすいもの違い見分け方
継承 / インスタンス生成継承しただけでは実体は作られない。newが必要。クラス定義と実行を分ける。
コンストラクタ / メソッドメソッドは継承されうるが、コンストラクタは継承されない。子の生成時は親コンストラクタを呼ぶが、子のコンストラクタとして使えるわけではない。
メソッド / フィールドオーバーライドはメソッド。フィールドは同名でも隠蔽であり動きが違う。呼び出しているのが()付きかを見る。

継承コードを判断する規則

  1. 変数の宣言型を確認し、コンパイル時に呼べるメンバを決める。
  2. 右辺のnewなどから実行時型を確認する。
  3. 対象がインスタンスメソッドで、オーバーライドされているか確認する。
  4. オーバーライドされていれば実行時型側の実装を呼ぶ。
  5. フィールドとstaticメソッドは、インスタンスメソッドの動的ディスパッチと分けて扱う。

確認用コード

class A {
    String name = "A";
    String getName() { return "A"; }
}
class B extends A {
    String name = "B";
    String getName() { return "B"; }
}
A obj = new B();
System.out.println(obj.name);
System.out.println(obj.getName());
見る点このコードでの判断結論
変数型A objフィールドはA側を見る
実体new B()メソッドはB側のオーバーライドが効く
出力obj.nameはA、obj.getName()はBA / B

宣言型と実体型を分ける

継承のコードでは、コンパイル時に見る変数の型と、実行時に存在するインスタンスの型を分けます。呼び出せるメソッドは宣言型で確認し、オーバーライドされた実行先は実体型で決まります。

class Animal {
    void speak() { System.out.println("?"); }
}
class Dog extends Animal {
    void speak() { System.out.println("wan"); }
    void run() { System.out.println("run"); }
}

Animal a = new Dog();
a.speak(); // wan
// a.run(); // コンパイルエラー

aの宣言型はAnimalなので、コンパイル時にAnimalとして扱われます。そのためrunは呼べません。一方、speakはAnimalに存在し、Dogでオーバーライドされているため、実行時にはDog側が呼ばれます。

見ること決まるタイミング例
そのメソッドを呼べるかコンパイル時変数の宣言型を見る
どの実装が動くか実行時実体のクラスを見る
フィールド参照基本的に宣言型で決まるフィールドはオーバーライドされない
コンストラクタ継承されない子の生成時に親のコンストラクタは呼ばれる
ここで整理する誤解

中身がDogでも、Animal型変数からはDog専用メソッドを呼べません。呼べるメソッドは、まず変数の宣言型で決まります。

定義と規則の整理

継承で誤解しやすい点

つまずき正しい考え方
継承はコードをコピーすることだと思う概念的には機能を引き継ぐが、単純なソースコードコピーではない。
コンストラクタも継承されると思うコンストラクタは継承されない。ただし生成時にスーパークラスのコンストラクタは呼ばれる。
フィールドも動的に切り替わると思うオーバーライドされるのはメソッド。フィールドアクセスは参照変数の型で決まる。

確認できれば次へ

  • extendsの意味を説明できる
  • オーバーライドとオーバーロードを区別できる
  • ポリモーフィズムで動的に選ばれる対象を説明できる
理解の基準

成立するコードと成立しないコード

成立する例：継承とオーバーライド

class Parent {
    void hello() { System.out.println("P"); }
}
class Child extends Parent {
    @Override
    void hello() { System.out.println("C"); }
}

成立しない例

class Child extends Parent {
    @Override
    void hello(String s) { System.out.println(s); }
}

成立する形

class Child extends Parent {
    @Override
    void hello() { System.out.println("C"); }
}
確認ポイント

オーバーライドはメソッド名だけでなく、引数リストまで一致する必要があります。

定義と規則を確認する

用語の確認

継承、スーパークラス、サブクラス、overrideを説明できますか。

説明を見る

継承は既存クラスをもとに新しいクラスを作る仕組みです。継承元がスーパークラス、継承先がサブクラス、overrideは親のメソッドを子で再定義することです。

コードの確認

次のコードはoverrideになりますか。

class A { void test(int n) {} }
class B extends A { void test(String s) {} }
説明を見る

overrideになりません。引数型が違うためオーバーロードです。

仕組みを説明する

Parent p = new Child(); で呼び出すメソッドがChild側になる場合がある理由を説明してください。

説明を見る

インスタンスの実体がChildで、メソッドは実行時の実体に基づいて動的に選択されるためです。フィールド直接参照とは分けます。

基本ルールのまとめ

  • extendsによる親子関係を説明できる。
  • 宣言型から呼び出せるメソッドを判断できる。
  • オーバーライドされたメソッドの実行先を判断できる。

コードに規則を当てはめる

次の出力を答える。

確認用コード

class Parent {
    void show() { System.out.println("P"); }
}
class Child extends Parent {
    void show() { System.out.println("C"); }
}
Parent p = new Child();
p.show();
説明を見る

出力はC。

変数の型はParentだが、実際のインスタンスはChild。showはオーバーライドされたメソッドなので、実行時の実体であるChild側が呼ばれる。
