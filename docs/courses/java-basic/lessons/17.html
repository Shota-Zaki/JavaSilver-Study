例外処理の入口

例外を通常の制御フローを中断するオブジェクトとして定義し、try・catch・finally・throw・throwsの規則を説明します。

このレッスンのゴール

checked例外とunchecked例外を区別し、例外発生後にどのcatchへ移るかを説明できること。

基本用語の定義

例外が発生すると、その地点から通常の次の文へは進まず、対応するcatchを探しながら呼び出し元へ制御が移ります。

例外とは通常の処理を続けられない状態を表すオブジェクト。
tryとは例外が発生する可能性がある処理を書くブロック。
catchとは発生した例外を受け取って処理するブロック。
finallyとは例外の有無にかかわらず実行したい処理を書くブロック。
checked例外とはコンパイラがcatchまたはthrowsを要求する例外。
unchecked例外とはcatch/throwsが必須ではないRuntimeException系の例外。

例外階層と処理構文の規則

項目定義・規則具体例
ThrowableJavaでthrowできるオブジェクトの最上位クラス。主な下位型はErrorとExceptionです。
checked例外RuntimeExceptionの系統ではないException。catchするかthrowsで宣言する必要があります。
unchecked例外RuntimeExceptionとそのサブクラス。コンパイラによるcatch・throwsの強制はありません。
try例外が発生する可能性のある処理を囲むブロック。例外発生時は残りの文を飛ばします。
catch型が一致する例外を受け取るブロック。上から最初に適合するcatchが実行されます。
finally正常終了でも例外終了でも原則として最後に実行されるブロック。JVMの強制終了などでは実行されない場合があります。

例外処理を読む順番

例外処理は、まずコンパイルできるかを確認し、次に例外が発生する行、対応するcatch、finallyの順に追います。例外が発生した後は、tryブロックの残りへ戻りません。

順番見ること判断例
1コンパイル可否checked例外の処理漏れ、catch順序、try構文
2例外発生位置例外が発生した行以降のtry内処理は実行されない
3catch一致発生した例外型を受け取れるcatchへ入る
4finallycatchされてもされなくても最後に実行される
try {
    System.out.print("A");
    int x = 10 / 0;
    System.out.print("B");
} catch (ArithmeticException e) {
    System.out.print("C");
} finally {
    System.out.print("D");
}
// ACD

try-catchの流れ

try {
    int[] a = new int[0];
    a[0] = 10;
    System.out.println("finish");
} catch (ArrayIndexOutOfBoundsException e) {
    System.out.println("error");
}

配列の範囲外アクセスで例外が発生した時点で、System.out.println("finish")には進まない。対応するcatchへ移動し、errorが表示される。

checked例外とunchecked例外

分類代表例コンパイラの扱い
checked例外IOExceptionなどcatchするかthrows宣言が必要
unchecked例外RuntimeException系catchやthrowsを強制されない
ErrorOutOfMemoryErrorなど通常のアプリ処理で回復を前提にしない

初心者段階では、RuntimeException系は「実行中の値や状態が悪いと起きることが多い」と読むと整理しやすい。

finally

finally は、例外発生の有無にかかわらず原則として実行したい後処理を書きます。ファイルや接続を閉じる処理の説明でよく使われます。

例外がなければ A, C

try {
    System.out.println("A");
} catch (Exception e) {
    System.out.println("B");
} finally {
    System.out.println("C");
}

catchの順序

複数のcatchを書く場合、狭い型から広い型へ並べる。先にExceptionで受けると、その後の具体的な例外型は到達不能になる。

悪い順序

try {
    // ...
} catch (Exception e) {
} catch (NullPointerException e) {
    // 前のcatchで捕捉される型なのでコンパイルエラー
}

例外が発生した位置とcatchへの移動

`Main.java`として保存する完全なコード

public class Main {
    public static void main(String[] args) {
        try {
            System.out.println("before");
            int value = 10 / 0;
            System.out.println(value);
        } catch (ArithmeticException e) {
            System.out.println("caught");
        } finally {
            System.out.println("finally");
        }
    }
}

10 / 0の評価時にArithmeticExceptionが発生します。直後のprintln(value)は実行されず、catch、finallyの順に進みます。

関連する用語との違い

混同しやすいもの違い見分け方
例外発生前 / 発生後例外が発生した行より後のtry内処理は飛ばされる。どの行で例外が起きるか見る。
catchする / しない対応するcatchがあれば処理される。なければ呼び出し元へ伝わる。例外クラスの型を見る。
finally / catchfinallyは例外処理そのものではなく、後始末用。catchされてもされなくても実行されうる。

例外発生後の制御を追う規則

  1. 例外が発生する式を特定する。
  2. その式より後のtry内の文は実行されないと判断する。
  3. catchを上から確認し、例外型に最初に適合するものを選ぶ。
  4. 適合するcatchがなければ呼び出し元へ例外を伝える。
  5. finallyがあれば、通常の制御移動の前に原則として実行する。

確認用コード

try {
    int[] a = new int[0];
    a[0] = 10;
    System.out.println("A");
} catch (ArrayIndexOutOfBoundsException e) {
    System.out.println("B");
} finally {
    System.out.println("C");
}
見る点このコードでの判断結論
例外発生a[0]で範囲外アクセスArrayIndexOutOfBoundsException
try残りSystem.out.println("A")は実行されないAは出ない
catch/finallyBを出してからC出力はB C

例外が起きた後の処理の進み方

例外処理は「エラーを除外する仕組み」ではありません。正確には、通常の流れで処理を続けられない事態が起きたとき、別の流れへ制御を移す仕組みです。

try {
    int[] nums = {1, 2, 3};
    System.out.println(nums[5]);
    System.out.println("end");
} catch (ArrayIndexOutOfBoundsException e) {
    System.out.println("out of range");
}

nums[5]で例外が発生すると、その時点でtryブロックの残りは実行されません。"end"は表示されず、対応するcatchへ移動します。

見ること見る場所判断
例外が発生する可能性tryブロック内どの行で止まるか
catchできるかcatchの型発生した例外の型と互換性があるか
catchの順序上から順に判定親クラスを先に書くと子クラス側に届かない
finally最後に実行例外の有無にかかわらず基本的に実行

checked例外とunchecked例外の違いは、まずコンパイラが処理を要求するかで見ます。Exceptionのサブクラスでも、RuntimeException系はuncheckedです。

大まかなcatchは理解を悪くする

catch (Exception e)だけで済ませると、何が起きたのかを読めなくなります。学習段階では、どの例外がどのcatchに入るかを必ず言語化してください。

定義と規則の整理

例外処理で混同しやすい点

つまずき正しい考え方
tryを書けば例外が消えると思うtryは例外が起きそうな範囲を示すだけ。対応するcatchやthrowsの設計が必要。
throwとthrowsを混同するthrowは例外を投げる処理。throwsはメソッド宣言で例外可能性を示す。
finallyはcatchの代わりだと思うfinallyは後始末用。例外を処理するブロックではない。

確認できれば次へ

  • checked例外とunchecked例外を区別できる
  • throwとthrowsの違いを説明できる
  • finallyが実行されるタイミングを説明できる
理解の基準

成立するコードと成立しないコード

成立する例：例外処理

try {
    int n = Integer.parseInt("10");
    System.out.println(n);
} catch (NumberFormatException e) {
    System.out.println("error");
}

成立しない例

try {
    int n = Integer.parseInt("x");
} catch (NullPointerException e) {
    System.out.println("error");
}

成立する形

try {
    int n = Integer.parseInt("x");
} catch (NumberFormatException e) {
    System.out.println("error");
}
確認ポイント

catchは発生する例外型に対応している必要があります。型が違うと捕捉できません。

定義と規則を確認する

用語の確認

例外、try、catch、finallyを説明できますか。

説明を見る

例外は実行中の異常状態、tryは例外が起きる可能性のある処理、catchは例外処理、finallyは例外有無に関係なく実行したい処理です。

コードの確認

次のコードは何を表示しますか。

try {
    int n = 10 / 0;
    System.out.print("A");
} catch (ArithmeticException e) {
    System.out.print("B");
}
説明を見る

Bです。0除算でArithmeticExceptionが発生し、tryの続きであるAは実行されません。

仕組みを説明する

catch後にtryの続きへ戻らない理由を説明してください。

説明を見る

例外が発生した時点でtryブロック内の残り処理はスキップされ、対応するcatchへ制御が移ります。catch後はtry-catch文の後へ進みます。

基本ルールのまとめ

  • checked例外とunchecked例外を区別できる。
  • 発生した例外がどのcatchに入るか判断できる。
  • finallyが実行される位置を説明できる。

コードに規則を当てはめる

次の出力を答える。

確認用コード

try {
    System.out.println("A");
} catch (RuntimeException e) {
    System.out.println("B");
} finally {
    System.out.println("C");
}
説明を見る

出力はA、C。

try内で例外は発生しないためcatchは実行されない。ただしfinallyは実行される。
