Java基礎 Lesson 18

エラーの種類

Javaの失敗は、コンパイル前に見つかるものと、実行中に起きるものに分かれます。まず発生した段階を確認します。

ここまでできればOK

コンパイルエラー、実行時例外、意図しない結果を分けて考えられるようになります。

まず押さえる言葉

「エラー」という一語では原因を特定できません。Javaでは、どの段階まで処理が進んだかを先に確認します。

コンパイルエラーとはjavacの段階で文法や型の問題により止まること。
実行時例外とはコンパイル後、実行中の値や状態が原因で発生する例外。
スタックトレースとは例外がどのメソッド呼び出し経路で発生したかを示す出力。
到達不能コードとは絶対に実行されないためコンパイルエラーになるコード。
原因切り分けとは文法・型・参照・実行順序のどれが原因か分けること。

失敗を発生段階で分類する

項目意味・使い方具体例
コンパイルエラー構文、型、アクセス、確実な代入などの規則に違反している状態。対象コードは通常の実行段階へ進みません。
クラス読み込み時の失敗JVMが必要なクラスを見つけられない、または整合しない状態。ClassNotFoundExceptionやNoClassDefFoundErrorなど。
実行時例外コンパイル後、文や式の評価中に例外オブジェクトが送出される状態。NullPointerExceptionなど。
論理エラー正常終了するが、要求した結果と異なる状態。コンパイラやJVMが必ず検出するものではありません。
エラー発生位置例外は特定の式の評価中に発生します。配列アクセス、除算、メソッド呼び出しなど。
後続処理例外が処理されない限り、その式以降の通常処理は実行されません。対応するcatchまたは呼び出し元へ移ります。

失敗した段階を先に確認する

同じ「動かない」でも、javacが止めたのか、javaで実行中に例外が起きたのかで調べる場所が変わります。出力を考える前に、まずコンパイルできるコードかを確認します。

分類発生タイミング代表例結果
コンパイルエラーjavac時型不一致、未定義変数、checked例外処理漏れ.classが作られず、実行できない
実行時例外java実行中null参照、配列範囲外、整数0除算、キャスト失敗途中まで実行され、例外発生地点で通常処理が止まる
論理ミス実行後出力が期待と違う、条件式が逆エラーは出ないが意図通りではない

コンパイルエラーの典型

原因例なぜ実行前に分かるか
文法ミスint a = ;文の形が成立していない
型不一致int a = "x";代入できない型だと分かる
未初期化ローカル変数int x; println(x);値が入っていない経路が分かる
到達不能コードreturn; println();絶対に実行されないと分かる
checked例外未処理throwsもcatchもない処理強制のルールに反する

実行時例外の典型

例外起きる状況
NullPointerExceptionnull参照からメソッドやフィールドを使う
ArrayIndexOutOfBoundsException配列の範囲外の添字を使う
NumberFormatException数値に変換できない文字列を変換する
ClassCastException実体に合わない型へキャストする
ArithmeticException整数の0除算など

同じように見えても違う

値がないことをコンパイラが判断できる

int x;
System.out.println(x); // コンパイルエラー

型としてはStringなのでコンパイルはできる

String s = null;
System.out.println(s.length()); // 実行時例外

確認ポイント

  1. 宣言されていない名前を使っていないか。
  2. 型が合っているか。
  3. アクセスできるか。
  4. ローカル変数は初期化済みか。
  5. checked例外を処理しているか。
  6. ここまで通るなら、実行時の値を追う。

コンパイル成功後に実行時例外が起きる例

`Main.java`として保存する完全なコード

public class Main {
    public static void main(String[] args) {
        String text = null;
        System.out.println("before");
        System.out.println(text.length());
        System.out.println("after");
    }
}

このコードはコンパイルできます。実行するとtext.length()の呼び出し時にNullPointerExceptionが発生し、"after"は出力されません。

似た言葉との違い

混同しやすいもの違い見分け方
型が合わない / 値が不正型が合わないならコンパイルエラー。値が実行時に不正なら例外になりやすい。int x = "1"とargs[0]を区別。
未初期化 / null未初期化ローカル変数はコンパイルエラー。null参照は実行時例外。代入されていないのか、nullが代入されているのかを見る。
エラーあり / 出力ありコンパイルエラーなら出力はない。「何が表示されるか」より先にコンパイル可否。

失敗の種類を見分ける順番

  1. 最初にソースコードがコンパイル可能か確認する。
  2. コンパイルできる場合、クラスが読み込まれてmainが開始できるか確認する。
  3. 実行中に例外が発生する具体的な式を特定する。
  4. 例外処理がある場合は対応するcatchを確認する。
  5. 例外もなく終了する場合は、出力や状態が要求どおりか確認する。

確認用コード

String s = null;
System.out.println(s.length());
見る点このコードでの判断結論
型sはString型コンパイルは可能
値sはnull参照先がない
実行length()呼び出しNullPointerException

ここまでの要点

  • コンパイルエラーと実行時例外を発生時点で区別できる。
  • 未初期化ローカル変数とnull参照の違いを説明できる。
  • コンパイルできないコードには実行結果がないと判断できる。
