コンパイルエラーと実行時例外

コンパイルエラー、クラス読み込み時の失敗、実行時例外、論理エラーを発生段階で分類します。

このレッスンのゴール

コードが実行開始前に止まる場合と、実行中に例外が発生する場合を明確に区別できること。

基本用語の定義

「エラー」という一語では原因を特定できません。Javaでは、どの段階まで処理が進んだかを先に確認します。

コンパイルエラーとはjavacの段階で文法や型の問題により止まること。
実行時例外とはコンパイル後、実行中の値や状態が原因で発生する例外。
スタックトレースとは例外がどのメソッド呼び出し経路で発生したかを示す出力。
到達不能コードとは絶対に実行されないためコンパイルエラーになるコード。
原因切り分けとは文法・型・参照・実行順序のどれが原因か分けること。

失敗を発生段階で分類する

項目定義・規則具体例
コンパイルエラー構文、型、アクセス、確実な代入などの規則に違反している状態。対象コードは通常の実行段階へ進みません。
クラス読み込み時の失敗JVMが必要なクラスを見つけられない、または整合しない状態。ClassNotFoundExceptionやNoClassDefFoundErrorなど。
実行時例外コンパイル後、文や式の評価中に例外オブジェクトが送出される状態。NullPointerExceptionなど。
論理エラー正常終了するが、要求した結果と異なる状態。コンパイラやJVMが必ず検出するものではありません。
エラー発生位置例外は特定の式の評価中に発生します。配列アクセス、除算、メソッド呼び出しなど。
後続処理例外が処理されない限り、その式以降の通常処理は実行されません。対応するcatchまたは呼び出し元へ移ります。

失敗した段階を先に確認する

同じ「動かない」でも、javacが止めたのか、javaで実行中に例外が起きたのかで調べる場所が変わります。出力を考える前に、まずコンパイルできるコードかを確認します。

分類発生タイミング代表例結果
コンパイルエラーjavac時型不一致、未定義変数、checked例外処理漏れ.classが作られず、実行できない
実行時例外java実行中null参照、配列範囲外、整数0除算、キャスト失敗途中まで実行され、例外発生地点で通常処理が止まる
論理ミス実行後出力が期待と違う、条件式が逆エラーは出ないが意図通りではない

コンパイルエラーの典型

原因例なぜ実行前に分かるか
文法ミスint a = ;文の形が成立していない
型不一致int a = "x";代入できない型だと分かる
未初期化ローカル変数int x; println(x);値が入っていない経路が分かる
到達不能コードreturn; println();絶対に実行されないと分かる
checked例外未処理throwsもcatchもない処理強制のルールに反する

実行時例外の典型

例外起きる状況
NullPointerExceptionnull参照からメソッドやフィールドを使う
ArrayIndexOutOfBoundsException配列の範囲外の添字を使う
NumberFormatException数値に変換できない文字列を変換する
ClassCastException実体に合わない型へキャストする
ArithmeticException整数の0除算など

同じように見えても違う

値がないことをコンパイラが判断できる

int x;
System.out.println(x); // コンパイルエラー

型としてはStringなのでコンパイルはできる

String s = null;
System.out.println(s.length()); // 実行時例外

確認ポイント

  1. 宣言されていない名前を使っていないか。
  2. 型が合っているか。
  3. アクセスできるか。
  4. ローカル変数は初期化済みか。
  5. checked例外を処理しているか。
  6. ここまで通るなら、実行時の値を追う。

コンパイル成功後に実行時例外が起きる例

`Main.java`として保存する完全なコード

public class Main {
    public static void main(String[] args) {
        String text = null;
        System.out.println("before");
        System.out.println(text.length());
        System.out.println("after");
    }
}

このコードはコンパイルできます。実行するとtext.length()の呼び出し時にNullPointerExceptionが発生し、"after"は出力されません。

関連する用語との違い

混同しやすいもの違い見分け方
型が合わない / 値が不正型が合わないならコンパイルエラー。値が実行時に不正なら例外になりやすい。int x = "1"とargs[0]を区別。
未初期化 / null未初期化ローカル変数はコンパイルエラー。null参照は実行時例外。代入されていないのか、nullが代入されているのかを見る。
エラーあり / 出力ありコンパイルエラーなら出力はない。「何が表示されるか」より先にコンパイル可否。

失敗の種類を判断する規則

  1. 最初にソースコードがコンパイル可能か確認する。
  2. コンパイルできる場合、クラスが読み込まれてmainが開始できるか確認する。
  3. 実行中に例外が発生する具体的な式を特定する。
  4. 例外処理がある場合は対応するcatchを確認する。
  5. 例外もなく終了する場合は、出力や状態が要求どおりか確認する。

確認用コード

String s = null;
System.out.println(s.length());
見る点このコードでの判断結論
型sはString型コンパイルは可能
値sはnull参照先がない
実行length()呼び出しNullPointerException

コンパイル前と実行中を分ける

Javaの失敗は、まずコンパイル前に見つかる失敗と実行中に起きる失敗に分けます。この区別をせずに「エラー」とまとめると、原因を探す場所を間違えます。

分類いつ起きるか例主な原因
コンパイルエラーjavac時cannot find symbol文法、型、名前、アクセス制御の問題
実行時例外java実行中NullPointerException実行して初めて決まる値や状態の問題
正常終了最後まで実行出力が出る文法も実行時状態も問題なし
// コンパイルエラーの例
String s = 10; // 型が合わない

// 実行時例外の例
String t = null;
System.out.println(t.length()); // NullPointerException

上の2つはどちらも「失敗」ですが、調べる段階が違います。String s = 10;は実行前にコンパイラが止めます。t.length()は文法としては正しいためコンパイルできますが、実行中にtがnullなので失敗します。

  1. まず文法・型・名前・アクセス修飾子でコンパイルできるか見る。
  2. コンパイルできるなら、実行時の値と参照を追う。
  3. 例外が起きるなら、どの行で、どの例外型かを見る。
  4. catchがあるなら、どのcatchに入るかを見る。
ここで整理する誤解

「nullだからコンパイルエラー」は間違いです。null参照からメソッドを呼ぶコードは文法上は成立します。問題は実行時に参照先がないことです。

定義と規則の整理

エラー段階の切り分け不足

つまずき正しい考え方
すべてを「エラー」とだけ呼ぶコンパイルエラー、実行時例外、出力違いを分けないと原因に辿り着けない。
コンパイルできたら正解だと思う実行時例外や期待出力との違いはコンパイル後に発覚する。
エラーメッセージの最初だけ見る行番号、記号の位置、symbol、locationを合わせて読む。

確認できれば次へ

  • コンパイルエラーと実行時例外の違いを説明できる
  • javacのエラー行と^の意味を説明できる
  • 出力を追う前の確認順を説明できる
理解の基準

成立するコードと成立しないコード

成立する例：実行時例外をtry-catchで扱う

String s = null;
try {
    System.out.println(s.length());
} catch (NullPointerException e) {
    System.out.println("null");
}

成立しない例：コンパイル時点で型が合わない

String s = 10;

成立する形：参照型には文字列リテラルを代入する

String s = "10";
System.out.println(s.length());
確認ポイント

コンパイルエラーは実行前に止まります。実行時例外はコンパイル後、実行中に起きます。

定義と規則を確認する

用語の確認

コンパイルエラーと実行時例外の違いを説明できますか。

説明を見る

コンパイルエラーはclassを作る前に止まる問題、実行時例外はコンパイル後に動かしたとき発生する問題です。

コードの確認

次のコードはどちらですか。

String s = null;
System.out.println(s.length());
説明を見る

コンパイルはできますが、実行時にNullPointerExceptionです。型としては呼べますが、参照先がnullです。

仕組みを説明する

実行結果を考える前に、コンパイル可否を確認する理由を説明してください。

説明を見る

コンパイルできなければ実行されないため、出力や実行時例外を考える必要がありません。判定順を固定するとミスが減ります。

基本ルールのまとめ

  • コンパイルエラーと実行時例外を発生時点で区別できる。
  • 未初期化ローカル変数とnull参照の違いを説明できる。
  • コンパイルできないコードには実行結果がないと判断できる。

コードに規則を当てはめる

次のコードはコンパイルエラーか、実行時例外か、正常終了か。

確認用コード

public class Main {
    public static void main(String[] args) {
        int[] a = new int[1];
        System.out.println(a[1]);
    }
}
説明を見る

コンパイルは通るが、実行時例外になる。

aは長さ1なので有効な添字は0だけ。a[1]は範囲外アクセスなのでArrayIndexOutOfBoundsExceptionが起きる。
