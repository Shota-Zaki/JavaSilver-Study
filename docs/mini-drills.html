ミニ演習

短いコードで、コンパイル可否・実行時例外・出力結果を素早く判定する訓練です。

一問ごとに異なる追跡表を使う

このミニ演習は、知識確認ではなく判定速度を上げる訓練です。各問を見たら、先に「コンパイルエラー / 実行時例外 / 出力あり」のどれかへ分類してください。その後に出力値や例外名を詰めます。

20秒以上かかる問題は、単に知識が足りないのではなく、確認順が固定されていない可能性が高いです。間違えたら、該当する学習記事へ戻ってください。

型・代入・リテラル

1. byte演算

byte b = 1;
b = b + 1;
解答

コンパイルエラー。b + 1 は int になり、byte へ暗黙代入できません。

2. 複合代入

byte b = 1;
b += 1;
System.out.println(b);
解答

2。複合代入は暗黙キャストを含みます。

3. 小数リテラル

float f = 1.0;
解答

コンパイルエラー。1.0 は double です。1.0F なら可。

4. 8進数

int x = 09;
解答

コンパイルエラー。先頭 0 は8進数表記で、8や9は使えません。

5. char演算

char c = 'A';
System.out.println(c + 1);
解答

66。char は演算時に数値として扱われ、結果は int です。

6. boolean非互換

boolean b = 1;
解答

コンパイルエラー。Javaでは boolean と数値に互換性はありません。

var・初期化・スコープ

7. varとnull

var x = null;
解答

コンパイルエラー。null だけでは推論する型が決まりません。

8. varと配列初期化子

var a = {1, 2, 3};
解答

コンパイルエラー。配列初期化子だけでは var の推論対象にできません。var a = new int[]{1,2,3}; なら可。

9. ローカル変数

int x;
System.out.println(x);
解答

コンパイルエラー。ローカル変数には初期値がありません。

10. フィールド初期値

class A {
    int x;
    void test() { System.out.println(x); }
}
解答

コンパイル可能。フィールド x は 0 に初期化されます。

String・StringBuilder・比較

11. String不変

String s = "abc";
s.toUpperCase();
System.out.println(s);
解答

abc。String は不変で、戻り値を受け取っていません。

12. substring

String s = "abcd";
s.substring(1, 3);
System.out.println(s);
解答

abcd。substring も新しい文字列を返すだけです。

13. 文字列プール

String a = "ja" + "va";
String b = "java";
System.out.println(a == b);
解答

true。コンパイル時定数同士の連結なので同じ文字列リテラルとして扱われます。

14. new String

String a = new String("java");
String b = "java";
System.out.println(a == b);
解答

false。new で別インスタンスを作っています。

15. StringBuilder可変

StringBuilder sb = new StringBuilder("A");
StringBuilder x = sb;
x.append("B");
System.out.println(sb);
解答

AB。同じ可変オブジェクトを参照しています。

16. StringBuilder.equals

var a = new StringBuilder("A");
var b = new StringBuilder("A");
System.out.println(a.equals(b));
解答

false。StringBuilder は内容比較の equals を提供していません。

配列・List

17. 長さ0配列

int[] a = new int[0];
System.out.println(a.length);
解答

0。長さ0の配列は作れます。

18. 長さ0配列の要素

int[] a = new int[0];
System.out.println(a[0]);
解答

実行時に ArrayIndexOutOfBoundsException。

19. removeのオーバーロード

var list = new java.util.ArrayList<Integer>();
list.add(10);
list.add(20);
list.remove(1);
System.out.println(list);
解答

[10]。remove(1) はインデックス1の削除です。

20. List.of

var list = java.util.List.of("A", "B");
list.add("C");
解答

実行時に UnsupportedOperationException。List.of は変更不可です。

演算子・制御構造

21. 後置インクリメント

int a = 10;
int b = a++ + a;
System.out.println(b);
解答

21。最初の a++ は10を使ってから a を11にします。次の a は11です。

22. 短絡評価

int a = 0;
if (false && ++a > 0) {}
System.out.println(a);
解答

0。&& は左がfalseなら右を評価しません。

23. 非短絡評価

int a = 0;
if (false & ++a > 0) {}
System.out.println(a);
解答

1。& はboolean同士にも使えますが、右辺も評価します。

24. booleanの大小比較

boolean a = true;
boolean b = false;
System.out.println(a > b);
解答

コンパイルエラー。boolean に大小比較は使えません。

継承・ポリモーフィズム・例外

25. 呼べるメソッド

class A {}
class B extends A { void test() {} }
A a = new B();
a.test();
解答

コンパイルエラー。呼べるメソッドは宣言型 A で決まります。

26. override実行

class A { void test(){ System.out.println("A"); } }
class B extends A { void test(){ System.out.println("B"); } }
A a = new B();
a.test();
解答

B。メソッド呼び出しは実体側のoverride実装が動きます。

27. フィールド

class A { String v = "A"; }
class B extends A { String v = "B"; }
A a = new B();
System.out.println(a.v);
解答

A。フィールドはoverrideされず、参照変数の型で決まります。

28. ダウンキャスト

Object o = "java";
Integer i = (Integer) o;
解答

実行時に ClassCastException。コンパイルはできますが実体は String です。

29. catch順序

try {
    throw new java.io.IOException();
} catch (Exception e) {
} catch (java.io.IOException e) {
}
解答

コンパイルエラー。後続の IOException catch は到達不能です。

30. finally

try {
    System.out.print("A");
    int x = 10 / 0;
    System.out.print("B");
} finally {
    System.out.print("C");
}
解答

AC が表示された後、ArithmeticException が送出されます。B には進みません。

複数論点が混ざるコードを分解する

追加1. 宣言型と実体

出力は何ですか。

class P { String x="P"; String m(){return "P";} }
      class C extends P { String x="C"; String m(){return "C";} }
      P v = new C();
      System.out.println(v.x + v.m());
宣言型と実行時型：出力はPCです。フィールドは宣言型P、メソッドは実体Cで選ばれます。判断根拠を分けて説明できることが重要です。
追加2. List.remove

結果のリストは何ですか。

var list = new java.util.ArrayList<Integer>();
      list.add(10); list.add(20); list.add(30);
      list.remove(1);
選ばれるオーバーロード：[10, 30]です。remove(1)は添字1の20を削除します。
追加3. finally

戻り値は何ですか。

static int test() {
          try { return 1; }
          finally { return 2; }
      }
確定する戻り値：2です。finallyのreturnがtryのreturnを置き換えます。
追加4. String不変

出力は何ですか。

String s = "Java";
      s.concat("17");
      System.out.println(s);
不変性の影響：Javaです。concatの戻り値を代入していません。
追加5. 配列共変

コンパイルと実行はどうなりますか。

Object[] a = new String[1];
      a[0] = 10;
配列の実行時型検査：代入はコンパイルできますが、実行時の格納でArrayStoreExceptionです。
追加6. var

var x = null;は有効ですか。

型推論の可否：無効です。具体的な推論型がありません。
