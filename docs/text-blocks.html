テキストブロック

ソースの見た目から実際のString値へ変換される順序を追い、インデント・末尾改行・空白・行継続を正確に読みます。

テキストブロックは記述どおりの文字列ではない

テキストブロックは複数行をそのまま囲むように見えますが、ソースからString値になるまでに整形処理が入ります。出力やlength()を求める問題では、見た目を数えるのではなく、変換後の文字列を作ります。

1開始区切りを除く

開始の"""直後に必要な改行は内容へ含めない。

2共通インデントを除く

閉じ区切りの位置も左端判定に影響する。

3エスケープを解釈する

\s、行継続、\nなどを最後に反映する。

開始区切りの直後には改行が必要

String ok = """
        hello
        """;
// String ng = """hello"""; // コンパイルエラー

開始区切りは3個の二重引用符の後、空白類を挟んでもよいですが、その行で改行して終える必要があります。文字列の内容は、その開始区切り行の次から始まります。

閉じ区切りの位置が末尾改行を決める

閉じ区切りを次の行に置く

String a = """
        red
        blue
        """;

概念上の値はred\nblue\nです。最後の内容行の改行が残ります。

最後の文字の直後に置く

String b = """
        red
        blue""";

概念上の値はred\nblueです。末尾改行はありません。

printlnで表示すると末尾改行の有無が見えにくいため、コード確認では角括弧を付ける、length()を使う、endsWith("\n")を使うなどして差を可視化します。

共通インデントは閉じ区切りを基準に考える

String text = """
        A
          B
        C
        """;

各非空行に共通する左側の空白が除かれ、値は次の形になります。

A
  B
C

Bの前にだけ追加されていた2文字分の空白は残ります。閉じ区切りをさらに左へ移動すると、除去される共通インデント量も変わります。

String text = """
        A
          B
    """;

閉じ区切りのインデントが4文字なら、それが左端候補となり、内容行には追加の空白が残ります。整形済みソースの見た目だけでなく、閉じ区切りの列を確認します。

空白を残したいときは\sを使う

行末の通常空白は、エディタや整形処理で判別しづらく、テキストブロックのインデント処理とも混同しやすいものです。\sは1個の空白文字を明示します。

String text = """
        A\s
        B
        """;

1行目はAの後に空白、その後に改行を持ちます。可視化するとA_\nB\nのような内容です（ここで_は説明用の空白表示）。

行末のバックスラッシュは改行を連結する

String text = """
        one \
        two \
        three
        """;

各行末の行継続エスケープは、バックスラッシュと直後の改行を取り除きます。値はone two three\nです。ソースは複数行でも、生成される文字列は1行になります。

通常の文字列リテラルはソース中の改行を含められないため、この行継続はテキストブロック固有の実用性を持ちます。

二重引用符は通常そのまま書ける

String json = """
        {"name":"Java"}
        """;

1個や2個の二重引用符は、通常の文字列リテラルのように毎回エスケープする必要はありません。ただし、内容中に3個連続する引用符を置く場合は、閉じ区切りと解釈されないよう一部をエスケープします。

通常のStringと同じ操作が適用される

String text = """
        Java
        Silver
        """;

System.out.println(text.length());
System.out.println(text.lines().count());
System.out.println(text instanceof String);

テキストブロックの評価結果はStringです。特別な複数行型ではありません。substring、replace、linesなど、Stringのメソッドをそのまま使用できます。定数式として得られる文字列は通常の文字列リテラルと同様にインターンされます。

文字数問題は不可視文字を記号へ置き換える

String s = """
        A
        BC
        """;

共通インデントを除いた後の値をA↵BC↵と書き直します。文字はA、改行、B、C、改行の5文字です。

次のように閉じ区切りを最後の内容と同じ行へ移すと、A↵BCとなり4文字です。

String s = """
        A
        BC""";

目視で数えず、空白を·、改行を↵などへ置換してから数えます。

変換順序を逆にしない

テキストブロックでは、改行の正規化と共通インデントの除去が行われた後に、エスケープシーケンスが解釈されます。そのため、\sで作った空白はインデント除去の対象として先に消されません。行継続も、インデント判定が済んだ後に改行を消します。

出力を求めるとき：ソース上の行を整える → 共通インデントを除く → \sや行継続を反映する → 最後に文字数・出力を判定します。

通常文字列との使い分け

用途向いている記述理由
短い1行通常の文字列リテラル区切りが簡潔
HTML・JSON・SQLなど複数行テキストブロック引用符と改行を読みやすく保持
動的な値の埋め込みテキストブロック＋formatted等テキストブロック自体に文字列補間機能はない
String sql = """
        SELECT id, name
        FROM users
        WHERE rank = %d
        """.formatted(rank);
