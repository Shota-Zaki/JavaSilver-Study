Java実行・宣言ルール

mainメソッド、ソースファイル名、package/import、起動引数を、コンパイル可否の観点で整理します。

実行できるクラスの条件

Javaコマンドで直接実行するには、実行対象クラスに正しい main メソッドが必要です。戻り値、static、引数、名前のどれかが違うと、起動用メソッドとして扱われません。

要素正しい形よくある誤り
メソッド名mainMain, mane
戻り値voidint, 戻り値省略
修飾子public staticstaticがない、publicがない
引数String[] args または String... argsString args, int[] args
public class Sample {
    public static void main(String[] args) {
        System.out.println("OK");
    }
}

publicクラスとファイル名

1つのソースファイルに複数のトップレベルクラスを書けます。ただし、public なトップレベルクラスは1つまでで、ファイル名と一致する必要があります。

// Sample.java
public class Sample { }
class Helper { }

これは可能です。Helper は public ではないので、同じファイルに置けます。

// Main.java
public class Sample { } // NG: publicクラス名とファイル名が一致しない

packageとimportの順序

パッケージ宣言は、コメントを除いてソースファイルの先頭に置きます。import は package の後、クラス宣言の前です。

順序書けるもの
1コメント
2package 宣言
3import 宣言
4クラス・interface・recordなどの宣言
package com.example;

import java.util.ArrayList;

public class Sample { }
import を package より前に書く、またはJavaに存在しない「packageブロック」を作るとコンパイルエラーです。

importで省略できるもの

java.lang パッケージと同じパッケージ内のクラスはimport不要です。java.util.* のワイルドカードは、そのパッケージ直下のクラスだけが対象です。サブパッケージまでは含みません。

import java.util.*;       // ArrayListは使える
// import java.util.stream.*; は別途必要

ArrayList<String> list = new ArrayList<>();

起動引数

main(String[] args) の args は、コマンドライン引数を文字列配列として受け取ります。引数がなければ null ではなく、長さ0の配列です。

// java Sample A B
System.out.println(args.length); // 2
System.out.println(args[0]);     // A
System.out.println(args[1]);     // B

引数なしで args[0] を読むと ArrayIndexOutOfBoundsException です。

起動引数の個数と内容を追う

シェルで引用符に囲まれた部分は、通常1つの引数として渡されます。Java側では引用符そのものではなく、分割後の文字列がargsに入ります。

java Main alpha "beta gamma" ""
      
      // args[0] = "alpha"
      // args[1] = "beta gamma"
      // args[2] = ""
実行時例外との境界：引数なしでもargs自体は通常nullではなく長さ0の配列です。args[0]へアクセスするとArrayIndexOutOfBoundsExceptionです。

ソース構造と起動条件を確かめる

Q. static public void main(String... args) は有効？

有効です。修飾子の順序は自由で、String... は可変長引数として String[] と同等に起動用mainとして使えます。

Q. import java.util.*; で java.util.stream.Stream は使える？

使えません。ワイルドカードは直下の型だけで、サブパッケージは対象外です。

mainメソッドの微妙な許容範囲

mainメソッドは形が固定に見えますが、修飾子の順序や可変長引数は許容されます。逆に、戻り値や引数型が違うものは、同名でも起動用mainではありません。

コード起動用mainか理由
public static void main(String[] args)可標準形
static public void main(String... args)可修飾子順序自由、可変長引数可
public void main(String[] args)不可staticがない
public static int main(String[] args)不可戻り値がvoidではない
public static void main(String args)不可引数が配列/可変長ではない

mainメソッドは「似ている形」ではなく条件で判定する

起動エントリーポイントとして認識される基本形は、public static void main(String[] args)です。引数名は自由で、配列記法はString args[]でも、可変長引数のString... argsでも構いません。

class Main {
          public static void main(String... values) {
              System.out.println(values.length);
          }
      }
要素必要条件誤答しやすい点
アクセスpublic省略すると通常メソッドとしては正しくても起動対象にならない
所属staticインスタンスを作らずに起動される
戻り値voidintなどは不可
引数String[]またはString...が1つ引数なし、Object[]、複数引数は不可
mainのオーバーロード

main(int value)も定義されています。どちらが起動時に呼ばれますか。

class Main {
          public static void main(String[] args) {
              main(10);
          }
          static void main(int value) {
              System.out.println(value);
          }
      }
起動対象：起動時に呼ばれるのはmain(String[])だけです。main(int)は通常のオーバーロードです。

ファイル名問題の切り方

ファイル名問題では、publicトップレベルクラスだけを探します。publicでないトップレベルクラスは、ファイル名と一致しなくても同じファイルに置けます。

// A.java
public class A { public static void main(String[] args) {} }
class B { public static void main(String[] args) {} }

この場合、javac A.java で A.class と B.class が生成されます。Bにも正しいmainがあれば java B で実行可能です。

演習で問われるソースファイルの判定順

Javaの実行問題は、コードの上から読む前に、コンパイル単位の形を確認します。本文の処理が正しくても、宣言順やファイル名が不正なら出力には到達しません。

  1. package宣言があるなら先頭の宣言位置かを確認する。コメントと空白は前に置けます。
  2. import宣言がpackageより後、型宣言より前にあるかを確認する。
  3. トップレベルのpublic型が複数ないか、ファイル名と一致するかを確認する。
  4. 起動対象に有効なmainメソッドがあるかを確認する。
  5. コンパイルコマンドと実行コマンドが、パッケージ名・出力先・クラスパスに合っているかを確認する。
確認対象正しい例典型的な誤り
宣言順package → import → 型宣言型宣言の後にimportを書く
公開型とファイル名public class MainをMain.javaへ保存Sample.javaにpublic class Mainを書く
起動名java p.Mainパッケージ付きクラスをjava Mainで起動
引数空白で分割された各トークンがargs要素になる引用符の役割を無視して個数を数える
public型とファイル名

次のコードをSample.javaとして保存できますか。

public class Main {
          public static void main(String[] args) { }
      }
保存可否：コンパイルエラーです。公開トップレベル型名Mainとファイル名が一致していません。

javac・java・ソースファイルモードを分ける

javacはソースをクラスファイルへ変換し、javaはクラスまたはソースファイルを実行します。問題では、どの時点で何が生成され、どの名前を指定するかが問われます。

# パッケージなし
      javac Main.java
      java Main
      
      # package p; がある場合
      javac -d out Main.java
      java -cp out p.Main
      
      # ソースファイルモード
      java Main.java one two
コンパイル結果と起動名：-dで作られたパッケージ階層のルートをクラスパスに置き、完全修飾クラス名で起動します。
package付きクラスの実行

package app;を持つMainをout/app/Main.classへ出力しました。どの名前で起動しますか。

実行コマンド：java -cp out app.Mainです。ファイルパスではなく完全修飾クラス名を使います。
