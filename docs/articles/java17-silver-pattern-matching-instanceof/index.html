instanceofパターン変数

Java 16で正式導入され、Java 17で利用できます。Java 11では利用できません。instanceofパターンでは、型判定とキャスト後の変数宣言を同時に書けます。パターン変数のスコープが主な判定点です。

基本形

Object obj = "Java";

if (obj instanceof String s) {
    System.out.println(s.length());
}

obj instanceof String s が true の場合、s は String として使えます。明示的なキャストは不要です。

使える範囲

コード可否理由
if (obj instanceof String s) { s.length(); }可条件がtrueのブロック内では s を使える。
if (!(obj instanceof String s)) { } else { s.length(); }可else側ではStringであることが確定する。
if (obj instanceof String s || s.length() > 0)不可|| の右側では s が存在するとは限らない。
if (obj instanceof String s && s.length() > 0)可&& の右側は左側がtrueのときだけ評価される。

再代入と名前の衝突

パターン変数はローカル変数のように扱われます。同じスコープ内に同じ名前の変数がある場合は宣言できません。

Object obj = "Java";
String s = "test";

if (obj instanceof String s) { // コンパイルエラー
    System.out.println(s);
}

既に同じ名前のローカル変数が見える範囲にあるため、パターン変数 s を宣言できません。

理解の確認

  • && の右側では、左側がtrueならパターン変数を使える。
  • || の右側では、左側がfalseの場合もあるためパターン変数を使えない。
  • 否定条件では、else 側や早期return後の範囲を見ます。
  • 同じスコープに同名変数があると宣言できない。
