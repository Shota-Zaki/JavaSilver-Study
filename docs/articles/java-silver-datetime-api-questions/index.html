Java Silverで出るLocalDate、LocalTime、LocalDateTime、Period、Duration、format、parseの問題を整理します。

先に結論

Date and Time APIは、型と不変性を分けて読みます。plus や minus は元のオブジェクトを書き換えません。戻り値を受け取っているか、Period と Duration の対象が合うかを確認します。

見る基準
  • LocalDate、LocalTime、LocalDateTimeのどれか
  • メソッドの戻り値を受け取っているか
  • PeriodとDurationの対象を混同していないか
  • formatとparseの型が合っているか
  • 存在しない日付や時刻で例外が出るか
Java Practice Labでの使い方日付APIは、暗記だけだと似たメソッドで崩れます。演習では、変数に再代入しているか、戻り値型が何かを必ず書き出します。

まず型を分ける

Date and Time APIでは、日付だけ、時刻だけ、日付時刻のどれを扱っているかを最初に分けます。LocalDate に時刻はなく、LocalTime に日付はありません。

immutableを前提に読む

plusDays や minusMonths は、元のオブジェクトを変更せず、新しい値を返します。戻り値を代入していないコードでは、元の変数は変わりません。

PeriodとDuration

型主な対象注意点
Period年・月・日日付ベースで扱う
Duration時・分・秒時間ベースで扱う

formatとparse

formatは日時オブジェクトを文字列へ変換し、parseは文字列から日時オブジェクトを作ります。書式と文字列が一致しない場合は例外が発生します。

練習方法

Date and Time APIは、表で型とメソッドを整理してから問題を解くと安定します。Goldではローカライズと絡むため、Silver段階で基本型を固めておきます。
