先に結論

Java Practice Labは、読む、解く、間違える、戻るためのサイトです。参考書の代わりではなく、参考書で得た知識をコード読解に変える場所として使います。単元別解説、通常演習、固定模試、復習導線を順に使います。

全体の使い方

Java Practice Labは、参考書を置き換えるサイトではなく、コード読解と復習を補強するための学習サイトです。基本の流れは、単元別解説で前提を確認し、通常演習で手を動かし、固定模試で総合確認し、結果画面から復習へ戻る形です。

  1. 単元別解説で論点を確認する。
  2. 通常演習で同じ論点の問題を解く。
  3. 間違えた問題の解説を読む。
  4. 復習対象に戻して後日解き直す。
  5. 固定模試で時間配分と総合力を確認する。

単元別解説の使い方

単元別解説は、問題を解く前後に使います。初めての論点は先に読み、間違えた論点は解説を読んだ後に戻ります。継承、例外、ラムダ式、Streamなど、判断順が必要な論点ほど効果があります。

通常演習の使い方

通常演習は、特定の単元を集中して確認するために使います。模試で失点した単元に戻るとき、参考書で読んだ章を確認するとき、同じ種類の問題を繰り返したいときに使いやすいです。

固定模試の使い方

固定模試は、範囲を一通り学習した後に使います。初期段階で模試だけを連続で解くと、弱点の原因が見えにくくなります。模試は点数確認ではなく、次に戻る単元を決めるために使います。

  • 1回目：現在地の確認。
  • 2回目：弱点復習後の伸びを確認。
  • 直前期：時間配分と見直し方を確認。

結果画面と復習導線の使い方

採点後は、正答率だけで判断しません。間違えた問題、迷った問題、正答したが根拠が曖昧だった問題を復習します。解説では、正答理由だけでなく、誤答選択肢の除外理由まで確認します。
