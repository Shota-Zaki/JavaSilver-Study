switch式・yieldの境界ケース

switch式はJava 14で正式導入され、Java 17では利用できます。Java 11では利用できません。ここでは文と式、アロー形式とコロン形式、網羅性を分けて判断します。

代入の右辺にあるswitchは値を返す必要がある

int level = 2;
String label = switch (level) {
    case 1 -> "LOW";
    case 2 -> "MID";
    default -> "HIGH";
};

switch全体がString値を返し、代入文の末尾にセミコロンがあります。アロー形式ではフォールスルーしません。

ブロックから値を返すときにyieldを使う

String label = switch (level) {
    case 1 -> "LOW";
    case 2 -> {
        String text = "MID";
        yield text;
    }
    default -> "HIGH";
};

yieldはswitch式の値を確定し、そのswitch式から抜けます。returnはメソッド全体から戻るため、用途が異なります。

コロン形式のswitch式でもbreakではなくyield

String label = switch (level) {
    case 1:
        yield "LOW";
    case 2:
        yield "MID";
    default:
        yield "HIGH";
};

従来のswitch文で使うbreakは値を返しません。switch式のコロン形式で結果を返す経路にはyieldが必要です。

switch式は網羅的でなければならない

// コンパイルエラー: levelが1以外のときの値がない
// String label = switch (level) {
//     case 1 -> "LOW";
// };

通常はdefaultを用意します。enum型など、コンパイラが全候補を網羅していると判断できる場合は、全定数を列挙してdefaultを省略できることがあります。

nullはJava 17の通常のswitchで特別に一致しない

String value = null;
// 次の式を実行するとNullPointerException
// int length = switch (value) {
//     case "A" -> 1;
//     default -> 0;
// };

Java 17の通常のswitch式では、セレクタがnullならcase照合の前にNullPointerExceptionです。後のJavaで追加されたswitchパターンの規則をJava 17のコードへ持ち込まないでください。

記事末尾の確認

  • switch文とswitch式を代入の有無だけでなく、値と網羅性で区別できる。
  • break、yield、returnの作用範囲を説明できる。
  • Java 11のコードへswitch式を持ち込まない。
  • Java 17の通常switchでnullをcaseとして扱わない。
