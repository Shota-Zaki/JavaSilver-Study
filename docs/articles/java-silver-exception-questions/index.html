Java Silverの例外処理問題の解き方

Java Silverの例外処理問題を、checked / unchecked、throw / throws、catch順序、finally、try-with-resourcesの観点で整理します。

先に結論

例外処理問題は、先にコンパイル可否を切ります。checked exceptionの処理漏れがあれば実行結果は追いません。コンパイルできる場合だけ、例外発生位置、catchの一致、finallyの実行順を追います。

見る基準
  • checked exceptionをcatchまたはthrowsで処理しているか
  • catchの順序が到達不能になっていないか
  • 例外発生後にtry内の後続処理を続けていないか
  • finallyの実行タイミングを見ているか
  • try-with-resourcesのclose順を追っているか
Java Practice Labでの使い方例外問題は、単元別解説で型と流れを整理してから演習に戻します。誤答した問題は、発生行、移動先catch、finallyの有無を短くメモすると再現性が上がります。

最初にコンパイル可否を見る

例外問題では、実行時にどのcatchへ行くかより前に、checked exceptionを処理しているかを確認します。throws宣言またはcatchがない場合、実行順を追う前にコンパイルエラーです。

checkedとunchecked

分類処理の必要性例
checked exceptionコンパイル時に処理が必要IOExceptionなど
unchecked exceptionコンパイル時の処理は不要NullPointerException、ArithmeticExceptionなど

catch順序

catchは上から判定します。親クラスの例外を先にcatchすると、後ろの子クラスcatchへ到達できないためコンパイルエラーになります。例外階層を見て、具体的な型を先に置くのが基本です。

finallyとreturn

finally は、通常終了でも例外発生でも実行されます。returnが絡む問題では、tryやcatchで戻る前にfinallyが実行される点を確認します。ただし、finally内でreturnすると読みづらく、試験では結果の上書きが問われます。

練習方法

例外問題は、例外処理の単元別解説で分類を確認してから、通常演習でcatch順序とfinallyの流れを練習します。
