先に結論

Silver模試は、参考書を一周した後に使います。最初から高得点を狙うより、どの単元で落としているかを見るために解きます。復習では正答率より、誤答理由を分類する方が得点につながります。

見る基準
  • 模試前に基礎単元を一通り解いたか
  • 時間内に60問を読めるか
  • 迷った問題を記録しているか
  • 正解したが根拠が薄い問題を復習対象にしているか
  • 得点後に次の行動を変えているか
Java Practice Labでの使い方模試結果は、単元別解説へ戻す入口です。50%未満なら基礎へ、合格ライン前後なら頻出ミスへ、80%以上なら時間配分と取りこぼしの確認へ進みます。

模試を解くタイミング

参考書を1周し、主要単元の演習を一通り解いた後に模試へ入ります。基礎文法が曖昧なまま模試を解くと、復習範囲が広すぎて効率が落ちます。

結果画面で見るべきこと

  • どの単元で間違えたか
  • どの選択肢を選んだか
  • 正答との差は何か
  • コンパイル可否を誤ったか
  • 実行順を誤ったか
  • 例外条件を見落としたか

点数だけでなく、誤答理由を分類すると次の復習が明確になります。

当サイトでの模試活用

固定模試を解いた後は、結果画面から解説を読みます。苦手単元は単元別解説へ戻り、見直し問題や間違えた問題を復習導線で解き直します。
