Java Silverに落ちた後にやること

Java Silverで不合格だった後に必要なのは、気合いで問題数を増やすことではありません。
落とした理由を分解し、次の受験までに「同じ判定ミスをしない状態」へ戻すことです。

先に結論

Silverに落ちた後は、すぐ同じ問題集を一周するより、失点の種類を分けます。時間切れ、論点不足、コード読解不足、模試復習不足では、次にやることが違います。

見る基準
  • 本番で時間が足りなかったか
  • コンパイル可否の誤答が多かったか
  • 継承・例外・ラムダのどこで落としたか
  • 模試で正解した問題の根拠まで説明できていたか
  • 再受験までの日数が何日あるか
Java Practice Labでの使い方再受験前は、全範囲を薄く回すより、失点単元を演習で戻します。模試の誤答を復習対象にし、同じ論点の通常演習で判断手順を確認します。

最初の1日は再受験の予約より分析を優先する

不合格直後にすぐ再受験日だけ決めても、原因が残ったままなら同じ失点を繰り返します。最初にやることは、点数を眺めることではなく、失点の種類を分けることです。

最初に残す情報
  • 総合点
  • 模試で安定していた点数
  • 本番で時間が足りたか
  • 解けるつもりで落とした単元
  • 迷った問題の種類

特に重要なのは「解けるつもりで落とした単元」です。ここは理解不足ではなく、読み方の癖で失点している可能性が高いです。

不合格の原因は4種類に分ける

原因起きていること戻る場所
知識不足用語・ルールを知らない。例: interfaceのメソッド修飾子、例外分類、varの使用可能箇所。単元別解説 / 覚える表
判定順ミス出力から考え始めて、コンパイルエラーや実行時例外を見落とす。通常演習 / 関連論点の読解
コード追跡ミス参照、スコープ、インクリメント、StringBuilderの変更有無を途中で取り違える。単元別演習
時間配分ミス難問に時間を使いすぎて、取れる問題を失点する。模試 / 見直しフラグ練習

全部を「勉強不足」でまとめると、次も同じ場所で失点します。原因は必ず分けると判定しやすくなります。

まず整理するべき読み方の癖

Java Silverで落ちた後は、知識を増やす前に読み方を直します。特に次の癖は失点に直結します。

  • 出力候補を先に見て、コンパイル可否を後回しにする
  • 参照コピーでオブジェクト本体も増えるように読んでしまう
  • StringBuilderのメソッドをすべて本体変更ありで処理する
  • オーバーロードとオーバーライドを名前だけで判断する
  • catchに入った後、tryブロックの続きへ戻るように読む

失点例1: 出力を見る前にコンパイル可否を見る

public class Sample {
    public static void main(String[] args) {
        byte a = 10;
        byte b = 20;
        byte c = a + b;
        System.out.println(c);
    }
}

このコードは出力を考える前に止まります。byte + byte の結果は int です。int を byte にそのまま代入できないため、コンパイルエラーです。

この手の問題で出力候補を探し始めると失点します。最初に見るのは、式の型と代入先の型です。

失点例2: コンストラクタとメソッドを分ける

class Item {
    void Item() {
        System.out.println("init");
    }
}

public class Main {
    public static void main(String[] args) {
        new Item();
    }
}

void Item() はコンストラクタになりません。戻り値型があるため、これはメソッドです。new Item(); ではこのメソッドは呼ばれません。出力はありません。

コンストラクタ問題では、名前だけで判断すると誤答になります。戻り値型がある時点でコンストラクタではありません。

失点例3: 参照コピーで本体は増えない

class Box {
    int value;
}

public class Main {
    public static void main(String[] args) {
        Box a = new Box();
        Box b = a;
        b.value = 10;
        System.out.println(a.value);
    }
}

b = a; は参照をコピーします。Boxオブジェクト本体は増えません。a と b は同じオブジェクトを指すため、出力は 10 です。

やり直しの順番

  1. 本番で落とした原因を4種類に分ける
  2. 模試の不正解だけを解き直す
  3. 同じタグの通常演習に戻る
  4. 対応する単元別解説で見るポイントを見る
  5. 別の模試で時間内に解けるかを見ます

この順番を崩すと、復習が「解説を読んで納得するだけ」で終わります。納得では点数は戻りません。次に同じ形が出たときに、同じ順番で判定できる状態を作ります。

再受験までの2週間プラン

期間やること目標
1〜3日目不正解をタグ別に分ける。弱点上位3つだけ通常演習で戻す。誤答理由を言語化する
4〜7日目弱点タグの総合問題を解く。解説は選択肢別に読む。同じミスを減らす
8〜10日目模試を1回解く。未回答、見直し、難易度別正答率を見る。時間内に解く
11〜13日目模試の不正解だけ再挑戦する。弱点単元別解説へ戻る。弱点を再発させない
前日覚える表と間違えた問題だけ判定する。新しいことを増やさない

  • 「この模試の不正解だけ解き直す」で、間違えた問題だけ再挑戦する
  • 同じタグの通常演習へ戻る
  • 対応する単元別解説で、判定順を見る
