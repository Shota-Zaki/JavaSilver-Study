Optionalの生成・変換・代替値

Optionalは「nullを完全に除外する型」ではありません。「値があるかもしれない処理」を、型として明示するための仕組みです。
Goldでは、作り方、取り出し方、遅延評価、mapとflatMap、Streamとの接続を型で読めないと、戻り値の型や重複キーの判定を誤ります。

最初に見るべき判断軸

Optionalのコードは、メソッド名だけで判断すると間違えます。
まず、Optionalが「空かもしれない」のか、「中身の型が何か」、「代替値をいつ評価するか」を判定します。

見る場所見ること混同しやすい点

取り出しget / orElse / orElseGet / orElseThrowgetは空ならNoSuchElementException。
変換map / flatMap / filter戻り値が通常値ならmap、OptionalならflatMap。
Stream連携findFirst / max / reduce / Optional.stream終端操作がOptionalを返すパターンを型で追う。

Optionalの作り方

Optionalを生成するときは、ofとofNullableのnull処理を分けます。値がnullでないと確定している場合だけofを使います。nullの可能性があるならofNullableです。

Optional<String> a = Optional.of("java");
Optional<String> b = Optional.ofNullable(null);
Optional<String> c = Optional.empty();

System.out.println(a); // Optional[java]
System.out.println(b); // Optional.empty
System.out.println(c); // Optional.empty
書き方値がnullでない値がnull
Optional.of(value)Optional[value]NullPointerException
Optional.ofNullable(value)Optional[value]Optional.empty
Optional.empty()常に空のOptional
 
String value = null;
// Optional<String> x = Optional.of(value); // NullPointerException
Optional<String> y = Optional.ofNullable(value); // Optional.empty
Optional.of(null)は「空のOptional」ではありません。of(null)は呼び出した時点でNullPointerExceptionを投げます。空を許容する生成にはofNullableを使います。

空のOptionalでget()を呼ぶと例外が発生する

get()は中身を直接取り出します。空のOptionalに対して呼ぶとNoSuchElementExceptionです。コンパイルエラーではありません。

Optional<String> opt = Optional.empty();
// System.out.println(opt.get()); // NoSuchElementException

isPresent()で判定してからget()するコードも出ます。ただし、実務上はmap、orElse、orElseGet、orElseThrowで処理する方が自然です。

Optional<String> opt = Optional.of("java");
if (opt.isPresent()) {
    System.out.println(opt.get().toUpperCase()); // JAVA
}

isPresent() と isEmpty()

isPresent()は値があるならtrue、isEmpty()は値がないならtrueです。isEmpty()はJava 11以降なので、Gold SE 11 / SE 17では出ても不自然ではありません。

Optional<String> a = Optional.of("A");
Optional<String> b = Optional.empty();

System.out.println(a.isPresent()); // true
System.out.println(a.isEmpty());   // false
System.out.println(b.isPresent()); // false
System.out.println(b.isEmpty());   // true

orElse と orElseGet の違い

どちらも空の場合の代替値を返します。違いは、代替値を作る処理がいつ実行されるかです。orElseの引数は、Optionalに値があっても先に評価されます。orElseGetのSupplierは、空の場合だけ実行されます。

static String fallback() {
    System.out.println("fallback");
    return "default";
}

public static void main(String[] args) {
    String a = Optional.of("A").orElse(fallback());
    System.out.println(a);

    String b = Optional.of("B").orElseGet(() -> fallback());
    System.out.println(b);
}
式Optionalに値がある場合Optionalが空の場合
orElse(fallback())fallback()は呼ばれるが、戻り値は使われないfallback()が呼ばれて、その値を返す
orElseGet(() -> fallback())Supplierは呼ばれないSupplierが呼ばれて、その値を返す
悪い判断

orElseは値があるなら引数が実行されないと読みます。

良い判断

orElseの引数は通常のメソッド引数なので、メソッド呼び出し前に評価される。

orElseThrow

値があれば中身を返し、空なら例外を投げます。Java 10以降は、引数なしのorElseThrow()も使えます。空の場合はNoSuchElementExceptionです。

String a = Optional.of("A").orElseThrow();
// String b = Optional.<String>empty().orElseThrow(); // NoSuchElementException

String c = Optional.of("C")
    .orElseThrow(() -> new IllegalStateException("missing"));

Supplier版のorElseThrowでチェック例外を投げる場合、その例外を処理または宣言する必要があります。

static String read(Optional<String> opt) throws IOException {
    return opt.orElseThrow(() -> new IOException("no value"));
}

or はOptionalそのものを代替する

orElseやorElseGetは中身の値を返します。一方、orは代替のOptionalを返します。Supplierが返す型はOptional<T>です。

Optional<String> primary = Optional.empty();
Optional<String> result = primary.or(() -> Optional.of("backup"));

System.out.println(result); // Optional[backup]
メソッド戻り値代替側の型
orElse(value)TT
orElseGet(supplier)TSupplier<? extends T>
or(supplier)Optional<T>Supplier<? extends Optional<? extends T>>

ifPresent と ifPresentOrElse

ifPresentは値がある場合だけConsumerを実行します。ifPresentOrElseは値がある場合と空の場合の両方を分けて処理できます。

Optional.of("java")
    .ifPresent(s -> System.out.println(s.toUpperCase())); // JAVA

Optional.empty()
    .ifPresentOrElse(
        v -> System.out.println(v),
        () -> System.out.println("empty")
    ); // empty

ifPresentは戻り値を返しません。値を変換したいならmap、値を取り出したいならorElse系を使います。

map は中身を変換する

mapは、値がある場合だけ関数を実行し、結果をOptionalで包み直します。空の場合は関数を実行せず、そのまま空です。

Optional<String> s = Optional.of("java");
Optional<Integer> len = s.map(String::length);

System.out.println(len); // Optional[4]

mapに渡した関数がnullを返した場合、結果はOptional.empty()になります。

Optional<String> x = Optional.of("A")
    .map(v -> null);

System.out.println(x); // Optional.empty
map内でnullが返った場合は、null入りOptionalではなく空Optionalになります。ここは代表的な混同しやすい点です。

flatMap はOptionalの入れ子を防ぐ

関数の戻り値が通常値ならmap、戻り値がOptionalならflatMapです。mapを使うとOptional<Optional<T>>になります。

record User(String name, Optional<String> email) {}

Optional<User> user = Optional.of(new User("taro", Optional.of("a@example.com")));

Optional<Optional<String>> wrong = user.map(User::email);
Optional<String> right = user.flatMap(User::email);

System.out.println(wrong); // Optional[Optional[a@example.com]]
System.out.println(right); // Optional[a@example.com]

flatMapに渡す関数は、nullではなくOptionalを返す必要があります。関数がnullを返すとNullPointerExceptionです。

// Optional<String> x = Optional.of("A").flatMap(v -> null);
// NullPointerException
関数の戻り値使うメソッド結果の型
RmapOptional<R>
Optional<R>flatMapOptional<R>
boolean判定filterOptional<T>

filter は条件に合わなければ空にする

filterは、値があり、かつPredicateがtrueを返した場合だけ、その値を残します。空の場合はPredicateを実行しません。

Optional<String> a = Optional.of("java")
    .filter(s -> s.length() > 3);

Optional<String> b = Optional.of("java")
    .filter(s -> s.length() > 10);

System.out.println(a); // Optional[java]
System.out.println(b); // Optional.empty

Predicate自体がnullならNullPointerExceptionです。

// Optional.of("A").filter(null); // NullPointerException

OptionalとStream終端操作

Streamの一部の終端操作は、結果が存在しない可能性があるためOptionalを返します。空StreamでmaxやfindFirstを呼んでも、例外ではなく空Optionalです。

終端操作戻り値空Streamの場合
findFirst()Optional<T>Optional.empty
findAny()Optional<T>Optional.empty
max(comparator)Optional<T>Optional.empty
min(comparator)Optional<T>Optional.empty
reduce(binaryOperator)Optional<T>Optional.empty
 
Optional<String> first = Stream.of("A", "B")
    .filter(s -> s.startsWith("Z"))
    .findFirst();

System.out.println(first); // Optional.empty

Stream内のnullとOptional

Streamは途中にnull要素を持てる場合があります。ただし、findFirstなどOptionalで包んで返す終端操作がnullを選ぶとNullPointerExceptionになる可能性があります。安全に扱うなら、先にnullを除外します。

// Optional<String> x = Stream.of((String) null, "A").findFirst();
// NullPointerException

Optional<String> y = Stream.of((String) null, "A")
    .filter(Objects::nonNull)
    .findFirst();

System.out.println(y); // Optional[A]

Optional.stream()

Optional.stream()は、値があれば1要素のStream、空なら0要素のStreamを返します。Stream<Optional<T>>から空を除いて中身だけ流したい場合に使います。

List<Optional<String>> list = List.of(
    Optional.of("A"),
    Optional.empty(),
    Optional.of("B")
);

List<String> result = list.stream()
    .flatMap(Optional::stream)
    .toList();

System.out.println(result); // [A, B]

Java 8ではOptional.stream()がないため、別の書き方が必要です。Gold SE 11 / SE 17では使える前提でよいです。

プリミティブ用Optional

Optional<int>は書けません。プリミティブ用にはOptionalInt、OptionalLong、OptionalDoubleがあります。ただし、Optional<Integer>とOptionalIntは別物です。

型値の取得空の場合
Optional<Integer>get()NoSuchElementException
OptionalIntgetAsInt()NoSuchElementException
OptionalLonggetAsLong()NoSuchElementException
OptionalDoublegetAsDouble()NoSuchElementException
 
OptionalInt max = IntStream.of(3, 1, 4).max();
System.out.println(max.orElse(-1)); // 4

OptionalInt empty = IntStream.empty().max();
System.out.println(empty.orElse(-1)); // -1

Optionalをメソッド引数やフィールドに乱用しない

ここでは主に戻り値やStream終端操作として出ます。Optionalは「戻り値がない可能性」を伝えるのに向いています。
一方、フィールドやメソッド引数にOptionalを乱用すると、コードの読みやすさが下がります。
理解上の成立可否の判断ではコンパイル可否が優先ですが、設計意図としては戻り値で見るのが基本です。

static Optional<String> findName(int id) {
    if (id == 1) {
        return Optional.of("Alice");
    }
    return Optional.empty();
}

Optional.of("A").orElse(f())f()は実行されるメソッド引数は呼び出し前に評価される。
Optional.of("A").orElseGet(f())コンパイルエラーになりやすい引数は値ではなくSupplier。通常は() -> f()。

読解ミニ演習 1：orElseとorElseGet

次のコードの出力を判定します。

import java.util.*;

public class Main {
    static String f(String s) {
        System.out.print(s);
        return s;
    }

    public static void main(String[] args) {
        String a = Optional.of("A").orElse(f("B"));
        String b = Optional.of("C").orElseGet(() -> f("D"));
        System.out.print(a + b);
    }
}
結果：BAC

orElse(f("B"))では、Optionalに値があってもf("B")が先に実行されます。その後、値としてはAが採用されます。orElseGet側は値があるのでf("D")は実行されません。最後にa + bでACが出力され、全体はBACです。

読解ミニ演習 2：mapとflatMap

import java.util.*;

public class Main {
    public static void main(String[] args) {
        Optional<String> a = Optional.of("java")
            .map(s -> null);

        Optional<String> b = Optional.of("java")
            .flatMap(s -> Optional.of(s.toUpperCase()));

        System.out.println(a + ":" + b);
    }
}
結果：Optional.empty:Optional[JAVA]

mapの関数がnullを返した場合は空Optionalになります。flatMapでは関数がOptional.of("JAVA")を返すため、入れ子にならずOptional[JAVA]になります。

読解ミニ演習 3：StreamとOptional

import java.util.*;
import java.util.stream.*;

public class Main {
    public static void main(String[] args) {
        var result = List.of(Optional.of("A"), Optional.empty(), Optional.of("B"))
            .stream()
            .flatMap(Optional::stream)
            .collect(Collectors.joining("-"));

        System.out.println(result);
    }
}
結果：A-B

Optional::streamにより、値があるOptionalは1要素Stream、空Optionalは0要素Streamになります。その結果、AとBだけが流れ、joining("-")で連結されます。

理解の確認

Optionalを使うコードは、感覚で読まずに型を紙に書くべきです。特にmapとflatMap、orElseとorElseGetは、暗記ではなく戻り値型と評価タイミングで判定します。

  • of(null)は空ではなくNullPointerException。
  • get()は空ならNoSuchElementException。
  • orElseは代替値を先に評価する。
  • orElseGetは空の場合だけSupplierを実行する。
  • mapは通常値をOptionalで包む。
  • flatMapはOptionalを返す関数に使う。
  • mapでnullを返すと空Optional。
  • flatMapでnullを返すとNullPointerException。
  • Optional.stream()は値ありなら1要素、空なら0要素。
  • Optional<Integer>とOptionalIntは別型。
誤った読み方

Optionalなら常に安全、orElseの引数は空のときだけ評価、mapとflatMapは同じ、と決めつける。

根拠を追う読み方

生成時のnull、空の場合の取り出し、代替値の評価タイミング、関数の戻り値型を順番に確認する。

Optionalの確認

Optionalは、値がある場合とない場合で分けて読みます。特に orElse と orElseGet の評価タイミングが扱われることが多いです。

  • get() は空の場合に例外です。
  • orElse(x) は、xが先に評価されます。
  • orElseGet(supplier) は、必要な場合だけSupplierが実行されます。
  • map と flatMap は戻り値の包まれ方が違います。
