Java Silver合格後にJava Goldへ進む前にやること

Silver合格直後にGoldへ進むなら、最初に見るべきなのは「Goldの新しいAPI」ではありません。
Silver範囲の読解力が残っているか、Goldで増える戻り値型・例外・ジェネリクスの読み方へ移れるかを判定します。

結論

Silverの出力問題を感覚で解いていた状態のままGoldへ入ると、Stream、Collectors、Genericsで止まります。
Goldでは「何が表示されるか」より先に、型、戻り値、例外、終端操作、変更可能性を読む必要があります。

Silver合格後すぐにGoldへ進める状態

Silverに合格していても、Goldへすぐ進める状態とは限りません。次の項目が説明できるなら、Goldへ入ってよい状態です。

見る項目進める状態戻るべき状態
コンパイル可否出力を考える前に、宣言・型・アクセス修飾子を判定できる出力候補から先に選んでしまう
型変換byte + byte が int になる理由を説明できる小さい型同士の計算結果も小さい型になると思っている
継承オーバーライドとフィールド隠蔽を分けて読めます変数の宣言型と実体型を混同する
例外checked例外、catch順序、finallyの実行順を説明できる例外発生後にtryの続きへ戻ると思っている
API読解メソッド名だけでなく戻り値型と例外を読むメソッド名の雰囲気だけで挙動を決める

Goldで最初に増える負荷

Goldでは、Silverよりも「APIの仕様を読む問題」が増えます。暗記量が増えるだけではなく、戻り値型と総称型を読み失点すると正解できない問題が増えます。

1. 戻り値型を読む量が増える

Goldでは、メソッド名を知っているだけでは足りません。戻り値型が Optional<T> なのか、List<T> なのか、Stream<T> なのかで次に呼べるメソッドが変わります。

var result = List.of("a", "bb", "ccc")
    .stream()
    .filter(s -> s.length() > 1)
    .map(String::length)
    .toList();

このコードでは、filter 後も Stream<String>、map 後は Stream<Integer>、toList() 後は List<Integer> です。Goldではこの型の流れを追います。

2. ジェネリクスで「入れられる」「取り出せる」が分かれる

List<? extends Number> a = List.of(1, 2, 3);
Number n = a.get(0);
// a.add(10); // コンパイルできない

? extends Number は、Numberとして取り出せます。ただし、具体的な要素型が Integer なのか Double なのか確定しないため、値を追加できません。

List<? super Integer> b = new ArrayList<Number>();
b.add(10);
Object value = b.get(0);

? super Integer は、Integerを追加できます。ただし、取り出すときは最低限 Object として扱います。Goldでは、この差を曖昧にすると失点します。

3. Streamは中間操作だけでは動かない

var s = Stream.of("a", "bb", "ccc")
    .filter(x -> {
        System.out.println(x);
        return x.length() > 1;
    });

この時点では何も表示されません。filter は中間操作です。終端操作が呼ばれるまで処理は実行されません。

long count = s.count();

count() が終端操作なので、この時点で評価されます。Goldでは、コードの上から順番に全部動くわけではありません。

Goldへ入る前にSilverで戻るべき単元

Goldへ進む前に、次の単元で崩れるなら先にSilverへ戻った方が効率的です。

崩れる単元Goldで困る理由戻る演習

String / ArrayListCollections、List、toListの変更可能性で迷うString・配列・コレクション基礎
継承・interface関数型インタフェース、defaultメソッド、Comparatorで崩れる継承・interface・ポリモーフィズム

Goldの最初の学習順

Goldは、いきなりStreamから入るより、土台になる順番で進めた方が安定します。

  1. ジェネリクス：<T>、? extends、? super、型消去を先に固める。
  2. ラムダ式・関数型インタフェース：Predicate、Function、Consumer、Supplierの形を覚える。
  3. Optional：empty、map、flatMap、orElse、orElseGetの差を判定する。
  4. Stream：中間操作、終端操作、遅延評価、再利用不可を読む。
  5. Collectors：toMap、groupingBy、partitioningBy、下流Collectorを読む。
  6. NIO / 日時 / JDBC：API仕様とchecked例外を整理する。
  7. 並行処理 / モジュール：出力順、Future、module-infoを整理する。

Gold模試へ進む基準

Gold模試は、最初から点数を取りに行くものではありません。最初の模試は、時間内にどの分類で崩れるかを見るために使います。

  • Generics、Lambda、Streamの通常演習で6割以上取れる
  • OptionalとCollectorsの戻り値型を説明できる
  • NIOと例外処理の関係を説明できる
  • 解説を読んだあと、なぜ誤答が誤りか説明できる

この条件を満たす前に模試だけ回すと、点数は出ても復習対象が多すぎて整理できません。

よくある誤解を整理する

Silverに合格しただけではGoldのAPI問題は解けません

Silverの読解力は前提です。Goldでは、戻り値型、総称型、例外、変更可能性を読む力が追加で必要です。

Streamはfor文の書き換えだけでは終わらない

Streamでは、遅延評価、終端操作、短絡操作、再利用不可を判定します。for文と同じ感覚で上から順に実行すると読み間違えます。

ジェネリクスは書き方の暗記だけでは足りない

? extends と ? super は、値を入れる側と取り出す側で制約が変わります。PECSを使って、どちらの立場で使っているか判定します。

Goldは問題文が長いだけの試験ではない

長文を読む力より、型とAPI仕様を順番に切る力が必要です。読解の中心は文章量ではなく、コード上の型の流れです。
