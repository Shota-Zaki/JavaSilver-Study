直前チェックモード

試験直前に見るべき頻出論点だけを短時間で整理します。

広告スポンサーリンク表示エリア

コンパイルエラーと実行時例外を混ぜない

分類例判断
コンパイルエラー型不一致、未初期化ローカル変数、アクセス不可実行前に止まる。出力はない。
実行時例外null参照、配列範囲外、キャスト失敗そこまでの出力は残る可能性がある。
正常出力型・アクセス・例外がすべて通る最後に評価順を追う。

この単元をコードに適用する手順

  1. まず宣言を読みます。型、修飾子、メソッド名、引数を固定する。
  2. 次に変更される値を探します。代入、メソッド呼び出し、副作用を表にする。
  3. 最後に出力を組み立てます。途中で止まる場合は、そこまでの出力だけを残します。
理解したつもりのままでは、似たコードに変えたときに誤答しやすくなります。本文の例を1行ずつ書き換えると、どこで結果が変わるか整理しやすいです。

よくある誤解と修正

誤解正しい見方判定方法
見た目が似ていれば同じ動きになる型、戻り値、参照、修飾子が1つ違うだけで結果は変わる。行ごとに「型」「値」「参照先」をメモする。
実行結果だけ追えばよいコンパイルできなければ実行されない。例外で止まれば後続行は動かない。コンパイル可否 → 実行時例外 → 出力の順に判定する。
暗記した用語だけで判断できる用語は入口にすぎない。コード上のどの行に効いているかまで見る。本文の例を1行変更し、結果が変わる理由を説明できる形にする。

ミニドリル

次の形式で、自分の答えを文章にします。正解だけを出しても、弱点は減りません。

  1. このコードはコンパイルできるか。
  2. 実行時に例外が起きるなら、どの行で起きるか。
  3. 正常に動くなら、出力は何か。
  4. 自分が間違えた場合、原因は「型」「参照」「戻り値」「アクセス」「例外」「評価順」のどれか。
学習時は、選択肢を見る前に自分の判定を作ると整理しやすいです。選択肢を先に見ると、もっともらしい誤答に引っ張られます。
