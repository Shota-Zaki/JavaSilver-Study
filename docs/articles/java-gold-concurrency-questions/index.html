Java Goldの並行処理問題の解き方

Java Goldの並行処理問題を、Thread、Runnable、Callable、ExecutorService、Future、synchronized、parallel streamで整理します。

先に結論

並行処理は、処理順を固定して読まないことが出発点です。ExecutorService、Future、synchronized、parallel streamは、いつ実行されるか、待機するか、順序が保証されるかを分けます。

見る基準
  • タスクをsubmitしただけで結果を待っているか
  • Future#getで待機しているか
  • shutdownとawaitTerminationの役割を分けているか
  • synchronizedの対象オブジェクトを見ているか
  • parallel streamで順序保証を決めつけていないか
Java Practice Labでの使い方並行処理の問題は、出力順を断定する前に保証の有無を確認します。解説では、確定することと確定しないことを分けて読みます。

並行処理は結果の固定性を見る

並行処理問題では、実行順が固定されるとは限りません。出力順、共有変数、同期、戻り値の取得、終了待ちの有無を分けて読みます。

ExecutorServiceとFuture

submit はFutureを返し、get() で結果取得や例外伝播が起きます。shutdown() は新規タスク受付を止めますが、すでに投入されたタスクの完了待ちとは別です。

synchronizedと共有状態

共有変数を複数スレッドから変更する場合、同期がないと結果は安定しません。synchronized は同じロックを使う範囲で排他制御します。

parallel stream

parallel streamでは処理順が固定されない場合があります。forEach と forEachOrdered、順序付きストリームかどうかを確認します。

練習方法

並行処理は、単にAPI名を覚えるだけでは足りません。タスク投入、実行、結果取得、終了処理の順に表へ分けて読むと安定します。
