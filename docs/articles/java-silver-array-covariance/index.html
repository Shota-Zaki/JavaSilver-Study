配列の共変性とArrayStoreException

Java 11とJava 17の共通範囲です。宣言上は代入できるのに、要素代入で実行時例外になる理由を、配列の実体型まで追って判断します。

共変性は「String[]をObject[]として扱える」という規則

StringはObjectのサブクラスなので、Javaの配列ではString[]をObject[]型の変数へ代入できます。この性質を配列の共変性と呼びます。

Object[] values = new String[2]; // コンパイルできる
values[0] = "A";                 // 実体がString[]なので格納できる
values[1] = Integer.valueOf(10); // ここでArrayStoreException

変数valuesの宣言型はObject[]ですが、newで作られた実体はString[]です。JVMは要素代入時に実体の要素型を確認するため、Integerは格納できません。

時点見る型判定
配列参照の代入String[] → Object[]共変性によりコンパイルできる。
values[0] = "A"実体はString[]Stringなので正常。
values[1] = 10実体はString[]Integerなので実行時に拒否。

例外は配列を代入した行ではなく、要素を書き込む行で起きる

public class ArrayStorePoint {
    public static void main(String[] args) {
        Number[] numbers = new Integer[2]; // 正常
        System.out.println("A");
        numbers[0] = 10;                   // 正常
        System.out.println("B");
        numbers[1] = 2.5;                  // ArrayStoreException
        System.out.println("C");          // 実行されない
    }
}

出力はA、Bまでです。2.5はDoubleとしてボクシングされますが、実体がInteger[]なので格納時に停止します。

ジェネリクスは配列と同じ規則ではない

List<String>をList<Object>へ代入することはできません。ジェネリクスは基本的に不変で、危険な要素追加をコンパイル時に防ぎます。

import java.util.List;

class GenericDifference {
    void sample(List<String> strings) {
        // List<Object> objects = strings; // コンパイルエラー
        List<? extends Object> readable = strings; // 読み取り用としては扱える
        Object first = readable.get(0);
        // readable.add("B"); // コンパイルエラー
    }
}
仕組み型の関係不一致を検出する時点
配列String[] → Object[]が可能要素代入時の実行時検査。
ジェネリクスList<String> → List<Object>は不可コンパイル時。

コンパイルエラーになる配列代入

配列なら何でも共変になるわけではありません。要素型に継承関係がない場合や、プリミティブ型配列では代入できません。

class ArrayCompileErrors {
    void sample() {
        // String[] a = new Object[2]; // コンパイルエラー: 逆方向
        // Object[] b = new int[2];    // コンパイルエラー: int[]はObject[]ではない
        Object c = new int[2];         // これは可能。配列自体はObjectのサブタイプ
    }
}

コードを読む順番

  1. 左辺と右辺の配列型に継承関係があり、参照の代入がコンパイルできるかを見る。
  2. newの右側から、実際の配列要素型を記録する。
  3. 要素代入ごとに、代入する値の実体型を確認する。
  4. 実体配列の要素型に入らない値なら、その式でArrayStoreException。
  5. 例外発生後の文は実行しない。

作成直後の初期値や参照型要素のnullは、配列要素の初期値とnullで扱います。

記事末尾の確認

  • 宣言型と配列の実体型を分けて書ける。
  • ArrayStoreExceptionをコンパイルエラーと混同しない。
  • 配列の共変性とジェネリクスの不変性を区別できる。
