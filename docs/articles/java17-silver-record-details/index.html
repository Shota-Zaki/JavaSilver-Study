recordを丁寧に整理

recordはJava 16で正式導入され、Java 17で利用できるクラス宣言です。値のまとまりを短く表します。アクセサ名、コンストラクタ、継承制限、同値比較を分けて判定します。

recordの基本

record を使うと、コンポーネントに対応するフィールド、アクセサ、正規コンストラクタ、equals、hashCode、toString が自動的に用意されます。

record Item(String name, int price) {}

public class Main {
    public static void main(String[] args) {
        Item item = new Item("apple", 100);
        System.out.println(item.name());
        System.out.println(item.price());
        System.out.println(item);
    }
}
項目判定読み方
アクセサname() / price()getName() や item.name ではない。
フィールド暗黙的に private final外部から直接読めない。再代入もできない。
toStringItem[name=apple, price=100] 形式クラス名、コンポーネント名、値が出る。
equalsコンポーネント値で比較== は参照比較のまま。

コンストラクタの判定

recordには、全コンポーネントと同じ並びの引数を受け取る正規コンストラクタがあります。値の検証や加工が必要な場合は、正規コンストラクタまたはコンパクトコンストラクタを書けます。

record User(String name) {
    public User {
        if (name == null) {
            throw new IllegalArgumentException();
        }
        name = name.trim();
    }
}
書き方可否理由
public User { ... }可コンパクトコンストラクタ。引数リストを書かない。
public User(String name) { ... }可正規コンストラクタ。全コンポーネントに対応する。
public void User { ... }不可戻り値型を書くとコンストラクタではなくメソッド扱いになる。

recordの制約

recordは通常クラスに近い形でメソッドを追加できます。ただし、状態の持ち方と継承には制限があります。

interface Printable {
    void print();
}

record User(String name) implements Printable {
    static int count = 0;

    public void print() {
        System.out.println(name);
    }

    // int age; // コンパイルエラー: 追加のインスタンスフィールド
}

// class Child extends User {} // コンパイルエラー: recordは継承不可

追加のインスタンスフィールドを持てない点がよく使うです。recordの状態はコンポーネントで表します。

理解の確認

  • アクセサは getXxx() ではなくコンポーネント名と同じメソッド。
  • == は参照比較、equals はコンポーネント値の比較。
  • 追加のインスタンスフィールド、別クラスの extends、recordの継承は不可。
  • コンパクトコンストラクタは引数リストを書かない。
