Javaコードを読む順番

Silverのコードを、コンパイル可否、実行時例外、出力の順に切り分けます。出力候補から逆算せず、プログラムが実行段階へ到達するかを先に判断します。

広告スポンサーリンク表示エリア

読解は5段階に分ける

  1. 対象バージョン: Java 11か17かを確認する。
  2. 宣言と型: ソース構造、名前解決、代入、引数、戻り値を確認する。
  3. 呼び出し可能性: 宣言型、アクセス修飾子、static/instanceを確認する。
  4. 実行時の停止位置: null、添字、キャスト、例外処理を確認する。
  5. 値と出力: 正常に実行される部分だけを順番に追う。

前の段階で失敗したら、後ろの段階は追いません。コンパイルできないコードに実行結果はありません。

一つの例を段階ごとに読む

public class ReadingOrder {
    static String change(String text) {
        return text + "B";
    }

    public static void main(String[] args) {
        String value = "A";
        value = change(value);
        System.out.println(value);
    }
}
段階確認内容このコードの判定
宣言publicクラス名、main、戻り値成立する。
呼び出しstaticなmainからstatic changeを呼べるか呼べる。
値valueの変化"A" → "AB"。
出力printlnへ渡る値AB。

コンパイルエラーは実行前に切る

public class CompileFirst {
    public static void main(String[] args) {
        int number;
        // System.out.println(number); // 未初期化のためコンパイルエラー
        System.out.println("A");
    }
}

コメントを外すと、printlnが後ろにあってもプログラム全体を実行できません。「Aだけ表示される」とは判断しません。

実行時例外では、発生行より前の処理は残る

public class RuntimeNext {
    public static void main(String[] args) {
        String text = null;
        System.out.print("A");
        System.out.print(text.length()); // NullPointerException
        System.out.print("B");
    }
}

コンパイルはできます。実行するとAを出力した後、text.length()で停止します。Bは出力されません。

宣言型と実体型を別に書く

class Parent {
    String name = "P";
    String getName() { return "Parent"; }
}
class Child extends Parent {
    String name = "C";
    @Override String getName() { return "Child"; }
}
class Main {
    public static void main(String[] args) {
        Parent p = new Child();
        System.out.println(p.name);      // P
        System.out.println(p.getName()); // Child
    }
}

フィールドnameは宣言型Parentから選ばれます。オーバーライドされたインスタンスメソッドは、実体Childの実装が実行されます。

後置・前置演算子、コンストラクタ連鎖、例外処理、Streamでは、式を一行ずつ分解すると誤読を減らせます。

記事末尾の確認

  • 出力を追う前にコンパイル可否を判定している。
  • 実行時例外の発生式と、その前後の出力を分けている。
  • 宣言型と実体型を一つの「型」として扱っていない。
  • Java 11と17の構文差を確認している。
