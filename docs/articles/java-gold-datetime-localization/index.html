日時APIとローカライズ

Date and Time APIとローカライズでは、保持している情報の種類を先に分けます。
日付、時刻、タイムゾーン、期間、書式、Locale、ResourceBundleは、似た名前でも役割が異なります。

Date and Time APIの全体像

Java 8以降の日時APIは、主にjava.timeパッケージにあります。多くのクラスは不変です。メソッドを呼んでも元のオブジェクトは変更されず、新しいオブジェクトを返します。

クラス表すもの例確認する点
LocalDate日付2026-06-01時刻を持たない
LocalTime時刻09:30:00日付を持たない
LocalDateTime日付と時刻2026-06-01T09:30タイムゾーンを持たない
ZonedDateTime日付・時刻・タイムゾーン2026-06-01T09:30+09:00[Asia/Tokyo]ゾーン差・夏時間に注意
InstantUTC基準の瞬間2026-06-01T00:30:00Z機械的な時刻に近い
Period年月日単位の期間P1Y2M3D日付向け
Duration時分秒単位の期間PT2H30M時刻・瞬間向け
LocalDateTimeを「タイムゾーン付き日時」と読まないことが判定の軸です。LocalDateTimeはローカルな日時であり、ゾーン情報を持ちません。

作成方法

now()は実行時点に依存するため、理解上の出力を追うコードではof(...)やparse(...)がよく使われます。

import java.time.*;

class Main {
    public static void main(String[] args) {
        LocalDate d = LocalDate.of(2026, 6, 1);
        LocalTime t = LocalTime.of(9, 30);
        LocalDateTime dt = LocalDateTime.of(d, t);
        ZonedDateTime zdt = ZonedDateTime.of(dt, ZoneId.of("Asia/Tokyo"));

        System.out.println(d);
        System.out.println(t);
        System.out.println(dt);
        System.out.println(zdt);
    }
}
作成方法意味注意
LocalDate.now()現在日付実行環境に依存
LocalDate.of(2026, 6, 1)指定日付月は1〜12。0始まりではない
LocalDate.parse("2026-06-01")ISO形式文字列から作成標準形式と合わないと例外
ZoneId.of("Asia/Tokyo")タイムゾーンID存在しないIDは例外

不変性

Date and Time APIの主要クラスは不変です。plusDaysやminusMonthsは元の値を変更せず、変更後の新しい値を返します。

LocalDate d = LocalDate.of(2026, 6, 1);
d.plusDays(10);
System.out.println(d);       // 2026-06-01

LocalDate next = d.plusDays(10);
System.out.println(next);    // 2026-06-11
Stringと同じく、戻り値を受け取らないと結果を捨てています。

plus / minus / with

日時APIでは、足し引きするメソッドと、特定の項目を置き換えるメソッドを区別します。

メソッド意味例
plusDays(3)3日後2026-06-01 → 2026-06-04
minusMonths(1)1か月前2026-06-01 → 2026-05-01
withYear(2030)年を2030へ置換2026-06-01 → 2030-06-01
withDayOfMonth(1)日を1へ置換2026-06-18 → 2026-06-01
LocalDate d = LocalDate.of(2026, 1, 31);
System.out.println(d.plusMonths(1)); // 2026-02-28

存在しない日付になる場合、APIが有効な日付へ調整するケースがあります。単純な文字列置換ではありません。

PeriodとDuration

Periodは年月日、Durationは時分秒を扱います。日付に時分秒を足す、時刻に年月日を足す、といった読み方をすると混乱します。

クラス単位向いている対象例
Period年・月・日LocalDatePeriod.of(1, 2, 3)
Duration時・分・秒・ナノ秒LocalTime / InstantDuration.ofHours(2)
LocalDate d = LocalDate.of(2026, 6, 1);
System.out.println(d.plus(Period.ofDays(3))); // 2026-06-04

LocalTime t = LocalTime.of(9, 0);
System.out.println(t.plus(Duration.ofMinutes(30))); // 09:30
Period.of(1, 2, 3)は1年2か月3日です。合計日数へ自動変換して考えません。

betweenの戻り値

betweenは、開始から終了までの差を返します。引数の順番が逆になると負の値になります。

LocalDate a = LocalDate.of(2026, 6, 1);
LocalDate b = LocalDate.of(2026, 6, 10);
System.out.println(Period.between(a, b)); // P9D

LocalTime x = LocalTime.of(9, 0);
LocalTime y = LocalTime.of(10, 30);
System.out.println(Duration.between(x, y).toMinutes()); // 90

DateTimeFormatter

DateTimeFormatterは、日時を文字列へ変換したり、文字列から日時を作ったりするための書式です。

import java.time.*;
import java.time.format.*;

LocalDate d = LocalDate.of(2026, 6, 1);
DateTimeFormatter f = DateTimeFormatter.ofPattern("yyyy/MM/dd");
System.out.println(d.format(f)); // 2026/06/01

LocalDate parsed = LocalDate.parse("2026/06/01", f);
System.out.println(parsed); // 2026-06-01
記号意味注意
y年yyyyで4桁年
M月大文字。小文字mは分
d日日付の日
H時0〜23
m分月ではない
s秒秒
MMとmmの取り違えは定番です。月は大文字M、分は小文字mです。

Locale

Localeは、言語・国・地域などのローカル設定を表します。表示形式やメッセージ選択に影響します。

import java.util.*;

Locale ja = Locale.JAPAN;
Locale us = Locale.US;
Locale fr = Locale.FRANCE;

System.out.println(ja.getLanguage()); // ja
System.out.println(ja.getCountry());  // JP
作成方法意味注意
Locale.JAPAN日本言語ja、国JP
Locale.US米国言語en、国US
new Locale("ja", "JP")言語・国を指定古い書き方として出る可能性
Locale.of("ja", "JP")言語・国を指定Java 19以降。Java 17では利用できない

NumberFormat

NumberFormatは、数値や通貨をLocaleに応じた文字列へ整形します。戻り値は文字列です。

import java.text.*;
import java.util.*;

NumberFormat nf = NumberFormat.getNumberInstance(Locale.US);
System.out.println(nf.format(1234567)); // 1,234,567

NumberFormat cf = NumberFormat.getCurrencyInstance(Locale.US);
System.out.println(cf.format(1234.5));
メソッド意味注意
getNumberInstance通常の数値形式桁区切りなどがLocale依存
getCurrencyInstance通貨形式通貨記号・小数桁がLocale依存
getPercentInstanceパーセント形式0.25 → 25%のように扱う
parse文字列から数値を読むParseExceptionに注意

ResourceBundle

ResourceBundleは、Localeに応じたメッセージを読み分ける仕組みです。プロパティファイル名とLocaleの対応を読みます。

// messages_ja_JP.properties
// hello=こんにちは

import java.util.*;

Locale locale = Locale.JAPAN;
ResourceBundle rb = ResourceBundle.getBundle("messages", locale);
System.out.println(rb.getString("hello"));
ファイル例対象注意
messages.properties基本最後の fallback
messages_ja.properties日本語言語だけ
messages_ja_JP.properties日本語・日本より具体的
messages_en_US.properties英語・米国Locale.US

キーが存在しない場合は、単にnullを返すのではなく例外になります。

よくある誤解を整理する

LocalDateTimeはタイムゾーンを持たない

LocalDateTimeは日付と時刻だけを持ちます。時差や夏時間を考えるならZonedDateTimeを見ます。

plusDaysは元の値を変更しない

日時APIの主要クラスは不変です。戻り値を受け取らないと変更後の値を使えません。

Periodは時間を扱わない

Periodは年・月・日です。時・分・秒はDurationで扱います。

DateTimeFormatterのMMは分にならない

月は大文字M、分は小文字mです。yyyy/mm/ddは月ではなく分を使います。

ResourceBundleはキーなしでnullを返さない

存在しないキーを読むと例外です。戻り値のnull判定だけで処理しません。

Localeは翻訳そのものをしない

Localeは地域設定です。メッセージ翻訳はResourceBundleなどのデータが必要です。

不可能・誤りパターン

コード・考え方結果理由
LocalDate.of(2026, 0, 1)実行時例外月は1〜12
LocalDate.parse("2026/06/01")実行時例外標準はyyyy-MM-dd
d.plusDays(1);だけでdが変わる変わらない不変オブジェクト
Period.ofHours(1)コンパイルエラーPeriodは時間単位を持たない
DateTimeFormatter.ofPattern("yyyy/mm/dd")意図と違う書式mmは分
ResourceBundle.getString("missing")実行時例外キーが存在しない

コードの読み方

  1. クラスが何を持つか判定します。日付だけか、時刻だけか、日時か、ゾーン付きか。
  2. メソッドが元の値を変更するか、戻り値を返すだけかを判定します。
  3. PeriodとDurationの単位を判定します。
  4. 書式では大文字・小文字を判定します。特にMとm。
  5. Localeは表示ルール、ResourceBundleはメッセージ取得と切り分けます。

日時・ローカライズの確認

日時APIはimmutableです。plusDays や minusMonths は元のインスタンスを変更せず、新しいインスタンスを返します。

  • LocalDate、LocalTime、LocalDateTime、ZonedDateTime の扱う情報量を分けます。
  • Period は日付ベース、Duration は時間ベースです。
  • フォーマットとパースでは、パターン不一致による例外を見ます。
  • ResourceBundle はLocaleに応じた探索順を見ます。
