Java Gold SE 11とSE 17の違い

Java Goldは、SE 11とSE 17で「勉強する単元の名前」よりも、問題の読み方と模試の時間配分が変わります。
共通部分を先に固めてから、受験するバージョンの追加要素へ進むのが安全です。

結論：最初は共通部分を優先する

SE 11とSE 17で迷っている段階でも、最初にやることはほぼ同じです。
Generics、ラムダ式、Stream、Collectors、Optional、NIO、並行処理、module-infoを先に固めます。
ここが弱いまま17追加分へ進むと、追加文法だけ覚えても点数が伸びません。

比較SE 11SE 17
中心になる学習Generics、Lambda、Stream、NIO、並行処理、module-infoSE 11共通部分に加えて、record、sealed、switch式、パターンマッチング
模試の重さ長時間で広く読む。集中力と戻り判定が必要問題数と時間は軽めでも、追加構文の判定が混ざる
向いている人既存教材や業務環境がJava 11寄りの人今後の学習や開発でJava 17以降を使う予定がある人
失点しやすい場所Stream、Collectors、Generics、NIO、並行処理共通部分に加えて、record / sealed / switch式 / instanceofパターン

先に決める3項目

見る項目SE 11寄りSE 17寄り
仕事・研修環境Java 11を使う予定があるJava 17以降を使う予定がある
教材状況手元の教材や研修資料がSE 11中心17追加分まで扱う教材を用意できる
学習時間共通部分に集中したい共通部分に加えて追加構文を判定できる

迷っているなら、先に共通部分を進めます。
Generics、Lambda、Stream、Collectors、NIO、Concurrency、Moduleはどちらでも必要です。
受験区分の判断で止まる時間は損です。

SE 11とSE 17で変わる読み方

Goldでは、単語を知っているだけでは足りません。SE 17では構文が増えるので、コードの形から意味を素早く判断する必要があります。

recordは、ただの短いクラス宣言ではありません。
フィールド、コンストラクタ、アクセサ、equals、hashCode、toStringの扱いまでまとめて判定します。
sealedは、継承できるクラスを制限します。
permitsに書かれていないクラスは継承できません。継承先はfinal、sealed、non-sealedのいずれかで宣言します。
switch式は値を返します。
switch文とは読み方が変わります。ブロック形式ではyieldを判定します。
instanceofパターンでは、変数のスコープを判定します。
型判定した後に使える変数が、どこまで有効かを見ます。

recordで見るポイント

recordは、値をまとめて扱うためのクラスです。見た目は短いですが、試験では「何が自動生成されるか」「何を自分で書けるか」を問われます。

record Item(String name, int price) { }

public class Main {
    public static void main(String[] args) {
        var item = new Item("book", 1200);
        System.out.println(item.name());
        System.out.println(item.price());
    }
}

アクセサはgetName()ではなくname()です。ここをJavaBeansの感覚で読むと失点します。

見る場所判断
アクセサコンポーネント名と同じメソッド名になる
継承recordは暗黙的にfinalなので、通常のクラスから継承できない
フィールドコンポーネントに対応するフィールドはprivate final
コンストラクタ正規コンストラクタ、コンパクトコンストラクタを区別する

sealedで見るポイント

sealedは「継承の許可リスト」です。許可されたクラス以外は継承できません。

sealed class Parent permits ChildA, ChildB { }
final class ChildA extends Parent { }
non-sealed class ChildB extends Parent { }

class ChildC extends Parent { } // コンパイルできない

Goldでは、sealedそのものよりも、継承先の修飾子を見落としやすいです。
許可されたサブクラスでも、final、sealed、non-sealedのどれも付いていなければコンパイルできません。

switch式で見るポイント

switch式は結果を返します。従来のswitch文と同じ感覚で読むと、yieldや網羅性を見落とします。

static String label(int n) {
    return switch (n) {
        case 1 -> "one";
        case 2 -> "two";
        default -> {
            var value = "other";
            yield value;
        }
    };
}

ブロックを使うcaseでは、最後にyieldで値を返します。returnではありません。

instanceofパターンで見るポイント

パターンマッチングでは、型判定と変数宣言を同時に書けます。ただし、パターン変数はどこでも使えるわけではありません。

static void print(Object obj) {
    if (obj instanceof String s) {
        System.out.println(s.length());
    }
    // System.out.println(s); // この場所では使えない
}

試験では、スコープ外でパターン変数を使っていないかを判定します。

SE 11を選ぶ判断

SE 11を選ぶのは、既存教材や職場環境がJava 11中心の場合です。教材量が多く、Gold共通部分に集中しやすい利点があります。

  • Java 11環境の仕事や研修に合わせたい
  • Generics、Stream、NIO、並行処理を優先して固めたい
  • Java 17追加構文に時間を使う余裕が少ない
  • 長めの模試で集中力と読解順を鍛えたい

SE 17を選ぶ判断

SE 17を選ぶのは、これからの開発や学習でJava 17以降を使う可能性が高い場合です。ただし、17追加分だけ先にやる進め方は不十分です。
共通部分が固まってから追加構文へ進みます。

  • Java 17以降の文法を試験対策と一緒に整理したい
  • record、sealed、switch式を読めるようにしたい
  • Silver 17の追加分を学習済み
  • Gold共通部分の演習で正答率が安定している

学習順はこう分ける

段階やること進む基準
1Silverの継承、例外、型変換を判定コンパイル可否を先に判断できる
2Generics、Lambda、Optionalを学習型と戻り値を追える
3Stream、Collectorsを学習中間操作と終端操作を区別できる
4NIO、日時、JDBC、並行処理、module-info例外、リソース、実行順を説明できる
5SE 17追加分共通部分の標準問題で大きく崩れない
6受験対象の模試カテゴリ別の弱点が見えている

共通部分と17追加分の接続

17追加分先に固める共通論点接続して見るポイント
recordクラス、コンストラクタ、equals、hashCode、toString自動生成されるメンバと、自分で書けるメンバを分ける
sealed継承、抽象クラス、ポリモーフィズム、アクセス修飾子許可リスト、サブクラス側のfinal / sealed / non-sealedを見る
switch式switch文、式と文、戻り値、網羅性値を返す構文として読み、yieldとdefaultを見る
instanceofパターン参照型、キャスト、スコープ、短絡評価パターン変数が使える範囲と条件式の流れを見る

模試の使い分け

SE 11を受けるならSE 11模試を中心に使います。SE 17を受けるなら、共通部分を通常演習で固めてからSE 17模試へ進みます。
SE 17模試だけを回すと、共通部分の穴を見落とします。

状態使う模試見る結果
Gold共通部分の判定中通常演習、Gold共通総合Stream、Generics、NIOの弱点
SE 11受験前SE 11模試80問を最後まで読み切れるか
SE 17受験前SE 17模試追加構文と共通APIの混合問題で崩れないか
点数が伸びない間違えた問題だけ復習カテゴリ別・難易度別の正答率

このサイトでの使い方

  1. 受験区分が未確定なら、このページでSE 11 / SE 17の判断基準を見る
  2. Java Gold学習ロードマップで、共通部分から模試までの順番を決める
  3. Java Gold教材の選び方で、参考書・問題集・模試・リファレンスの役割を分ける

よくある誤解を整理する

  • SE 17は追加文法だけ覚えても点数にならない
  • SE 11対策でもStreamとGenericsは避けられない
  • recordはただの省略記法にならない
  • sealedは継承先の修飾子まで見る
  • switch式はswitch文と同じ読み方にならない
  • 模試の点数だけでは受験判断できない。カテゴリ別の崩れを見る
