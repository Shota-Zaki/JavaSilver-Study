Java Silverの継承問題の解き方

Java Silverで頻出の継承問題を、参照型と実体型、override、overload、private、static、フィールド隠蔽、コンストラクタ順に分けて整理します。

広告スポンサーリンク表示エリア

最終更新日：2026年7月7日。Java SE 17の試験範囲を中心に、演習前後の読み方を整理しています。

先に結論

継承問題は、宣言型、実体、メンバ種別、override成立条件を分けて読みます。new Child() だけを見て子クラスの処理だと決めると、private、static、フィールド隠蔽で失点します。

見る基準
  • 参照型からそのメンバを呼べるか
  • 実体型でoverrideメソッドが選ばれるか
  • privateメソッドをoverride扱いにしていないか
  • staticメソッドとフィールドを動的ディスパッチ扱いしていないか
  • コンストラクタと初期化順を追っているか

読む順番

継承問題は、宣言を見る → 代入を見る → 呼び出しを見る → overrideか判定する → 実行順を見るの順で読みます。最初から出力だけを探すと、privateメソッドやstaticメソッドで失点します。

参照型と実体型

Parent p = new Child(); では、コンパイル時に使う型は Parent、実行時の実体は Child です。呼び出せるかは参照型で見て、overrideされたインスタンスメソッドは実体型で選ばれます。

overrideとoverload

論点見ること誤答原因
override親のインスタンスメソッドを子で再定義しているか戻り値、アクセス修飾子、例外条件を見落とす
overload同名でも引数リストが違うかoverrideされたと思い込む
private継承されないためoverrideにならない子の同名メソッドが呼ばれると思う
staticoverrideではなく隠蔽実体型で選ばれると思う

コンストラクタ順

インスタンス生成では、親クラスの初期化が先に行われます。super() が明示されていなくても、条件を満たせば暗黙に呼ばれます。出力問題では、フィールド初期化、初期化ブロック、コンストラクタの順も確認します。

練習方法

通常演習では、継承・クラス設計・メソッド解決の問題をまとめて解きます。迷った問題は、継承の単元別解説とメソッド解決の記事へ戻ると整理しやすくなります。
