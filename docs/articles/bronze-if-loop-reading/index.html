if・for・whileの実行順

分岐と繰り返しは、条件式だけを眺めると順序を見失います。条件が評価される時点と、変数が更新される時点を一行ずつ追います。

広告スポンサーリンク表示エリア

if文はbooleanの条件で分岐する

public class IfBasic {
    public static void main(String[] args) {
        int score = 75;

        if (score >= 80) {
            System.out.println("A");
        } else if (score >= 60) {
            System.out.println("B");
        } else {
            System.out.println("C");
        }
    }
}

最初の条件75 >= 80はfalseです。次の75 >= 60がtrueなので、Bを表示し、その後のelseは実行しません。

if (1)のように数値を条件へ置くことはできません。Javaの条件式はboolean型である必要があり、これはコンパイル時に判定されます。

for文は4段階で追う

for (int i = 0; i < 3; i++) {
    System.out.print(i);
}
順番処理この例
1初期化i = 0。最初の1回だけ。
2条件判定i < 3。
3本体iを表示。
4更新i++。

この順番を繰り返すので、出力は012です。iが3になると条件がfalseになり、本体へ入りません。

whileとdo-whileは最初の条件判定位置が違う

int a = 0;
while (a > 0) {
    System.out.print("W");
}

do {
    System.out.print("D");
} while (a > 0);

whileは本体より前に条件を見るため、Wは表示しません。do-whileは本体を一度実行してから条件を見るため、Dを1回表示します。

breakとcontinueの移動先

for (int i = 0; i < 5; i++) {
    if (i == 1) continue;
    if (i == 3) break;
    System.out.print(i);
}
i処理出力
0どちらの条件にも該当しない0
1continueで更新式へなし
2通常実行2
3breakでループ終了なし

最終出力は02です。continueはループ全体を終了せず、for文では更新式へ進みます。

コードを読む順番

  1. 条件式がboolean型として成立するか確認する。
  2. 変数の初期値を書く。
  3. 条件判定、本体、更新の順を表へ記録する。
  4. breakとcontinueがどのループへ作用するか確認する。
  5. 例外が起きる式があれば、それ以後の処理を除く。

記事末尾の確認

  • for文の初期化・条件・本体・更新の順を言える。
  • whileとdo-whileの最低実行回数の違いを説明できる。
  • continue後にfor文の更新式が実行されることを追える。
