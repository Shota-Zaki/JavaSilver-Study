Java Silverのラムダ式・関数型インターフェース問題の解き方

Java Silverで出るラムダ式と関数型インターフェースを、ターゲット型、引数、戻り値、Predicate、Consumer、Function、Supplierで整理します。

広告スポンサーリンク表示エリア

最終更新日：2026年7月7日。Java SE 17の試験範囲を中心に、演習前後の読み方を整理しています。

先に結論

ラムダ式は、単体では型が決まりません。代入先、引数、メソッド呼び出しからターゲット型を決め、抽象メソッドの引数と戻り値に合うかを見ます。見た目の短さに反して、型の判定が得点差になります。

見る基準
  • 関数型インターフェースか
  • 抽象メソッドの引数数と戻り値型が合っているか
  • ブロックラムダでreturnの要否を見ているか
  • ローカル変数が実質的finalか
  • Predicate、Function、Consumer、Supplierを混同していないか
Java Practice Labでの使い方ラムダ式は、選択肢だけで判断しない方が安定します。単元別解説でターゲット型を確認し、演習では各選択肢がどの抽象メソッドに合うかを1つずつ切ります。

ラムダ式は代入先の型から読む

ラムダ式だけを見ても型は確定しません。代入先、引数、メソッドの仮引数などのターゲット型を見て、抽象メソッドの引数型と戻り値を確認します。

主要な関数型インターフェース

型抽象メソッドのイメージ戻り値
Predicate<T>Tを受け取り判定するboolean
Consumer<T>Tを受け取って処理するvoid
Function<T,R>TからRへ変換するR
Supplier<T>値を供給するT

構文の判定

引数が1つなら括弧を省略できますが、型を書く場合や引数が複数ある場合は省略条件が変わります。処理本体が式なら戻り値に使いますが、ブロックにした場合は必要に応じてreturnを書きます。

よくある誤答

  • Consumerなのに戻り値を返している。
  • Predicateなのにboolean以外を返している。
  • ブロック本体でreturnを書き忘れる。
  • effectively finalでないローカル変数をラムダ内から参照する。

練習方法

ラムダ式は暗記よりターゲット型の確認が外せません。通常演習で型を先に書き出し、Silverで間違えやすい問題パターンと合わせて復習します。
