数値リテラル・ラッパークラス・Numberの関係

数値に関するコードは、ソースコード上の表記、評価後のプリミティブ値、ボクシング後のオブジェクト、変数の宣言型を分けて読みます。Number value = 10;の一文にも、intリテラル、Integerオブジェクト、Number型という異なる段階があります。

Number value = 100;

  2. int値の100が、オートボクシングによってIntegerオブジェクトになります。
  3. IntegerはNumberを継承しているため、参照型の拡大変換によってNumber型変数へ代入されます。

区別する点: 100というリテラル自体がNumberオブジェクトなのではありません。右辺の値に必要な変換が適用された結果、IntegerオブジェクトがNumber型として保持されます。

1. 数値プリミティブ型とラッパークラス

Javaの数値プリミティブ型は6種類です。各型には対応するラッパークラスがあります。

分類プリミティブ型ラッパークラス代表的なリテラル主な注意点
整数byteByte専用接尾辞なし整数リテラルは通常int。範囲内定数だけ直接代入できる。
整数shortShort専用接尾辞なし算術演算では原則intへ昇格する。
整数intInteger100接尾辞なし整数リテラルの基本型。
整数longLong100L大文字Lを使うと数字の1と区別しやすい。
浮動小数点floatFloat1.5F接尾辞なし小数はdouble。
浮動小数点doubleDouble1.5小数リテラルの基本型。

プリミティブ値はオブジェクトではありません。ラッパークラスのインスタンスはオブジェクトであり、メソッドを呼び出せます。

int primitive = 100;
Integer wrapper = 100;

System.out.println(wrapper.doubleValue()); // 100.0
System.out.println(wrapper.toString());    // 100

2. Numberは数値オブジェクトの共通親クラス

Numberは抽象クラスです。数値を表す複数のクラスを共通の型として扱うために使います。

java.lang.Object
    └── java.lang.Number
          ├── Byte
          ├── Short
          ├── Integer
          ├── Long
          ├── Float
          └── Double

標準ラッパークラス以外にも、次のようなクラスがNumberを継承しています。

Number
├── BigInteger
├── BigDecimal
├── AtomicInteger
├── AtomicLong
├── LongAdder
└── DoubleAdder

Number a = Integer.valueOf(10);
Number b = Long.valueOf(20L);
Number c = new java.math.BigInteger("12345678901234567890");
Number d = new java.math.BigDecimal("12.50");
Numberの役割: 共通の受け皿と基本的な値取り出し方法を提供します。異なる数値クラスに共通する加算演算を提供するクラスではありません。
Number a = 10;
Number b = 20;

// Number result = a + b; // コンパイルエラー

+はプリミティブ数値に対する演算子です。Number型のままでは、実体がIntegerなのかBigDecimalなのか、どの精度と規則で加算するのかを決められません。

3. 数値リテラルの型は表記で決まる

代入先の型より先に、右辺のリテラル自体の型が決まります。

表記リテラルの型例補足
接尾辞なし整数int100intで表せない10進値にはLが必要。
L/l付き整数long100L通常は大文字Lを使う。
接尾辞なし小数double1.5floatへは暗黙に縮小されない。
F/f付き小数float1.5Ffloat変数へ直接代入できる。
D/d付き小数double1.5D接尾辞なしと同じ型を明示する。
 
var a = 100;    // int
var b = 100L;   // long
var c = 1.5F;   // float
var d = 1.5;    // double

long a = 100;        // int値をlongへ拡大変換
long b = 3_000_000_000L;

// long c = 3_000_000_000; // コンパイルエラー: intリテラルとして大きすぎる
// float d = 1.5;          // コンパイルエラー: doubleからfloatへの縮小
float e = 1.5F;

浮動小数点リテラルの形式

形式例型・値注意点
小数点表記1.5, .5, 5.double小数点の前後どちらかには数字が必要。
10進指数表記1.5e2, 1E-3doublee/Eの指数は10の累乗。
float接尾辞1.5F, 1e3f, 10Ffloat小数点がなくてもFがあれば浮動小数点リテラル。

double b = 5.;       // 5.0
double c = 1.5e2;    // 150.0

16進浮動小数点の指数: 仮数部は16進表記ですが、指数はpの後に10進数字で書き、その値を2の累乗として適用します。0x1.8p1は1.5 × 2です。

4. 2進・8進・10進・16進の整数リテラル

基数書き方例10進値
2進数0bまたは0B0b101010
8進数先頭を0にする01210
10進数通常の数字1010
16進数0xまたは0X0xA10
 
int a = 0b1010;
int b = 012;
int c = 10;
int d = 0xA;

System.out.println(a == b && b == c && c == d); // true
8進数の注意: 018はコンパイルエラーです。先頭の0によって8進数として解析されますが、8進数には8と9を使えません。

非10進リテラルはビット列として負数を表せる

16進・8進・2進のintリテラルは、32ビットのビット列として解釈される場合があります。

int x = 0xFFFF_FFFF;
System.out.println(x); // -1

long y = 0xFFFF_FFFF;
long z = 0xFFFF_FFFFL;
System.out.println(y); // -1: intの-1をlongへ拡大
System.out.println(z); // 4294967295: longリテラル

同じ桁並びでも、Lの有無でリテラルの型と値が変わります。

5. 数値リテラル内のアンダースコア

数字同士の間には、可読性のための_を置けます。値には影響しません。

int price = 1_000_000;
long population = 8_000_000_000L;
int bits = 0b1010_1100;
int mask = 0xFF_FF;
double rate = 1_234.567_89;
double scientific = 1.2e3_0;

基本規則は「数字と数字の間だけ」です。

位置例結果
数字の間1_000有効
先頭・末尾_100, 100_無効
小数点の直前・直後1_.5, 1._5無効
基数接頭辞の直後0x_FF, 0b_10無効
型接尾辞の直前100_L, 1.0_F無効
指数記号や符号の隣1_e3, 1e_3, 1e+_3無効
8進数の例外に見える書き方: 0_52は有効です。8進数の先頭0も数字なので、0と5の間に_を置いた形です。

6. 負号はリテラルの一部ではない

-100は、負の数を表す単一のリテラルではありません。正の整数リテラル100へ単項マイナス演算子-を適用した式です。

-100
│└── intリテラル 100
└─── 単項マイナス演算子

int value = -(100);

最小値には特別な規則があります。

int min = -2147483648;
long longMin = -9223372036854775808L;

10進表記の2147483648と9223372036854775808Lは、それぞれ最小値を表す単項マイナスのオペランドとして使う場合に限って許されます。単独の正の値としては対応型の上限を超えます。

// int x = 2147483648;              // コンパイルエラー
// long y = 9223372036854775808L;    // コンパイルエラー

7. byte・short・charへの定数代入

接尾辞のない整数リテラルはintですが、コンパイル時に値が確定し、代入先の範囲内であれば、小さい整数型へ直接代入できます。

byte b = 100;
short s = 30_000;
char c = 65;
型範囲範囲内範囲外
byte-128〜127byte b = 127;byte b = 128;
short-32768〜32767short s = 32767;short s = 32768;
char0〜65535char c = 65535;char c = -1;

変数を経由すると値だけでは判断されない

int value = 100;
// byte b = value; // コンパイルエラー

valueの現在値が100でも、式の型はintです。通常のint変数は実行中に別の値へ変わる可能性があるため、暗黙の縮小変換は行われません。

finalでもコンパイル時定数かを確認する

final int COMPILE_TIME = 100;
byte a = COMPILE_TIME; // 有効

final int RUNTIME = Integer.parseInt("100");
// byte b = RUNTIME; // コンパイルエラー

finalは再代入を禁止しますが、それだけでコンパイル時定数になるわけではありません。定数式だけで初期化されている必要があります。

8. オートボクシングとアンボクシング

プリミティブ値から対応するラッパークラスへの自動変換がボクシング、逆方向がアンボクシングです。

Integer boxed = 100; // int → Integer
int value = boxed;    // Integer → int

Integer boxed = Integer.valueOf(100);
int value = boxed.intValue();

概念上は上の明示的な呼び出しに近いですが、言語仕様上はボクシング変換・アンボクシング変換として定義されています。

プリミティブ型ボクシング先Numberのサブクラスか
byteByteはい
shortShortはい
intIntegerはい
longLongはい
floatFloatはい
doubleDoubleはい
charCharacterいいえ
booleanBooleanいいえ

nullをアンボクシングすると実行時例外

Integer boxed = null;
int value = boxed; // 実行時にNullPointerException

コンパイル時にはIntegerからintへのアンボクシングが許可されます。しかし実行時には、nullから値を取り出せないため例外になります。

9. 代入変換の組み合わせ

数値代入では、どの変換をどの順番で適用できるかが重要です。

コード結果変換
long x = 10;有効intからlongへのプリミティブ拡大変換
Integer x = 10;有効intからIntegerへのボクシング
Number x = 10;有効int→Integer→Number
Object x = 10;有効int→Integer→Object
Long x = 10;無効拡大してからボクシングする変換は代入時に行われない
Long x = 10L;有効longからLongへのボクシング
long x = Integer.valueOf(10);有効Integer→int→long
int x = (Number) 10;無効Numberは直接アンボクシングできない

Long value = 10;が成立しない理由

必要になる変換
int → long → Long
      拡大   ボクシング

代入変換は、プリミティブ拡大変換の後にボクシングする組み合わせを自動では行いません。

Long a = 10L;
Long b = Long.valueOf(10); // メソッド引数でint→longの拡大後、Longを返す
Long c = (long) 10;        // キャスト結果はlong。それをLongへボクシング

Number型は直接アンボクシングできない

Number n = 10;
// int x = n; // コンパイルエラー
int x = n.intValue();

変数nの宣言型はNumberです。実体がIntegerであっても、コンパイラは宣言型に基づいて直接アンボクシングできるかを判定します。

10. Number型変数の宣言型と実体型

Number型変数へ異なるリテラルを代入すると、リテラルの型に対応したラッパークラスが実体になります。

Number a = 10;
Number b = 10L;
Number c = 10.0F;
Number d = 10.0;

System.out.println(a.getClass().getSimpleName()); // Integer
System.out.println(b.getClass().getSimpleName()); // Long
System.out.println(c.getClass().getSimpleName()); // Float
System.out.println(d.getClass().getSimpleName()); // Double
右辺プリミティブ型ボクシング後の実体変数の宣言型
10intIntegerNumber
10LlongLongNumber
10.0FfloatFloatNumber
10.0doubleDoubleNumber

Numberから特定のラッパークラスへ戻すには、宣言型上はダウンキャストが必要です。

Number n = 100;
Integer i = (Integer) n; // 実体がIntegerなので成功

Number m = 100L;
// Integer j = (Integer) m; // 実行時にClassCastException

単に整数値が必要なら、実体クラスへキャストせずintValue()などを使います。ただし、変換による桁あふれや小数部の消失を確認する必要があります。

11. Numberの値取り出しメソッド

Numberは、保持している数値を各プリミティブ型として取り出すためのメソッドを定義しています。

メソッド戻り値Numberでの定義
byteValue()byte具象メソッド
shortValue()short具象メソッド
intValue()int抽象メソッド
longValue()long抽象メソッド
floatValue()float抽象メソッド
doubleValue()double抽象メソッド
 
Number n = 123.75;

System.out.println(n.intValue());    // 123
System.out.println(n.longValue());   // 123
System.out.println(n.floatValue());  // 123.75
System.out.println(n.doubleValue()); // 123.75

値が完全に保持されるとは限らない

Number n = 300;
System.out.println(n.byteValue()); // 44

300をbyteへ変換すると、byteで表せる8ビットだけが残ります。

Number n = 9.99;
System.out.println(n.intValue()); // 9

浮動小数点値を整数型へ変換すると、小数部分は0方向へ丸められます。正数では切り捨てに見え、負数では切り上げに見える点に注意します。

Number n = -9.99;
System.out.println(n.intValue()); // -9

12. 算術演算時の数値昇格

byte、short、charは、算術演算の前に原則としてintへ昇格します。

byte a = 10;
byte b = 20;

int result = a + b;
// byte small = a + b; // コンパイルエラー

二項演算では、概ね次の優先順位で共通の型へ揃えます。

double
  ↑
float
  ↑
long
  ↑
int  ← byte・short・charはここへ昇格
式式全体の型理由
byte + byteint両方とも単項数値昇格でintになる
int + longlongintをlongへ揃える
long + floatfloatlongをfloatへ揃える
float + doubledoublefloatをdoubleへ揃える
範囲と精度は別: longからfloatへの変換は言語上の拡大変換ですが、すべての整数値を正確に保持できるわけではありません。型の変換方向が「拡大」でも、精度を失う場合があります。

13. 複合代入は暗黙の型変換を含む

byte value = 10;
value += 20; // 有効

複合代入は、左辺の値を使って演算した後、左辺の型へ暗黙に変換して代入します。

value = (byte) (value + 20);

通常の代入では同じ変換を自動適用しません。

byte value = 10;
// value = value + 20; // コンパイルエラー: 右辺はint

暗黙の縮小変換を含むため、範囲を超えるとオーバーフローします。

byte value = 127;
value += 1;
System.out.println(value); // -128

14. 整数除算か小数除算かはオペランドの型で決まる

System.out.println(5 / 2);     // 2
System.out.println(5.0 / 2);   // 2.5
System.out.println(5 / 2.0);   // 2.5
System.out.println(5F / 2);    // 2.5

5 / 2は両方がintなので、先に整数除算が行われます。代入先をdoubleにしても、除算後の結果は戻りません。

double result = 5 / 2;
System.out.println(result); // 2.0

小数除算にするには、除算時点で少なくとも片方を浮動小数点型にします。

double result = 5 / 2.0;

0除算の違い

// int x = 10 / 0;       // 実行時にArithmeticException
System.out.println(10.0 / 0.0); // Infinity
System.out.println(0.0 / 0.0);  // NaN

InfinityとNaNには数値リテラルがありません。Double.POSITIVE_INFINITYやDouble.NaNなどの定数でも参照できます。

15. charは数値演算に参加するがCharacterはNumberではない

charは0〜65535の整数値を保持するプリミティブ型です。文字リテラルはchar型であり、数値演算ではintへ昇格します。

char c = 'A';
System.out.println(c);          // A
System.out.println((int) c);    // 65
System.out.println(c + 1);      // 66
System.out.println((char)(c + 1)); // B

一方、charのラッパークラスであるCharacterはNumberを継承していません。

Object
├── Number
│   ├── Integer
│   └── Long など
└── Character

Character c = 'A';
// Number n = c; // コンパイルエラー

Number n = (int) 'A'; // int 65 → Integer → Number

16. BigInteger・BigDecimal専用のリテラルはない

任意精度整数や10進小数を扱うクラスですが、専用のリテラル構文はありません。コンストラクタやファクトリメソッドで生成します。

java.math.BigInteger integer =
        new java.math.BigInteger("123456789012345678901234567890");

java.math.BigDecimal decimal =
        new java.math.BigDecimal("123.45");

これらはNumberのサブクラスですが、+や*ではなくメソッドで演算します。

var a = new java.math.BigDecimal("0.1");
var b = new java.math.BigDecimal("0.2");
var sum = a.add(b);
System.out.println(sum); // 0.3

BigDecimalへdoubleリテラルを渡す違い

var a = new java.math.BigDecimal(0.1);
var b = new java.math.BigDecimal("0.1");
var c = java.math.BigDecimal.valueOf(0.1);

System.out.println(a); // 0.10000000000000000555... のような値
System.out.println(b); // 0.1
System.out.println(c); // 0.1

0.1は先にdoubleとして評価されます。2進浮動小数点で表現された値をそのまま正確に取り込むのがnew BigDecimal(0.1)です。10進表記どおりの値が必要なら、文字列コンストラクタかBigDecimal.valueOf(double)を使います。

17. varは右辺の実際の型を推論する

varは数値をすべてNumberとして扱う機能ではありません。右辺のコンパイル時型が、そのままローカル変数の型になります。

var a = 10;                    // int
var b = 10L;                   // long
var c = Integer.valueOf(10);   // Integer
var d = (Number) 10;           // Number
var e = new java.math.BigDecimal("10"); // BigDecimal

同じ数値10でも、右辺の式が何型かによって推論結果が変わります。

18. ラッパークラスの==と数値比較

ラッパークラスは参照型なので、==は原則として参照の同一性を比較します。ただし、ボクシング時に同じインスタンスが再利用される値があるため、値比較と誤認しやすくなります。

Integer a = 100;
Integer b = 100;
Integer c = 1000;
Integer d = 1000;

System.out.println(a == b); // trueになる
System.out.println(c == d); // 通常false
System.out.println(c.equals(d)); // true

Integerのボクシングでは少なくとも-128〜127の値が共有されます。実装によって共有範囲が広がる可能性があるため、ラッパーの値比較に==を使いません。

異なるラッパークラスのequalsは数値が同じでもfalse

Integer i = 10;
Long l = 10L;

System.out.println(i.equals(l)); // false

Integer.equalsは相手もIntegerであり、保持値が同じ場合だけtrueです。数値クラスをまたいだ大小比較が必要なら、型と精度を統一してから比較します。

19. IntegerがNumberでもList<Integer>はList<Number>ではない

クラス単体の継承関係は、ジェネリック型の型引数へそのまま引き継がれません。

Integer value = 10;
Number number = value; // 有効

java.util.List<Integer> integers = java.util.List.of(1, 2, 3);
// java.util.List<Number> numbers = integers; // コンパイルエラー

読み取り専用として複数の数値サブクラスを受け取るなら、上限境界ワイルドカードを使います。

static double sum(java.util.List<? extends Number> values) {
    double total = 0;
    for (Number value : values) {
        total += value.doubleValue();
    }
    return total;
}
精度設計: 上の例は説明用です。金額計算へdoubleValue()を使うと10進精度を失う可能性があります。実務では扱う数値型を統一し、型に合った演算を行います。

20. オーバーロードでは拡大変換がボクシングより優先される

同じ実引数に複数のメソッドが適用可能な場合、プリミティブ拡大変換だけで呼べるメソッドが、ボクシングを必要とするメソッドより先に選ばれます。

static void print(long value) {
    System.out.println("long");
}

static void print(Integer value) {
    System.out.println("Integer");
}

public static void main(String[] args) {
    print(10); // long
}

10はintリテラルです。long版はプリミティブ拡大変換だけで呼べるため、Integerへのボクシングより優先されます。

ただし、代入のLong x = 10;が有効になるわけではありません。メソッド選択の規則と代入変換の規則は分けて考えます。

21. コンパイル可否と実行時結果の整理

コード判定理由・結果
int a = 10;正常10はint。
long a = 10;正常intからlongへ拡大。
Long a = 10;コンパイルエラーint→long→Longを自動適用しない。
Long a = 10L;正常longからLongへボクシング。
Number a = 10;正常int→Integer→Number。
Number a = 'A';コンパイルエラーcharはCharacterへボクシングされ、CharacterはNumberではない。
float a = 1.5;コンパイルエラーdoubleからfloatへの縮小。
byte a = 100;正常範囲内コンパイル時定数の特例。
byte a = 128;コンパイルエラーbyteの範囲外。
Integer a = null; int b = a;実行時例外nullのアンボクシングでNullPointerException。
Number a = 100L; Integer b = (Integer)a;実行時例外実体はLongなのでClassCastException。

22. 数値コードを読む順番

  1. リテラルの表記を確認する。 L、F、小数点、基数接頭辞を確認します。
  2. 各部分式の型を決める。 byte・short・charの昇格、二項数値昇格を適用します。
  3. 定数式か変数式かを分ける。 小さい整数型への定数代入特例が使えるかを確認します。
  4. 代入・引数・戻り値で必要な変換を列挙する。 拡大、縮小、ボクシング、アンボクシング、参照型変換を順に追います。
  5. 参照型なら宣言型と実体型を分ける。 Number変数に入っている実体クラスを確認します。
  6. 値の欠落と実行時例外を確認する。 桁あふれ、小数部消失、精度低下、null、不正キャストを確認します。

この単元の最終チェック

  • 整数リテラルは基本int、小数リテラルは基本doubleと判断できる。
  • 10、10L、10F、10.0から生成されるラッパークラスを説明できる。
  • Number n = 10;を、int→Integer→Numberに分解できる。
  • Long n = 10;がコンパイルエラーになる理由を説明できる。
  • Number型を直接アンボクシングできないことを判断できる。
  • Numberの変換メソッドで値や精度を失う場合を説明できる。
  • CharacterがNumberのサブクラスではないことを説明できる。
  • BigIntegerとBigDecimalに専用リテラルがないことを説明できる。
  • 定数式の縮小代入、数値昇格、複合代入を区別できる。
  • ラッパークラスの==とequalsの役割を区別できる。
