配列要素の初期値とnull

Java 11とJava 17の共通範囲です。配列を作った直後の値、ローカル変数との違い、参照型配列で例外が起きる位置をコードで追います。

広告スポンサーリンク表示エリア

配列を作ると、要素は型ごとの値で初期化される

new int[3] は、intを3個置ける配列オブジェクトを作ります。作成された3要素には、すぐに0が入ります。変数numbersが保持するのは、配列そのものではなく配列への参照です。

public class ArrayDefaults {
    public static void main(String[] args) {
        int[] numbers = new int[2];
        boolean[] flags = new boolean[2];
        char[] letters = new char[2];
        String[] names = new String[2];

        System.out.println(numbers[0]);       // 0
        System.out.println(flags[0]);         // false
        System.out.println((int) letters[0]); // 0
        System.out.println(names[0]);         // null
    }
}

charの初期値は空の文字リテラルではありません。値は'\u0000'で、数値として表示すると0です。''という文字リテラルはJavaには存在しません。

要素型初期値表示を読むときの注意
整数型0byte、short、int、longで共通。
浮動小数点型0.0float、doubleで共通。
booleanfalse数値の0とは別の値。
char'\u0000'画面上では見えにくいので、必要ならintへ変換して確認する。
参照型nullオブジェクトが自動生成されるわけではない。

ローカル変数は同じ扱いではない

配列の要素には初期値がありますが、メソッド内で宣言したローカル変数は使用前に値を代入する必要があります。次のコードは実行時例外ではなく、コンパイルエラーです。

public class LocalVariableError {
    public static void main(String[] args) {
        int value;
        // System.out.println(value); // コンパイルエラー: valueが初期化されていない
    }
}

コードを読むときは「型の初期値」を暗記するだけでなく、その値が入る場所を見ます。フィールドと配列要素は初期化され、ローカル変数は自動初期化されません。

参照型配列では、要素のnullを使った時点で例外になる

public class ReferenceArray {
    public static void main(String[] args) {
        String[] names = new String[2];
        System.out.println(names[0]);          // null
        System.out.println(names[0].length()); // ここでNullPointerException
    }
}
手順式結果
1new String[2]要素が2個ある配列を作る。各要素はnull。
2names[0]nullを取り出す。この操作だけでは例外は出ない。
3names[0].length()nullに対してメソッドを呼ぶためNullPointerException。

多次元配列では、行そのものがnullの場合がある

public class JaggedArray {
    public static void main(String[] args) {
        int[][] table = new int[2][];
        table[0] = new int[3];

        System.out.println(table[0].length); // 3
        System.out.println(table[1]);        // null
        // System.out.println(table[1].length); // NullPointerException
    }
}

new int[2][]が作るのは、int[]への参照を2個持つ外側の配列です。内側の配列は別途作るまでnullです。

コードを読む順番

  1. 変数そのものか、配列の要素か、フィールドかを区別する。
  2. 配列が作成済みかを確認する。
  3. 参照型要素なら、要素がnullのまま使われていないかを見る。
  4. 添字が0からlength - 1の範囲内かを確認する。
  5. 例外が起きる式より前の出力だけを残す。

配列の代入とArrayStoreExceptionは、配列の共変性で扱います。ここでは「作成直後の要素」と「nullを使う位置」に絞っています。

記事末尾の確認

  • char[]の初期値を'\u0000'と説明できる。
  • ローカル変数未初期化を実行時例外と誤認しない。
  • String[]の各要素が自動で空文字になると思わない。
  • 多次元配列の外側と内側を別の配列として追える。
