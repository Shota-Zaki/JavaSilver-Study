switch文のcase・break・フォールスルー

このページはJava 11とJava 17で利用できる従来のswitch文を扱います。Java 17のswitch式とyieldは、別記事で扱います。

一致したcaseから実行を開始する

public class SwitchStatement {
    public static void main(String[] args) {
        int number = 2;

        switch (number) {
            case 1:
                System.out.print("A");
            case 2:
                System.out.print("B");
            case 3:
                System.out.print("C");
                break;
            default:
                System.out.print("D");
        }
    }
}

numberは2なので、case 2から実行します。case 2の末尾にbreakがないため、次のcase 3へ続きます。出力はBCです。

フォールスルーは条件を再判定しない

手順処理
1switch式を1回評価し、一致するcaseを探す。
2一致したラベルの文から実行する。
3break、return、例外、switch末尾のいずれかまで続く。

case 3へ流れたとき、number == 3を再判定するわけではありません。

caseラベルはコンパイル時定数である必要がある

int value = 1;
final int constant = 2;

switch (value) {
    // case value:    // コンパイルエラー: valueは変数
    case constant:   // コンパイル時定数なので使用可能
        break;
}

finalを付ければ常に使えるわけではありません。宣言時に定数式で初期化され、値がコンパイル時に確定する必要があります。

使用できる型とnull

Java 11/17の従来switch文では、byte、short、char、intと対応するラッパー型、String、enum型を主に扱います。long、float、double、booleanはswitch対象にできません。

String command = null;
switch (command) { // 実行時にNullPointerException
    case "start":
        break;
    default:
        break;
}

型としてはStringを使えますが、switch対象の値がnullだと、一致判定へ進む時点でNullPointerExceptionが発生します。

defaultの位置は自由だが、実行順はソース順

int value = 9;
switch (value) {
    default:
        System.out.print("D");
    case 1:
        System.out.print("A");
}

一致するcaseがないためdefaultから開始し、その後にbreakがないのでcase 1の文へ流れます。出力はDAです。

コードを読む順番

  1. switch対象の型が使用可能か確認する。
  2. caseラベルが重複せず、コンパイル時定数か確認する。
  3. switch対象の値を1回だけ評価する。
  4. 一致したcaseまたはdefaultの位置を探す。
  5. breakまでソースコードの順に実行する。

値を返すswitch式、アロー形式、yieldはswitch式とyieldで扱います。Java 11ではswitch式を利用できません。

記事末尾の確認

  • 一致したcase以降で条件を再判定しないと説明できる。
  • defaultが途中にあっても実行順を追える。
  • コンパイル時定数と単なるfinal変数を区別できる。
