配列とStringの基礎

配列は同じ型の値を決まった個数だけ並べる仕組みです。Stringは文字列を表す参照型で、作成後に内容を書き換えられません。まず、配列の添字とStringの戻り値を別々に追います。

広告スポンサーリンク表示エリア

配列は作成時に要素数が決まる

public class ArrayBasic {
    public static void main(String[] args) {
        int[] scores = new int[3];
        scores[0] = 70;
        scores[1] = 80;
        scores[2] = 90;

        System.out.println(scores.length); // 3
        System.out.println(scores[1]);     // 80
    }
}

new int[3]は、int型の要素を3個持つ配列を作ります。有効な添字は0、1、2です。scores[3]はコンパイルできますが、実行するとArrayIndexOutOfBoundsExceptionが発生します。

式意味結果
scores.length要素数3
scores[0]先頭要素70
scores[3]4番目の要素を要求実行時例外

配列変数を代入すると参照がコピーされる

int[] first = {10, 20};
int[] second = first;
second[0] = 99;
System.out.println(first[0]); // 99

second = firstで配列の中身を複製したわけではありません。2つの変数が同じ配列を参照します。そのため、second[0]の変更はfirst[0]からも見えます。

Stringの加工メソッドは新しい文字列を返す

public class StringBasic {
    public static void main(String[] args) {
        String text = "Java";
        text.toLowerCase();
        System.out.println(text); // Java

        text = text.toLowerCase();
        System.out.println(text); // java
    }
}

Stringオブジェクトは不変です。toLowerCase()は元の文字列を変更せず、変換後のStringを戻り値として返します。1回目は戻り値を受け取っていないため、textは"Java"のままです。

lengthの書き方を区別する

対象書き方理由
配列array.lengthlengthはフィールド。
Stringtext.length()length()はメソッド。
String[] words = {"A", "Java"};
System.out.println(words.length);    // 2
System.out.println(words[1].length()); // 4

コードを読む順番

  1. 配列の要素数と有効な添字を書く。
  2. 配列変数同士の代入では、同じ配列を参照していないかを見る。
  3. Stringメソッドの戻り値を変数へ代入しているかを見る。
  4. 配列のlengthとStringのlength()を区別する。
  5. 範囲外アクセスがあれば、その式で実行が止まる。

配列要素の型ごとの初期値は、配列要素の初期値とnullで扱います。

記事末尾の確認

  • 要素数3の配列で有効な添字を答えられる。
  • 配列の参照コピーを中身の複製と混同しない。
  • Stringの加工結果を使うには戻り値が必要だと説明できる。
