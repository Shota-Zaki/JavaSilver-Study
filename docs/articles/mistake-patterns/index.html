理解のつまずきから戻る確認表

結論だけを覚えず、「どの前提を取り違えたか」から復習先を決める補助記事です。仕様の詳細は各解説記事へ分離しています。

広告スポンサーリンク表示エリア

結果を間違えたときの分類

誤りの症状取り違えた前提戻る記事
出力を選んだが実際はコンパイルエラー型、初期化、アクセス、checked例外を後回しにしたコンパイルエラー
例外後の文も出力へ含めた例外発生後も同じブロックを続行すると思った実行時例外
==とequalsの結果を混同同一性と同値性、クラスごとのequals実装を分けていないStringと参照比較
親型変数なら親メソッドが動くと判断オーバーロード選択とオーバーライド実行を一段階で考えた継承
Stringのメソッド後に元変数が変わったと判断不変オブジェクトと可変オブジェクトを分けていないString / StringBuilder
順序が明示されていないのに一つへ断定APIが保証する順序と現在の実装結果を区別していない仕様保証の境界

理解を確認するときに一文で根拠を書く

  • 「未初期化ローカル変数を読むため、実行前にコンパイルエラー」
  • 「実体がString[]なので、Integer格納時にArrayStoreException」
  • 「宣言型Parentでオーバーロードを選び、実体Childのoverrideを実行」
  • 「終端操作がないためStreamの中間操作は評価されない」

この一文が書けない場合、結論が合っていても判断手順が再現できていません。

記事末尾の確認

  • 誤りを「暗記不足」だけでまとめていない。
  • コンパイル時・実行時・API仕様・評価順のどこで誤ったか分類できる。
  • 同じ誤りを別のコードへ持ち込まないための判断文を書ける。
