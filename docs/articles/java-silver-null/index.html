nullとNullPointerException

Java 11とJava 17の共通範囲です。nullは「空文字」や「値0」ではなく、参照型変数がオブジェクトを参照していない状態を表します。

nullを保持・比較・表示するだけでは例外にならない

public class NullBasic {
    public static void main(String[] args) {
        String text = null;
        System.out.println(text);        // null
        System.out.println(text == null); // true
    }
}

textはString型の参照変数ですが、どのStringオブジェクトも参照していません。nullの代入、== nullによる比較、printlnへの受け渡しは正常に実行できます。

メンバへ進もうとした式でNullPointerExceptionになる

public class NullCall {
    public static void main(String[] args) {
        String text = null;
        System.out.print("A");
        System.out.println(text.length()); // ここでNullPointerException
        System.out.print("B");             // 実行されない
    }
}
式判定
textnullという参照値を取り出すだけ。
text.length()参照先オブジェクトのメソッドへ進もうとするため例外。

Aは例外より前に表示されます。例外を捕捉していないためプログラムは異常終了し、Bは表示しません。

配列自体のnullと、配列要素のnullを分ける

String[] first = null;
String[] second = new String[2];

// first.length;       // NullPointerException: 配列自体がない
System.out.println(second.length); // 2
System.out.println(second[0]);     // null
// second[0].length(); // NullPointerException: 要素がnull

secondは配列オブジェクトを参照しているため、second.lengthは読めます。ただし、参照型配列の各要素は初期状態でnullです。

アンボクシングでも例外が起きる

Integer boxed = null;
// int value = boxed; // 自動アンボクシング時にNullPointerException

代入先がintなので、コンパイラはboxed.intValue()に相当する処理を必要とします。boxedがnullのため、アンボクシング時に例外が発生します。

nullを受け取るオーバーロード

static void print(Object value) { System.out.print("O"); }
static void print(String value) { System.out.print("S"); }

public static void main(String[] args) {
    print(null); // S
}

nullはどちらの参照型にも渡せますが、StringはObjectより具体的な型なのでprint(String)が選ばれます。互いに継承関係のない候補が残ると、曖昧な呼び出しとしてコンパイルエラーになります。

判定手順

  1. 対象がプリミティブ型か参照型かを見る。プリミティブ型へnullは直接代入できない。
  2. 参照値を保持・比較・表示するだけか、参照先のメンバへ進むかを分ける。
  3. 配列自体と配列要素のどちらがnullかを確認する。
  4. 自動アンボクシングが発生していないかを見る。
  5. 例外が捕捉されるなら、対応するcatchとfinallyへ進む。

記事末尾の確認

  • System.out.println(null参照)とnull参照.toString()を区別できる。
  • 配列自体のnullと要素のnullを分けて説明できる。
  • Integer nullのアンボクシングで例外が起きる理由を説明できる。
