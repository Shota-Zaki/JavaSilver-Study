Java Goldの入出力・NIO.2問題の解き方

Java GoldのPath、Files、readString、lines、walk、try-with-resources、IOExceptionの問題を整理します。

広告スポンサーリンク表示エリア

最終更新日：2026年7月7日。Java SE 17の試験範囲を中心に、演習前後の読み方を整理しています。

先に結論

NIO.2は、Path と Files を分けて読みます。Path はパス表現、Files はファイルシステム操作です。存在確認、例外、Streamを返すメソッドのclose忘れが出題されやすいです。

  • 実ファイルの存在が必要な処理か
  • IOExceptionを処理しているか
  • Files.linesやwalkのStreamをcloseできるか
  • 相対パスと絶対パスの扱いを混同していないか
Java Practice Labでの使い方NIO.2の演習では、コードを読む前に「文字列操作ではなくファイル操作か」を切ります。例外条件が絡む問題は、コンパイル可否と実行時の存在条件を分けます。

Pathは文字列ではなくパス表現

Path はファイルシステム上のパスを表すオブジェクトです。文字列結合だけで判断せず、resolve、normalize、getFileName、getParentなどの戻り値を確認します。

Filesの代表操作

操作見ること注意点
readString / writeString文字列の読み書きIOExceptionの扱い
linesStreamを返すリソースclose
walk階層をたどる遅延評価と例外

try-with-resources

ファイルやStreamを扱う問題では、try-with-resourcesでcloseされるかを確認します。複数リソースは後から作られたものからcloseされます。

例外条件

存在しないファイル、権限、ディレクトリとファイルの取り違えなどでIOExceptionが発生します。Gold問題では、コンパイル可否と例外発生位置を分けて読みます。

練習方法

NIO.2は実行環境依存の結果を断定しすぎないことも外せません。試験では、問題文に与えられたファイル構造と条件だけで判断します。
