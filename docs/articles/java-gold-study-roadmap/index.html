Java Gold学習ロードマップ

Java Goldは、API名を覚える試験ではありません。
型、戻り値、例外、評価タイミング、リソース管理、モジュール境界をコードから判定する試験です。
Silverの読解力を土台にして、Generics、Lambda、Stream、NIO、並行処理、Moduleへ順番に進めます。

結論

最初にGenericsとLambdaを固めています。
ここが弱いままStreamやCollectorsへ進むと、APIの暗記量だけが増えて、出力問題・コンパイル可否問題で点が伸びません。

Gold学習で最初に整理する前提

Gold対策は、Silverの延長ではあります。ただし、問われ方はかなり変わります。
Silverは文法の可否を見抜く問題が中心でしたが、Goldは複数APIを組み合わせたときの型の流れ、実行タイミング、副作用、例外処理を読ませます。

最初に見る前提弱い状態先に直すこと
継承・ポリモーフィズム変数の型と実体の型を混同するメソッド呼び出しは実体、フィールド参照は変数の型で読む
例外チェック例外、非チェック例外、catch順序が曖昧throws、try-with-resources、catchの到達可能性を先に整理する
型変換暗黙変換、キャスト、ラッパー型の扱いが不安定Genericsへ入る前に参照型・プリミティブ型・ラッパー型を分ける
メソッド解決オーバーロードとオーバーライドを混ぜるコンパイル時の選択と実行時の動的ディスパッチを分ける
スコープラムダ内のローカル変数、effectively finalを失点する変数がどこで宣言され、どこで変更されるかを追う

標準ルート：8週間

学習時間が十分に取れる場合は、8週間で分野別理解と模試復習を分けます。平日は短時間で記事と小問、休日にまとまった演習を入れる構成です。

週主テーマやること完了条件
1週目Silver橋渡し継承、例外、型変換、static、オーバーロードを復習コンパイル可否と実行時例外を分けて説明できる
2週目Generics型パラメータ、境界型、ワイルドカード、PECS、型消去? extends と ? super の使い分けを説明できる
3週目Lambda関数型インタフェース、メソッド参照、ローカル変数制約ラムダの戻り値型と例外をターゲット型から判断できる
4週目Stream / Optional遅延評価、短絡評価、終端操作、Optionalの戻り値Stream再利用不可、Null回避、例外発生位置を追える
5週目CollectorstoMap、groupingBy、partitioningBy、下流Collector戻り値のネスト構造を型で書ける
6週目NIO / DateTime / JDBCPath、Files、try-with-resources、日時API、ResultSet戻り値型と例外を表で整理できる
7週目Concurrency / ModuleExecutor、Callable、Future、synchronized、requires、exports、opens実行順の保証有無とモジュール境界を説明できる
8週目模試・弱点補強模試を解き、カテゴリ別に復習、再演習不正解理由をタグ単位で説明できる

短期ルート：4週間

短期で詰める場合は、全範囲を浅く回すより、点に直結する分野を優先しています。
Generics、Lambda、Stream、Collectorsを失点すると、後半の複合問題で失点が連鎖します。

  1. 1週目：Silver橋渡し、Generics、Lambdaを集中して固める。
  2. 2週目：Stream、Optional、Collectorsを毎日解く。戻り値型を必ず書き出す。
  3. 3週目：NIO、DateTime、JDBC、Concurrency、Moduleを表で整理する。
  4. 4週目：模試を解いて、弱点タグ単位で戻る。新しい教材を増やさず、同じ他の選択肢の理由を除外する。

SE 11とSE 17の分岐

このサイトでは、Goldを共通部分と17追加分に分けています。先に共通部分を固め、受ける試験に応じて17側の言語機能・API差分を追加します。

受験区分優先順位注意点
Gold SE 11Generics、Lambda、Stream、Collectors、NIO、Concurrency、Moduleを中心に進めるModuleは暗記で処理しにくい。requires、exports、opens の効果をコードで見る
Gold SE 17SE 11の土台に加えて、17側の言語機能とAPI差分を見るrecord、sealed、switch式、パターンマッチングはSilver 17側の理解も再利用する

Java Gold SE 11とSE 17の違いで受験区分を決め、教材を買う前にJava Gold教材の選び方で役割を分けると判定しやすくなります。

演習の回し方

Goldの演習は、問題数をこなすだけでは伸びません。
間違えた単元を「APIを知らなかった」「型を追えなかった」「実行順を誤読した」「例外を見落とした」に分類します。

  1. 1周目：正答率よりも分類を優先する。知らないAPIは単元別解説へ戻る。
  2. 2周目：不正解タグだけを解く。解説を読んだ直後に、なぜ他の選択肢が誤りかを説明する。
  3. 3周目：時間制限を付ける。読解に時間がかかる分野を洗い出す。
  4. 模試後：カテゴリ別分析を見て、下位3カテゴリを先に復習します。

模試後は、点数ではなく失点した段階へ戻します。Gold対策ページの模試結果では、弱点タグとあわせてロードマップの戻り先を表示します。

模試結果で弱点段階を判定し、単元別解説一覧のGold読む順番から該当単元別解説へ戻ります。記事を読んだら対応する通常演習で同じ確認ポイントを使えるか見ると分かりやすいです。
