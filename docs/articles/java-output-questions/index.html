先に結論

出力問題は、目についたprintlnだけを読むと崩れます。コンパイルできるかを先に確認し、mainから順に変数、参照先、メソッド呼び出し、例外発生位置を追います。

見る基準
  • コンパイルできるコードか
  • 変数の値がどこで変わるか
  • 文字列結合やインクリメントの評価順を見たか
  • 呼び出されるメソッドがどれか
  • 例外発生後に後続処理を続けていないか

出力問題の基本手順

  1. コンパイルできるかを確認する
  2. main メソッドの先頭を探す
  3. 変数に入る値と参照先を書く
  4. メソッド呼び出しの順序を追う
  5. 条件分岐のtrue / falseを決める
  6. 例外が発生する行を確認する
  7. 出力される文字列を順に並べる

値の変化を追う

インクリメント、文字列結合、配列、StringBuilder、Listは、途中の値を表にして追うと安定します。特に i++ と ++i、Stringの不変性、StringBuilderの破壊的変更は混同しやすくなります。

メソッド呼び出しを追う

継承が絡む場合は、参照型と実体型を分けます。overrideされたインスタンスメソッドは実行時の実体型で選ばれます。一方、staticメソッドやフィールドは同じようには扱いません。

継承・interface・overrideを復習する

例外が出たら後続処理は止まる

try内で例外が発生すると、try内の後続処理は実行されず、対応するcatchへ移ります。finallyがある場合は、正常終了でも例外発生時でも実行されます。
