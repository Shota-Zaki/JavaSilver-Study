メソッド・引数・戻り値の基礎

基礎・Bronze向けの記事です。メソッドを「処理に名前を付けたもの」として捉え、宣言、呼び出し、引数、戻り値、値渡しを最小コードから説明します。

メソッドは入力を受け取り、処理し、必要なら結果を返す

public class MethodBasics {
    static int add(int left, int right) {
        int result = left + right;
        return result;
    }

    public static void main(String[] args) {
        int answer = add(2, 3);
        System.out.println(answer); // 5
    }
}
部分この例意味
戻り値型int呼び出し元へ返す値の型。
メソッド名add呼び出すときに使う名前。
仮引数int left, int rightメソッド側が受け取る変数。
実引数2, 3呼び出し側から渡す値。
returnreturn result;メソッドを終了し、値を呼び出し元へ返す。

voidは値を返さない

static void printMessage(String message) {
    System.out.println(message);
    return; // 値を付けないreturnは使用できる。省略も可能。
}

voidメソッドでreturn 10;のように値を返すとコンパイルエラーです。逆に、戻り値型がintなのに正常終了する経路で値を返さない場合もコンパイルエラーです。

引数は左から評価され、仮引数へコピーされる

public class ArgumentOrder {
    static int show(String label, int value) {
        System.out.print(label);
        return value;
    }

    static void use(int a, int b) {
        System.out.print(a + b);
    }

    public static void main(String[] args) {
        use(show("A", 1), show("B", 2));
    }
}

実引数は左から評価されるため、A、B、合計の3の順に表示され、出力はAB3です。

Javaは参照型でも値渡し

class Box { int value; }
public class PassByValue {
    static void change(Box box) {
        box.value = 10; // 同じオブジェクトの状態を変更
        box = new Box(); // 仮引数だけを別の参照へ変更
        box.value = 20;
    }

    public static void main(String[] args) {
        Box original = new Box();
        change(original);
        System.out.println(original.value); // 10
    }
}

呼び出し時に参照値が仮引数へコピーされます。最初のbox.value = 10は共有している実体を変更します。box = new Box()は仮引数のコピーだけを変更するため、originalの参照先は変わりません。

staticメソッドとインスタンスメソッド

class Counter {
    static int total;
    int value;

    static void resetTotal() { total = 0; }
    void increment() { value++; total++; }
}

staticメソッドはクラスに属し、インスタンスを作らずに呼べます。インスタンスメソッドは特定のオブジェクトに対して呼び出します。static文脈からインスタンスフィールドvalueをオブジェクトなしで直接参照することはできません。

コンストラクタはメソッドではない

コンストラクタはクラス名と同じ名前で、戻り値型を書きません。void Sample()はコンストラクタではなく、Sampleという名前のメソッドです。

class Sample {
    Sample() { }       // コンストラクタ
    void Sample() { }  // メソッド
}

次に学ぶ内容

同名メソッドが複数ある場合の選択、nullや可変長引数による曖昧さは、メソッド呼び出し解決で扱います。コンストラクタ連鎖はクラス・コンストラクタへ進んでください。

記事末尾の確認

  • 仮引数と実引数を区別できる。
  • voidと値を返すメソッドのreturnを区別できる。
  • 参照型を「参照渡し」と説明せず、参照値のコピーとして追える。
  • コンストラクタに戻り値型を書かない理由を説明できる。
