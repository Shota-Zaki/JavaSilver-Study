実行時例外の発生位置を読む

コンパイルに成功した後、どの式を評価した瞬間に例外が発生するかを整理します。例外名だけでなく、発生条件、発生時点、後続処理が実行されるかを追います。

広告スポンサーリンク表示エリア

nullは代入時ではなく、参照を使う式で問題になる

String text = null;           // 正常
System.out.println(text);     // "null"を表示
System.out.println(text.length()); // ここでNullPointerException

nullを参照型変数へ代入すること自体は有効です。フィールド参照、メソッド呼び出し、配列長取得、アンボクシングなど、実体を必要とする操作で例外になります。

配列と文字列は上限が「長さ - 1」

int[] values = {10, 20};
System.out.println(values[1]); // 20
System.out.println(values[2]); // ArrayIndexOutOfBoundsException

長さが2なら有効な添字は0と1です。文字列のcharAtやsubstringも境界条件を別に確認します。

キャストは実体を変換しない

class Parent {}
class Child extends Parent {}
class Other extends Parent {}

Parent value = new Other();
Child child = (Child) value; // 実体はOtherなのでClassCastException

このコードは、型の継承関係があるためコンパイルできます。実行時に実体OtherをChildとして扱えないことが判明し、キャスト式で停止します。

整数0除算と浮動小数点0除算を分ける

System.out.println(10 / 0);     // ArithmeticException
System.out.println(10.0 / 0.0); // Infinity

浮動小数点演算はIEEE 754の値としてInfinityやNaNを作るため、整数演算と同じ例外にはなりません。

catchされた場合は、そのcatchから処理が続く

try {
    System.out.print("A");
    Integer.parseInt("x");
    System.out.print("B");
} catch (NumberFormatException e) {
    System.out.print("C");
}
System.out.print("D");

出力はACDです。例外後のtry内Bは飛ばされ、catch終了後にtry-catchの次の文Dへ進みます。

コードを読むときの記録

  1. 例外候補となる式へ印を付ける。
  2. 値・参照・添字をその式の直前まで追う。
  3. 最初に発生する例外だけを確定する。
  4. 対応するcatchがあるか、型の親子関係を確認する。
  5. 例外発生前、catch、finally、後続文の出力を並べる。

記事末尾の確認

  • 例外名と発生する式をセットで説明できる。
  • 実行時例外とコンパイルエラーを区別している。
  • 整数と浮動小数点の0除算を区別できる。
  • catch後にどこへ進むかを追える。
