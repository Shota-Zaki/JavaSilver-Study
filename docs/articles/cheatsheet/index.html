最初の30秒で行う分類

確認見る場所次に進む条件
1. 対象バージョンrecord、sealed、switch式、Stream.toList()などJava 11と17のどちらで成立するか確定。
2. コンパイル可否宣言、型、アクセス、戻り値、checked例外エラーがなければ実行へ。
3. 実行時例外null、添字、キャスト、0除算、変更不可コレクション例外発生行とcatch先を確定。
4. 出力追跡評価順、値、参照、呼び出し順例外より前に実行された出力だけを並べる。

コンパイル前チェック

  • ローカル変数は使用前に初期化されているか。
  • 式の型を代入先、引数、戻り値の型へ変換できるか。
  • オーバーライドでアクセス範囲を狭めていないか。
  • 抽象メソッドやdefaultメソッドの衝突を解決しているか。
  • checked例外をcatchまたはthrowsで処理しているか。
  • this(...)またはsuper(...)がコンストラクタ本体の先頭にあるか。

コンパイルエラーの詳しい解説 / コンパイルと実行コマンド

実行時チェック

  • nullからフィールドやメソッドへ進んでいないか。
  • 配列・文字列・Listの有効範囲を越えていないか。
  • ダウンキャスト先と実体型が一致するか。
  • 整数の0除算がないか。
  • 変更不可または固定サイズのListへ変更操作をしていないか。
  • StreamやOptionalを空・消費済みの状態で使っていないか。

提出前の4問

  1. そのコードは対象バージョンでコンパイルできるか。
  2. 最初に例外が起きる式はどれか。
  3. その式より前に何が出力されたか。
  4. 順序や実装型を、仕様の保証なしに断定していないか。
