switch式とyield

switch式はJava 14で正式導入され、Java 17で利用できます。Java 11では利用できません。Java 17 Silverでは、switch文とswitch式を分け、戻り値の有無、yield、網羅性を判定します。

switch文とswitch式の違い

項目switch文switch式
値を返すか返さない返す
代入に使えるか不可可
網羅性必須ではない必要
yield通常使わないブロックで値を返すときに使う

アロー形式のswitch式

int score = 2;
String result = switch (score) {
    case 1 -> "A";
    case 2 -> "B";
    default -> "C";
};
System.out.println(result);

switch 式全体が値を返すため、最後にセミコロンが必要です。各 case の右側は、式、ブロック、throw のいずれかになります。

yieldを使うケース

case の右側をブロックにした場合、switch式として返す値を yield で示します。

int score = 2;
String result = switch (score) {
    case 1 -> "A";
    case 2 -> {
        String text = "B";
        yield text;
    }
    default -> "C";
};

return ではなく yield を使う点が判定材料です。return はメソッドから戻るため、switch式の値を返す目的には合いません。

コロン形式とbreak

従来のコロン形式でもswitch式を書けます。この場合は yield で値を返します。

String result = switch (score) {
    case 1:
        yield "A";
    case 2:
        yield "B";
    default:
        yield "C";
};

switch式では、従来の break で値を返す形ではなく、yield を使う形で判定します。

理解の確認

  • 代入の右辺にあるswitchは式として値を返す必要がある。
  • switch式は全ケースで値を返す必要がある。
  • ブロック形式では yield を使う。
  • switch式全体の末尾にはセミコロンが必要。
