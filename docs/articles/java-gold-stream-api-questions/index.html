Java GoldのStream API問題の解き方

Java Goldで頻出のStream API問題を、中間操作、終端操作、遅延評価、Optional、Collectors、並列ストリームの観点で整理します。

先に結論

Stream API問題は、パイプラインを左から眺めるだけでは解けません。中間操作、終端操作、戻り値型、Optionalの有無、遅延評価を分けて見ます。終端操作がない処理は実行されません。

見る基準
  • 中間操作と終端操作を分けたか
  • mapとflatMapの戻り値の違いを見たか
  • reduceやcollectの戻り値型を確認したか
  • Optionalを返す可能性があるか
  • 並列ストリームで順序を決めつけていないか
Java Practice Labでの使い方Stream問題は、解説を読んだ後に処理の表を作ると安定します。入力、各操作後の要素、終端操作の結果を分けて確認します。

Streamは流れを分けて読む

Stream問題では、ソース、中間操作、終端操作を分けて読みます。終端操作がない場合、中間操作は実行されません。出力問題では、どのタイミングで処理が走るかを確認します。

中間操作と終端操作

分類例読み方
中間操作filter, map, flatMap, sorted, distinctStreamを返し、遅延評価される
終端操作forEach, collect, reduce, count, anyMatch結果を返し、ここで処理が実行される

Optionalとの組み合わせ

findFirst、max、min などはOptionalを返すことがあります。値がない可能性があるため、get() を無条件に呼ぶ問題では例外条件を確認します。

Collectors

groupingBy、partitioningBy、mapping、joining では戻り値の型が問われます。Mapのキー型、値型、下流collectorの結果型を分けて見ます。

練習方法

Streamは単元別解説で操作分類を確認し、Gold演習で戻り値と評価順を追います。模試では、長いパイプラインを表に分けて読む練習をします。
