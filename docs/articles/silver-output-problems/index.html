出力結果を表で追う方法

このページは、コンパイルできることを確認した後の追跡に限定します。変数、参照、呼び出し順、例外を別々の欄へ書き、表示された文字だけを最後に結合します。

広告スポンサーリンク表示エリア

前提: 出力追跡は最後の段階

コンパイル可否が未確定なら、先にJavaコードを読む順番へ戻ります。出力候補があっても、コンパイルエラーなら何も実行されません。

値の変化は「式が返す値」と「実行後の値」を分ける

public class IncrementTrace {
    public static void main(String[] args) {
        int i = 1;
        System.out.print(i++);
        System.out.print(++i);
        System.out.print(i);
    }
}
式式が返す値実行後のi出力
i++121
++i333
i333

最終出力は133です。

参照型は、変数とオブジェクトを別に描く

class Box { int value; }
public class ReferenceTrace {
    public static void main(String[] args) {
        Box a = new Box();
        Box b = a;
        b.value = 5;
        System.out.println(a.value);
    }
}
行abBoxのvalue
new Box()Box#1を参照未宣言0
b = aBox#1を参照同じBox#1を参照0
b.value = 5Box#1を参照Box#1を参照5

aとbは同じ実体を参照するため、出力は5です。

コンストラクタは呼び出しを深く進めてから本体へ戻る

class Parent {
    Parent() { System.out.print("P"); }
}
class Child extends Parent {
    Child() { this(2); System.out.print("A"); }
    Child(int n) { System.out.print(n); }
}
public class ConstructorTrace {
    public static void main(String[] args) { new Child(); }
}
  1. Child()からthis(2)でChild(int)へ進む。
  2. Child(int)の先頭には暗黙のsuper()があり、Parent()がPを出力する。
  3. Child(int)へ戻って2を出力する。
  4. Child()へ戻ってAを出力する。

出力はP2Aです。

例外処理は移動先を矢印で記録する

public class ExceptionTrace {
    public static void main(String[] args) {
        try {
            System.out.print("A");
            int n = 10 / 0;
            System.out.print("B");
        } catch (ArithmeticException e) {
            System.out.print("C");
        } finally {
            System.out.print("D");
        }
    }
}

10 / 0でArithmeticExceptionが発生し、try内のBは実行されません。catchのC、finallyのDへ進むため、出力はACDです。

ループは反復ごとに条件と更新を記録する

反復開始時i条件本体更新後i
100 < 3 → true0を出力1
211 < 3 → true1を出力2
322 < 3 → true2を出力3
終了33 < 3 → false実行しない-

記事末尾の確認

  • 後置演算子の「渡す値」と「残る値」を分けている。
  • 参照変数の代入をオブジェクトの複製と思っていない。
  • コンストラクタ本体へ入る前のthis()/super()を追っている。
  • 例外発生後の同じtryブロックを続行していない。
