先に結論

Silverで失点しやすい原因は、知識不足より読み方の混同です。コンパイル時に見る条件、実行時に見る条件、例外が発生した後の流れを分けるだけで、同じ論点の問題を安定して処理できます。

見る基準
  • コンパイルエラーなのに実行結果を追っていないか
  • 参照型と実体型を混同していないか
  • StringとStringBuilderの値変化を同じに扱っていないか
  • checked exceptionの処理漏れを見落としていないか
  • ラムダ式のターゲット型を飛ばしていないか

間違えやすい10パターン

  1. 参照型と実体型を混同する
  2. overrideとoverloadを混同する
  3. staticメソッドやフィールドを動的に選ばれると考える
  4. privateメソッドをoverrideできると考える
  5. checked exceptionの処理漏れを見落とす
  6. StringとStringBuilderの値変化を混同する
  7. 配列とListの要素数・添字を誤る
  8. ラムダ式のターゲット型を確認しない
  9. Date and Time APIの不変性を見落とす
  10. switch式、record、sealedなどJava 17論点を曖昧に読む

継承問題は参照型と実体型を分ける

A obj = new B(); のようなコードでは、参照型は A、実体型は B です。コンパイル時は参照型から呼び出せるかを見ます。実行時はoverrideされたインスタンスメソッドだけが実体型で選ばれます。

例外は発生タイミングを分ける

コンパイルエラー、実行時例外、正常終了を同じものとして扱うと誤答します。checked exceptionはコンパイル時に処理します。配列範囲外やNullPointerExceptionは実行時に発生します。
