学習記事

問題を解くための判定基準を、単元別に整理した詳説版です。読む順番、使い分け、復習導線までここから確認します。

目的別に参照先を選ぶ

1. 判定フロー

問題文を読んだら最初に何を見るかを固定します。出力を追う前にコンパイル可否を切るのが最優先です。

2. コンパイル/実行時例外

コンパイルエラー、実行時例外、正常終了を分けます。ここが曖昧だと選択肢を消せません。

3. 弱点別復習

間違え方から読む記事を逆引きします。全部読むより、弱点から潰すほうが効率的です。

論点に合う読み方

  1. 型・継承の記事では、宣言型、実行時型、対象メンバーの順に分ける。
  2. 数値・演算子の記事では、式の型、評価順、副作用、代入先を表にする。
  3. String・配列・Listの記事では、生成されたオブジェクト、参照共有、変更可否、境界を追う。
  4. 例外の記事では、発生地点、捕捉先、finally、リソース解放、伝播先を経路として書く。
  5. API記事では、戻り値、副作用、許される添字、発生し得る例外をメソッドごとに確認する。
