ループ制御

while、do-while、for、拡張for、break、continue、ラベルを、コンパイル可否と出力追跡の両面から整理します。

ループは次に実行される地点を追う

ループ問題は「何回回るか」ではなく、「各回の入口・本体・出口で変数がどう変わるか」を表で追う。
特に continue の後に更新式へ行くか、条件式へ戻るかを間違えると、ほぼ確実に失点します。
  1. コンパイル可否を見る。条件式がbooleanか、ラベルの飛び先が正しいか、到達不能文がないか確認する。
  2. ループの種類を判定する。while、do-while、通常for、拡張forで評価順が違う。
  3. 初期値を表に書く。i、j、sum、flagなど、変わる変数を列にする。
  4. 1周分の流れを固定する。条件 → 本体 → continue/break → 更新式 → 次の条件、の順を確認する。
  5. 終了条件を確認する。条件false、break、return、例外、無限ループのどれで終わるか見る。

while文は条件を先に見る

while は本体を実行する前に条件式を評価します。最初から条件がfalseなら、本体は1回も実行されません。

int a = 10;
while (a < 10) {
    System.out.println("A");
}
System.out.println("B");
// B

a < 10 は最初からfalseなので、A は出ません。

do-while文は最低1回実行する

do-while は本体を実行した後に条件式を評価します。そのため、条件が最初からfalseでも本体は1回実行されます。

int a = 10;
do {
    System.out.println("A");
} while (a < 10);
System.out.println("B");
// A
// B
do-whileの末尾にはセミコロンが必要。
do { ... } while (condition); の形です。ここは構文問題として狙われます。

通常for文の実行順序

for文は、初期化式・条件式・更新式を1行に書きますが、実行順序は固定です。

  1. 初期化式を1回だけ実行する。
  2. 条件式を評価する。falseなら終了。
  3. 本体を実行する。
  4. 更新式を実行する。
  5. 再び条件式へ戻る。
for (int i = 0; i < 3; i++) {
    System.out.print(i);
}
// 012
タイミングi出力説明
初期化0-int i = 0
1周目00本体後に i++ で1。
2周目11本体後に2。
3周目22本体後に3。
条件判定3-i < 3 がfalseで終了。

通常for文は4地点を固定して追跡する

通常for文は、初期化、条件判定、本体、更新の順に進みます。continueが実行された場合でも、更新式を経由して次の条件判定へ進みます。

for (int i = 0; i < 3; i++) {
          if (i == 1) {
              continue;
          }
          System.out.print(i);
      }
      // 02
構文continue後の移動先頻出事故
for更新式更新が実行されることを忘れる
while条件式本体末尾の更新を飛ばして無限ループ
do-while末尾の条件式最低1回実行される点を忘れる
拡張for次要素の取得添字を直接操作できない
whileのcontinue

このコードは終了しますか。

int i = 0;
      while (i < 3) {
          if (i == 1) continue;
          i++;
      }
制御の戻り先：i == 1になった後、i++を通らないため無限ループです。

for文の3要素は省略できる

通常for文の初期化式、条件式、更新式は省略できます。条件式を省略するとtrue扱いになり、無限ループになります。

for (;;) {
    break;
}
System.out.println("end"); // OK

無限ループでも、内部に到達可能な break があれば後続文が到達可能と判断される場合があります。

for文内で宣言した変数のスコープ

for文の初期化部で宣言した変数は、そのfor文の中だけで有効です。

for (int i = 0; i < 3; i++) {
    System.out.println(i);
}
// System.out.println(i); // NG: iはfor文の外では使えない

一方、for文の外で宣言した変数を更新式で使った場合、その変数はループ後も残ります。

int i = 0;
for (; i < 3; i++) { }
System.out.println(i); // 3

continueの飛び先はループの種類で違う

continue は現在の周回の残りを飛ばします。ただし、その後にどこへ行くかはループの種類で違います。

ループcontinue 後の移動先注意点
for更新式へ進むi++ は実行される。
while条件式へ戻る更新処理を本体末尾に書いていると、飛ばされる可能性がある。
do-while条件式へ進む本体末尾を飛ばして条件を見る。
 
for (int i = 0; i < 3; i++) {
    if (i == 1) continue;
    System.out.print(i);
}
// 02

while文のcontinueは無限ループを作りやすい

while文では、更新処理を continue より後ろに書くと、その更新が飛ばされます。

int i = 0;
while (i < 3) {
    if (i == 1) {
        i++;       // これがないと無限ループになりやすい
        continue;
    }
    System.out.print(i);
    i++;
}
// 02
whileでcontinueが出たら、更新処理が実行されるかを必ず確認。 ここを見落とすと、終了するループを無限ループと誤判定したり、その逆をします。

breakは最も近いループまたはswitchを抜ける

ラベルなしの break は、最も内側のループまたはswitchを1つだけ抜けます。ネストした外側のループまでは抜けません。

for (int i = 0; i < 2; i++) {
    for (int j = 0; j < 2; j++) {
        break; // jのループだけ抜ける
    }
    System.out.print(i);
}
// 01

ラベル付きbreak/continue

ラベルは文に名前を付ける機能です。break label; は指定したラベル付き文から抜けます。一方、continue label; の対象にできるのはループだけです。

構文飛び先対象
break label;ラベル付き文の外ループ、switch、通常ブロックなど
continue label;ラベル付きループの次周回ループのみ
 
outer:
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (j == 1) break outer;
    }
}
System.out.println("end");

j == 1 になった時点で、内側だけでなく outer が付いた外側のfor文ごと抜けます。

ラベル付きbreakとcontinue

ラベルは任意の文に付けられますが、continue ラベル名の対象はループ文でなければなりません。breakはラベル付き文全体を抜けます。

outer:
      for (int i = 0; i < 3; i++) {
          for (int j = 0; j < 3; j++) {
              if (j == 1) {
                  continue outer;
              }
              System.out.print(i + ":" + j + " ");
          }
      }
      // 0:0 1:0 2:0
制御対象：ラベルなしのbreak/continueは、最も内側の対応する文だけを対象にします。
ラベル付きbreak

break outer;は内側ループだけを抜けますか。

脱出する範囲：いいえ。outerラベルが付いた文全体を終了します。

continue labelの対象はループでなければならない

ラベル自体はブロックにも付けられます。しかし、continue で飛べるのはループだけです。

label: {
    // continue label; // NG: labelはループではなくブロック
    break label;       // OK: ラベル付きブロックから抜ける
}

この違いは細かいですが、Java Silverでは狙われます。break と continue を同じものとして覚えるのは危険です。

拡張for文の基本

拡張for文は、配列やIterableの要素を順に取り出す構文です。添字を使わないため、範囲外アクセスは起きにくくなりますが、制約もあります。

int[] nums = {1, 2, 3};
for (int n : nums) {
    System.out.print(n);
}
// 123
項目内容
対象配列、または Iterable を実装したオブジェクト。
ループ変数各要素を受け取る一時変数。型互換性が必要。
添字直接は取れない。必要なら通常forを使う。
更新ループ変数へ再代入しても、配列要素そのものは置き換わらない。

拡張forのループ変数は要素そのものではない

プリミティブ配列では、ループ変数を書き換えても配列要素は変わりません。

int[] nums = {1, 2, 3};
for (int n : nums) {
    n = 10;
}
System.out.println(nums[0]); // 1

参照型の場合、ループ変数は参照のコピーを受け取ります。変数に別オブジェクトを代入しても元の配列要素は変わりませんが、参照先オブジェクトの中身を変更すれば影響します。

拡張for文の変数は要素そのものではなく受け取り用変数

int[] values = {1, 2, 3};
      for (int value : values) {
          value *= 10;
      }
      System.out.println(values[0]); // 1

プリミティブ配列では要素値がコピーされます。参照型配列では参照がコピーされるため、参照先オブジェクトの状態変更は反映されますが、ループ変数へ別の参照を代入しても配列要素は差し替わりません。

StringBuilder[] values = {
          new StringBuilder("A")
      };
      for (StringBuilder value : values) {
          value.append("B");              // 同じオブジェクトを変更
          value = new StringBuilder("C"); // 配列要素は差し替わらない
      }
      System.out.println(values[0]); // AB
拡張for中のremove

ArrayListを拡張forで走査しながら、同じリストから要素を削除するとどうなりますか。

構造変更時：構造変更の検出によりConcurrentModificationExceptionとなる可能性が高いです。イテレータのremoveなど適切な方法を使います。

Collectionを拡張forで走査中に構造変更しない

ArrayList などを拡張forで走査中に、同じリストへ add や remove を行うと、実行時に ConcurrentModificationException が発生する典型パターンです。

var list = new java.util.ArrayList<String>();
list.add("A");
list.add("B");

for (String s : list) {
    list.remove(s); // 実行時例外になり得る
}

通常for文で添字を調整する、Iteratorを使うなどの対処が必要ですが、試験では「実行時例外」として判定できれば十分なことが多いです。

到達不能文を判定する

Javaでは、明らかに実行されない文があるとコンパイルエラーになります。

while (true) {
}
// System.out.println("A"); // NG: 到達不能

ただし、ループ内部に到達可能な break がある場合、後続文は到達可能と判断されます。

while (true) {
    break;
}
System.out.println("A"); // OK
if (false) と while(false) は扱いが違う。 while(false) はループ本体が到達不能として問題になります。定数条件が出たらコンパイル可否を先に見てください。

到達不能コードと無限ループ

コンパイラが必ず制御が戻らないと判断できる文の後には、到達不能コードを書けません。ただし、実行時に無限になりそうな一般の条件式をすべて解析するわけではありません。

while (true) {
          break;
      }
      System.out.println("reachable");
      
      // while (true) { }
      // System.out.println("unreachable"); // コンパイルエラー

ネストループの追跡例

outer:
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (i == 1 && j == 1) continue outer;
        System.out.print(i + "" + j + " ");
    }
}
ijの動き出力理由
00,1,200 01 02条件に該当しない。
10,1で中断10j==1 で continue outer。外側forの更新へ。
20,1,220 21 22条件に該当しない。

全体の出力は 00 01 02 10 20 21 22 です。ラベル付きcontinueは「外側ループの次周回へ移動する」と考えると崩れません。

反復位置と脱出先を確かめる

Q1. int a = 10; while (a < 10) { System.out.print("A"); } System.out.print("B");

while は条件を先に見るため本体は0回です。出力は B。

Q2. int a = 10; do { System.out.print("A"); } while (a < 10);

do-while は最低1回実行されるため、A が出力されます。

Q3. for文で continue したら更新式は実行される？

実行されます。for文では、continue後に更新式へ進み、その後条件式を評価します。

Q4. continue はswitchを抜けるために使える？

使えません。switchを抜けるのは break です。continue の対象はループです。

Q5. 拡張forで for (int n : nums) { n = 0; } とすると配列要素は0になる？

なりません。n は各要素の値を受け取る一時変数です。
