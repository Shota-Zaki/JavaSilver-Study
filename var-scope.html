var・スコープ・初期化

varが使える場所、使えない場所、ローカル変数とフィールドの初期化規則を整理します。

varの正体

var は「型をなくす」機能ではありません。右辺からコンパイラが型を推論し、コンパイル時点で型が確定します。

使える場所・使えない場所

場所使用可否例/注意
ローカル変数OKvar s = "java";
フィールドNGprivate var value; は不可。
メソッド引数NGvoid m(var x) は不可。
戻り値型NGvar m() は不可。
コンストラクタ引数NGSample(var x) は不可。
初期化なしNGvar x; は型推論できない。
nullだけNGvar x = null; は型推論できない。
配列初期化子のみNGvar a = {{1,2}}; や {1,2} は不可。

ローカル変数とフィールドの違い

変数初期値注意
フィールド型ごとのデフォルト値intは0、booleanはfalse、参照型はnull。
ローカル変数デフォルト値なし代入前に読むとコンパイルエラー。
配列要素型ごとのデフォルト値配列を作ると要素は初期化される。

スコープの典型例

if (true) {
    int x = 10;
}
// System.out.println(x); // コンパイルエラー
