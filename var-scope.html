var・スコープ・初期化

varの使用可能範囲、型推論の限界、ローカル変数のスコープと初期化を整理します。

varはローカル変数の型推論

var は「型を書かない」構文ではなく、右辺からコンパイラが型を決める構文です。決まった後は普通の静的型付き変数です。

var n = 10;       // int
var s = "abc";    // String
// n = "abc";     // NG。nはint

varを使えない場所

Java Silverでは、var の禁止場所がよく問われます。フィールド、引数、戻り値、配列要素型、catchパラメータなどに使えません。

class Sample {
    // private var value = 10;      // NG。フィールド

    // void test(var x) { }         // NG。引数
    // var method() { return 10; }  // NG。戻り値

    void ok() {
        var value = 10;             // OK。ローカル変数
    }
}

初期化式なし

var a; は不可。型を推論できません。

null単独

var a = null; は不可。型を決められません。

配列初期化子

var a = {1,2,3}; は不可。new int[]{...} が必要です。

ラムダ単独

var f = () -> 1; はターゲット型がないため不可です。

スコープと初期化

ローカル変数は宣言されたブロック内でのみ有効です。また、使用前に確実に初期化されている必要があります。

if (true) {
    int x = 10;
}
// System.out.println(x); // NG。スコープ外

int y;
// System.out.println(y); // NG。未初期化
