String

String・StringBuilder

文字列問題は、参照の同一性、同値比較、不変性、メソッドの戻り値を落とさないことが中心です。

== と equals

比較対象例
==参照が同じかnew String("a") == "a" は通常false
equals内容が同じか"a".equals(new String("a")) はtrue
equalsIgnoreCase大文字小文字を無視null側から呼ぶとNullPointerException

宣言別の比較例を見る

宣言で結果が変わる例

String a = "sample";
String b = "sample";
String c = new String("sample");
String d = c;
System.out.println(a == b);      // true
System.out.println(a == c);      // false
System.out.println(c == d);      // true
System.out.println(a.equals(c)); // true
  • リテラル同士は同じ文字列プールを参照する。
  • new String(...) は別インスタンスを作る。
  • 参照代入はオブジェクトをコピーしない。

Stringは不変

String s = "abc";
s.concat("d");
System.out.println(s); // abc
s = s.concat("d");
System.out.println(s); // abcd
  • replace、substring、trim、toUpperCaseなどは元のStringを変えず、新しいStringを返す。
  • 戻り値を受け取っていない処理は、結果が捨てられる。
