String・StringBuilder

文字列の不変性、可変文字列、頻出メソッドを例題で確認します。

Stringは不変

String s = "java";
s.concat("silver");
System.out.println(s); // java

s = s.concat("silver");
System.out.println(s); // javasilver

Stringの操作メソッドは新しいStringを返します。戻り値を代入しなければ変数は変わりません。

頻出メソッド

メソッド例注意
length()"abc".length() → 3添字ではなく文字数。
charAt(int)"abc".charAt(0) → a範囲外はStringIndexOutOfBoundsException。
substring"abcd".substring(1,3) → bc開始含む、終了含まない。
indexOf見つからないと -10始まり。
equals内容比較null側から呼ぶと例外。
replace置換後の新しいString元のStringは変わらない。

StringBuilderは可変

StringBuilder sb = new StringBuilder("Java");
sb.append(" Silver");
sb.insert(0, "Oracle ");
sb.delete(0, 7);
System.out.println(sb); // Java Silver

StringBuilderは同じインスタンスの中身を変更します。文字列を何度も加工する問題で出ます。

StringBuilderのcapacity

StringBuilder sb = new StringBuilder();
System.out.println(sb.length());   // 0
System.out.println(sb.capacity()); // 16

StringBuilder sb2 = new StringBuilder("abc");
System.out.println(sb2.capacity()); // 19

capacityは現在の文字数ではなく内部バッファ容量です。lengthと混同しないこと。

比較は専用ページで確認

==、equals、intern()、new String、final の比較は 同一性・同値性比較 にまとめています。
