文字列・StringBuilder・null

Stringの不変性、文字列プール、intern、StringBuilderの破壊的操作、nullで起きる例外を整理します。

Stringは不変

String は不変です。連結や置換のように見える操作も、元の文字列を変えるのではなく、新しい文字列を返します。

String s = "java";
        s.concat("17");
        System.out.println(s); // java

        s = s.concat("17");
        System.out.println(s); // java17
戻り値を受け取っていない String メソッド呼び出しは、たいてい結果に影響しません。

文字列プールとnew String

文字列リテラルは文字列プール上のインスタンスを参照します。一方、new String("x") は明示的に新しいインスタンスを作ります。

String a = "sample";
        String b = "sample";
        String c = new String("sample");

        System.out.println(a == b); // true
        System.out.println(a == c); // false
        System.out.println(a.equals(c)); // true

== は参照の同一性、equals は内容比較です。出力問題ではまずここを分けてください。

internの見方

intern() は、文字列プール上の同じ内容の文字列への参照を返します。呼び出し元のインスタンス自体をプール上のものに変えるわけではありません。

String a = new String("sample");
        String b = a;
        String c = a.intern();
        String d = "sample";

        System.out.println(a == b); // true
        System.out.println(a == c); // false
        System.out.println(c == d); // true

a と b はヒープ上の同じ new String を参照します。c と d は文字列プール上の同じインスタンスを参照します。

StringBuilderは可変

StringBuilder は可変です。append、delete、insert、reverse は対象インスタンスを変更します。

StringBuilder sb = new StringBuilder("abc");
        StringBuilder ret = sb.append("de");
        System.out.println(sb);       // abcde
        System.out.println(sb == ret); // true

一方、substring は String を返し、StringBuilder 本体は変わりません。

StringBuilder sb = new StringBuilder("abcdef");
        String s = sb.substring(1, 4); // bcd
        System.out.println(sb);       // abcdef

主なメソッドの境界

メソッド要点注意
charAt(i)i番目の文字範囲外は StringIndexOutOfBoundsException
substring(begin,end)begin以上end未満endは含まない
indexOf(x)見つからなければ -1例外ではない
replace置換結果を返すString本体は変わらない
trim前後の空白除去結果を返す
equalsIgnoreCase大文字小文字を無視呼び出し元がnullならNPE

nullと文字列

null は文字列ではなく、参照がない状態です。メソッド呼び出しの左側が null だと NullPointerException です。

String s = null;
        System.out.println(s);        // null と表示
        // System.out.println(s.length()); // 実行時例外

        System.out.println("x".equals(s)); // false。安全
        // System.out.println(s.equals("x")); // NPE

比較では、定数文字列を左に置くとNPEを避けられます。

ミニ判定

Q1. "abc".substring(1, 1) は？

空文字列です。開始位置と終了位置が同じで、範囲としては0文字です。

Q2. new String("x").intern() == "x" は？

通常は true です。intern() が文字列プール上の参照を返すためです。

Q3. StringBuilder.reverse() は元を変える？

変えます。StringBuilder の主要な編集メソッドは破壊的操作です。
