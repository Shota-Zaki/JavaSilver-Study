文字列・比較

String、StringBuilder、文字列プール、intern、==、equals、nullの扱いを、出力結果の追跡に使える形で整理します。

Stringは不変

String は不変です。メソッドを呼んでも元の文字列は変わりません。戻り値を受け取っていないコードは、見た目ほど何も起きていないことがあります。

String s = "java";
s.toUpperCase();
System.out.println(s); // java

s = s.toUpperCase();
System.out.println(s); // JAVA

concat / replace

新しいStringを返します。元のStringは変更されません。

substring

指定範囲の新しいStringを返します。開始は含み、終了は含みません。

trim / strip

空白処理の結果を返します。元の参照先は変わりません。

文字列結合

+ でStringが絡むと文字列結合になります。左から評価される点も重要です。

== と equals

== は同じオブジェクトを参照しているかを見ます。equals は内容の同値性を見ます。String問題の多くは、ここが中心です。

String a = new String("sample");
String b = a;
String c = "sample";
String d = "sample";

System.out.println(a == b);      // true
System.out.println(a == c);      // false
System.out.println(c == d);      // true
System.out.println(a.equals(c)); // true
new String("sample") はヒープ上に別インスタンスを作ります。文字列リテラル "sample" は文字列プールの参照です。

intern()

intern() は、文字列プール上の同じ内容のString参照を返します。戻り値を受け取らなければ、元の変数の参照は変わりません。

String a = new String("sample");
String b = a.intern();
String c = "sample";

System.out.println(a == b); // false
System.out.println(b == c); // true

試験では a.intern(); とだけ書いて、a 自体がプール参照に変わったように見せる選択肢が出ます。変わるのは戻り値を代入した変数です。

StringBuilderは可変

StringBuilder は可変です。append、insert、delete、reverse などは、同じインスタンスの中身を変えます。

StringBuilder sb = new StringBuilder("abc");
StringBuilder same = sb.append("de");

System.out.println(sb);        // abcde
System.out.println(sb == same); // true

append

末尾に追加し、自分自身を返します。

insert

指定位置に挿入します。位置は0始まりです。

delete

開始位置を含み、終了位置を含みません。

toString

StringBuilderからStringを得ます。StringBuilderとStringは別型です。

nullと文字列

参照型変数は null を持てます。null に対してインスタンスメソッドを呼ぶと NullPointerException です。一方、文字列結合では null は文字列 "null" として扱われます。

String s = null;
System.out.println("value=" + s); // value=null
// System.out.println(s.length()); // NullPointerException
