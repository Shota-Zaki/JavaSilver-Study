文字列・StringBuilder

Stringの不変性、文字列プール、intern、StringBuilder、頻出メソッドの戻り値と副作用を整理します。

Stringは不変

String のメソッドは、元の文字列を変更しません。変更したように見えるメソッドは、新しい文字列を返します。

String s = "abc";
s.toUpperCase();
System.out.println(s); // abc

s = s.toUpperCase();
System.out.println(s); // ABC
戻り値を受け取っているかを必ず確認してください。受け取っていなければ、元のStringは変わりません。

文字列プールとnew String

文字列リテラルは文字列プールを使います。new String("abc") は別のStringオブジェクトを作ります。

String a = "abc";
String b = "abc";
String c = new String("abc");

System.out.println(a == b); // true
System.out.println(a == c); // false
System.out.println(a.equals(c)); // true

internの見方

intern() は、文字列プール上の同じ内容のStringへの参照を返します。== と絡めた問題で出やすいです。

String a = new String("sample");
String b = a;
String c = a.intern();
String d = "sample";

System.out.println(a == b); // true
System.out.println(a == c); // false
System.out.println(c == d); // true

StringBuilderは可変

StringBuilder は自身を変更します。append、insert、delete、reverse は同じオブジェクトを変える点がStringと違います。

StringBuilder sb = new StringBuilder("ab");
sb.append("c");
System.out.println(sb); // abc

StringBuilder same = sb.append("d");
System.out.println(sb == same); // true

ミニ演習

Q. "abc".substring(1, 1) は？

空文字です。beginとendが同じなら長さ0になります。

Q. String s=null; System.out.println(s + "x"); は？

nullx と表示されます。文字列連結ではnullが文字列化されます。

Stringメソッドの戻り値を捨てる罠

Stringは不変なので、メソッドの戻り値を受け取っていなければ結果は使われません。選択肢に変更後の文字列がある場合、代入の有無を必ず見ます。

String s = " Java ";
s.trim();
s.replace("J", "j");
System.out.println(s); // " Java "

StringBuilderの容量と長さ

length() は現在の文字数、capacity() は内部バッファの容量です。Silverではcapacityの細かい計算よりも、StringBuilderが可変であることと、メソッドが自分自身を返すことが重要です。

StringBuilder sb = new StringBuilder("abc");
StringBuilder r = sb.reverse();
System.out.println(sb);     // cba
System.out.println(sb == r); // true
