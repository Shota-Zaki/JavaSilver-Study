数値・リテラル・型昇格

byte/short/charのint昇格、定数式、複合代入、リテラル表記を、試験で使える判定手順に落とします。

数値問題の判定順

  1. リテラルの型を確認する。整数は基本 int、小数は基本 double。
  2. 変数を含む演算か、リテラルだけの定数式かを分ける。
  3. 演算後の型を確認する。byte/short/char は原則 int へ昇格。
  4. 代入先へ暗黙変換できるか確認する。

リテラルの基本型

int a = 10;       // 10はint
long b = 10;      // intからlongへ拡大変換
long c = 10L;     // 10Lはlong
float d = 1.0F;   // OK
// float e = 1.0; // NG: 1.0はdouble

L や F はソースコード上の型指定です。出力結果に L や F が付くわけではありません。

byte/short/charのint昇格

小さい整数型同士の演算は、基本的に int になります。値が小さく見えても、式の型が int なら小さい型へそのまま代入できません。

byte a = 1;
byte b = 2;
// byte c = a + b; // NG: a + b はint
byte d = 1 + 2;    // OK: リテラルだけの定数式で範囲内

char x = 'A';
// char y = x + 1; // NG: x + 1 はint
char z = (char)(x + 1); // OK

複合代入は暗黙キャストを含む

+=、-= などの複合代入は、内部的に代入先の型へのキャストを含みます。通常の代入とは結果が変わります。

byte b = 1;
// b = b + 1; // NG: b + 1 はint
b += 1;       // OK: (byte)(b + 1) 相当
複合代入が便利だから安全、という意味ではありません。オーバーフローしてもコンパイルは通る場合があります。

アンダースコアと進数表記

数値リテラルには _ を使えますが、先頭・末尾・小数点の隣・接尾辞の直前などには置けません。

int a = 123_456;   // OK
// int b = _123;   // NG
// int c = 123_;   // NG
// float d = 3_.14F; // NG
int e = 0b1010;    // 2進数
int f = 017;       // 8進数。10進で15
int g = 0x10;      // 16進数。10進で16
// int h = 089;    // NG: 8進数に8/9は使えない

ミニ演習

Q. short s = 10 + 20; は？

OKです。リテラルだけの定数式で、結果がshortの範囲内です。

Q. short a=10,b=20; short c=a+b; は？

コンパイルエラーです。a+b は int です。

定数式と変数式の違い

Javaコンパイラは、リテラルだけで計算できる定数式については範囲内か判断できます。一方、変数を含むと、たとえfinalでも文脈によって扱いが変わるため、まず「コンパイル時定数か」を見ます。

byte a = 10 + 20;       // OK
final int x = 10;
byte b = x + 20;        // OK: コンパイル時定数
int y = 10;
// byte c = y + 20;     // NG

オーバーフローはコンパイルエラーとは限らない

複合代入やキャストでは、値が範囲を超えてもコンパイルできる場合があります。試験では「コンパイルできるか」と「結果の値」を分けてください。

byte b = 127;
b += 1;
System.out.println(b); // -128
