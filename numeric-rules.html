数値・リテラル・型昇格

数値型の代入、演算、リテラル表記、複合代入の罠を、コンパイル判定のために整理します。

数値変換の基本

数値型の問題は、代入時の変換と演算時の変換を分けて考えます。代入では左辺の型に収まるか、演算ではオペランドがどの型に昇格するかを見ます。

拡大変換

int から long など、値域が広い型へは基本的に暗黙変換できます。

縮小変換

long から int など、値域が狭い型へは明示キャストが必要です。

整数演算

byte、short、char は演算時に原則 int になります。

小数演算

double が混ざると結果は double 寄りに広がります。小数リテラルも基本 double。

byte / short / char の演算

狭い整数型は、演算すると int になります。ここを覚えていないと、見た目では範囲内なのにコンパイルエラーになる問題を落とします。

byte a = 10;
byte b = 20;
// byte c = a + b;      // NG。a + b はint
byte c = (byte)(a + b); // OK

char ch = 'A';
int code = ch + 1;      // OK。charは演算でintへ

ただし、リテラルだけで構成された定数式なら例外的に狭い型へ入ることがあります。

byte x = 10 + 20;     // OK。定数式で範囲内
short y = 128 + 128;  // OK
// byte z = 100 + 100; // NG。byte範囲外

複合代入演算子の罠

+=、-= などの複合代入は、内部的にキャストを含むように扱われます。そのため、通常の代入ではエラーになるものが通ることがあります。

byte b = 10;
// b = b + 1; // NG。b + 1 はint
b += 1;       // OK。暗黙のキャスト相当が入る
複合代入を b = b + 1 と完全に同じと覚えるのは弱いです。意味は近いですが、型変換の扱いが違います。

charは文字でもあり数値でもある

char は16ビットの文字型ですが、数値的な扱いも受けます。'A' は文字リテラルで、"A" はStringです。ここを混同しないこと。

char c1 = 'A';
int n = c1;       // OK。65
char c2 = 65;     // OK。定数式でchar範囲内
// char c3 = -1;  // NG。charは負数不可
// String s = 'A'; // NG。charとStringは別物
