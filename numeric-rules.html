数値・リテラル・型昇格

整数リテラル、浮動小数点リテラル、アンダースコア、型昇格、複合代入をまとめます。

primitive型と参照型

Javaの型は大きくprimitive型と参照型に分かれます。primitive型は値そのものを持ち、参照型はオブジェクトへの参照を持ちます。この違いは、代入、比較、null、初期値、例外で何度も問われます。

primitive型

byte short int long float double char boolean。値そのものを保持します。null は代入できません。

参照型

クラス、配列、interface、recordなど。参照先がない状態を null で表せます。

ローカル変数

自動初期化されません。代入前に読むとコンパイルエラーです。

フィールド・配列要素

型ごとの初期値で初期化されます。参照型はnull、数値は0系、booleanはfalseです。
class Sample {
    int field;        // 0
    String text;      // null

    void test() {
        int local;
        // System.out.println(local); // コンパイルエラー
        System.out.println(field);    // OK
    }
}

整数リテラルと表記

整数リテラルは基本的にint型です。longとして扱いたい場合は末尾にLまたはlを付けます。小文字lは数字1と紛らわしいため、試験でも実務でも大文字Lで覚える方が安全です。

int a = 10;
long b = 10L;
int bin = 0b1010;
int oct = 012;    // 8進数。10進数では10
int hex = 0xA;    // 16進数。10進数では10

2進数

0b または 0B。使える数字は0と1だけ。

8進数

先頭が 0。08 は8が使えないのでコンパイルエラー。

16進数

0x または 0X。A〜Fも使えます。

アンダースコア

数字の間だけ。先頭、末尾、小数点や接尾辞の隣は不可。
int ok1 = 1_000_000;
int ok2 = 0b1010_0101;

// int ng1 = _100;   // 先頭は不可
// int ng2 = 100_;   // 末尾は不可
// float ng3 = 3_.14F; // 小数点の隣は不可
// long ng4 = 100_L;   // 接尾辞の直前は不可

型変換と数値昇格

byte、short、charは演算時にintへ昇格します。byte同士の足し算でも結果はintです。ここを落とすと、代入可否の問題で間違えます。

byte a = 10;
byte b = 20;
// byte c = a + b; // コンパイルエラー。a + b は int
byte d = (byte)(a + b); // OK

ただし、リテラルや定数式が代入先の範囲に収まる場合は、byteやshortへ代入できます。

byte a = 127;       // OK
// byte b = 128;    // コンパイルエラー
short s = 100 + 20; // OK。コンパイル時に120と分かる

複合代入は暗黙のキャストを含みます。<code>a += 1</code> は単純に <code>a = a + 1</code> と同じではありません。

byte a = 10;
a += 1;      // OK。内部的に(byte)(a + 1)に近い
// a = a + 1; // コンパイルエラー。a + 1 は int

var

varは型ではなく、ローカル変数の型推論に使う予約された型名です。初期値から型を決めるため、初期値がない場所や型が決められない場所では使えません。

使える場所

ローカル変数。初期値が必要。

使えない場所

フィールド、メソッド引数、戻り値型、コンストラクタ引数、catchパラメータ、配列要素型。

null単独

var a = null; は型が決められないため不可。

ラムダ単独

var f = () -> {} はターゲット型がないため不可。
var name = "Java"; // String
var list = new java.util.ArrayList<String>();

// var x;          // 初期値なしは不可
// var y = null;   // 型を決められない
// private var n;  // フィールドでは不可
