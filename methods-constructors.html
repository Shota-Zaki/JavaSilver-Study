メソッド・コンストラクタ

戻り値、引数、オーバーロード、this/superをまとめて確認。

メソッド宣言

項目ルール注意
戻り値void値を返さないreturn; は可。return 値; は不可。voidメソッド呼び出しを変数に代入できない。
戻り値ありすべての経路で値を返すifだけreturnしてelseなしはコンパイルエラーになり得る。
引数呼び出し時に個数・型が合う必要型変換可能なら候補になる。
可変長引数型... name引数リストの最後だけ。1メソッドに1つだけ。
throws呼び出し元に例外処理を要求チェック例外で重要。

オーバーロード

判定材料使う使わない
メソッド名同じ名前-
引数リスト個数・型・順序が違えば別メソッド引数名は関係ない。
戻り値型判定に使わない戻り値だけ違う同名同引数は不可。
アクセス修飾子判定に使わないprivate/publicだけ違うのは不可。
選択タイミングコンパイル時宣言型と引数のコンパイル時型で決まる。

オーバーロード解決

優先例注意
完全一致test(int)最優先。
拡大変換int → long → float → double小さい整数型はintへ上がりやすい。
ボクシングint → Integer拡大変換より後に見られる。
可変長引数int...最後の候補。
曖昧test(long,float) と test(float,long)どちらも同程度ならコンパイルエラー。

コンストラクタ

項目ルール注意
名前クラス名と同じ、戻り値なしvoid Sample() はコンストラクタではなくメソッド。
暗黙super()明示しなければ先頭に入る親に引数なしコンストラクタがないとエラー。
this() / super()コンストラクタの先頭1文だけ両方同時には書けない。
アクセス修飾子public/private等を付けられるprivateコンストラクタは外部new不可。

例題

class A {
    A(int n) {}
}
class B extends A {
    B() { } // コンパイルエラー。暗黙 super() が呼ばれるが A() がない
}

class C {
    C() { this(10); }
    C(int n) { }
}
