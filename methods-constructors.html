メソッド・コンストラクタ

メソッド定義、オーバーロード、可変長引数、コンストラクタ、this/super、引数渡しを整理します。

メソッドシグネチャ

Javaでメソッドを区別する基準は、メソッド名と引数リストです。戻り値型だけでは区別できません。

void test(int a) {}
        void test(String s) {} // OK。引数が違う
        // int test(int a) { return a; } // NG。戻り値だけ違っても不可

引数名はシグネチャに含まれません。test(int a) と test(int b) は同じです。

オーバーロード解決

オーバーロードでは、コンパイル時に最も適合するメソッドが選ばれます。基本順は、完全一致、拡大変換、ボクシング、可変長引数です。

void call(int x) { System.out.println("int"); }
        void call(long x) { System.out.println("long"); }
        void call(Integer x) { System.out.println("Integer"); }

        call(10); // int

候補が曖昧になるとコンパイルエラーです。null を渡す問題では、参照型の継承関係を見ます。

void test(String s) {}
        void test(StringBuilder sb) {}
        // test(null); // NG。どちらにも合い、優劣がない

可変長引数 varargs

可変長引数は、メソッド定義の最後の引数に1つだけ使えます。呼び出し側から見ると、0個以上の引数または配列を渡せます。

void print(String prefix, int... nums) {}

        print("A");
        print("A", 1, 2, 3);
        print("A", new int[]{1, 2});

次の形は不可です。

// void ng(int... a, String b) {} // NG。最後ではない
        // void ng(int... a, String... b) {} // NG。可変長引数は1つだけ

コンストラクタの基本

コンストラクタはクラス名と同じ名前で、戻り値型を書きません。戻り値型を書いたら普通のメソッドです。

class Item {
            Item() {}          // コンストラクタ
            void Item() {}     // メソッド。コンストラクタではない
        }

コンストラクタを1つも定義しなければ、引数なしのデフォルトコンストラクタが暗黙に用意されます。1つでも定義したら、デフォルトコンストラクタは自動生成されません。

this() と super()

コンストラクタの先頭では、同じクラスの別コンストラクタを呼ぶ this(...)、またはスーパークラスのコンストラクタを呼ぶ super(...) を書けます。どちらも同時には書けません。

class A {
            A(int x) {}
        }
        class B extends A {
            B() {
                super(10); // 先頭文
            }
        }

明示しなければ super() が暗黙に挿入されます。スーパークラスに引数なしコンストラクタがない場合、サブクラス側で明示的に super(引数) を呼ばないとコンパイルエラーです。

引数渡しは値渡し

Javaの引数渡しは常に値渡しです。参照型の場合、参照値のコピーが渡されます。呼び出し先でオブジェクトの中身を変えれば呼び出し元にも見えますが、引数変数に別オブジェクトを代入しても呼び出し元の変数は変わりません。

static void change(StringBuilder sb) {
            sb.append("B");       // 参照先を変更
            sb = new StringBuilder("C"); // ローカル変数の向きだけ変更
        }

        StringBuilder x = new StringBuilder("A");
        change(x);
        System.out.println(x); // AB

ミニ判定

Q1. 戻り値型だけ違う同名同引数メソッドはオーバーロード？

違います。コンパイルエラーです。戻り値型だけではメソッドを区別できません。

Q2. コンストラクタに void を書いたら？

コンストラクタではなくメソッドになります。

Q3. this() と super() は同じコンストラクタ内に両方書ける？

書けません。どちらもコンストラクタの先頭文でなければならないためです。
