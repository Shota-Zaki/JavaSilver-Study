メソッド・コンストラクタ

戻り値、引数、可変長引数、オーバーロード、コンストラクタ、this/superをまとめます。

メソッドの戻り値

戻り値型がvoidなら値を返しません。戻り値を変数へ代入することもできません。戻り値型がvoid以外なら、すべての実行経路で値を返す必要があります。

void print() {
    System.out.println("A");
}

int get() {
    return 10;
}

// int x = print(); // コンパイルエラー
int y = get();      // OK

<code>return;</code> はvoidメソッドで使えます。値を返すメソッドでは、戻り値型に代入可能な値を返す必要があります。

引数と可変長引数

引数は呼び出し時に個数と型が合う必要があります。可変長引数は配列として扱われますが、宣言できるのは引数リストの最後だけです。

void test(String... values) {
    System.out.println(values.length);
}

test();
test("A");
test("A", "B");
// void ng(String... values, int n) {} // NG。可変長引数は最後だけ
void ok(int n, String... values) {}    // OK

オーバーロード

オーバーロードは、同じ名前で引数リストが違うメソッドを複数定義することです。戻り値型やアクセス修飾子だけを変えてもオーバーロードにはなりません。

void test(int a) {}
void test(String a) {}
void test(int a, int b) {}

// int test(int a) { return a; } // 戻り値だけ違うのでNG

呼び出すメソッドはコンパイル時に、引数の型から選ばれます。完全一致、拡大変換、ボクシング、可変長引数などの優先順位があります。曖昧になるとコンパイルエラーです。

void test(long n) { System.out.println("long"); }
void test(Integer n) { System.out.println("Integer"); }

test(10); // long。intからlongへの拡大変換が使われる

コンストラクタ

コンストラクタはクラス名と同じ名前で、戻り値型を書きません。戻り値型を書いた時点で、それはコンストラクタではなく通常メソッドです。

class Sample {
    Sample() { }

    void Sample() { } // コンストラクタではなく、戻り値voidのメソッド
}

コンストラクタの先頭では、同じクラスの別コンストラクタを呼ぶ <code>this(...)</code> か、親クラスのコンストラクタを呼ぶ <code>super(...)</code> を使えます。書くなら必ず最初の文です。同時には書けません。

class A {
    A(int n) {}
}
class B extends A {
    B() {
        super(10); // 先頭文
    }
}
