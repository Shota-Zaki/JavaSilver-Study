メソッド・コンストラクタ

シグネチャ、オーバーロード、可変長引数、コンストラクタ、this/super、引数渡しを整理します。

メソッドシグネチャ

メソッドの識別に使われるのは、メソッド名と引数リストです。戻り値だけが違うメソッドはオーバーロードになりません。

class Sample {
    void test(int n) {}
    void test(String s) {}  // OK。引数が違う

    // int test(int n) { return n; } // NG。戻り値だけ違う
}

引数の数

違えばオーバーロード可能です。

引数の型

違えばオーバーロード可能です。

引数の順番

(int, String) と (String, int) は別シグネチャです。

戻り値

シグネチャに含まれません。戻り値だけで区別できません。

オーバーロード解決

呼び出すメソッドはコンパイル時に決まります。完全一致、拡大変換、ボクシング、可変長引数などの候補から、より具体的なものが選ばれます。

void test(int n) { System.out.println("int"); }
void test(long n) { System.out.println("long"); }
void test(Integer n) { System.out.println("Integer"); }

short s = 10;
test(s); // int。shortはintへ拡大される
オーバーロードは「実体クラスで動的に選ばれる」わけではありません。それはオーバーライドの話です。

可変長引数

可変長引数は配列として受け取られます。1つのメソッドに書ける可変長引数は1つだけで、必ず最後の引数に置きます。

void print(String label, int... nums) {
    System.out.println(nums.length);
}

print("A");        // 0
print("B", 1, 2);  // 2
print("C", new int[]{1, 2, 3}); // 3

コンストラクタ

コンストラクタはインスタンス生成時の初期化処理です。戻り値型を書きません。戻り値型を書いたら、それはコンストラクタではなくメソッドです。

class Item {
    String name;

    Item(String name) {      // コンストラクタ
        this.name = name;
    }

    void Item() {            // メソッド。コンストラクタではない
    }
}

this(...) は同じクラスの別コンストラクタ呼び出し、super(...) は親クラスのコンストラクタ呼び出しです。どちらも書くならコンストラクタの先頭文でなければなりません。

class A {
    A(int n) {}
}
class B extends A {
    B() {
        super(10); // 先頭文
    }
}

引数渡し

Javaは常に値渡しです。ただし、参照型変数の値は「参照」です。メソッドにオブジェクト参照のコピーが渡るため、参照先オブジェクトの状態変更は呼び出し元にも見えますが、仮引数への再代入は呼び出し元の変数を変えません。

class Box { int value; }

static void change(Box b) {
    b.value = 10;      // 参照先の状態変更
    b = new Box();     // 仮引数の参照を付け替えるだけ
    b.value = 20;
}

Box box = new Box();
change(box);
System.out.println(box.value); // 10
