メソッド・コンストラクタ

オーバーロード、戻り値、varargs、this/super、コンストラクタ連鎖、引数渡しを整理します。

メソッドシグネチャ

Javaでオーバーロード判定に使うシグネチャは、メソッド名と引数リストです。戻り値型だけが違うメソッドは同時に定義できません。

void test(int a) {}
void test(String s) {}   // OK: 引数が違う
// int test(int a) { return a; } // NG: 戻り値だけ違う

オーバーロード解決

呼び出しに最も適合するメソッドが選ばれます。完全一致、拡大変換、ボクシング、可変長引数の順に考えると整理しやすいです。

void test(long x) { System.out.print("long"); }
void test(Integer x) { System.out.print("Integer"); }
void test(int... x) { System.out.print("varargs"); }

test(10); // long: intからlongへの拡大が優先されやすい

varargsのルール

可変長引数はメソッドの最後の引数に1つだけ書けます。内部的には配列として扱われます。

void test(String label, int... values) { }
// void bad(int... values, String label) { } // NG
// void bad2(int... a, String... b) { }      // NG

コンストラクタの基本

コンストラクタはクラス名と同じ名前で、戻り値型を書きません。戻り値型を書いたら普通のメソッドです。

class Sample {
    Sample() { }        // コンストラクタ
    void Sample() { }   // メソッド。コンストラクタではない
}

this()とsuper()

コンストラクタ内で別のコンストラクタを呼ぶ this()、親クラスのコンストラクタを呼ぶ super() は、コンストラクタの先頭行にしか書けません。同時には使えません。

class A {
    A(int n) { }
}
class B extends A {
    B() {
        this(10);
    }
    B(int n) {
        super(n);
    }
}

引数渡し

Javaは常に値渡しです。参照型の場合も、参照値がコピーされます。オブジェクトの中身を変えることと、仮引数を差し替えることを分けてください。

static void change(StringBuilder sb) {
    sb.append("X");
    sb = new StringBuilder("Y");
}

StringBuilder s = new StringBuilder("A");
change(s);
System.out.println(s); // AX

ミニ演習

Q. コンストラクタは継承される？

継承されません。サブクラスのコンストラクタから super(...) で呼ぶことはありますが、コンストラクタそのものを引き継ぐわけではありません。

Q. 戻り値型だけ違う同名メソッドはオーバーロード？

違います。コンパイルエラーです。

オーバーロードでnullを渡す場合

null は参照型に代入できます。複数の参照型オーバーロードがあり、どちらがより具体的か決められない場合はコンパイルエラーになります。

void test(String s) {}
void test(StringBuilder sb) {}
// test(null); // NG: どちらとも決められない

void sample(Object o) {}
void sample(String s) {}
sample(null); // Stringがより具体的

コンストラクタの暗黙super()

コンストラクタの先頭に this() も super() も書かない場合、コンパイラが super() を挿入します。親に無引数コンストラクタがなければコンパイルエラーです。

class A { A(String s) {} }
class B extends A {
    // B() { super(); } が暗黙に必要になるが、A()がないのでNG
}
