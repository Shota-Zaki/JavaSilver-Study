Java Silver Study

Data Types

データ型・リテラル・型変換

Java Silverでは「代入できるか」「演算後の型は何か」「実行時に落ちるか」を分けて判定します。

プリミティブ型

型サイズ範囲/性質注意
byte8bit-128〜127整数演算ではintへ昇格しやすい
short16bit-32768〜32767リテラル代入と演算結果代入を分ける
int32bit約±21億整数リテラルの基本型
long64bit大きい整数L/l接尾辞。小文字lは1と紛らわしい
float32bit単精度F/f接尾辞が必要になりやすい
double64bit倍精度浮動小数点リテラルの基本型
char16bit0〜65535文字だが数値として演算される
booleantrue/false2値数値と互換性なし

初期値の違い

場所初期化例
フィールド/配列要素自動初期化されるintは0、booleanはfalse、参照型はnull
ローカル変数自動初期化されない使う前に必ず代入が必要
finalローカル変数一度だけ代入可代入前使用はコンパイルエラー

リテラルと接尾辞

書き方意味注意
10int整数リテラルの基本。
10Llonglongを明示。
3.14doublefloatにはそのまま代入不可。
3.14Ffloatfloatリテラル。
0b10102進数b/B可。
0108進数10進数の10ではなく8。
0x1016進数16。
'A'char文字1つ。Stringではない。
"A"String参照型。charではない。

暗黙変換・明示キャスト

byte b = 10;      // リテラルが範囲内なので可
int i = b;        // 小さい型から大きい型へ
byte x = (byte)i; // 大きい型から小さい型へはキャスト
ケース結果
byte + byte結果はint。byteへ戻すならキャストが必要。
int + longintがlongへ昇格し、結果はlong。
float + double結果はdouble。
char + intcharが数値として扱われ、結果はint。
booleanと数値相互変換不可。

var の使用可否

場所可否例
ローカル変数可var s = "abc";
初期化なし不可var x;
null単独不可var x = null;
フィールド不可private var value;
メソッド引数不可void test(var x)
戻り値型不可var test()
ラムダ式だけ不可var f = () -> {}; は対象型がない
配列初期化子だけ不可var a = {1,2,3};
