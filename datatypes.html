型・変数・リテラル

プリミティブ型、参照型、リテラル、初期化、型変換、varを、コンパイル可否の判断に使える形で整理します。

プリミティブ型と参照型

Javaの型は大きくプリミティブ型と参照型に分かれます。プリミティブ型の変数は値そのものを持ち、参照型の変数はオブジェクトへの参照を持ちます。この違いは、代入、比較、null、メソッド呼び出しのすべてに効きます。

整数型

byte、short、int、long。整数リテラルは基本 int、末尾 L で long。

浮動小数点型

float、double。小数リテラルは基本 double、末尾 F で float。

boolean

true / false のみ。数値の0/1とは互換性がありません。

char

16ビットのUnicode文字。文字リテラルは単一引用符、文字列は二重引用符です。

boolean と数値型を混ぜられる言語の感覚をJavaに持ち込むと失敗します。if (1) も boolean b = 1; も不可です。

リテラルの型と代入

整数リテラルは基本的に int ですが、定数式で値が範囲内なら byte や short へ代入できます。変数を含む式になると、もう単なる定数ではないため扱いが変わります。

byte a = 127;        // OK
// byte b = 128;     // NG。byteの範囲外
short c = 128 + 128; // OK。定数式でshort範囲内

byte x = 10;
byte y = 20;
// byte z = x + y;   // NG。演算結果はint

小数リテラルは基本 double です。float f = 1.0; は不可で、1.0F またはキャストが必要です。

表記: 2進数・8進数・16進数・アンダースコア

数値リテラルは表記が違っても、値として扱われます。0b は2進数、先頭 0 は8進数、0x は16進数です。

int a = 267;        // 10進数
int b = 0413;       // 8進数
int c = 0x10B;      // 16進数
int d = 0b100001011;// 2進数
// int e = 0827;    // NG。8進数に8は使えない

アンダースコアは桁区切りに使えますが、先頭・末尾・小数点や接頭辞の直後などには置けません。

int ok1 = 123_456;
int ok2 = 0b10_10;
// int ng1 = _123;
// int ng2 = 123_;
// float ng3 = 3_.14F;
// int ng4 = 0x_52;

ローカル変数・フィールド・初期値

フィールドにはデフォルト値がありますが、ローカル変数にはありません。ローカル変数は、使用前に確実に初期化されている必要があります。

class Sample {
    int field;       // 0で初期化される
    String name;     // nullで初期化される

    void test() {
        int local;
        // System.out.println(local); // NG。初期化前
    }
}

フィールド

インスタンス生成時にデフォルト値が入ります。数値は0、booleanはfalse、参照型はnull。

ローカル変数

メソッド内の変数。デフォルト値はなく、使用前に明示的な初期化が必要です。

ブロックスコープ

{} 内で宣言した変数は、そのブロックの外では使えません。

同名変数

近いスコープの変数が優先されます。フィールドに明示的に触るなら this.field。

varの正しい位置

var は型推論であって、型がなくなるわけではありません。コンパイル時に右辺から型を決めるだけです。使える場所はローカル変数に限定されます。

var s = "abc";       // String
var list = new java.util.ArrayList<String>();

// var a;           // NG。初期化式がない
// var b = null;    // NG。型を決められない
// private var x;   // NG。フィールドには使えない
// void test(var x) {} // NG。引数には使えない
var を「何でも入る型」と考えるのは誤りです。var n = 10; の n は int です。後から "abc" は入りません。
