型・変数・リテラル

プリミティブ型、参照型、初期値、数値リテラル、varを、Java Silverで落とされる境界条件まで整理します。

型の大分類

Javaの型は、プリミティブ型と参照型に分かれます。プリミティブ型の変数は値そのもの、参照型の変数はオブジェクトへの参照を保持します。この違いは null、==、代入、メソッド呼び出し、ガベージコレクションの判断に直結します。

分類例変数が持つものnull
プリミティブ型int, double, boolean, char値そのもの不可
参照型String, 配列, ArrayList, 自作クラスオブジェクトへの参照可
String s = null; は可能ですが、int n = null; は不可です。Integer n = null; は参照型なので可能です。

プリミティブ型の範囲と試験で必要な覚え方

すべての範囲を丸暗記する必要は薄いですが、byte と short、整数リテラルのデフォルト型、浮動小数点リテラルのデフォルト型は必須です。

型サイズ要点
byte8ビット-128〜127。byte b = 128; は不可。
short16ビット-32768〜32767。定数式なら範囲内代入可。
char16ビットUnicode文字。数値型として扱われる場面もある。
int32ビット整数リテラルの基本型。
long64ビットリテラル末尾に L。
float32ビットリテラル末尾に F。
double64ビット小数リテラルの基本型。
boolean規定なしtrue/false のみ。0/1と互換性なし。

リテラルのデフォルト型

整数リテラルは基本 int、小数リテラルは基本 double です。ここを外すと、型変換問題の大半で誤答します。

long a = 10;       // OK。intリテラル10をlongへ拡大変換
        long b = 10L;      // OK。longリテラル
        float c = 1.0F;    // OK
        // float d = 1.0;  // NG。1.0はdouble
        // int e = 10L;    // NG。longからintへ暗黙変換不可

L や F はリテラルの型を表す記号です。出力時に 10L のように表示されるわけではありません。

初期値: フィールドとローカル変数

フィールドにはデフォルト値があります。ローカル変数にはありません。宣言されたかではなく、使用前に確実に代入されたかを見ます。

class Sample {
            int field;       // 0
            boolean flag;    // false
            String text;     // null

            void test(boolean condition) {
                int local;
                if (condition) {
                    local = 10;
                }
                // System.out.println(local); // NG。condition=falseなら未初期化
            }
        }

分岐がある場合は、すべての経路で初期化されるか確認してください。

識別子と予約語

変数名・クラス名・メソッド名などを識別子と呼びます。英字、数字、_、$ を使えますが、数字から始めることはできません。予約語は使えません。

int $a = 100;   // OK。ただし実務では非推奨
        int _a = 200;   // OK
        // int 1a = 300; // NG。数字始まり
        // int a.b = 400;// NG。ドットは不可
        // int class = 5;// NG。予約語
_ 単独の識別子は使えません。_a のように他の文字と組み合わせるのとは別です。

varは型推論であって動的型ではない

var はローカル変数の型を右辺から推論する仕組みです。型がなくなるわけではなく、コンパイル時点で確定します。

var n = 10;      // int
        var s = "abc";   // String
        // n = "abc";    // NG。nはint

使えない場所も頻出です。

場所var使用理由
ローカル変数可右辺から型を推論できる
フィールド不可ローカル変数ではない
メソッド引数不可右辺がない
戻り値型不可戻り値型を推論する仕組みではない
初期化式なし不可型を決められない
null 初期化不可nullだけでは型を決められない

ミニ判定

Q1. boolean b = 1; は？

コンパイルエラーです。Javaの boolean は数値と互換性がありません。

Q2. long x = 10L; System.out.println(x); の表示は？

10 です。L はソースコード上のリテラル接尾辞で、値の表示には含まれません。

Q3. var list = new ArrayList<>(); の型は？

コンテキストがなければ ArrayList<Object> 相当として扱われます。何でも安全に入れられるという意味ではありません。
