型・変数・リテラル

プリミティブ型、参照型、初期値、識別子、nullを、代入可否の観点で詳しく整理します。

プリミティブ型と参照型

プリミティブ型の変数は値そのものを保持します。参照型の変数はオブジェクトへの参照を保持します。null、==、メソッド呼び出し、ガベージコレクションの判断はこの違いから始まります。

分類例変数が持つものnull
プリミティブ型int, double, boolean, char値そのもの不可
参照型String, 配列, 自作クラス, ラッパークラスオブジェクトへの参照可

基本型の範囲で覚えるべきもの

型試験での要点
byte-128〜127。byte b = 128; は不可。
short-32768〜32767。範囲内の定数式なら代入できる。
char0〜65535相当のUnicode文字。演算に入るとintへ昇格。
int整数リテラルの基本型。
long末尾に L を付けるとlongリテラル。
float小数を直接入れるなら F が必要。
double小数リテラルの基本型。
booleantrue/false のみ。数値と互換性なし。
Javaの boolean は数値ではありません。if (1)、boolean b = 0 はコンパイルエラーです。

初期値: フィールドとローカル変数

フィールドにはデフォルト値があります。ローカル変数にはありません。問題では「宣言しただけのローカル変数」を使わせるひっかけが多いです。

class Sample {
    int field;       // 0
    boolean flag;    // false
    String text;     // null

    void test(boolean condition) {
        int local;
        if (condition) {
            local = 10;
        }
        System.out.println(local); // NG: false経路で未初期化
    }
}

nullと空文字は別

null は参照先がない状態です。空文字 "" は長さ0のStringオブジェクトです。

String a = null;
String b = "";
System.out.println(a); // null
System.out.println(b); // 空文字なので見た目は何も出ない
// a.length();         // NullPointerException
System.out.println(b.length()); // 0

識別子のルール

識別子は英字、数字、_、$ を使えますが、数字始まりは不可です。予約語も使えません。_ 単独の識別子も不可です。

int $a = 100;    // OK。ただし実務では避ける
int _a = 200;    // OK
int a1 = 300;    // OK
// int 1a = 400; // NG
// int class = 5;// NG
// int _ = 1;    // NG

ミニ演習

Q. Object obj = null; System.out.println(obj); の表示は？

null と表示されます。obj.toString() を呼ぶと NullPointerException です。

Q. char c = 65; は可能？

可能です。char の範囲内の整数定数は代入できます。ただし変数を含む演算結果はintになりやすい点に注意します。

関連: 判定フロー / コンパイル・実行時例外 / ひっかけ集

代入可否を決める4パターン

パターン例判定
拡大変換long x = 10;OK
縮小変換int x = 10L;NG。明示キャストが必要
範囲内定数代入byte b = 127;OK
変数を含む演算byte c = b + 1;NG。式の型がint

「値が小さいから入る」ではなく、「式の型が何か」で判定します。

ラッパークラスとnull

ラッパークラスは参照型なので null を代入できます。ただし、アンボクシングが発生する場所でnullだと NullPointerException になります。

Integer n = null; // OK
// int x = n;     // 実行時にNullPointerException
