データ型・リテラル・型変換

型の範囲、リテラル表記、暗黙変換を出題目線で整理します。

primitive型の範囲と初期値

型サイズ/値域の目安デフォルト値注意
booleantrue/falsefalse数値との互換性なし。0/1は代入不可。
byte8bit -128〜1270リテラルが範囲内なら代入可能。演算するとint。
short16bit0演算結果はintになりやすい。
char16bit Unicode\u0000数値として演算に参加するとintへ昇格。
int32bit0整数リテラルの基本型。
long64bit0LリテラルにはL推奨。小文字lは読みにくい。
float32bit0.0f小数リテラルにはFが必要。
double64bit0.0d小数リテラルの基本型。

ローカル変数とフィールドの初期値

class Sample {
  int field; // 0で初期化される
  void test() {
    int local;
    // System.out.println(local); // コンパイルエラー
  }
}

フィールドはデフォルト初期化されます。ローカル変数は明示的に初期化するまで使えません。

整数リテラル

int a = 267;       // 10進数
int b = 0413;      // 8進数
int c = 0x10B;     // 16進数
int d = 0b100001011; // 2進数
// int e = 0827;   // 8進数として不正

先頭0は8進数です。8や9を含むとコンパイルエラーになります。

アンダースコア

int a = 123_456;
int b = 5_____2;
// int c = _123;
// int d = 123_;
// float e = 3_.14F;

アンダースコアは数字と数字の間だけに置けます。

型変換と昇格

byte a = 10;
byte b = 20;
// byte c = a + b; // intになるためコンパイルエラー
byte c = (byte)(a + b);

short s = 10;      // 範囲内のintリテラルなのでOK
// short t = 40000; // 範囲外

byte/short/charは演算時にintへ昇格します。複合代入は暗黙キャストを含む点も頻出です。

varの使い方

var x = 10;          // int
var s = "java";      // String
// var n = null;      // 型推論不可
// private var field; // フィールド不可

varは型を省略する構文であり、動的型付けではありません。コンパイル時に型は確定します。
