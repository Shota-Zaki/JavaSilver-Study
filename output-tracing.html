出力追跡トレーニング

値・参照・例外の流れを表で追い、出力問題の計算ミスを減らすためのページです。

出力問題は「表」で追う

頭の中だけで追うと、後置インクリメント、参照の上書き、catch/finallyで崩れます。試験では紙に小さい表を書く前提で練習してください。

問題型表に書くもの最初に見ること
++/--式に渡す値 / 変数の現在値前置か後置か
短絡評価左辺結果 / 右辺を評価したか&& と || の停止条件
参照変数名 / 参照先ID / フィールド値a = b で参照がコピーされる点
継承宣言型 / 実体 / 呼ぶメンバメソッドかフィールドか
例外実行行 / 例外 / catch / finally例外発生後のtry内は飛ぶ

++/--：式に渡す値と現在値を分ける

int a = 10;
int b = a++ + a + --a;
System.out.println(a + "," + b);
部分式式に渡す値評価後のa理由
a++1011後置なので元の値を渡してから増える
a1111現在値を読む
--a1010前置なので減ってから渡す

b は 10 + 11 + 10 = 31、出力は 10,31。

短絡評価：右辺が実行されたかを見る

int a = 10;
int b = 10;
if (a < 5 && ++b > 10) {
    a++;
}
System.out.println(a + "," + b);

&& は左辺がfalseなら右辺を評価しません。a < 5 はfalseなので ++b は実行されず、出力は 10,10。

危険：& と &&、| と || は違います。boolean演算で &/| を使うと右辺も評価されます。

参照：変数にはインスタンスそのものではなく参照が入る

class Item { String name; }

Item a = new Item(); // #1
a.name = "A";
Item b = new Item(); // #2
b.name = "B";
a = b;
a.name = "C";
System.out.println(b.name);
  1. a は最初 #1、b は #2 を参照。
  2. a = b; で a も #2 を参照。
  3. a.name = "C"; は #2 のnameを書き換える。
  4. b も #2 を見ているため、出力は C。

継承：メソッドは実体、フィールドは宣言型

class A {
    String name = "A";
    String getName() { return name; }
}
class B extends A {
    String name = "B";
    String getName() { return name; }
}
A obj = new B();
System.out.println(obj.name + "," + obj.getName());

obj.name はフィールドなので宣言型 A 側。obj.getName() はオーバーライドメソッドなので実体 B 側。出力は A,B。

例外：止まる行を確定する

try {
    System.out.println("A");
    String s = null;
    System.out.println(s.length());
    System.out.println("B");
} catch (RuntimeException e) {
    System.out.println("C");
} finally {
    System.out.println("D");
}

A を出力後、s.length() で NullPointerException。try内の B は実行されません。対応するcatchで C、最後にfinallyで D。出力は A C D の順です。
