出力追跡トレーニング

出力問題を感覚で解かず、変数表・参照表・例外表で追うための練習ページです。

出力追跡の原則

出力問題は、頭の中だけで追うほどミスが増えます。特に ++/--、参照代入、StringBuilder、例外は表にしてください。

問題タイプ書く表
数値演算行番号 / 式 / 変数の値 / 出力
参照型変数 / 参照先 / オブジェクトの中身
例外実行行 / 例外発生有無 / catch/finally / 出力
継承宣言型 / 実体 / 呼ばれるメソッド

++/--の追跡例

int a = 10;
int b = a++ + ++a;
System.out.println(a + "," + b);
時点a式で使う値説明
開始10-初期値
a++1110現在値10を使ってから増える
++a1212増やしてから使う
代入後1222b=10+12

出力は 12,22 です。

参照の追跡例

class Item { String name; }
Item a = new Item();
Item b = new Item();
a.name = "A";
b.name = "B";
a = b;
a.name = "C";
System.out.println(b.name);

a = b の後、aとbは同じインスタンスを参照します。したがって a.name="C" はbから見える同じインスタンスの値も変え、出力は C です。

例外の追跡例

try {
    System.out.print("A");
    int[] a = new int[0];
    a[0] = 1;
    System.out.print("B");
} catch (ArrayIndexOutOfBoundsException e) {
    System.out.print("C");
} finally {
    System.out.print("D");
}

A を表示後、配列アクセスで例外が発生します。B は表示されません。catchで C、finallyで D。出力は ACD です。

メソッド呼び出しの追跡

参照型の引数は、参照値がコピーされます。メソッド内でオブジェクトの中身を変えれば呼び出し元にも見えますが、仮引数そのものへ別インスタンスを代入しても呼び出し元の変数は変わりません。

static void test(StringBuilder sb) {
    sb.append("B");
    sb = new StringBuilder("C");
}

StringBuilder x = new StringBuilder("A");
test(x);
System.out.println(x); // AB

ミニ演習

Q. 例外が発生したtryブロックの残りは追う？

追いません。対応するcatch、finallyへ進みます。

Q. 参照型引数を受け取ったメソッド内で param = new X() すると呼び出し元の変数も変わる？

変わりません。参照値のコピーが差し替わるだけです。

参照表の書き方

参照型の問題では、変数名の横に「どのインスタンスを見ているか」を書きます。値を変えたのか、参照を付け替えたのかを分けます。

行ab説明
1Item#1-aが新規Itemを参照
2Item#1Item#2bが別Itemを参照
3Item#2Item#2a=bで同じ参照
4Item#2.name変更Item#2.name変更同じ実体なのでb側にも見える

例外表の書き方

try/catch/finallyでは、出力文字列と制御移動を分けて書きます。例外が発生した瞬間にtryの残りを消すのがコツです。

行出力制御
try開始-通常実行
例外前printlnA続行
例外発生-対応catchへ
catchCfinallyへ
finallyD終了
