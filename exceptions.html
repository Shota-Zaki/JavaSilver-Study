例外処理

try/catch/finally、チェック例外、非チェック例外、throw/throws、try-with-resourcesを整理します。

例外処理の基本構造

tryブロックで例外が発生すると、tryの残りは実行されず、対応するcatchへ移動します。finallyがあれば、例外の有無に関係なく最後に実行されます。

try {
    System.out.print("A");
    String s = null;
    s.length();
    System.out.print("B");
} catch (NullPointerException e) {
    System.out.print("C");
} finally {
    System.out.print("D");
}
// ACD

チェック例外と非チェック例外

チェック例外は、コンパイラが処理または宣言を要求します。非チェック例外は RuntimeException 系で、明示的な処理は必須ではありません。

分類例処理必須
チェック例外Exception, IOException必須
非チェック例外RuntimeException, NullPointerException, ArithmeticException必須ではない
ErrorOutOfMemoryError など通常はアプリで処理しない

catchの順序

catchは上から評価されます。サブクラス例外を先、スーパークラス例外を後に書きます。逆にすると到達不能でコンパイルエラーです。

try {
    throw new java.io.FileNotFoundException();
} catch (java.io.FileNotFoundException e) {
} catch (java.io.IOException e) {
} catch (Exception e) {
}

throwとthrows

throw は例外インスタンスを投げる文です。throws はメソッド宣言で、呼び出し元へ例外処理の責任を伝えます。

static void test() throws java.io.IOException {
    throw new java.io.IOException();
}

try-with-resources

AutoCloseable を実装したリソースは、tryブロック終了時に自動でcloseされます。複数ある場合、closeは宣言と逆順です。

class R implements AutoCloseable {
    private final String name;
    R(String name) { this.name = name; }
    public void close() { System.out.print(name); }
}

try (R a = new R("A"); R b = new R("B")) {
    System.out.print("T");
}
// TBA

finallyの注意

finallyは原則実行されますが、finally内でreturnや例外を発生させると、try/catch側の結果を上書きすることがあります。Silverでは、まず「finallyは通る」と判断できれば十分な問題が多いです。

ミニ演習

Q. catchの後にfinallyを書ける？

書けます。try-catch-finally の形です。

Q. finallyの後にcatchを書ける？

書けません。try-finally-catch の順序はコンパイルエラーです。

Q. RuntimeException はthrows宣言必須？

必須ではありません。非チェック例外です。

チェック例外のoverrideルール

オーバーライド時、チェック例外は親メソッドより広くできません。非チェック例外は制約が緩いです。

class A {
    void test() throws java.io.IOException {}
}
class B extends A {
    void test() throws java.io.FileNotFoundException {} // OK: より狭い
    // void test() throws Exception {} // NG: より広い
}

try-with-resourcesとfinallyの順序

try-with-resourcesでは、tryブロックを抜けるときにリソースのcloseが呼ばれ、その後にfinallyが実行されます。リソースが複数ならcloseは逆順です。

try (R a = new R("A"); R b = new R("B")) {
    System.out.print("T");
} finally {
    System.out.print("F");
}
// TBAF
