例外処理

チェック例外、catch順序、finally、try-with-resourcesを整理します。

例外の分類

分類例処理の必要
チェック例外IOException, Exceptioncatchまたはthrowsが必要。
非チェック例外RuntimeException, NullPointerExceptionコンパイル時の処理強制なし。
ErrorOutOfMemoryErrorなど通常はアプリで処理対象にしない。

try-catch-finally

try {
  System.out.println("A");
  int x = 10 / 0;
  System.out.println("B");
} catch (ArithmeticException e) {
  System.out.println("C");
} finally {
  System.out.println("D");
}
// A C D

例外発生後、try内の残りはスキップされます。finallyは原則実行されます。

catch順序

try {
  throw new NullPointerException();
} catch (RuntimeException e) {
  System.out.println("R");
} catch (Exception e) {
  System.out.println("E");
}

子から親へ、または狭い例外から広い例外へ並べます。親を先に書くと子catchが到達不能になります。

throwsとthrow

void read() throws IOException {
  throw new IOException();
}

void test() {
  try {
    read();
  } catch (IOException e) {
    System.out.println("handled");
  }
}

throw は例外を投げる文、throws はメソッド宣言で呼び出し元へ伝える指定です。

try-with-resources

class R implements AutoCloseable {
  String name;
  R(String name) { this.name = name; }
  public void close() { System.out.println(name); }
}
try (R a = new R("A"); R b = new R("B")) {
  System.out.println("try");
}
// try B A

リソースは宣言と逆順にcloseされます。
