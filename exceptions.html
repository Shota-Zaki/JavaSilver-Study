例外処理

チェック例外、非チェック例外、catch順序、finally、try-with-resourcesを整理します。

例外の分類

分類例処理必須か
チェック例外IOException, Exception原則としてcatchまたはthrowsが必要。
非チェック例外RuntimeException配下コンパイル時の処理必須ではない。
ErrorOutOfMemoryErrorなど通常のアプリで捕まえる対象ではない。

try-catch-finallyの順序

try {
    System.out.println("A");
} catch (RuntimeException e) {
    System.out.println("B");
} finally {
    System.out.println("C");
}

例外がなければtry→finally。例外がcatchされればtry→catch→finally。catchされなくてもfinallyは原則実行され、その後に例外が伝播します。

catch順序

try {
    throw new java.io.FileNotFoundException();
} catch (java.io.IOException e) {
} // catch (java.io.FileNotFoundException e) { } // NG: すでに上位で捕まる

サブクラス例外を先、スーパークラス例外を後に書きます。

try-with-resources

class R implements AutoCloseable {
    String name;
    R(String name){ this.name = name; }
    public void close(){ System.out.println(name); }
}
try (R a = new R("A"); R b = new R("B")) {
    System.out.println("try");
}
// try, B, A の順。closeは宣言と逆順。
