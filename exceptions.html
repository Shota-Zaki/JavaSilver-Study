例外処理

try-catch-finally、checked/unchecked、throw/throws、try-with-resourcesをまとめます。

例外の種類

例外は、コンパイル時に処理を強制されるchecked例外と、強制されないunchecked例外に分かれます。RuntimeExceptionとそのサブクラスはunchecked例外です。

checked例外

Exception のサブクラスだが RuntimeException ではないもの。例: IOException。

unchecked例外

RuntimeException とそのサブクラス。例: NullPointerException、NumberFormatException。

Error

通常アプリ側で処理する対象ではありません。

throws

メソッドが例外を外へ投げる可能性を宣言します。

try-catch-finally

try内で例外が発生すると、その後のtry内の文は実行されず、対応するcatchへ移ります。finallyは例外の有無にかかわらず基本的に実行されます。

try {
    System.out.println("A");
    int n = 10 / 0;
    System.out.println("B");
} catch (ArithmeticException e) {
    System.out.println("C");
} finally {
    System.out.println("D");
}

// A
// C
// D

catchは上から順に判定されます。親例外を先に書くと、子例外のcatchへ到達できなくなりコンパイルエラーです。

try {
} catch (Exception e) {
// } catch (IOException e) { // NG。すでにExceptionで捕まる
}

throw と throws

<code>throw</code> は実際に例外を投げる文です。<code>throws</code> はメソッド宣言に書き、呼び出し元へ例外処理を要求または通知します。

void read() throws IOException {
    throw new IOException();
}

try-with-resources

try-with-resourcesでは、tryの括弧内で宣言したリソースが自動でcloseされます。closeは宣言した順番とは逆順に呼ばれます。

try (A a = new A("A"); A b = new A("B")) {
    System.out.println("body");
}
// body
// B
// A

finallyを書く場合、リソースのcloseが呼ばれた後にfinallyが実行されます。
