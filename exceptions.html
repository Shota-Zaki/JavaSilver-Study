Exceptions

例外処理

例外は「コンパイルで処理を要求されるか」「実行時にどのcatchへ行くか」「finally/closeの順序」で解きます。

例外の分類

分類代表例処理
チェック例外Exception, IOExceptioncatchまたはthrowsが必要。
非チェック例外RuntimeException, NullPointerException, ArrayIndexOutOfBoundsExceptionコンパイラは処理を強制しない。
ErrorOutOfMemoryErrorなど通常アプリで捕捉対象にしない。

catch順序

try {
  sample();
} catch (SubException e) {
} catch (SuperException e) {
}
  • サブクラス例外を先に書く。
  • 親例外を先に書くと、後続の子例外catchは到達不能でコンパイルエラー。

finallyの順序

状況流れ
例外なしtry → finally
catchありtry → catch → finally
catchなしで外へ投げるtry → finally → 呼び出し元へ
returnありreturn値評価 → finally → return

try-with-resources

try (A a = new A("A"); A b = new A("B")) {
  System.out.println("try");
} finally {
  System.out.println("finally");
}
// close順は B → A → finally
  • AutoCloseableが必要。
  • リソースは宣言の逆順にclose。
  • closeはfinallyより前。

throw と throws

キーワード意味例
throw実際に例外を投げるthrow new IOException();
throwsメソッド宣言で例外可能性を示すvoid test() throws IOException
