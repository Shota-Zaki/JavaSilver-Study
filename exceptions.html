例外処理

try/catch/finally、チェック例外、非チェック例外、throw/throws、try-with-resources、multi-catchを整理します。

例外問題の判定順

  1. コンパイルできるか。チェック例外の処理漏れ、catch順、到達不能catchを確認。
  2. try内で例外が発生するか。発生した行より後のtry内処理は実行されない。
  3. どのcatchが最初に受け取るか。上から順に評価される。
  4. finallyがあるなら最後に実行される。
  5. try-with-resourcesなら、リソースのclose順を確認する。

try-catchの基本

tryブロック内で例外が発生すると、その時点でtry内の残りはスキップされ、対応するcatchへ移動します。

try {
            int[] a = {};
            a[0] = 10;
            System.out.println("finish");
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("error");
        }
        // error

finish は表示されません。例外発生行の後は実行されないためです。

finallyの実行

finally は、例外の有無にかかわらず実行されます。catchが実行された後にも実行されます。

try {
            System.out.println("A");
        } catch (RuntimeException e) {
            System.out.println("B");
        } finally {
            System.out.println("C");
        }
        // A C

ただし、System.exit など極端なケースは例外です。Java Silverでは通常、finallyは通ると考えて処理順を追います。

catchの順番

複数のcatchは上から順に評価されます。親クラスの例外を先にcatchすると、子クラスのcatchは到達不能になりコンパイルエラーです。

try {
            throw new java.io.FileNotFoundException();
        } catch (java.io.FileNotFoundException e) {
            System.out.println("file");
        } catch (java.io.IOException e) {
            System.out.println("io");
        }

逆にすると FileNotFoundException のcatchが到達不能です。

チェック例外と非チェック例外

チェック例外は、catchするかthrowsで宣言する必要があります。非チェック例外は強制されません。

分類代表例処理義務
チェック例外Exception のうち RuntimeException 系以外、IOExceptionあり
非チェック例外RuntimeException、NullPointerException、IllegalArgumentExceptionなし
ErrorOutOfMemoryError など通常処理対象外
Exception はチェック例外ですが、RuntimeException は非チェック例外です。名前だけで雑に判断しないでください。

throwとthrows

throw は例外インスタンスを投げる文、throws はメソッド宣言で「この例外を投げる可能性がある」と示す句です。

static void read() throws java.io.IOException {
            throw new java.io.IOException();
        }

チェック例外を throw するなら、その場でcatchするか、メソッドに throws を付ける必要があります。

try-with-resources

try-with-resources では、AutoCloseable なリソースがtryを抜けるときに自動でcloseされます。close順は宣言の逆順です。

try (A a = new A("A"); A b = new A("B")) {
            System.out.println("body");
        } finally {
            System.out.println("finally");
        }
        // body
        // B
        // A
        // finally

closeはfinallyより前に呼ばれます。ここは出力順問題で狙われます。

multi-catch

multi-catchでは、複数の例外型を | で並べます。ただし、親子関係にある例外を同時に並べることはできません。

try {
            // ...
        } catch (java.io.IOException | IllegalArgumentException e) {
            System.out.println(e.getMessage());
        }
// catch (Exception | java.io.IOException e) {} // NG。親子関係がある

ミニ判定

Q1. tryで例外が発生した後、try内の次の行は実行される？

実行されません。対応するcatchへ移動します。

Q2. catch(Exception) の後に catch(IOException) は？

コンパイルエラーです。IOException はすでに Exception で捕まるため到達不能です。

Q3. try-with-resourcesのclose順は？

宣言と逆順です。A a; B b; なら b.close()、a.close() の順です。
