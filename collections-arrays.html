配列・ArrayList・List

配列、ArrayList、List.of、Arrays.asList、拡張for、removeのオーバーロードまで、頻出の罠を判断順で整理します。

この単元で固定する判断

配列・ArrayList・List は、見た目が似ていても性質が違います。最初に「固定長か可変長か」「要素を変更できるか」「API名が何か」を分けます。

対象長さ/サイズ要素アクセス変更可否典型ミス
配列lengtha[0]長さ固定・要素変更可length() と書く
Stringlength()charAt(0)不変配列と混同する
ArrayListsize()get(0)可変長[0] で取ろうとする
List.of(...)size()get(0)変更不可add がコンパイルエラーだと思う

配列はオブジェクトですが、ArrayList とはAPIが違います。ここを雑にまとめると、問題文の1行目で落ちます。

配列は固定長のオブジェクト

配列は作成時に長さが決まり、その後に長さを変えることはできません。要素は変更できます。インデックスは0から始まり、最後の要素は length - 1 です。

int[] nums = new int[3];
System.out.println(nums.length); // 3
System.out.println(nums[0]);     // 0
nums[1] = 10;
System.out.println(nums[1]);     // 10
// System.out.println(nums[3]);  // ArrayIndexOutOfBoundsException

new int[3] は要素を3つ持つ配列です。アクセスできる添字は 0、1、2 だけです。

配列要素の初期値

配列の要素は、作成時に型ごとのデフォルト値で初期化されます。ローカル変数そのものは自動初期化されませんが、配列オブジェクトの要素は初期化される点を分けてください。

配列型要素の初期値
int[] などの整数型0
double[] などの浮動小数点型0.0
boolean[]false
char[]''
String[] などの参照型null
 
String[] names = new String[2];
System.out.println(names[0]); // null

int[] scores = new int[2];
System.out.println(scores[0]); // 0

配列宣言の形

配列の [] は型側にも変数名側にも書けます。ただし、読みやすさと誤読防止のため、基本は型側に寄せて読むのが安全です。

int[] a;      // 推奨される読み方: int配列型のa
int b[];      // これもOK
int[] x, y;   // xもyもint[]
int m[], n;   // mはint[]、nはint

int m[], n; は、m だけが配列で、n はただの int です。試験ではこの宣言の混在がよく狙われます。

配列初期化子 {...} の使い方

配列初期化子は、宣言と同時に使うか、new 型[] {...} の形で使います。宣言後に単独で {...} を代入することはできません。

int[] a = {1, 2, 3};       // OK
int[] b;
// b = {1, 2, 3};          // コンパイルエラー
b = new int[]{1, 2, 3};    // OK

var c = new int[]{1, 2, 3}; // OK
// var d = {1, 2, 3};       // コンパイルエラー

var と配列初期化子の組み合わせも頻出です。var d = {1, 2, 3}; では型推論に必要な型情報が不足します。

長さ0の配列

長さ0の配列は作れます。ただし、要素が存在しないため、どの添字にアクセスしても実行時例外です。

int[] a = new int[0];
System.out.println(a.length); // 0
// System.out.println(a[0]);  // ArrayIndexOutOfBoundsException

「長さ0の配列は作れない」と判断すると誤りです。作成可否とアクセス可否を分けます。

多次元配列は「配列の配列」

Javaの多次元配列は、厳密には配列の中に配列への参照が入っている構造です。行ごとに長さが違っても構いません。

int[][] data = new int[2][];
data[0] = new int[]{1, 2};
data[1] = new int[]{3, 4, 5};

System.out.println(data.length);    // 2
System.out.println(data[0].length); // 2
System.out.println(data[1].length); // 3

new int[2][] の時点では、外側の配列だけが作られます。内側の配列を作る前に data[0][0] へアクセスすると、data[0] が null なので NullPointerException になります。

int[][] data = new int[2][];
// System.out.println(data[0][0]); // NullPointerException

配列の代入と共変性

参照型配列では、サブクラス配列をスーパークラス配列の変数で扱えます。ただし、実体の配列型に合わない要素を入れると、コンパイルではなく実行時に ArrayStoreException が発生します。

Object[] data = new String[2];
data[0] = "A";      // OK
// data[1] = 100;    // ArrayStoreException

変数の型は Object[] でも、実体は String[] です。実体の配列に Integer を格納できないため実行時例外になります。

ArrayList は可変長

ArrayList は要素数を増減できます。配列と違い、要素数は size()、要素取得は get(index) です。

import java.util.ArrayList;

ArrayList<String> list = new ArrayList<>();
list.add("A");
list.add("B");
System.out.println(list.size()); // 2
System.out.println(list.get(0)); // A

list[0] は使えません。ArrayList は配列ではなくクラスです。

ArrayList の頻出メソッド

メソッド意味戻り値/注意
add(e)末尾に追加boolean を返す
add(index, e)指定位置に挿入戻り値は void。後ろがずれる
set(index, e)指定位置を置換古い要素を返す。サイズは変わらない
get(index)指定位置を取得範囲外なら IndexOutOfBoundsException
remove(index)指定位置を削除削除した要素を返す

var list = new java.util.ArrayList<String>();
list.add("A");       // [A]
list.add("B");       // [A, B]
list.add(1, "C");    // [A, C, B]
String old = list.set(0, "X");
System.out.println(old);  // A
System.out.println(list); // [X, C, B]

remove のオーバーロード

ArrayList の remove は、引数が int ならインデックス削除、Object なら要素削除です。ArrayList<Integer> で特に間違えます。

var list = new java.util.ArrayList<Integer>();
list.add(1);
list.add(2);
list.add(3);

list.remove(1); // インデックス1を削除 → 要素2が消える
System.out.println(list); // [1, 3]

list.remove(Integer.valueOf(1)); // 値1を削除
System.out.println(list); // [3]

remove(1) は「値1を消す」ではありません。int 引数として最も合うため、インデックス削除になります。

List 型で受ける意味

List はインタフェース、ArrayList は実装クラスです。変数の型が List でも、実体が ArrayList なら可変リストとして動作します。

import java.util.ArrayList;
import java.util.List;

List<String> list = new ArrayList<>();
list.add("A");
System.out.println(list); // [A]

ただし、変数の型で呼べるメソッドが決まります。List に存在しない ArrayList 固有メソッドは、List 型変数からは呼べません。

List.of は変更不可

List.of は変更不可のリストを返します。add や remove はメソッドとして存在するためコンパイルは通りますが、実行時に UnsupportedOperationException が発生します。

import java.util.List;

List<String> list = List.of("A", "B");
System.out.println(list.get(0)); // A
// list.add("C");              // UnsupportedOperationException

また、List.of は null 要素を許可しません。

// List<String> list = List.of("A", null); // NullPointerException

Arrays.asList の固定サイズリスト

Arrays.asList で作ったリストは、サイズ変更ができません。ただし、既存要素の置換はできます。List.of と同じ「完全な変更不可」とは雑に同一視しないでください。

import java.util.Arrays;
import java.util.List;

List<String> list = Arrays.asList("A", "B");
list.set(0, "X");
System.out.println(list); // [X, B]
// list.add("C");        // UnsupportedOperationException

可変の ArrayList にしたいなら、new ArrayList<>(...) で包みます。

var list = new java.util.ArrayList<>(java.util.List.of("A", "B"));
list.add("C");
System.out.println(list); // [A, B, C]

ジェネリクスとダイヤモンド演算子

ArrayList<String> のように要素型を指定すると、その型以外の要素は原則として追加できません。右辺の <> は、左辺から型を推論するダイヤモンド演算子です。

ArrayList<String> list = new ArrayList<>();
list.add("A");
// list.add(10); // コンパイルエラー

一方、raw typeを使うと型安全性が崩れます。Silverでは深追いしすぎる必要はありませんが、警告とコンパイルエラーは別物です。

拡張for: 変数は要素そのものではなくコピー

拡張forのループ変数は、各要素の値または参照のコピーです。ループ変数に別の値を代入しても、配列やリストのスロットは置き換わりません。

int[] nums = {1, 2, 3};
for (int n : nums) {
    n = 0;
}
System.out.println(nums[0]); // 1

参照型の場合、参照先オブジェクトの中身を変更すれば影響しますが、ループ変数に別オブジェクトを代入しても元の配列要素は差し替わりません。

StringBuilder[] arr = { new StringBuilder("A") };
for (StringBuilder sb : arr) {
    sb.append("B");
}
System.out.println(arr[0]); // AB

for (StringBuilder sb : arr) {
    sb = new StringBuilder("X");
}
System.out.println(arr[0]); // AB

走査中の構造変更

拡張forで ArrayList を走査している最中に、リストの構造を変更すると実行時例外につながることがあります。Silverでは、まず「拡張for中に add / remove していないか」を確認します。

var list = new java.util.ArrayList<String>();
list.add("A");
list.add("B");

for (String s : list) {
    // list.remove(s); // ConcurrentModificationExceptionになる可能性が高い
}

単にループ変数へ代入するだけなら構造変更ではありません。ただし、それでリストの中身が変わるわけでもありません。

ミニ演習

Q1. 次の出力は？
int[] a = new int[2]; System.out.println(a[1]);

0。int 配列の要素は0で初期化されます。

Q2. 次のコードは？
int[] a; a = {1, 2};

コンパイルエラー。配列初期化子を代入だけで使うことはできません。a = new int[]{1, 2}; ならOKです。

Q3. 次の出力は？
var list = new ArrayList<Integer>(); list.add(1); list.add(2); list.remove(1); System.out.println(list);

[1]。remove(1) は値1ではなく、インデックス1の要素を削除します。

Q4. 次のコードは？
List<String> list = List.of("A"); list.add("B");

実行時例外。add メソッド呼び出し自体はコンパイルされますが、変更不可リストなので UnsupportedOperationException です。
