配列・ArrayList・List

配列の宣言・初期化・参照、ArrayListの操作、List.ofの制約、拡張forの落とし穴を整理します。

配列はオブジェクト

配列は複数の値を入れる箱ではありますが、Java上はオブジェクトです。配列変数は配列インスタンスへの参照を持ちます。

int[] a = new int[3];
        System.out.println(a.length); // 3
        System.out.println(a[0]);     // 0

length はフィールドです。length() ではありません。String は length() メソッドです。

要素数0の配列は作れます。new int[0] はOKです。ただし a[0] は実行時例外です。

配列宣言の形

角括弧の位置には複数の書き方があります。試験では読めれば十分ですが、変な形に見えても有効なものがあります。

int[] a;
        int b[];
        int[][] c;
        int[] d[];

初期化子 {} は、宣言と同時に使うか、new 型[] と組み合わせます。

int[] a = {1, 2, 3};
        int[] b;
        b = new int[]{1, 2, 3};
        // b = {1, 2, 3}; // NG

多次元配列とジャグ配列

Javaの多次元配列は「配列の配列」です。行ごとに長さが違っても構いません。

int[][] data = new int[2][];
        data[0] = new int[]{1, 2};
        data[1] = new int[]{3, 4, 5};

        System.out.println(data.length);    // 2
        System.out.println(data[1].length); // 3

new int[2][] の時点では内側の配列はまだ null です。data[0][0] に触る前に内側を生成しているか確認します。

ArrayListの基本

ArrayList は要素数を変更できるリストです。配列と違い、要素数は size() で取得します。

ArrayList<String> list = new ArrayList<>();
        list.add("A");
        list.add("B");
        System.out.println(list.size()); // 2
        System.out.println(list.get(0)); // A
操作戻り値/挙動注意
add(e)boolean末尾に追加
add(index,e)void指定位置に挿入
remove(index)削除した要素intならインデックス扱い
remove(Object)boolean一致する要素を1つ削除
set(index,e)置換前の要素要素数は増えない

removeのオーバーロード

ArrayList<Integer> では remove(1) が「値1を消す」ではなく「インデックス1を消す」になる点が頻出です。

ArrayList<Integer> nums = new ArrayList<>();
        nums.add(10);
        nums.add(20);
        nums.add(30);
        nums.remove(1);              // index 1の20を削除
        nums.remove(Integer.valueOf(10)); // 値10を削除
remove(int) と remove(Object) のどちらが選ばれるかは、引数の型で決まります。

List.ofと固定・不変リスト

List.of は変更不可のリストを返します。add、remove、set は実行時例外です。また、null 要素も不可です。

List<String> list = List.of("A", "B");
        // list.add("C"); // UnsupportedOperationException
        // List.of("A", null); // NullPointerException

Arrays.asList はサイズ固定のリストです。set はできますが、add/remove はできません。

拡張forの罠

拡張forのループ変数は、要素のコピーまたは参照のコピーです。プリミティブ配列の要素そのものを書き換えるわけではありません。

int[] nums = {1, 2, 3};
        for (int n : nums) {
            n *= 2;
        }
        System.out.println(nums[0]); // 1

参照型の要素では、参照先オブジェクトを変更するメソッドを呼ぶと中身は変わります。ただし、ループ変数に別の参照を代入しても配列・リスト側は変わりません。

ミニ判定

Q1. int[] a; System.out.println(a.length); は？

ローカル変数ならコンパイルエラーです。配列変数を宣言しただけで、配列インスタンスはありません。

Q2. new int[2][] の a[0] は？

null です。内側の配列はまだ生成されていません。

Q3. List.of("A").add("B") は？

コンパイルはできますが、実行時に UnsupportedOperationException です。
