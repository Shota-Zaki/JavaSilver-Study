配列・ArrayList・List

配列、ArrayList、List.of、拡張for、要素数とインデックスの境界を整理します。

配列の基本

配列は固定長のオブジェクトです。作成後に長さは変わりません。インデックスは0始まりです。

int[] a = new int[3];
System.out.println(a.length); // 3
System.out.println(a[0]);     // 0
// System.out.println(a[3]);  // ArrayIndexOutOfBoundsException

配列宣言と初期化

配列初期化子 {...} は、宣言と同時に使うか、new 型[]{...} の形で使います。

int[] a = {1, 2, 3};       // OK
int[] b;
// b = {1, 2, 3};          // NG
b = new int[]{1, 2, 3};    // OK

int c[] = new int[2];      // OK。ただし int[] c が読みやすい

多次元配列

Javaの多次元配列は、配列の配列です。行ごとに長さが違っても構いません。

int[][] data = new int[2][];
data[0] = new int[]{1, 2};
data[1] = new int[]{3, 4, 5};
System.out.println(data.length);    // 2
System.out.println(data[1].length); // 3

ArrayListの基本操作

ArrayList は可変長です。add、remove、set の戻り値と副作用を区別します。

操作意味注意
add(e)末尾に追加戻り値はboolean
add(index,e)指定位置に挿入後ろの要素がずれる
set(index,e)置換サイズは変わらない。古い要素を返す
remove(index)指定位置を削除削除した要素を返す
remove(Object)一致する要素を削除削除できたかbooleanを返す
var list = new java.util.ArrayList<String>();
list.add("A");
list.add("B");
list.add(1, "C");
System.out.println(list); // [A, C, B]

List.ofの注意

List.of で作ったリストは変更不可です。add や remove はコンパイルはできても、実行時に UnsupportedOperationException が発生します。

List<String> list = java.util.List.of("A", "B");
// list.add("C"); // 実行時例外

拡張forの注意

拡張forの変数は要素のコピー、または参照のコピーです。変数へ別の値を代入しても、配列やリストのスロット自体は置き換わりません。

int[] nums = {1, 2, 3};
for (int n : nums) {
    n = 0;
}
System.out.println(nums[0]); // 1

ミニ演習

Q. new int[0] は作れる？

作れます。長さ0の配列です。ただし [0] にアクセスすると実行時例外です。

Q. remove(1) と remove(Integer.valueOf(1)) は同じ？

違います。前者はインデックス削除、後者は要素値1の削除です。

削除しながら走査する罠

拡張for中にリスト構造を変更するコードは、実行時例外につながる場合があります。Silverでは複雑なIteratorよりも、拡張forの変数が要素のコピーである点を優先して押さえます。

var list = new java.util.ArrayList<String>();
list.add("A");
list.add("B");
for (String s : list) {
    s = "X";
}
System.out.println(list); // [A, B]
