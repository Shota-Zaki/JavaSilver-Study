配列・ArrayList・List

配列、ArrayList、List.of、拡張for、境界例外を、出題される違いに絞って整理します。

配列は固定長のオブジェクト

配列はオブジェクトです。生成時に長さが決まり、その長さは後から変えられません。要素にはデフォルト値が入ります。

int[] a = new int[3];
System.out.println(a.length); // 3
System.out.println(a[0]);     // 0

String[] s = new String[2];
System.out.println(s[0]);     // null

length

配列はフィールド length。メソッドではないので length() は不可です。

添字

0始まり。長さ3なら有効な添字は0,1,2です。

固定長

要素の中身は変えられますが、配列自体の長さは変えられません。

参照型配列

要素の初期値は null。メソッド呼び出し前に参照先の有無を確認します。

配列宣言と初期化

配列の宣言は複数の書き方がありますが、初期化子 {} の使える位置には制限があります。

int[] a = {1, 2, 3};        // OK
int[] b = new int[]{1, 2, 3}; // OK

int[] c;
// c = {1, 2, 3};            // NG
c = new int[]{1, 2, 3};       // OK

多次元配列は「配列の配列」です。各行の長さが揃っていなくても構いません。

int[][] x = new int[2][];
x[0] = new int[]{1, 2, 3};
x[1] = new int[]{4};
System.out.println(x[0][2]); // 3

ArrayListとList

ArrayList はサイズ可変のリストです。配列とは違い、要素数は size() メソッドで取得します。

import java.util.ArrayList;

ArrayList<String> list = new ArrayList<>();
list.add("A");
list.add("B");
System.out.println(list.size()); // 2
System.out.println(list.get(0)); // A

add

末尾追加、または指定位置に挿入。戻り値の有無はオーバーロードで異なります。

set

既存要素を置換します。要素数は増えません。

remove

インデックス指定とオブジェクト指定があります。Integer リストでは特に混同注意。

contains

equals による同値判定で含有を見ます。

List.of の注意

List.of で作ったリストは変更不可です。add や remove はコンパイルは通っても実行時に UnsupportedOperationException が発生します。

List<String> list = List.of("A", "B");
System.out.println(list.get(0)); // A
// list.add("C");              // 実行時例外
「メソッドが存在するからコンパイルできる」ことと、「そのインスタンスで実行できる」ことは別です。List.ofはここを狙われます。

拡張for文

拡張for文のループ変数は、各要素の値または参照を一時的に受け取る変数です。ループ変数に別の値を代入しても、配列やリストの要素そのものが必ず変わるわけではありません。

int[] nums = {1, 2, 3};
for (int n : nums) {
    n = 0;
}
System.out.println(nums[0]); // 1
