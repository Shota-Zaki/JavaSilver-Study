配列・ArrayList・List

配列宣言、初期化、List操作、拡張for文をまとめて確認。

配列

項目書き方注意
宣言int[] a; / int a[];要素数は宣言時に書かない。int[3] a は不可。
生成new int[3]要素は0で初期化。参照型配列ならnull。
初期化int[] a = {1,2,3};宣言と同時でない場合は new int[]{...} が必要。
lengtha.lengthフィールド。()は付けない。
範囲0〜length-1範囲外でArrayIndexOutOfBoundsException。

多次元配列

int[][] a = new int[2][];
a[0] = new int[]{1, 2};
a[1] = new int[]{3};
System.out.println(a[1][0]); // 3

ArrayList / List

操作戻り値/挙動注意
add(E)boolean末尾追加。
add(index, E)void指定位置に挿入。既存要素は後ろへ。
set(index, E)置き換え前の要素indexが存在しないとIndexOutOfBoundsException。追加ではない。
remove(index)削除した要素Integerリストでremove(1)はインデックス削除。
remove(Object)booleanInteger.valueOf(1)なら値削除。
size()要素数配列はlength、Stringはlength()。
List.of変更不可Listadd/set/removeでUnsupportedOperationException。

拡張for文

対象ルール注意
primitive配列取り出し変数へ代入しても元配列は変わらない値のコピー。
参照型配列/List参照先オブジェクトの中身変更は反映取り出し変数に別インスタンスを代入しても元要素は差し替わらない。
ArrayListを走査中にremoveConcurrentModificationExceptionの可能性試験では実行時例外として問われる。
