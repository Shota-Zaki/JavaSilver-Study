ポリモーフィズム・キャスト

宣言型、実体、メソッド解決、キャスト、instanceof、ClassCastExceptionを整理します。

宣言型と実体を分ける

ポリモーフィズム問題では、変数の宣言型と実際のインスタンス型を必ず分けて書きます。呼べるメソッドは宣言型で決まり、実行されるoverrideメソッドは実体で決まります。

class A { void test() { System.out.println("A"); } }
class B extends A { void test() { System.out.println("B"); } void onlyB() {} }

A obj = new B();
obj.test();  // B
// obj.onlyB(); // NG: 宣言型AにonlyBがない

アップキャストとダウンキャスト

サブクラスからスーパークラスへのアップキャストは暗黙にできます。スーパークラス型からサブクラス型へのダウンキャストは明示キャストが必要です。

B b = new B();
A a = b;       // アップキャスト。OK
B b2 = (B) a;  // ダウンキャスト。実体がBなのでOK

ClassCastException

コンパイル時に型の関連があればキャスト式は書けます。しかし、実行時の実体がその型でなければ ClassCastException です。

class A {}
class B extends A {}
class C extends A {}

A a = new C();
B b = (B) a; // コンパイルは通るが、実行時にClassCastException

instanceofとパターンマッチング

instanceof は、実体が指定型として扱えるかを調べます。パターン変数を使うと、trueの範囲内でキャスト済み変数を使えます。

Object obj = "abc";
if (obj instanceof String s) {
    System.out.println(s.length());
}

メソッド呼び出しとキャストの優先

キャストしてからメソッドを呼びたい場合、括弧の位置が重要です。

Object obj = "abc";
// (String)obj.length(); // NG: 先にobj.length()を探してしまう形
System.out.println(((String) obj).length()); // OK

ミニ演習

Q. A a = new B(); のとき、Bだけにあるメソッドは呼べる？

宣言型Aにそのメソッドがなければ呼べません。呼ぶには適切なキャストが必要です。

Q. キャストを書けば必ず実行時も安全？

違います。実体がその型として扱えなければ ClassCastException です。

関連: 判定フロー / コンパイル・実行時例外 / ひっかけ集

キャストは「呼べるようにする」だけではない

キャストすると、コンパイル時にその型としてメンバを探せます。ただし、実行時に実体がその型でなければ落ちます。つまり、キャストはコンパイル時の見え方と実行時の安全性の両方を考えます。

A a = new B();
((B) a).onlyB(); // コンパイルも実行もOK。実体がBだから。

A x = new C();
// ((B) x).onlyB(); // コンパイルは可能でも実行時にClassCastException

instanceofで安全に狭める

ダウンキャスト前に instanceof を使うと、実行時例外を避けられます。

if (a instanceof B b) {
    b.onlyB();
}
