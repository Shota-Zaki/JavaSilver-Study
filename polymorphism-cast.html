ポリモーフィズム・キャスト・instanceof

宣言型と実体、メソッド呼び出し、フィールド参照、キャスト、instanceofパターンを整理します。

宣言型と実体を分ける

参照型変数には、宣言型と実体があります。コンパイル時に見えるメンバは宣言型で決まり、オーバーライドされたインスタンスメソッドの実行内容は実体で決まります。

Animal a = new Dog();
        a.eat();      // 呼べるかはAnimal型にeatがあるかで決まる
                      // 実行される内容はDogのoverride実装
まず「そのメソッド名が宣言型に存在するか」を見ます。存在しないなら、実体にあってもコンパイルエラーです。

メソッドは実体、フィールドは宣言型

オーバーライドされたメソッドは実体側で決まります。一方、フィールドはオーバーライドされません。参照変数の宣言型で決まります。

class A {
            String name = "A";
            String getName() { return name; }
        }
        class B extends A {
            String name = "B";
            @Override String getName() { return name; }
        }
        A obj = new B();
        System.out.println(obj.name);     // A
        System.out.println(obj.getName()); // B

同名フィールドは隠蔽されるだけで、ポリモーフィズムの対象ではありません。

アップキャストとダウンキャスト

サブクラスからスーパークラスへの代入はアップキャストで、暗黙にできます。スーパークラス型の参照をサブクラス型へ戻すにはダウンキャストが必要です。

Dog d = new Dog();
        Animal a = d;       // OK。アップキャスト
        Dog d2 = (Dog) a;   // OK。実体がDogなら成功

コンパイルできるキャストでも、実体が合わなければ ClassCastException です。

Animal a = new Cat();
        // Dog d = (Dog) a; // 実行時例外

instanceofとパターンマッチング

instanceof は、実体が指定型として扱えるかを判定します。Java 17ではパターン変数も使えます。

Object obj = "java";
        if (obj instanceof String s) {
            System.out.println(s.length());
        }

パターン変数 s は、条件がtrueの範囲でだけ使えます。obj が null の場合、instanceof はfalseです。

キャスト式の優先順位

キャストしてからメソッドを呼びたい場合、カッコの位置が重要です。

Object obj = "abc";
        // (String)obj.length(); // NG。lengthを先に探してしまう
        System.out.println(((String)obj).length()); // OK

メソッド呼び出しの対象をどの型として解釈させるかを、カッコで明確にします。

ミニ判定

Q1. Parent p = new Child(); p.childOnly(); は？

childOnly が Parent にないならコンパイルエラーです。実体が Child でも、宣言型で見えません。

Q2. obj instanceof String s で obj がnullなら？

falseです。例外は発生しません。

Q3. フィールドはオーバーライドされる？

されません。同名フィールドは隠蔽され、参照変数の宣言型で参照先が決まります。
