ポリモーフィズム・キャスト・instanceof

参照型と実体の違い、キャスト、オーバーライド呼び出しを整理します。

参照型と実体を分ける

Parent p = new Child();

左辺の Parent は「使えるメンバを決める型」です。右辺の Child は「実際に動くインスタンス」です。

メソッドとフィールドの違い

class A { String name = "A"; String getName(){ return "A"; } }
class B extends A { String name = "B"; String getName(){ return "B"; } }

A obj = new B();
System.out.println(obj.name);      // A
System.out.println(obj.getName()); // B

メソッドは実体側のオーバーライドが効きます。フィールドは変数の型で決まります。

キャストの判定

段階見るもの例
コンパイル時型同士に継承・実装関係があり得るか無関係なクラス同士はコンパイルエラーになりやすい。
実行時実体がキャスト先として扱えるか実体が違えばClassCastException。
Object o = "java";
String s = (String) o;   // OK
// Integer i = (Integer) o; // 実行時ClassCastException

instanceof パターンマッチング

Object obj = "java";
if (obj instanceof String s) {
    System.out.println(s.length());
}

条件がtrueのブロック内で、パターン変数 s が使えます。nullに対する instanceof はfalseです。
