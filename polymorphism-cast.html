ポリモーフィズム・キャスト

宣言型と実体、キャスト、instanceof、フィールドとメソッドの解決差をまとめます。

宣言型と実体

参照変数には型があります。実際に入っているインスタンスにも型があります。コンパイル時に呼び出せるかは宣言型、オーバーライドされたメソッドの実行結果は実体型で決まります。

class A { void x() { System.out.println("A"); } }
class B extends A { void x() { System.out.println("B"); } void y() {} }

A a = new B();
a.x();   // B
// a.y(); // コンパイルエラー

キャスト

キャストは、参照の見え方を変えるものです。実体そのものが変わるわけではありません。コンパイル時に型関係があり、実行時に実体がその型として扱える場合だけ成功します。

A a = new B();
B b = (B) a; // OK。実体はB

A onlyA = new A();
// B bad = (B) onlyA; // 実行時にClassCastException

instanceof パターンマッチング

instanceofは、実体がその型として扱えるかを判定します。Java 17では、判定と同時に変数へ束縛できます。

Object obj = "java";

if (obj instanceof String s) {
    System.out.println(s.length());
}

パターン変数は、条件がtrueになる範囲でだけ使えます。スコープの問題として問われることがあります。
