ポリモーフィズム・キャスト

宣言型、実体、動的ディスパッチ、フィールド解決、キャスト、instanceofを整理します。

宣言型と実体

参照変数には宣言型と実体があります。コンパイル時に見えるメンバは宣言型で決まり、オーバーライドされたインスタンスメソッドの実行先は実体で決まります。

class Animal {
    void sound() { System.out.println("animal"); }
}
class Dog extends Animal {
    void sound() { System.out.println("dog"); }
    void run() { System.out.println("run"); }
}

Animal a = new Dog();
a.sound(); // dog
// a.run(); // NG。宣言型Animalにはrunがない
「実体がDogだからDogの全メソッドを呼べる」は誤りです。呼び出せるかはまず宣言型で判定します。

フィールドは多態的に解決されない

インスタンスメソッドはオーバーライドにより実体で解決されますが、フィールドは宣言型で解決されます。同名フィールドをサブクラスで定義すると、メソッドとフィールドで見え方が変わります。

class A {
    String name = "A";
    String getName() { return name; }
}
class B extends A {
    String name = "B";
    String getName() { return name; }
}

A obj = new B();
System.out.println(obj.name);      // A
System.out.println(obj.getName()); // B

キャスト

キャストは、参照の見え方を変える構文です。実体が変わるわけではありません。互換性のない実体へダウンキャストすると ClassCastException です。

Animal a = new Dog();
Dog d = (Dog) a;  // OK。実体がDog

a = new Animal();
// Dog bad = (Dog) a; // 実行時ClassCastException

アップキャスト

サブクラス参照をスーパークラス型へ。通常は明示キャスト不要です。

ダウンキャスト

スーパークラス型の参照をサブクラス型へ。明示キャストが必要です。

コンパイル可否

継承関係など、型として変換可能性がなければコンパイルエラーです。

実行時可否

実体がその型として扱えるかで ClassCastException の有無が決まります。

instanceof とパターンマッチング

instanceof は実体が指定型として扱えるかを判定します。Java 17ではパターン変数を同時に宣言できます。

Object obj = "hello";

if (obj instanceof String s) {
    System.out.println(s.length());
}

パターン変数は、判定がtrueだと分かる範囲でだけ使えます。スコープを雑に扱うとコンパイルエラーになります。
