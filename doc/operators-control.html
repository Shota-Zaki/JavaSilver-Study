演算子・制御構造

演算子問題は「知識」だけでなく、評価順を崩さず追跡できるかで差が出ます。暗算ではなく、型・副作用・短絡評価・分岐の順に機械的に判定します。

式はグループ化・評価順・結果型の順で読む

演算子・制御構造の問題は、出力を考える前にコンパイル可否を確定する。
特に、booleanでない条件式、boolean同士の大小比較、数値型の縮小変換、caseラベルの非定数、switch対象型の不適合は、実行前に落ちます。
  1. 構文として成立するか。括弧、セミコロン、switch/case、三項演算子の形を見る。
  2. 型として成立するか。if/whileの条件はboolean、代入先の型、演算後の型を確認する。
  3. 評価される式だけを追う。&& と ||、三項演算子、switchの分岐で評価されない部分を消す。
  4. 副作用を表に書く。++、--、+=、代入式で変数値が変わるタイミングを追う。
  5. 出力を最後に組み立てる。String結合は左から、数値演算は型昇格後に計算する。

演算子の優先順位は「全部暗記」ではなく頻出だけ押さえる

Java Silverで現実的に重要なのは、次の順序です。細かい優先順位表を丸暗記するより、括弧・単項・算術・比較・論理・代入の大枠を覚えた方が点に直結します。

優先度分類例試験での見方
高括弧・メソッド呼び出し(), method()まずここを確定する。
高後置/前置a++, --b値として使うタイミングと変数更新を分ける。
中算術*, /, %, +, -整数除算、String結合、型昇格に注意。
中関係・等価<, <=, ==, !=大小比較と等価比較で使える型が違う。
低論理&&, ||, &, |, ^短絡評価の有無で右辺の副作用が変わる。
低三項・代入?:, =, +=代入式は値を持つ。複合代入は暗黙キャストを含む。

前置・後置インクリメントの正しい追跡

++ と -- は、変数の値を変える副作用を持ちます。ここで雑に読むと出力問題は崩れます。

形値として使われるもの変数の更新例
++a増えた後の値先に増えるint b = ++a;
a++増える前の値値を渡した後に増えるint b = a++;
--a減った後の値先に減るint b = --a;
a--減る前の値値を渡した後に減るint b = a--;
 
int a = 10;
int b = ++a; // aは11になってからbへ渡る。a=11, b=11
int c = a++; // まず11がcへ渡り、その後aが12になる。
System.out.println(a + "," + b + "," + c); // 12,11,11
雑な覚え方は危険。「前置は先、後置は後」という言葉だけでは不十分です。必ず「式に渡る値」と「変数に残る値」を分けてください。

複雑な ++ / -- は一時値で分解する

同じ変数が1つの式に複数回出る場合、左から読んで、各地点で「式に渡す値」と「変数の現在値」を表にします。

int a = 10;
int b = a++ + a + a-- - a-- + ++a;
System.out.println(b);
部分式式に渡る値評価後のa理由
a++1011後置なので10を使ってから増える。
a1111現在値を使う。
a--1110後置なので11を使ってから減る。
a--109後置なので10を使ってから減る。
++a1010先に9から10へ増えて、その10を使う。

したがって、b = 10 + 11 + 11 - 10 + 10 です。こういう問題は、頭の中だけで解くほどミスが増えます。試験本番でも小さく表を書いた方が速いです。

評価順序と演算子優先順位を分ける

優先順位は式のグループ化を決め、評価順序は実際にどの副作用が先に起こるかを決めます。Javaではオペランドは原則として左から評価されますが、優先順位の高い演算が先にグループ化されます。

int x = 1;
      int result = x++ + ++x * 2;
      
      // グループ化: x++ + ((++x) * 2)
      // 評価は左から
      // x++ の値は1、xは2
      // ++x の値は3、xは3
      // 1 + 3 * 2 = 7
副作用を含む式の追跡：演算子が返す式の値と、変数に残る更新後の値を分けて記録します。
後置インクリメントと代入

出力は何ですか。

int x = 1;
      x = x++;
      System.out.println(x);
値の推移：1です。x++は更新前の1を返し、その値を同じ変数へ代入するため、増加後の2が上書きされます。

代入式は値を持つ

= や += は単なる文ではなく、式です。代入後の値を式全体の値として返します。

int a = 3;
int b = a += 5;
System.out.println(a + b); // 16
  1. a += 5 が先に評価され、a は8になる。
  2. a += 5 という式全体の値も8になる。
  3. その8が b に代入される。
  4. a + b は 8 + 8 で16。

複合代入は暗黙のキャストを含む

+= などの複合代入は、単純な a = a + b と完全に同じではありません。代入先の型への暗黙キャストが入ります。

byte b = 1;
// b = b + 1; // NG: b + 1 は int
b += 1;       // OK: 内部的に (byte)(b + 1) のように扱われる
ここは頻出。 byte、short、char は演算時に基本的に int へ昇格します。単純代入なら落ちるが、複合代入なら通る、という差が狙われます。

算術演算とString結合は左から分ける

+ は数値加算にも文字列結合にも使われます。どちらになるかは、その時点で片方がStringかどうかで決まります。

System.out.println(1 + 2 + "A"); // 3A
System.out.println("A" + 1 + 2); // A12
System.out.println("A" + (1 + 2)); // A3

一度String結合に入ると、以降の + は文字列結合として進みやすくなります。左から順に、現在の式が数値なのかStringなのかを追ってください。

比較演算子で使える型を混同しない

演算子主な用途boolean同士参照型同士注意点
<, <=, >, >=大小比較不可不可数値やcharなどの比較。booleanには使えない。
==, !=等価比較可可参照型では同じオブジェクトを指すかの比較。
&&, ||短絡論理可不可左右はboolean式。
&, |, ^非短絡論理/ビット演算可不可booleanなら非短絡、整数ならビット演算。
 
boolean a = true;
boolean b = false;
System.out.println(a == b); // OK: false
// System.out.println(a <= b); // NG: booleanに大小比較はない

短絡評価と非短絡評価

&& と || は、左辺だけで結果が決まる場合、右辺を評価しません。右辺に ++ やメソッド呼び出しがある問題では、ここが正答を分けます。

式左辺右辺評価理由
false && exprfalseしない全体は必ずfalse。
true || exprtrueしない全体は必ずtrue。
false & exprfalseする& はbooleanでは非短絡。
true | exprtrueする| はbooleanでは非短絡。
 
int a = 10;
int b = 10;
if (a < 10 && ++b > 10) { }
System.out.println(b); // 10: 右辺は評価されない

if (a < 10 & ++b > 10) { }
System.out.println(b); // 11: 右辺も評価される

短絡評価は結果だけでなく副作用を変える

int x = 0;
      boolean a = false && (++x > 0); // 右辺を評価しない
      boolean b = true  || (++x > 0); // 右辺を評価しない
      boolean c = false &  (++x > 0); // 右辺を評価する
      boolean d = true  |  (++x > 0); // 右辺を評価する
      System.out.println(x);          // 2
演算子右辺を省略する条件用途
&&左辺がfalsenull確認とメソッド呼び出しを安全に連結
||左辺がtrue代替条件を順番に判定
& / |省略しないbooleanでも両辺を必ず評価。整数ではビット演算
null安全性

s != null && s.length() > 0は安全ですか。

短絡評価の結果：安全です。左辺がfalseなら右辺を評価しません。&に変えるとNullPointerExceptionになり得ます。

if文の条件はbooleanだけ

Javaでは、C言語系のように 0 をfalse、非0をtrueとして扱うことはありません。条件式には必ずboolean式が必要です。

int n = 1;
// if (n) { }      // NG
if (n == 1) { }    // OK

このルールは while、do-while、三項演算子の条件部でも同じです。

三項演算子は「条件」「選ばれる式」「型」を見る

三項演算子は 条件 ? 式1 : 式2 です。条件はbooleanでなければいけません。また、実行時に評価されるのは選ばれた側だけです。

int a = 10;
int b = 20;
String result = a > b ? "A" : "B";
System.out.println(result); // B

出題では、選ばれない側に例外が起きそうな式を置いてくることがあります。評価されない式なら実行時例外は起きません。ただし、型や構文として不正ならコンパイルエラーです。

int x = 10;
int y = x > 0 ? 1 : 10 / 0; // OK。実行時に10/0は評価されない。
System.out.println(y);      // 1

条件演算子は結果の型も問われる

condition ? expression1 : expression2は値を持つ式です。第2・第3オペランドの型から、条件式全体の型が決まります。

boolean flag = true;
      var x = flag ? 10 : 20L;    // xはlong
      var y = flag ? "Java" : null; // yはString
      
      // var z = flag ? null : null; // 型を推論できずコンパイルエラー
非選択側の検査：実行されない式もコンパイル時の型検査と構文検査を受けます。

switch文の対象型

switchは何でも使えるわけではありません。Java Silverでは特に、long、float、double、boolean が使えない点を押さえてください。

使える使えない
byte, short, char, intlong, float, double, boolean
対応するラッパークラス、String、enum任意のObject、配列、Listなど
 
String s = "B";
switch (s) {
    case "A": System.out.println("A"); break;
    case "B": System.out.println("B"); break;
    default:  System.out.println("other");
}

caseラベルは互換性のあるコンパイル時定数

case に置けるのは、switch対象型と互換性があり、かつコンパイル時に値が確定するものです。単に final が付いていれば何でもよい、ではありません。

final int A = 1;      // コンパイル時定数
final int B;
B = 2;                // 実行時に代入される扱い
int n = 1;

switch (n) {
    case A: break;    // OK
    // case B: break; // NG: コンパイル時定数ではない
}
重複caseもコンパイルエラー。 表記が違っても値が同じなら重複です。case 1: と case 01: はどちらも値としては1です。

switch文はbreakがなければフォールスルーする

switch文では、一致したcaseから実行が始まり、break などで抜けるまで下へ流れます。default の位置は最後でなくても構いません。

int n = 2;
switch (n) {
    case 1: System.out.print("A");
    case 2: System.out.print("B");
    default: System.out.print("C");
}
// BC

case 2 に一致して B を出力し、breakがないため default へ流れて C も出力します。

switch式の詳細

値を生成するswitch式は、switch文とは網羅性、yield、結果型の判定が異なります。switch式で、セレクタ評価から式全体の型まで独立して扱います。

副作用と短絡評価の誤読

コード判定理由
if (1) {}コンパイルエラー条件はbooleanのみ。
true < falseコンパイルエラーbooleanに大小比較はない。
byte b = 1; b = b + 1;コンパイルエラーb + 1 はint。
byte b = 1; b += 1;OK複合代入は代入先型への変換を含む。
false && ++a > 0右辺未評価短絡評価。
false & ++a > 0右辺評価booleanの & は非短絡。
switch (1L)コンパイルエラーlong はswitch対象不可。

式の値と変数の値を別々に追う

Q1. int a = 1; int b = a += 2; System.out.println(a + b);

a += 2 でaは3。式全体の値も3なのでbも3。出力は6です。

Q2. boolean x = true; boolean y = false; System.out.println(x > y);

コンパイルエラーです。boolean同士に大小比較は使えません。

Q3. int a = 0; if (false && ++a > 0) {} System.out.println(a);

&& の左辺がfalseなので右辺は評価されません。出力は0です。

Q4. int a = 0; if (false & ++a > 0) {} System.out.println(a);

& は右辺も評価します。++a が実行され、出力は1です。

Q5. switch の default は最後に書かないといけない？

最後でなくても構文上は可能です。ただしbreakがなければそこから下にフォールスルーします。
