instanceofパターン変数

型判定が成功した経路だけに広がるパターン変数のスコープを、短絡評価と制御フローから説明します。

型判定とキャストを1つの条件にまとめる

型パターン付きinstanceofは、条件が真なら安全なキャスト結果を変数へ束縛します。構文を省略形として暗記するより、従来コードのどの2段階をまとめたものかを見る方が理解しやすくなります。

従来の書き方

if (obj instanceof String) {
    String text = (String) obj;
    System.out.println(text.length());
}

型パターン

if (obj instanceof String text) {
    System.out.println(text.length());
}

変数textが使えるのは、objがStringであることをコンパイラが保証できる経路だけです。この保証を基準にスコープを読みます。

&&では右辺へ進む時点で型判定が成功している

if (obj instanceof String text && text.length() > 3) {
    System.out.println(text.toUpperCase());
}

&&の右辺は左辺がtrueのときだけ評価されます。右辺へ到達した時点でパターン照合は成功しているため、textを使用できます。

if (obj instanceof String text && check(text)) {
    // textは使用可能
}

条件式を左から読むと、変数が導入される位置と、その後に保証が続く範囲を追えます。

||の右辺では成功が保証されない

if (obj instanceof String text || text.length() > 3) {
    // コンパイルエラー
}

||の右辺が評価されるのは、左辺がfalseのときです。つまりobj instanceof String textが失敗した経路でtextを使おうとしているため、不正です。

同じ変数名が条件内に見えても、短絡評価の方向が逆ならスコープも逆になります。

否定条件の後でも、失敗経路を終了させれば使える

static int length(Object obj) {
    if (!(obj instanceof String text)) {
        return 0;
    }
    return text.length();
}

ifの後へ到達できるのは、否定条件がfalseだった場合だけです。したがって元のパターン照合は成功しており、textを使用できます。

一方、失敗経路が終了しない場合は、ifの後で使用できません。

if (!(obj instanceof String text)) {
    System.out.println("not string");
}
// System.out.println(text); // Stringでない経路も到達するためエラー

if-elseでは成功側だけに変数が存在する

if (obj instanceof String text) {
    System.out.println(text.length()); // 可
} else {
    // System.out.println(text);       // 不可
}

elseへ進むのはパターン照合が失敗した場合です。パターン変数は単にif文全体へ宣言されたローカル変数ではありません。真偽によって確実に照合済みとなる範囲へだけ広がります。

nullはどの型パターンにも一致しない

Object obj = null;
if (obj instanceof String text) {
    System.out.println(text.length());
} else {
    System.out.println("no match");
}

出力はno matchです。instanceofはnullに対してfalseを返すため、判定前のnullチェックを必ず別に書く必要はありません。ただし、条件の別部分でobjのメソッドを先に呼べばNullPointerExceptionになります。

// obj.toString()を先に評価するためnullなら例外
if (obj.toString().length() > 0 && obj instanceof String text) {}

宣言型と実行時型を分けて判定する

Object value = "Java";
if (value instanceof CharSequence seq) {
    System.out.println(seq.length());
}

変数valueの宣言型はObjectですが、参照先はStringです。StringはCharSequenceを実装するため照合が成功します。パターン変数seqの型はCharSequenceなので、その型から見えるメソッドを呼びます。

if (value instanceof CharSequence seq) {
    seq.length();
    // seq.substring(1); // CharSequenceにはないためコンパイルエラー
}

実体がStringでも、パターン変数をCharSequenceとして宣言したなら、呼び出せるメンバーはCharSequence型を基準に決まります。

同じ参照を指すが、別オブジェクトを生成するわけではない

Object obj = new StringBuilder("A");
if (obj instanceof StringBuilder builder) {
    builder.append("B");
}
System.out.println(obj); // AB

パターン変数builderには、objが参照している同じインスタンスへの参照が入ります。キャストと同様に、型判定によって新しいオブジェクトが生成されるわけではありません。

複合条件は到達経路へ印を付ける

if (obj instanceof String text
        && !text.isEmpty()
        && text.charAt(0) == 'J') {
    System.out.println(text);
}

この条件は次の順で読みます。

  1. objがStringならtextが導入される。
  2. Stringでない場合は短絡してif本体へ進まない。
  3. textが空でない場合だけcharAt(0)を評価する。
  4. 先頭がJの場合だけ本体へ進む。

型安全だけでなく、charAt(0)の境界安全性も短絡評価によって保証されています。型パターン問題では、変数スコープとAPIの実行条件を同時に追うことがあります。

instanceofの型パターンとswitchの型パターンを分ける

switchのcaseへ型パターンを書く構文は、Java 17ではプレビュー機能です。通常のコンパイル設定を前提とする問題では、instanceof Type variableの型パターンと、switchのパターンマッチングを同一視しません。

スコープを判断する方法

変数名の見た目ではなく、その行へ到達したときに型判定成功が必ず成り立つかを確認します。

位置使用可否理由
&&の右辺可左辺成功時だけ到達
||の右辺不可左辺失敗時に到達
ifのthenブロック可条件がtrue
対応するelseブロック不可条件がfalse
否定判定後、失敗側がreturnした後可残る経路は照合成功のみ
