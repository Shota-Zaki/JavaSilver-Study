Java実行・宣言ルール

main、ファイル名、package/import、識別子をまとめて確認。

実行可能なクラス

確認項目正しい形間違いやすい点
mainメソッドpublic static void main(String[] args) / public static void main(String... args)String args[] も配列として成立するが、戻り値void、static、引数の型が崩れると起動対象にならない。
ファイル名public class A は A.java に保存publicでないトップレベルクラスは同一ファイルに複数書ける。publicは1つだけ。
javaコマンドjava Main a bargs[0] が a、args[1] が b。クラス名に .class は付けない。
javacコマンドjavac -d build src/ex/Main.java-d は出力先。package宣言に従ってディレクトリが作られる。

package / import

ルール例注意
package宣言は先頭package ex.sample;コメント以外をpackageより前に書けない。importより前。
importは型名を省略するだけimport java.util.ArrayList;importしてもクラス自体が作られるわけではない。
* はそのパッケージ直下だけimport java.util.*;java.util.concurrent は含まない。サブパッケージは別。
java.langは自動importString, System, Integerjava.util.Listなどは自動ではない。
同名クラス衝突java.util.Date と java.sql.Date両方を単純名で同時使用できない。片方は完全修飾名を使う。

識別子・予約語・コメント

項目OKNG / 注意
識別子の開始$name, _name, name11name は不可。予約語も不可。
var変数名として使える場面があるローカル変数型推論として使う場合は初期化必須。
コメント//, /* */, /** */文字列リテラル中の // はコメントではない。
