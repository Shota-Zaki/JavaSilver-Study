Java実行・宣言ルール

main、ソースファイル名、package/import、起動引数を、コンパイル可否と実行可否の順で判定するためのページです。

まず見る順番

この分野は暗記で押し切るより、チェック順を固定した方が速いです。問題文に複数のファイル・パッケージ・コマンドが出る場合、次の順で潰してください。

  1. ソースファイル名: public トップレベルクラス名と一致しているか。
  2. package宣言: あるならコメントを除いた先頭にあるか。
  3. import宣言: package の後、クラス宣言の前にあるか。
  4. mainメソッド: 起動対象クラスに正しい形式の main があるか。
  5. javaコマンドの引数: クラス名と起動引数を取り違えていないか。
最初から出力を追うのは遅いです。 先にコンパイル不可・起動不可を潰してください。出力計算は最後です。

ソースファイル名とトップレベルクラス

1つのソースファイルには複数のトップレベルクラスを書けます。ただし、public なトップレベルクラスは最大1つで、ファイル名はそのクラス名と一致する必要があります。

// A.java
        public class A {}
        class B {}

この場合、javac A.java で A.class と B.class が生成されます。B に正しい main があれば java B で起動できます。public class B を同じファイルに追加するとコンパイルエラーです。

OK

A.java に public class A と非publicの class B。

NG

Sample.java に public class Main。名前不一致。

NG

1ファイルに public class A と public class B。

package / importの位置

package 宣言は、コメントを除いてソースファイルの先頭に1回だけ書けます。import は package の後、クラス宣言の前です。

package ex.sample;

        import java.util.ArrayList;
        import java.util.*;

        public class Main {}

import より前に package が必要です。import が先に出てきたら、それだけでコンパイルエラー候補です。

java.lang と同一パッケージのクラスは自動的に見えますが、サブパッケージは自動では見えません。java.util.* で見えるのは java.util 直下のクラスだけです。

importのワイルドカードとstatic import

通常の import はクラス名を省略するためのものです。ワイルドカード * はパッケージ直下のクラスにだけ効き、パッケージ階層の途中には使えません。

import java.util.*;        // OK。java.util直下
        // import java.*;          // java.util.ArrayListまでは届かない
        // import ex.*.Sample;     // NG。途中に*は置けない

import static は、クラスの static メンバを省略して使うための宣言です。クラスそのものを読み込む宣言ではありません。

import static java.lang.Math.PI;
        import static java.lang.Math.*;

        class Main {
            double area(double r) { return PI * r * r; }
        }

mainメソッドの成立条件

アプリケーションのエントリーポイントになる main は、次の条件を満たします。

条件必要な形よくある誤り
アクセス修飾子publicprivate static void main
クラスメソッドstaticインスタンスメソッドにする
戻り値voidint main
引数String[] args または String... argsString args、int[] args
public static void main(String[] args) {}
        public static void main(String... args) {}

args という名前は変更できます。重要なのは型です。

起動引数の見方

java Main java one two の Main は起動対象クラス名であり、args には入りません。args[0] は java、args[1] は one です。

// command: java Main java one two
        System.out.println(args[0] + " " + args[1]); // java one
起動引数なしなら args は null ではなく、要素数0の配列です。args.length は0、args[0] は実行時例外です。

ミニ判定

Q1. public static int main(String[] args) は起動用mainになる？

なりません。戻り値が void ではないためです。普通のstaticメソッドとしては定義できますが、エントリーポイントではありません。

Q2. package より前に書けるものは？

コメントだけです。import、変数宣言、クラス宣言などは不可です。

Q3. java Sample 5 2 の args.length は？

2です。Sample は起動対象クラス名で、引数には含まれません。
