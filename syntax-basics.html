Java実行・package/import

Javaファイル、main、package/import、javac/javaの最小ルールを整理します。

mainメソッド

形式可否注意
public static void main(String[] args)OK標準形。
public static void main(String... args)OK可変長引数。String[] と同等に扱える。
static public void main(String[] args)OK修飾子順は問わない。
public void main(String[] args)NGstaticでないため起動用mainではない。
public static int main(String[] args)NG戻り値はvoidが必要。

package/importの順序

// 1. package宣言があるなら最初。コメントは前に書ける。
package ex.sample;

// 2. import宣言
import java.util.ArrayList;

// 3. class/interface/recordなど
public class Main { }
  • packageより前にimportを書くとコンパイルエラー。
  • java.lang は自動importされる。
  • import ex.*; はex直下の型だけ。サブパッケージまでは対象外。
  • import static はクラスのstaticメンバ用。通常のクラスimportとは違う。

javac / java

目的例見る点
コンパイルjavac -d build src/ex/Main.java-dで出力先を指定できる。packageに応じたフォルダが作られる。
実行java -cp build ex.Mainクラス名は完全修飾名。.classや.javaは付けない。
起動引数java Main A Bargs[0]がA、args[1]がB。
