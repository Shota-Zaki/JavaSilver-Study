Java実行・宣言ルール

ファイル名、package/import、mainメソッド、コマンド実行、起動引数を、試験で使う順番で整理します。

最初に見る判定順

Java Silverのコード問題は、出力結果を追う前に「そのコードが実行できる状態か」を潰します。実行結果だけを先に考える癖があると、package宣言、publicクラス名、mainメソッド、static文脈で簡単に落ちます。

1. ファイル単位

publicなトップレベルクラスがあるなら、ソースファイル名はそのクラス名と一致します。複数の非publicクラスは1ファイルに書けます。

2. 宣言順

package 宣言は先頭。コメントを除き、import より前です。import の後に package は書けません。

3. エントリーポイント

public static void main(String[] args) または public static void main(String... args) が基本形です。

4. 実行時引数

java Main a b なら args[0] は "a"。クラス名 Main は配列に入りません。

判断を急ぐな。コンパイルできないコードに実行結果は存在しません。 まず宣言・修飾子・型・到達不能・例外処理を確認してから、出力順を追います。

package / import

packageはクラスの所属を決めます。importは別パッケージの型名を短く書くための宣言です。importしてもクラスが読み込まれるわけではなく、単に名前解決を補助します。

package app.sample;     // 先頭に書く

import java.util.ArrayList;
import java.util.*;       // java.util直下の型だけ。サブパッケージは対象外

public class Main { }

java.lang

String、System、Integer など java.lang の型は自動インポートされます。

同じパッケージ

同じパッケージ内のクラスはimport不要です。別パッケージならimportまたは完全修飾名が必要です。

ワイルドカード

import java.util.*; は java.util.ArrayList などに効きますが、java.util.stream.* までは含みません。

static import

import static は型ではなく、staticメンバを短く書くためのものです。通常のクラスimportと混同しないこと。

mainメソッドの形

Javaコマンドから起動されるmainは、外部から呼ばれるため public、インスタンス生成なしで呼ばれるため static、戻り値を受け取らないため void です。引数は String[] または String... です。

public class Main {
    public static void main(String[] args) {
        System.out.println(args.length);
    }
}

class SubMain {
    public static void main(String... args) {
        System.out.println("OK");
    }
}

1つのソースファイルに複数のクラスを書き、それぞれが正しいmainを持つ場合、コンパイル後は各クラスを指定して実行できます。ただしpublicクラス名とファイル名の一致条件は別問題です。

コマンド実行で落とされる点

javac はソースファイルをコンパイルし、java はクラス名を指定して実行します。実行時に .class や .java を付ける選択肢は基本的に誤りです。

javac Main.java
java Main one two

// args[0] == "one"
// args[1] == "two"
  • java Main java one two なら args[0] は "java"。
  • 起動引数が0個でも args は通常 null ではなく、長さ0の配列です。
  • args[0] を参照した時点で、長さ0なら ArrayIndexOutOfBoundsException です。
