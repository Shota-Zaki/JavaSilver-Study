Java実行・package/import

最初の章で落とされるのは、Javaの実行単位、ファイル名、main、package/importの順序です。

public class とファイル名

条件正しい理解試験での罠
publicなトップレベルクラスソースファイル名と一致する必要がある。public class MainならMain.java。
publicでないクラス1ファイルに複数書ける。mainがあればjavaコマンドで実行対象にできる場合がある。
トップレベルクラスprivate/protected/staticは基本不可。static classはインナークラスで使う。

mainメソッド

形可否注意
public static void main(String[] args)可標準形。
public static void main(String... args)可可変長引数でも可。
static public void main(String[] args)可修飾子の順序は固定ではない。
public void main(String[] args)不可staticがないため開始点ではない。
public static int main(String[] args)不可戻り値がvoidではない。
public class Main {
  public static void main(String[] args) {
    System.out.println(args.length);
  }
}

package / import の順序

// コメントは可
package com.example;

import java.util.ArrayList;

public class Sample {}
項目ルール
packageある場合はソースの先頭。importより前。
importpackageの後、クラス宣言の前。
java.langimport不要。String/System/Integerなど。
ワイルドカードimport java.util.*; はjava.util直下だけ。サブパッケージは含まない。
static importstaticメンバを短く書くため。クラスそのもののimportではない。
