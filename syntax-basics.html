Java実行・宣言ルール

mainメソッド、ソースファイル名、package/import、起動引数を、コンパイル可否の観点で整理します。

実行できるクラスの条件

Javaコマンドで直接実行するには、実行対象クラスに正しい main メソッドが必要です。戻り値、static、引数、名前のどれかが違うと、起動用メソッドとして扱われません。

要素正しい形よくある誤り
メソッド名mainMain, mane
戻り値voidint, 戻り値省略
修飾子public staticstaticがない、publicがない
引数String[] args または String... argsString args, int[] args
public class Sample {
    public static void main(String[] args) {
        System.out.println("OK");
    }
}

publicクラスとファイル名

1つのソースファイルに複数のトップレベルクラスを書けます。ただし、public なトップレベルクラスは1つまでで、ファイル名と一致する必要があります。

// Sample.java
public class Sample { }
class Helper { }

これは可能です。Helper は public ではないので、同じファイルに置けます。

// Main.java
public class Sample { } // NG: publicクラス名とファイル名が一致しない

packageとimportの順序

パッケージ宣言は、コメントを除いてソースファイルの先頭に置きます。import は package の後、クラス宣言の前です。

順序書けるもの
1コメント
2package 宣言
3import 宣言
4クラス・interface・recordなどの宣言
package com.example;

import java.util.ArrayList;

public class Sample { }
import を package より前に書く、またはJavaに存在しない「packageブロック」を作るとコンパイルエラーです。

importで省略できるもの

java.lang パッケージと同じパッケージ内のクラスはimport不要です。java.util.* のワイルドカードは、そのパッケージ直下のクラスだけが対象です。サブパッケージまでは含みません。

import java.util.*;       // ArrayListは使える
// import java.util.stream.*; は別途必要

ArrayList<String> list = new ArrayList<>();

起動引数

main(String[] args) の args は、コマンドライン引数を文字列配列として受け取ります。引数がなければ null ではなく、長さ0の配列です。

// java Sample A B
System.out.println(args.length); // 2
System.out.println(args[0]);     // A
System.out.println(args[1]);     // B

引数なしで args[0] を読むと ArrayIndexOutOfBoundsException です。

ミニ演習

Q. static public void main(String... args) は有効？

有効です。修飾子の順序は自由で、String... は可変長引数として String[] と同等に起動用mainとして使えます。

Q. import java.util.*; で java.util.stream.Stream は使える？

使えません。ワイルドカードは直下の型だけで、サブパッケージは対象外です。

mainメソッドの微妙な許容範囲

mainメソッドは形が固定に見えますが、修飾子の順序や可変長引数は許容されます。逆に、戻り値や引数型が違うものは、同名でも起動用mainではありません。

コード起動用mainか理由
public static void main(String[] args)可標準形
static public void main(String... args)可修飾子順序自由、可変長引数可
public void main(String[] args)不可staticがない
public static int main(String[] args)不可戻り値がvoidではない
public static void main(String args)不可引数が配列/可変長ではない

ファイル名問題の切り方

ファイル名問題では、publicトップレベルクラスだけを探します。publicでないトップレベルクラスは、ファイル名と一致しなくても同じファイルに置けます。

// A.java
public class A { public static void main(String[] args) {} }
class B { public static void main(String[] args) {} }

この場合、javac A.java で A.class と B.class が生成されます。Bにも正しいmainがあれば java B で実行可能です。
