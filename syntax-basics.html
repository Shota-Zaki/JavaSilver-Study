Java実行・宣言ルール

mainメソッド、ファイル名、package/import、classpath、起動引数の基本をまとめます。

Javaプログラムの実行単位

Javaはソースファイルをそのまま実行するのではなく、まずjavacでコンパイルしてclassファイルを作ります。その後、javaコマンドでJVM上に読み込んで実行します。試験では、コンパイル時に落ちるのか、実行時に落ちるのかを分けて見る必要があります。

// Sample.java
public class Sample {
    public static void main(String[] args) {
        System.out.println("OK");
    }
}

// コマンド
javac Sample.java
java Sample
publicなトップレベルクラスがある場合、ソースファイル名はそのpublicクラス名と一致する必要があります。public class Sample なら Sample.java です。

mainメソッド

通常の実行開始点として認識されるmainメソッドは、基本的に次の形です。

public static void main(String[] args) { }
public static void main(String... args) { }

public

JVMが外部から呼び出せる必要があります。省略すると実行開始点として扱えません。

static

インスタンスを作らずに呼び出すために必要です。

void

戻り値は返しません。int main などはJavaのmainとしては不正です。

String[] args / String... args

起動パラメータをString配列として受け取ります。String... は可変長引数ですが、実体は配列です。

起動パラメータはすべてStringです。数値として使うには変換が必要です。

public class Main {
    public static void main(String[] args) {
        System.out.println(args[0]);
        int num = Integer.parseInt(args[1]);
        System.out.println(num + 10);
    }
}

// java Main A 20
// args[0] = "A"
// args[1] = "20"
起動パラメータを何も渡していないのに args[0] を読むと、コンパイルエラーではなく実行時に ArrayIndexOutOfBoundsException です。

package と import

package宣言は、そのクラスがどのパッケージに属するかを示します。書く場合はソースファイルの先頭、importより前です。package宣言より前に書けるのはコメントだけです。

package ex.sample;

import java.util.ArrayList;

public class Main {
}

package宣言

書くなら先頭。1ファイルに1つだけ。

import宣言

package宣言の後、クラス宣言の前に書きます。

java.lang

String、System、Object などは自動importされます。

ワイルドカード

import java.util.*; はjava.util直下のクラスだけ。サブパッケージまでは含みません。
import java.util.*;       // ArrayList は使える
import java.util.stream.*; // streamパッケージは別に必要
ワイルドカードは「クラス名」の部分にだけ使えます。import ex.*.A; のようにパッケージ途中では使えません。
