Java Silver Reference

ミニ演習

短いコードで、Java Silverの判定順を確認します。答えだけでなく、なぜそうなるかを1つずつ潰します。

演習1: var

class Sample {
    private var value = 10;
}

フィールドに var は使えないためコンパイルエラーです。varはローカル変数限定です。

演習2: String

String s = "abc";
s.concat("d");
System.out.println(s);

abc が出力されます。Stringは不変で、concat の戻り値を受け取っていません。

演習3: オーバーライド

interface A {
    void test();
}
class B implements A {
    void test() { }
}

コンパイルエラーです。interfaceの抽象メソッドはpublicなので、実装メソッドもpublicが必要です。

演習4: try-with-resources

try (A a = new A("A"); A b = new A("B")) {
    System.out.println("C");
}
// closeは名前を出力するものとする

出力順は C B A です。リソースは宣言と逆順にcloseされます。

演習5: キャスト

Object obj = "abc";
Integer n = (Integer) obj;
System.out.println(n);

コンパイルは通りますが、実行時に ClassCastException です。実体はStringでありIntegerではありません。
