Java Silver Study

Java Silver Reference

ミニ演習 強化版

短いコードを使って、コンパイル可否・実行時例外・出力を即判定する練習ページです。

使い方

答えを開く前に、必ず「コンパイル可否 → 実行時例外 → 出力」のどこに分類されるかを口に出してください。分類できないまま答えを見ると、同じ論点でまた落ちます。

目標：1問あたり20秒以内。遅い問題は知識ではなく判定手順が曖昧です。

型・リテラル

byte b = 127; b++; System.out.println(b);

コンパイル可。出力は -128。byteの範囲でオーバーフローします。

byte b = 127; b = b + 1;

コンパイルエラー。b + 1 は int。

byte b = 127; b += 1;

コンパイル可。複合代入は暗黙キャストを含む。

short s = 10 + 20;

コンパイル可。int定数式で範囲内。

short s = 10; s = s + 20;

コンパイルエラー。変数を含む演算結果はint。

float f = 10;

コンパイル可。intからfloatは暗黙変換可。

float f = 10.0;

コンパイルエラー。10.0はdouble。

int x = 012;

コンパイル可。8進数なので10進で10。

int x = 09;

コンパイルエラー。8進数に9は使えない。

var x = null;

コンパイルエラー。型推論できない。

String・null

String s = "A"; s.concat("B"); System.out.println(s);

A。Stringは不変で戻り値を捨てている。

StringBuilder sb = new StringBuilder("A"); sb.append("B"); System.out.println(sb);

AB。StringBuilderは可変。

String s = null; System.out.println(s);

null と表示。メソッド呼び出しではない。

String s = null; System.out.println(s.length());

NullPointerException。

String a = new String("x"); String b = "x"; System.out.println(a == b);

false。newで別インスタンス。

String a = new String("x").intern(); String b = "x"; System.out.println(a == b);

true。internでプール参照。

配列・List

int[] a = new int[0]; System.out.println(a.length);

0。長さ0の配列は作れる。

int[] a = new int[0]; System.out.println(a[0]);

ArrayIndexOutOfBoundsException。

String[] s = new String[1]; System.out.println(s[0]);

null。参照型配列の要素初期値。

int[] a = {1,2}; int[] b = a; b[0] = 9; System.out.println(a[0]);

9。同じ配列を参照。

List<String> list = List.of("A"); list.add("B");

コンパイル可、実行時にUnsupportedOperationException。

for (String x : list) { x = "Z"; }

ループ変数を差し替えてもListの要素自体は差し替わらない。

演算子・制御

int a=1; int b=a++ + ++a; System.out.println(a + "," + b);

3,4。a++は1を渡して2、++aは3を渡す。

int a=0; if (false && ++a > 0) {} System.out.println(a);

0。&&短絡で右辺は評価されない。

int a=0; if (false & ++a > 0) {} System.out.println(a);

1。&は右辺も評価する。

do { System.out.println("A"); } while(false);

Aを1回表示。do-whileは最低1回。

switchでbreakなし

一致case以降をフォールスルーする。

switch式のブロックで値を返す

returnではなくyieldを使う。

メソッド・コンストラクタ

void Sample() {}

戻り値型があるためコンストラクタではなくメソッド。

コンストラクタ内で this(); の前に文を書く

コンパイルエラー。this()/super()は最初の文。

void m(int... a, String s) {}

コンパイルエラー。varargsは最後。

int m(int a) と long m(int a)

コンパイルエラー。戻り値だけではoverload不可。

親のpublicメソッドを子でprotected override

コンパイルエラー。アクセスを狭められない。

継承・interface

interface A { void run(); } class B implements A { void run(){} }

コンパイルエラー。実装メソッドはpublicが必要。

Parent p = new Child(); p.childOnly();

コンパイルエラー。宣言型ParentにchildOnlyがない。

Parent p = new Child(); p.overrideMethod();

宣言型にメソッドがあれば、実行されるのはChild側。

フィールドを親子で同名定義

フィールドはoverrideされず、宣言型で解決。

abstract class A implements I でIのメソッド未実装

OK。抽象クラスは未実装を残せる。

Java 17

record Person(String name) のアクセサ

getName()ではなくname()。

recordをextendsする通常クラス

不可。recordは暗黙的にfinal。

sealed class A permits B; class B extends A {}

コンパイルエラー。Bはfinal/sealed/non-sealedが必要。

if (obj instanceof String s) { } の外でsを使う

基本的にスコープ外で使えない。

switch式の全分岐で値が返らない

コンパイルエラー。

例外

catch(Exception e) の後に catch(RuntimeException e)

コンパイルエラー。後続catchが到達不能。

throw new IOException(); をcatch/throwsなし

コンパイルエラー。checked例外。

throw new RuntimeException(); をcatch/throwsなし

コンパイル可。非チェック例外。

try内で例外後の文

実行されず、対応catchへ移る。

try-with-resourcesで A a; B b;

close順は b → a。宣言の逆順。
