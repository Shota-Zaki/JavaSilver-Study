直前確認

模擬問題や試験直前に、全単元の判定基準だけを短時間で確認する総仕上げページです。

直前確認の使い方

このページは新知識を増やすためではなく、解答時の判定順を固定するためのページです。試験直前に全部を深く読み込むより、ここを使って「どの順番で切るか」を固めてください。

1コンパイル可否

宣言・型・アクセス・override・例外処理。

2実行時例外

null・境界外・cast・parse・変更不可。

3出力

評価順・参照・動的束縛・finally。

String・StringBuilder・比較

  • String は不変。s.replace(...) の戻り値を受けないと元は変わらない。
  • StringBuilder は可変。append、reverse、delete などは自分を変える。
  • == は参照比較。内容比較は原則 equals。
  • StringBuilder は equals を内容比較としてオーバーライドしていない。
  • "a" + "b" はコンパイル時に "ab" になり得る。変数を含むと実行時結合。
  • intern() は文字列プール上の参照を返す。
  • substring(begin, end) はbegin以上end未満。
  • indexOf は見つからなければ -1。

演算子・制御構造

  • 後置 a++ は古い値を使ってから増やす。前置 ++a は増やしてから使う。
  • && と || は短絡評価。右辺が評価されないことがある。
  • & と | はbooleanにも使えるが、右辺も評価する。
  • 代入式は値を持つ。int b = a += 5; のような式に注意。
  • String 結合は左から評価。途中で文字列になったら以後は連結。
  • switch のcaseはコンパイル時定数。long、boolean などは制御式に使えない。
  • switch 文の : は break がなければフォールスルー。
  • switch 式のブロックでは yield。

クラス・メソッド・コンストラクタ

項目直前チェック
コンストラクタ戻り値型なし。継承されない。最初に this() または super()。
デフォルトコンストラクタコンストラクタを1つも書かない場合だけ自動生成。
overload引数リストで決まる。戻り値だけ違うのは不可。
override同じシグネチャ。戻り値は同じか共変。アクセスは狭めない。
staticstaticからインスタンスメンバを直接参照できない。
値渡し参照型でも参照値のコピーを渡す。呼び出し元変数の参照先差し替えはできない。
GCどこからも参照されないインスタンスが対象。実行タイミングは保証されない。

継承・interface・ポリモーフィズム

  • コンストラクタと private メンバは継承されない。
  • 抽象メソッドを実装しない具象クラスはコンパイルエラー。
  • interfaceの抽象メソッドは暗黙に public abstract。
  • interfaceのフィールドは暗黙に public static final。
  • defaultメソッドの衝突は実装クラス側で解決が必要。
  • 呼べるメソッドは宣言型で決まる。実行されるoverrideメソッドは実体で決まる。
  • フィールドはoverrideされない。宣言型で決まる。
  • staticメソッドもoverrideではなく隠蔽。
  • ダウンキャストはコンパイル可でも、実体が合わなければ ClassCastException。

例外

最後に見る15項目

  1. 出力を追う前にコンパイル可否を見る。
  2. ローカル変数にデフォルト値はない。
  3. byte/short/char の演算結果は基本 int。
  4. float f = 1.0; は不可。1.0F。
  5. var はローカル変数のみ。
  6. String は不変、StringBuilder は可変。
  7. == は参照比較、equals は実装確認。
  8. 配列は固定長、ArrayListは可変長。
  9. 後置 ++ は古い値、前置 ++ は新しい値。
  10. && と || は右辺を評価しないことがある。
  11. コンストラクタは継承されない。
  12. overrideでアクセスを狭められない。
  13. フィールドはoverrideされない。
  14. checked例外はcatchまたはthrows。
  15. try-with-resourcesのcloseは逆順。
