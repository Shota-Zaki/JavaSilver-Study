直前確認チートシート

試験直前に見る前提で、頻出の判断基準だけを圧縮しています。

コンパイル判定

  • ローカル変数は使用前に必ず初期化。
  • var はローカル変数のみ。フィールド・引数・戻り値不可。
  • 整数リテラルは基本 int、小数は基本 double。
  • byte/short/char の演算結果は基本 int。
  • interface実装メソッドは public が必要。
  • overrideでアクセスを狭めない。
  • 親例外catchを子例外catchより先に書かない。

実行時例外

  • null参照でメソッド呼び出し: NullPointerException。
  • 配列範囲外: ArrayIndexOutOfBoundsException。
  • 文字列範囲外: StringIndexOutOfBoundsException。
  • 不正キャスト: ClassCastException。
  • 数値変換失敗: NumberFormatException。
  • List.of の変更: UnsupportedOperationException。

出力判定

  • String は不変。戻り値を捨てたら変わらない。
  • StringBuilder は可変。append/reverse は本体を変える。
  • == は参照比較、equals は内容比較。
  • オーバーライドメソッドは実体で決まる。
  • フィールドは宣言型で決まる。
  • &&/|| は短絡評価。&/| は短絡しない。
  • try-with-resourcesのcloseは宣言の逆順、finallyより前。

Java 17固有寄り

  • recordは暗黙にfinal。アクセサ名は name() 形式。
  • sealedの許可サブクラスは final/sealed/non-sealed を明示。
  • instanceof String s の s はtrue側で使える。
  • switch式は値を返す。すべての経路で値が必要。
