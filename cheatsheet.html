直前確認

試験直前に確認する細かい仕様、引っかけ、判定順をまとめます。

直前に見るべき仕様

  • String は不変。戻り値を受け取らないと結果は捨てられる。
  • StringBuilder は可変。appendなどは自分自身を返す。
  • == は参照比較。equals はクラス実装による内容比較。
  • byte/short/char は演算時にintへ昇格。
  • += は暗黙キャストを含む。
  • var はローカル変数だけ。フィールドや引数では不可。
  • do-while は最低1回実行し、最後にセミコロンが必要。
  • break はfor更新式へ行かない。continue はfor更新式へ行く。
  • switch式 は値を返す。ブロック内では yield。
  • interfaceの抽象メソッドはpublic。実装側もpublic。
  • recordのアクセサは name()。getName() ではない。
  • sealedの子は final / sealed / non-sealed が必要。

よくあるミス

String s = "abc";
s.replace("a", "x");
System.out.println(s); // abc

byte b = 1;
// b = b + 1; // NG
b += 1;       // OK

record Person(String name) {}
Person p = new Person("A");
// p.getName(); // NG
p.name();       // OK
