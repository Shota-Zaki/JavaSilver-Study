型・リテラル

項目要点
byte-128〜127。整数定数が範囲内なら代入可。
short-32768〜32767。演算すると基本int。
char0〜65535。数値としても扱えるが演算ではint。
float1.0F が必要。1.0 はdouble。
long10L。小文字lは見間違いが多い。
boolean数値と互換性なし。0/1 は使えない。
varローカル変数のみ。null単独、配列初期化子単独、ラムダ単独は不可。

文字列・配列・List

  • String は不変。戻り値を受けないと変わらない。
  • StringBuilder は可変。append等は自分を変える。
  • == は参照比較。equals は内容比較。
  • intern() はプール上の参照を返す。
  • 配列の長さは length、Stringの長さは length()。
  • 配列は固定長。ArrayListは要素数を変えられる。
  • Arrays.asList と List.of は変更可否が違う。
  • 拡張forのループ変数代入では元要素は差し替わらない。

オブジェクト指向

項目要点
コンストラクタ継承されない。戻り値型を書いたらメソッド。
overload引数リストが違う。同じ引数で戻り値だけ違うのは不可。
override同じシグネチャ。戻り値は同じか共変戻り値。アクセスは狭めない。
staticインスタンスなしで使える。staticからinstanceを直接読めない。
フィールドoverrideされない。宣言型で決まる。
メソッドoverrideされたインスタンスメソッドは実体で決まる。
interface抽象メソッドはpublic abstract扱い。フィールドはpublic static final扱い。
abstract抽象クラスはインスタンス化不可。抽象メソッド未実装なら子もabstract。

Java 17系

  • record: アクセサは name() 型。暗黙的にfinal。
  • sealed: permits で許可。サブクラス側は final/sealed/non-sealed 必須。
  • switch式: 値を返す。ブロックでは yield。
  • instanceof pattern: trueの範囲でだけパターン変数が使える。

例外

項目要点
checked例外catchまたはthrowsが必要。未処理はコンパイルエラー。
RuntimeException非チェック例外。処理しなくてもコンパイル可。
catch順子から親。親が先だと後続catchが到達不能。
finally例外の有無に関わらず基本実行。
try-with-resourcescloseは宣言の逆順。finallyより前。

さらに深掘り：直前に見る最小チェック

  • String は不変、StringBuilder は可変。
  • == は参照比較、equals は実装依存の内容比較。
  • ローカル変数にデフォルト値はない。
  • byte/short/char の演算結果は基本 int。
  • var はローカル変数のみ。null 単独不可。
  • interface の抽象メソッドは暗黙に public。
  • オーバーライドでアクセスを狭くできない。
  • チェック例外はcatchまたはthrowsが必要。
  • try-with-resourcesはcloseが逆順。
  • 出力追跡はコンパイル可否を確認した後。
