エラー・例外カタログ完全版

コンパイルエラーと実行時例外を、原因・コード形・試験での見分け方に分けて整理します。

このページの使い方

「なぜ間違えたか」を、単なる不注意で終わらせないためのページです。エラー名を覚えることより、どのコード形で起きるかを覚える方が点につながります。

分類：コンパイルエラーは実行前に止まる。RuntimeException系は実行中に発生する。checked例外は処理漏れがコンパイルエラーになる。

例外階層で混乱しやすい点

Exception

多くの例外の親。catch順では後ろに置く。

RuntimeException

非チェック例外。catch/throwsがなくてもコンパイルは通る。

IOExceptionなど

checked例外。処理しないとコンパイルエラー。

Error

通常アプリで処理対象にしない重大な問題。Exceptionとは別系統。

見分けミニ問題

String s = null; System.out.println(s.toString());

実行時に NullPointerException。println(s) ではなく、先に s.toString() を呼んでいる。

int[] a = {{1,2}};

コンパイルエラー。左辺は1次元配列なのに、右辺は2次元配列の初期化子。

List<String> list = List.of("A"); list.add("B");

コンパイルは通るが実行時に UnsupportedOperationException。

Object o = new StringBuilder("A"); String s = (String)o;

コンパイルは通るが実行時に ClassCastException。実体はStringBuilder。
