エラー・例外カタログ

コンパイルエラーと実行時例外を、原因・例・試験で見る場所に分けて整理します。

コンパイルエラーと実行時例外の境界

コンパイルエラーは「実行前に文法・型・宣言上の矛盾が見つかる」ものです。実行時例外は「文法上は正しいが、実行して初めて不正な状態になる」ものです。

コンパイルエラー

種類例見るポイント
未初期化ローカル変数int a; System.out.println(a);ローカル変数にデフォルト値はない。
型不一致byte b = 128;範囲外、long/doubleを小さい型へ代入。
boolean以外を条件に使うif (1) {}Javaの条件式はbooleanのみ。
チェック例外未処理throw new IOException();catchまたはthrowsが必要。
catch順序不正Exceptionを先にcatchし、その後にIOException狭い例外から広い例外へ。
staticから非static参照static void m(){ field; }インスタンスなしでインスタンスメンバは使えない。
オーバーライド不正アクセスを狭くする、戻り値が非互換override条件を満たさない。

実行時例外

例外例見るポイント
NullPointerExceptionString s = null; s.length();参照先がないのに操作する。
ArrayIndexOutOfBoundsExceptionnew int[2][2] に [2]添字は0からlength-1。
ClassCastExceptionObject o = "x"; Integer i = (Integer)o;参照型の実体を見る。
NumberFormatExceptionInteger.parseInt("12A")変換不能文字列。
ArithmeticException10 / 0整数除算のみ。doubleはInfinity等。

例題比較

int[] a = new int[0];
System.out.println(a[0]);

実行時例外。 配列は作れる。実行時に範囲外アクセス。

int a;
System.out.println(a);

コンパイルエラー。 ローカル変数が初期化されていない。
