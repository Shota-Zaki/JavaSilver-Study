エラー・例外カタログ

コンパイルエラーと実行時例外を、原因・典型コード・切り方で逆引きします。

コンパイルエラー早見

原因典型コード判断
未初期化ローカル変数int a; println(a);ローカル変数にはデフォルト値なし
型不一致float f = 1.0;小数リテラルはdouble
小さい整数型の演算byte c = a + b;演算結果はint
override違反アクセスを狭くする同じか広くする必要
catch順序不正catch(Exception) の後に catch(IOException)後続catchが到達不能

実行時例外早見

例外典型原因試験での見分け方
NullPointerExceptionnull参照でメソッド/フィールドアクセスprintln(null) は例外ではない点に注意
ArrayIndexOutOfBoundsException配列範囲外アクセス長さ0配列の [0]
StringIndexOutOfBoundsException文字列の範囲外charAt, substring
ClassCastException実体と合わないダウンキャストコンパイル可でも実行時に落ちる
NumberFormatException数値変換できない文字列Integer.parseInt("A")

迷ったときの切り方

  1. コンパイルエラー候補を先に潰す。
  2. コンパイル可能なら、左から実行して最初に発生する例外を見る。
  3. 例外がcatchされるなら、catch後のfinallyや後続処理を見る。
  4. 例外がcatchされないなら、そこで異常終了。後続出力はない。
