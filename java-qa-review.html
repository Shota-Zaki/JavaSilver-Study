質問まとめ

これまで質問していた論点を、学習記事の関係順に再配置しました。

try-catch-finally

try内で例外が発生したら、対応するcatchへ移ります。例外発生後のtry内の残りは実行されません。finallyは例外の有無にかかわらず原則実行されます。

interface Worker { void work(); }

これは「Workerとして扱うならwork()を呼べる」という契約です。interface内の抽象メソッドはpublic abstract扱いなので、実装クラスではpublicで実装します。

抽象メソッド

本体を持たないメソッドです。抽象クラスやinterfaceに定義し、具象クラスで実装します。抽象クラスは未実装のままでもよいですが、最終的な具象クラスでは実装が必要です。

配列宣言と初期化

int[] a;              // 宣言だけ。要素数なし
int[] b = new int[3]; // 生成時に要素数
int[] c = {1,2,3};    // 宣言と同時なら初期化子OK

intern()

intern() は文字列定数プール上の同値文字列の参照を返します。new String("x") はヒープ上に別インスタンスを作るため、== では通常falseです。

ラベル

ラベルは文に付けられます。ループに付けると、break label や continue label で外側制御に使えます。変数宣言やメソッド宣言のラベルと混同しないこと。
