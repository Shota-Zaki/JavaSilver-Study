try-catch-finally：例外が出た後にtryの続きは実行されるか

try {
  System.out.println("A");
  int[] a = {};
  a[0] = 10;
  System.out.println("B");
} catch (ArrayIndexOutOfBoundsException e) {
  System.out.println("C");
} finally {
  System.out.println("D");
}

出力は A C D です。例外が発生した時点で、同じtryブロック内の残りは実行されません。対応するcatchへ移り、その後finallyが実行されます。

場面実行されるもの実行されないもの
try内で例外なしtryの最後まで → finallycatch
try内でcatch可能な例外例外発生前のtry → catch → finally例外発生後のtry残り
catchできない例外例外発生前のtry → finallycatch、例外発生後のtry残り

interface Worker { void work(); } は何を意味するか

interface Worker {
  void work();
}

これは「Worker型として扱うならwork()を呼べる」という契約です。インタフェース内の抽象メソッドは暗黙に public abstract です。

class Engineer implements Worker {
  @Override
  public void work() {
    System.out.println("develop");
  }
}

Worker w = new Engineer();
w.work();

Worker 型の変数では、Workerに定義されたメソッドだけを呼べます。実行時に動くのは、実体である Engineer の実装です。

抽象メソッドと抽象クラス

abstract class Employee {
  abstract void work();
}

class Engineer extends Employee {
  @Override
  void work() {
    System.out.println("engineer");
  }
}

抽象メソッドは本体を持たないメソッドです。通常クラスが抽象メソッドを継承した場合、そのメソッドを実装しなければコンパイルエラーです。実装しないなら、そのクラス自身も abstract にします。

クラス種別抽象メソッドを持てるかnewできるか
通常クラス不可可
抽象クラス可不可
インタフェース可。通常は暗黙にpublic abstract不可

配列宣言と初期化

int[] a;          // OK。まだ配列インスタンスはない
int b[];          // OK。ただしJavaではint[] aが読みやすい
int[] c = new int[3];
int[] d = {1, 2, 3};
// int[3] e;      // NG。宣言時に要素数は書かない
// int f[3];      // NG

int[] a; の時点では要素数はありません。変数が宣言されただけです。要素数は new int[3] のように配列インスタンス生成時に決まります。

intern() とString比較

String a = new String("java");
String b = a.intern();
String c = "java";

System.out.println(a == b); // false
System.out.println(b == c); // true
System.out.println(a.equals(c)); // true

intern() は文字列プール上の同じ内容のString参照を返します。== は参照比較、equals は内容比較です。

ラベル、break、continue

outer:
for (int i = 0; i < 3; i++) {
  for (int j = 0; j < 3; j++) {
    if (j == 1) break outer;
    System.out.println(i + ":" + j);
  }
}

break outer; はouterラベルが付いた文全体を抜けます。continue outer; はouterラベルが付いたループの次の周回へ進みます。ラベルは「ループ専用」ではなく文に付けられますが、continue の飛び先はループでなければいけません。

コンパイルエラーと実行時例外の切り分け

  1. まず型・宣言・修飾子を見る。ここで不正なら実行されない。
  2. 次に未初期化ローカル変数、到達不能コード、catch順序を見る。これもコンパイルエラー。
  3. コンパイルできる前提になってから実行時の値を追う。null、配列範囲外、キャスト失敗、0除算などを見る。
