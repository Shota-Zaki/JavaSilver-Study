関係順ロードマップ

重複していた内容は、各論点の主ページに寄せました。読む順番は「判定手順 → 基本文法 → 型 → 制御 → 文字列/コレクション → OOP → 例外 → 仕上げ」です。

0まず判定手順

コード問題は、出力を見る前にコンパイル可否を判定する。

1実行単位

main、ファイル名、package/import、起動引数を固定する。

2型と値

型変換、初期化、var、スコープでコンパイルエラーを拾う。

3評価順と制御

演算子、switch、ループ、break/continueで実行順を追う。

4参照とデータ構造

String、配列、List、同一性/同値性をまとめて理解する。

5クラスの土台

フィールド、メソッド、static、this/super、コンストラクタ。

6OOPの関係

継承、interface、overload/override、ポリモーフィズム。

7例外

コンパイル時に必要な処理と実行時に起きる例外を分ける。

8仕上げ

頻出メソッド、エラー、ひっかけ、ミニ演習で確認する。

学習記事

関係順に並べています。苦手な箇所だけ読んでもよいですが、OOPは「クラス → メソッド → 修飾子 → 継承/interface → ポリモーフィズム」の順が崩れると理解が曖昧になります。

重複整理の方針

  • 型変換・数値昇格は「データ型」と「数値ルール」に集約。
  • switchの基本は「演算子・制御構造」、yieldや境界は「細かい挙動」に分離。
  • == / equals / intern は「同一性・同値性比較」に集約。
  • 継承、interface、overload/overrideの関係は「OOP関係整理」に集約。
  • 例外の基本は「例外処理」、失敗段階の分類は「コンパイル/例外判定」と「エラー・例外カタログ」に集約。
