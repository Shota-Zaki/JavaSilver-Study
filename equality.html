同一性比較・同値比較

==、equals、recordのequals、nullとの比較を整理します。

同一性と同値性

Javaの比較は、まず「何を比較しているか」を決めます。プリミティブ値なら値比較、参照型に対する == なら参照比較、equals ならクラスごとの同値比較です。

Integer a = 100;
Integer b = 100;
System.out.println(a.equals(b)); // true

String s1 = new String("A");
String s2 = "A";
System.out.println(s1 == s2);      // false
System.out.println(s1.equals(s2)); // true

recordのequals

recordはコンポーネント値に基づく equals、hashCode、toString を自動生成します。ただし == は通常通り参照比較です。

record Point(int x, int y) {}

Point p1 = new Point(1, 2);
Point p2 = new Point(1, 2);

System.out.println(p1 == p2);      // false
System.out.println(p1.equals(p2)); // true

equalsを呼ぶ側

equals はメソッド呼び出しです。呼び出し元が null なら NullPointerException です。比較対象の引数が null なら、多くの実装ではfalseを返します。

String s = null;
// s.equals("A");       // NullPointerException
"A".equals(s);          // false
nullの可能性がある変数に対して 変数.equals(...) と呼ぶコードは危険です。試験ではNPEの発生箇所として狙われます。
