同一性比較・同値比較

==、equals、null、recordのequalsを、参照型とプリミティブ型に分けて整理します。

==の意味は型で変わる

プリミティブ型では値の比較です。参照型では参照先が同じオブジェクトかどうかを比較します。

対象== の意味
プリミティブ型値が同じか
参照型同じインスタンスを参照しているか
int a = 10;
int b = 10;
System.out.println(a == b); // true

String x = new String("A");
String y = new String("A");
System.out.println(x == y);      // false
System.out.println(x.equals(y)); // true

equalsはメソッド

equals はメソッドです。左辺が null なら呼び出し時に NullPointerException です。

String s = null;
// s.equals("A");      // NullPointerException
"A".equals(s);          // false。null安全に書ける

Objectのequalsとオーバーライド

Object の equals は、基本的に同一性比較です。Stringやrecordなどは内容比較になるようにオーバーライドされています。

class Item {
    int id;
    Item(int id) { this.id = id; }
}

Item a = new Item(1);
Item b = new Item(1);
System.out.println(a.equals(b)); // false: equalsをoverrideしていない

recordのequals

recordは、全コンポーネントを使った equals、hashCode、toString が自動生成されます。

record Point(int x, int y) {}

Point p1 = new Point(1, 2);
Point p2 = new Point(1, 2);
System.out.println(p1 == p2);      // false
System.out.println(p1.equals(p2)); // true

ラッパークラスの注意

ラッパークラスではオートボクシングにより見た目が単純になりますが、== は原則として参照比較です。値比較なら equals またはアンボクシング後の比較として読む必要があります。

Integer a = 1000;
Integer b = 1000;
System.out.println(a == b);      // 実装依存のキャッシュに頼らない。試験ではequalsで考える
System.out.println(a.equals(b)); // true

ミニ演習

Q. String a="x"; String b="x"; a==b は？

trueです。同じ文字列リテラルは文字列プール上の同じインスタンスを参照します。

Q. new String("x") == "x" は？

falseです。new は別インスタンスを作ります。

比較前に型を見る

== は、比較対象の型によって意味が変わります。プリミティブ同士なら値比較、参照型同士なら参照比較です。プリミティブとラッパーの比較ではアンボクシングが絡むことがあります。

Integer a = 10;
int b = 10;
System.out.println(a == b); // true: aがアンボクシングされる

Integer c = null;
// System.out.println(c == b); // NullPointerException

equalsを安全に呼ぶ形

左辺がnullの可能性があるなら、定数やnullでないことが明らかな値を左に置くと安全です。

String input = null;
System.out.println("yes".equals(input)); // false
// System.out.println(input.equals("yes")); // NullPointerException
