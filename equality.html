同一性・同値性比較

== は同じ参照か、equals は内容として等しいかを問う。

Stringの基本

操作戻り値/副作用注意
concat / replace / substring / trim新しいStringを返す元のStringは変わらない。戻り値を代入しないと結果を失う。
charAt(index)charを返す範囲外でStringIndexOutOfBoundsException。
indexOf見つかれば位置、なければ-10始まり。
equals内容比較null.equals(...) はNullPointerException。"abc".equals(s) は安全。

StringBuilder

操作戻り値/副作用注意
append / insert / delete / reverse同じインスタンスを変更して自分を返すStringと違い可変。
toStringStringを返すStringBuilder同士のequalsは内容比較ではない。
capacity容量lengthとは別。試験ではlengthとの混同に注意。

宣言==equals

intern()

String a = new String("java");
String b = a.intern();
String c = "java";
System.out.println(a == c); // false
System.out.println(b == c); // true

nullと比較

式結果理由
s == nullOK参照先がないかを比較。
s.equals("x")sがnullなら例外nullに対するメソッド呼び出し。
"x".equals(s)OK左辺が非nullリテラルなので安全。

recordと配列/List

対象equalsの意味注意
record全コンポーネントの値で比較recordはequals/hashCode/toStringを自動生成。
配列Objectのequalsなので参照比較内容比較はArrays.equals。
ArrayList要素の順番と内容を比較List.ofもListとしてequals可能。
