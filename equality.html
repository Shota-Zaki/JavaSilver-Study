同一性比較・同値比較

宣言の仕方で比較結果が変わる論点を、例題で丁寧に確認します。

同一性と同値性の違い

比較見るもの使うもの例
同一性同じインスタンスを指しているか==a == b
同値性値・内容が同じとみなせるかequals()a.equals(b)
プリミティブ値そのもの==10 == 10

参照型の == は原則として「同じ箱を見ているか」です。Stringの内容が同じかどうかは equals で判定します。

Stringリテラル同士

String a = "java";
String b = "java";
System.out.println(a == b);      // true
System.out.println(a.equals(b)); // true

同じ文字列リテラルは文字列プールの同じインスタンスを参照します。

new Stringを使う場合

String a = "java";
String b = new String("java");
System.out.println(a == b);      // false
System.out.println(a.equals(b)); // true

new は新しいインスタンスを作ります。内容が同じでも参照は別です。

参照を代入する場合

String a = new String("java");
String b = a;
System.out.println(a == b);      // true
System.out.println(a.equals(b)); // true

b = a はオブジェクトのコピーではなく参照のコピーです。

連結：リテラルだけか、変数を含むか

String a = "ja" + "va";
String b = "java";
System.out.println(a == b); // true

String part = "ja";
String c = part + "va";
System.out.println(c == b); // false

リテラル同士はコンパイル時に結合されます。通常変数を含むと実行時に新しいStringが作られます。

final変数の罠

final String part1 = "ja";
String a = part1 + "va";
System.out.println(a == "java"); // true

final String part2 = new String("ja");
String b = part2 + "va";
System.out.println(b == "java"); // false

final でも、コンパイル時定数でなければリテラル結合扱いにはなりません。

intern

String a = new String("java");
String b = a.intern();
String c = "java";
System.out.println(a == c); // false
System.out.println(b == c); // true

intern() は文字列プール上の参照を返します。newで作ったa自体が同じ参照になるわけではありません。

StringBuilder・配列・List

StringBuilder sb1 = new StringBuilder("java");
StringBuilder sb2 = new StringBuilder("java");
System.out.println(sb1.equals(sb2)); // false

int[] a = {1, 2};
int[] b = {1, 2};
System.out.println(a.equals(b)); // false

ArrayList<String> x = new ArrayList<>();
x.add("java");
ArrayList<String> y = new ArrayList<>();
y.add("java");
System.out.println(x.equals(y)); // true

StringBuilderと配列は内容比較になりません。ArrayListは要素の内容で比較します。

自作クラスとrecord

class Item {
  int id;
  Item(int id) { this.id = id; }
}
System.out.println(new Item(1).equals(new Item(1))); // false

record RecItem(int id) {}
System.out.println(new RecItem(1).equals(new RecItem(1))); // true

通常クラスはequalsを自分で実装しない限り同一性比較です。recordは成分に基づくequalsが自動生成されます。

nullとequals

String s = null;
System.out.println(s == null);      // true
System.out.println(s.equals("x"));  // NullPointerException
System.out.println("x".equals(s));  // false

null に対してメソッドは呼べません。定数側からequalsを呼ぶと安全です。

判定手順

  1. 比較対象がプリミティブか参照型かを見る。
  2. == なら参照比較か値比較かを決める。
  3. equals なら、そのクラスが内容比較を実装しているかを見る。
  4. Stringならリテラル、new、連結、final、internを分ける。
  5. nullが絡む場合、メソッド呼び出し側がnullでないか確認する。
