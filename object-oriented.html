クラス・static・record

インスタンス、参照、static、final、record、GC、フィールド解決をまとめます。

クラスとインスタンス

クラスは設計図で、インスタンスはnewで作られる実体です。参照変数にはインスタンスそのものではなく、インスタンスへの参照が入ります。代入すると参照がコピーされるだけで、オブジェクトのコピーは作られません。

class Item {
    int price;
}

Item a = new Item();
Item b = a;
b.price = 100;
System.out.println(a.price); // 100

static

staticメンバはインスタンスではなくクラスに属します。staticフィールドは全インスタンスで共有されます。インスタンス経由でもアクセスできてしまいますが、実体は1つです。

class Counter {
    static int count;
}

Counter a = new Counter();
Counter b = new Counter();
a.count = 10;
b.count++;
System.out.println(Counter.count); // 11

staticメソッドはインスタンスなしで呼べます。その代わり、直接インスタンスフィールドやインスタンスメソッドにはアクセスできません。

record

Javaのrecordは、データを入れるだけのクラスを短く書くための仕組みです。Java 16以降で正式導入され、Java Silver 17では重要です。

普通のクラスで名前と年齢を持つデータクラスを書くと、フィールド、コンストラクタ、アクセサ、toString、equals、hashCodeなどを自分で書く必要があります。

public class Person {
    private final String name;
    private final int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String name() {
        return name;
    }

    public int age() {
        return age;
    }
}

recordなら、これだけで同じようなデータ保持クラスを作れます。

public record Person(String name, int age) {
}

自動生成されるフィールド

private final String name; と private final int age; に相当する状態を持ちます。

正規コンストラクタ

Person(String name, int age) が作られます。

アクセサ

name() と age()。getName() ではありません。

toString / equals / hashCode

中身に基づく実装が自動生成されます。
record Person(String name, int age) {}

public class Main {
    public static void main(String[] args) {
        Person p = new Person("Tanaka", 20);
        System.out.println(p.name());
        System.out.println(p.age());
        System.out.println(p);
    }
}

// Tanaka
// 20
// Person[name=Tanaka, age=20]
recordのアクセサは name() です。getName() は自動生成されません。セッターも自動生成されません。

recordにもメソッドは書けます。データ保持用の形を簡潔にしつつ、関連する処理を追加できます。

record Person(String name, int age) {
    public boolean isAdult() {
        return age >= 20;
    }
}

コンストラクタも書けます。recordでは、すべてのコンポーネントを初期化する正規コンストラクタと、代入をJavaに任せるコンパクトコンストラクタが重要です。

record Person(String name, int age) {
    public Person {
        if (age < 0) {
            throw new IllegalArgumentException();
        }
    }
}

recordは暗黙的にfinalです。recordを継承するクラスは作れません。また、recordは内部的にjava.lang.Recordを継承するため、他のクラスをextendsすることもできません。ただし、interfaceの実装はできます。

interface Printable {
    void print();
}

record Person(String name, int age) implements Printable {
    public void print() {
        System.out.println(name + ":" + age);
    }
}

追加インスタンスフィールド

不可。recordの状態はレコードヘッダーに書いたものだけです。

staticフィールド

可。インスタンスごとの状態ではなく、クラス共通だからです。

extends

不可。recordは他クラスを継承できず、record自身も継承されません。

implements

可。実装すべきメソッドは通常通り実装します。
record Item(String name, int price) {
    static int count = 0;     // OK
    // private int stock;     // NG。追加インスタンスフィールド不可
}

record同士の比較では、<code>==</code> は参照比較、<code>equals()</code> は中身比較です。recordはequalsを自動生成するので、同じ値ならtrueになります。

record Person(String name, int age) {}

Person p1 = new Person("A", 10);
Person p2 = new Person("A", 10);

System.out.println(p1 == p2);      // false
System.out.println(p1.equals(p2)); // true

ガベージコレクション

Javaでは、どこからも参照されなくなったインスタンスがガベージコレクションの対象になります。対象になることと、すぐ回収されることは別です。

Object a = new Object();
Object b = new Object();
Object c = a;
a = null;
b = null;

// bが参照していたインスタンスは対象
// aが参照していたインスタンスはcがまだ参照しているので対象外
