クラス・static・record

インスタンス、参照、static、final、record、ガベージコレクションを、出力問題で迷わない形に整理します。

クラスとインスタンス

クラスは設計図、インスタンスは実体です。変数に入っているのはインスタンスそのものではなく、インスタンスへの参照です。

Item a = new Item();
        Item b = new Item();
        a == b; // false。別インスタンス

同じクラスから作ったインスタンスでも、フィールドの値は別々に保持されます。

参照の代入

参照型変数の代入は、インスタンスをコピーするのではなく、参照をコピーします。

Item a = new Item();
        Item b = new Item();
        a.name = "apple";
        b.name = "banana";
        a = b;
        System.out.println(a.name); // banana

a = b 後は、a と b が同じインスタンスを参照します。元の a が参照していたインスタンスは、他から参照されていなければGC対象です。

staticメンバ

static はインスタンスではなくクラスに属します。staticメソッドからインスタンスフィールドへ直接アクセスすることはできません。

class Value {
            private String contents;

            public static void print() {
                // System.out.println(contents); // NG
            }
        }

インスタンスメンバに触るには、インスタンス参照が必要です。

Value v = new Value();
        // v.contents など。ただしprivateなら同一クラス外から不可

finalの意味

final は文脈によって意味が変わります。

対象意味
変数再代入不可。参照先オブジェクトの変更不可ではない。
メソッドオーバーライド不可。
クラス継承不可。
final StringBuilder sb = new StringBuilder("A");
        sb.append("B"); // OK。参照先の中身変更
        // sb = new StringBuilder("C"); // NG。再代入

recordの基本

record は、データを運ぶためのクラスを簡潔に書く仕組みです。コンポーネントに対応する private final フィールド、アクセサ、equals、hashCode、toString が生成されます。

record Point(int x, int y) {}

        Point p = new Point(10, 20);
        System.out.println(p.x());      // 10
        System.out.println(p);          // Point[x=10, y=20]

recordは暗黙に final です。他のクラスを継承できません。ただしインタフェースは実装できます。

ガベージコレクション

GC対象になるのは、どこからも到達できなくなったインスタンスです。null 代入や参照の上書きによって到達不能になります。

Object a = new Object(); // インスタンス1
        Object b = new Object(); // インスタンス2
        Object c = a;
        a = null;
        b = null;
        // この時点でGC対象はインスタンス2だけ。インスタンス1はcから参照されている。

System.gc() はGC実行を保証しません。試験で問われるのは「対象になるか」であって「いつ回収されるか」ではありません。

ミニ判定

Q1. staticメソッドからインスタンスフィールドを直接読める？

読めません。インスタンス参照が必要です。

Q2. final ArrayList は要素追加不可？

いいえ。変数への再代入が不可なだけで、参照先リストの変更は可能です。

Q3. recordは継承できる？

record自身は暗黙にfinalなので、他クラスから継承できません。
