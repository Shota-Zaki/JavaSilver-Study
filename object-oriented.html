クラス・インスタンス・static・参照

クラスからインスタンスを作り、参照型変数がその参照を保持します。

参照型変数はリンクを持つ

Item a = new Item();
Item b = new Item();
a = b; // aとbは同じインスタンスを参照する

代入でコピーされるのはインスタンス本体ではなく参照です。ガベージコレクション対象は「どこからも参照されないインスタンス」です。

staticとinstance

種類所属アクセス
インスタンスフィールド各インスタンス参照変数から使う。
staticフィールドクラス全インスタンスで共有。クラス名アクセスが基本。
インスタンスメソッド各インスタンスstaticメソッドから直接呼べない。
staticメソッドクラスthisは使えない。

フィールドとメソッドの解決差

class A { String name = "A"; String getName(){ return name; } }
class B extends A { String name = "B"; String getName(){ return name; } }
A obj = new B();
System.out.println(obj.name);    // A: フィールドは宣言型
System.out.println(obj.getName());// B: メソッドは実体型でoverride

record

record Item(String name, int price) {}
Item i = new Item("apple", 100);
System.out.println(i.name());
System.out.println(i); // Item[name=apple, price=100]

recordはコンストラクタ、アクセサ、equals、hashCode、toStringが自動生成されます。
