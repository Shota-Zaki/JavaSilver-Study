継承で引き継がれるもの/引き継がれないもの

対象扱い注意
フィールドprivate以外はアクセス条件次第で使える同名フィールドは隠蔽。メソッドとは解決が違う。
メソッドoverride可能なものは再定義できるprivate/static/finalは通常のoverride対象ではない。
コンストラクタ引き継がれないサブクラスのコンストラクタからsuper(...)で呼ぶ。
privateメンバサブクラスから直接アクセス不可インスタンス内部には存在するが直接見えない。

interface

interface Worker {
    void work(); // public abstract扱い
}
class Engineer implements Worker {
    public void work() { System.out.println("work"); }
}
  • interfaceの抽象メソッドは暗黙にpublic abstract。
  • 実装クラスのメソッドはpublicでなければならない。
  • interfaceのフィールドは暗黙にpublic static final。
  • クラスは複数のinterfaceをimplementsできる。

抽象クラスとinterface

interface A { void test(); }
abstract class B implements A { } // test未実装でもOK
class C extends B {
    public void test() { }
}

抽象クラスは、interfaceの抽象メソッドをすべて実装しなくてもよいです。ただし、最終的な具象クラスでは実装が必要です。

default/privateメソッド

interfaceのdefaultメソッドは実装を持ち、実装クラスがそのまま利用できます。Java 9以降はinterface内にprivateメソッドも書けますが、interface内部の共通処理用です。
