継承・インタフェース・sealed

継承問題は「変数の型」「実体の型」「アクセス可能なメンバ」「override対象」を分けます。

継承で引き継がないもの

対象理由/注意
コンストラクタサブクラス用ではないため継承されない。
privateメンバクラス外からアクセス不可。存在はしても直接使えない。
パッケージプライベート別パッケージのサブクラスからは不可。

オーバーライド条件

条件説明
シグネチャ同一メソッド名・引数型・順序。
戻り値同じかサブクラス型。
アクセス修飾子同じか緩くする。狭くできない。
例外チェック例外は広げられない。
staticstaticメソッドはoverrideではなく隠蔽。

ポリモーフィズム

Parent p = new Child();
p.method(); // Child側のoverrideメソッド
// p.childOnly(); // 変数型Parentにないので不可
見る対象決まり方
呼び出せるメンバ変数の型で決まる。
実行されるoverrideメソッド実体の型で決まる。
フィールドアクセス変数の型で決まる。

sealed

public sealed class A permits B, C {}
public final class B extends A {}
public non-sealed class C extends A {}
  • sealed親はpermitsで許可先を指定。
  • 直接の子はfinal/sealed/non-sealedのいずれかを明示。
  • 許可されていないクラスは継承不可。
