継承とインタフェース詳細

継承、interface、defaultメソッド、抽象クラス、sealedを、問題で迷いやすい形に寄せて詳しく整理します。

継承で「使える」と「直接アクセスできる」は違う

サブクラスのインスタンスは親クラス部分を持ちますが、privateメンバへサブクラスから直接アクセスできるわけではありません。public/protected/defaultなどのアクセス制御を通じて使えるかを判断します。

class A {
    private String value;
    public String getValue() { return value; }
}
class B extends A {
    void test() {
        // value = "x";       // NG
        System.out.println(getValue()); // OK
    }
}

フィールドは多態的に解決されない

メソッドは実体のクラスで動的に選ばれますが、フィールドアクセスは変数の宣言型で決まります。ここは継承問題の重要な罠です。

class A {
    String name = "A";
    String getName() { return name; }
}
class B extends A {
    String name = "B";
    String getName() { return name; }
}
A obj = new B();
System.out.println(obj.name);      // A
System.out.println(obj.getName()); // B

defaultメソッドの衝突

複数interfaceから同じシグネチャのdefaultメソッドを受け継ぐと、実装クラス側で解決が必要です。

interface A { default void test() { System.out.println("A"); } }
interface B { default void test() { System.out.println("B"); } }
class C implements A, B {
    public void test() {
        A.super.test();
    }
}

クラスのメソッドが優先される

スーパークラスに具象メソッドがあり、interfaceに同じシグネチャのdefaultメソッドがある場合、クラス側のメソッドが優先されます。

class A { public void test() { System.out.println("A"); } }
interface B { default void test() { System.out.println("B"); } }
class C extends A implements B { }
new C().test(); // A

sealed階層の読み方

sealedクラス/インタフェースは、許可されたクラスだけが継承・実装できます。許可された側は、継承を止めるなら final、さらに制限するなら sealed、開放するなら non-sealed を選びます。

sealed interface Shape permits Circle, Rectangle { }
final class Circle implements Shape { }
non-sealed class Rectangle implements Shape { }

ミニ演習

Q. A a = new B(); a.field はAとBどちらのfield？

宣言型Aのfieldです。フィールドは多態的に解決されません。

Q. defaultメソッド同士が衝突したら自動でどちらか選ばれる？

選ばれません。実装クラスで明示的にoverrideして解決する必要があります。

superの使いどころ

super は、親クラスのコンストラクタ、親クラスのメソッド、親クラスのフィールドへアクセスするために使います。ただしprivateメンバにはアクセスできません。

class A {
    String name = "A";
    void print() { System.out.println(name); }
}
class B extends A {
    String name = "B";
    void print() {
        System.out.println(name);       // B
        System.out.println(super.name); // A
        super.print();                  // A側のprint。ただし中で使うnameはAのフィールド
    }
}

抽象メソッドの実装責任

具象クラスは、継承してきた抽象メソッドをすべて実装する必要があります。抽象クラスなら未実装のまま残せます。

abstract class A { abstract void test(); }
abstract class B extends A { } // OK
class C extends B {
    void test() { } // ここで実装が必要
}
