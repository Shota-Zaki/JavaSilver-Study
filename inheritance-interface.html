継承・インタフェース

継承、インタフェース、抽象クラス、オーバーライド、sealedを、問題判定に使える形で整理します。

extends と implements

extends は既存の型を拡張し、implements はinterfaceの契約を満たします。クラスは1つのクラスしか継承できませんが、複数のinterfaceを実装できます。

interface Worker {
    void work();
}

class Employee { }

class Engineer extends Employee implements Worker {
    public void work() {
        System.out.println("work");
    }
}

class extends class

クラスの継承。単一継承です。

class implements interface

interfaceの実装。複数指定できます。

interface extends interface

interface同士の継承。複数継承できます。

class extends interface

不可。クラスはinterfaceを implements します。

サブクラスが引き継ぐもの・引き継がないもの

継承しても、スーパークラスのすべてがそのまま使えるわけではありません。特にコンストラクタとprivateメンバは試験で狙われます。

フィールド・メソッド

アクセス可能なメンバは継承関係で利用できます。

privateメンバ

サブクラスから直接アクセスできません。継承しているというより、外から見えないと考える方が安全です。

コンストラクタ

継承されません。サブクラスのコンストラクタから super(...) で呼び出します。

staticメンバ

インスタンスの多態的なオーバーライド対象ではありません。隠蔽として扱います。

abstract

抽象メソッドを持つクラスはabstractでなければなりません。abstractクラスは直接 new できません。具象サブクラスにするなら、継承した抽象メソッドをすべて実装します。

abstract class Animal {
    abstract void sound();
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("bow");
    }
}

interfaceの抽象メソッドは暗黙的に public abstract です。実装クラスでは public を付ける必要があります。

interface A {
    void test(); // public abstract
}

class B implements A {
    public void test() { } // publicが必要
}

default / static / private interface method

interfaceにも、抽象メソッド以外のメソッドを定義できます。Java Silver 17ではdefaultメソッドやprivateメソッドの扱いを問われます。

abstract method

暗黙的に public abstract。本体なし。

default method

本体あり。実装クラスに継承され、必要ならオーバーライドできます。

static method

interface名で呼びます。実装クラスのインスタンスから継承される通常メソッドではありません。

private method

interface内の共通処理用。外部や実装クラスから呼び出せません。

interface Loggable {
    default void log(String msg) {
        format(msg);
    }

    private void format(String msg) {
        System.out.println("[LOG] " + msg);
    }
}

override

オーバーライドは、親のインスタンスメソッドを子で再定義することです。判定条件は、シグネチャ、戻り値、アクセス修飾子、例外です。

シグネチャ

メソッド名と引数リストが同じ。違えばオーバーロードです。

戻り値

同じ型、またはサブクラス型の共変戻り値が使えます。

アクセス

親より狭くできません。public をpackage-privateにするのは不可。

static

staticメソッドはオーバーライドではなく隠蔽です。

class A {
    Number value() { return 1; }
}

class B extends A {
    @Override
    Integer value() { return 1; } // OK。IntegerはNumberのサブクラス
}

sealed

sealed は継承・実装できる直接のサブタイプを限定します。親側の permits と、子側の final / sealed / non-sealed の両方を確認します。

public sealed class A permits B, C { }
public final class B extends A { }
public non-sealed class C extends A { }

// class D extends A { } // NG。permitsされていない
sealedは「抽象」という意味ではありません。abstract と混同すると、new可否の問題で外します。
