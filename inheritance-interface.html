継承・インタフェース・sealed

多態性、オーバーライド、interface、sealedを判定できるようにします。

継承の基本

class Parent {
  void hello() { System.out.println("Parent"); }
}
class Child extends Parent {
  @Override
  void hello() { System.out.println("Child"); }
}
Parent p = new Child();
p.hello(); // Child

変数の型はParentでも、オーバーライドメソッドは実体Childのものが動きます。

フィールドは多態的ではない

class A { String name = "A"; }
class B extends A { String name = "B"; }
A obj = new B();
System.out.println(obj.name); // A

フィールドアクセスは宣言型で決まります。メソッド呼び出しと区別します。

オーバーライド条件

条件説明
シグネチャメソッド名と引数リストが同じ。
戻り値同じ型または共変戻り値。
アクセス親より狭くできない。
例外チェック例外は親より広げられない。

インタフェース

interface A {
  void test(); // public abstract
  default void log() { System.out.println("log"); }
}
class B implements A {
  public void test() {}
}

インタフェースの抽象メソッドはpublic扱いです。実装側でpublicを省くとアクセスを弱めることになりエラーです。

sealed

sealed class A permits B, C {}
final class B extends A {}
non-sealed class C extends A {}
class D extends C {}

sealedの直接の子は許可リストに必要で、さらにfinal/sealed/non-sealedのいずれかを宣言します。
