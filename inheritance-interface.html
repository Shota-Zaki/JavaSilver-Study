継承・インタフェース

継承、interface、abstract、default/static/privateメソッド、sealed interfaceをまとめます。

extends / implements / abstract / interface

interfaceは、クラスに対して「このメソッドを持っているべき」という契約を定義する型です。処理の中身を持たない抽象メソッドを中心に、実装するクラスへルールを要求します。

interface Worker {
    void work();
}

class Engineer implements Worker {
    public void work() {
        System.out.println("設計する");
    }
}

interfaceの抽象メソッドは暗黙的にpublic abstractです。そのため、実装クラスではpublicを付ける必要があります。アクセス修飾子を弱めるとコンパイルエラーです。

interface A {
    void test(); // public abstract void test(); と同じ
}

class B implements A {
    public void test() { } // OK
}

// class C implements A {
//     void test() { } // NG。publicより弱い
// }

未実装メソッドが残るクラスはabstractにする必要があります。abstractクラスは未完成なので、直接newできません。

interface A {
    void test();
}

abstract class B implements A {
    // test() を未実装のままにできる
}

class C extends B {
    public void test() { }
}

class → class

extends。Javaではクラスの多重継承は不可。

class → interface

implements。複数実装可能。

interface → interface

extends。複数継承可能。

abstract

未完成を許す。抽象メソッドを持つクラスはabstract必須。

継承で引き継がれるもの

子クラスは親クラスのメンバを利用できます。ただし、何でも直接使えるわけではありません。privateメンバは継承関係にあっても直接アクセスできません。コンストラクタも継承されません。

引き継がれるもの

アクセス可能なフィールド、メソッド。protectedやpublicは子クラスから扱いやすい。

直接アクセスできないもの

privateフィールド、privateメソッド。親の内部実装として隠れます。

継承されないもの

コンストラクタ。子クラスの生成時には親コンストラクタが呼ばれるが、子のコンストラクタとして使えるわけではありません。

staticメンバ

継承されたように見えるが、インスタンスの多態性とは別。staticメソッドはオーバーライドではなく隠蔽です。
class Parent {
    private int secret;
    protected int value;

    Parent() { }

    void show() { }
}

class Child extends Parent {
    void test() {
        value = 10;   // OK
        show();       // OK
        // secret = 1; // NG。privateは直接アクセス不可
    }
}

オーバーロードとオーバーライド

オーバーロードは同じクラス内または継承関係の中で、同じ名前・違う引数リストのメソッドを複数持つことです。オーバーライドは、親クラスのメソッドを子クラスで再定義することです。目的も判定タイミングも違います。

オーバーロード

引数リストが違う。呼び出し先はコンパイル時に決まる。戻り値だけ違うのは不可。

オーバーライド

シグネチャが同じ。実行時の実体クラスに応じて呼ばれる。

戻り値

オーバーライドでは同じ型または共変戻り値。

アクセス修飾子

オーバーライドでは親より狭くできない。
class A {
    void test(int n) { System.out.println("A int"); }
    void test(String s) { System.out.println("A String"); } // overload
}

class B extends A {
    @Override
    void test(int n) { System.out.println("B int"); } // override
}

A obj = new B();
obj.test(10);   // B int
obj.test("x"); // A String

ポリモーフィズム

ポリモーフィズムでは、親型の変数で子クラスのインスタンスを扱えます。メソッド呼び出しは実体クラス側のオーバーライドが動きます。ただし、呼び出せるメソッドは宣言型に存在するものだけです。

class A {
    void hello() { System.out.println("A"); }
}
class B extends A {
    void hello() { System.out.println("B"); }
    void bye() { System.out.println("bye"); }
}

A a = new B();
a.hello(); // B
// a.bye(); // コンパイルエラー。宣言型Aにbyeがない

フィールドはメソッドと違い、宣言型で解決されます。親と子に同名フィールドがある場合、変数の型でどちらを見るかが決まります。

sealed class / sealed interface

sealedは、継承や実装できる直接のサブクラスを制限する仕組みです。sealedは「インスタンス化できない」という意味ではありません。インスタンス化できないのはabstractです。

public sealed class Animal permits Dog, Cat {
}

public final class Dog extends Animal {
}

public final class Cat extends Animal {
}

sealedクラスを継承したクラスは、必ず <code>final</code>、<code>sealed</code>、<code>non-sealed</code> のどれかを付けます。

final

これ以上継承させません。

sealed

さらに継承先を制限します。子側にもpermitsが必要です。

non-sealed

そのクラスから先は自由に継承できます。

permits

直接の許可クラスを書きます。許可されていないクラスは継承不可。
sealed class Animal permits Dog, Bird {}

final class Dog extends Animal {}

non-sealed class Bird extends Animal {}
class Sparrow extends Bird {} // OK。Birdがnon-sealedだから

interfaceもsealedにできます。実装できるクラスを限定できます。

sealed interface Worker permits Engineer, Designer {
    void work();
}

final class Engineer implements Worker {
    public void work() { }
}

final class Designer implements Worker {
    public void work() { }
}
sealedの子クラスに final / sealed / non-sealed がない場合はコンパイルエラーです。
