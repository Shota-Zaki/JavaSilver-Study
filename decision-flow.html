例外処理・失敗段階

コンパイルエラー、実行時例外、try/catch/finally、try-with-resourcesをまとめて確認。

失敗段階の分類

段階代表例見るポイント
コンパイルエラー型不一致、未初期化ローカル変数、未処理チェック例外実行されない。出力も例外も発生しない。
実行時例外NullPointerException、ArrayIndexOutOfBoundsException、ClassCastExceptionコンパイルは通るが実行中に止まる。
正常出力出力文まで到達途中で例外やreturn/breakがないか確認。

例外の種類

種類例処理必須か
チェック例外Exception, IOExceptioncatchまたはthrowsが必要。
非チェック例外RuntimeException系コンパイル時処理は不要。
ErrorOutOfMemoryErrorなど通常アプリで捕まえる前提ではない。

try / catch / finally

構文実行順注意
tryのみ不可catchまたはfinallyが必要。
try-catch例外発生で対応catchへtry内の例外発生後の文は実行されない。
try-finallyfinallyは原則実行catchがないので例外は処理されず外へ。
try-catch-finallytry → catch → finally例外なしならtry → finally。
catch順序子 → 親親を先に書くと子catchは到達不能。

try-with-resources

項目ルール注意
対象AutoCloseable実装close()が自動呼び出し。
close順宣言と逆順A; B なら B.close() → A.close()。
finallyclose後に実行tryブロック終了 → close → finally。
例外closeがthrowsする可能性mainでthrowsするかcatchが必要。

finallyとreturn

static int test() {
    try {
        return 1;
    } finally {
        System.out.println("finally");
    }
}
// returnで抜ける前にfinallyは実行される。
