判定フロー

コード問題を「コンパイル可否 → 実行時例外 → 出力」の順に機械的に処理するための手順です。

最初に決めること

Java Silverのコード問題は、感覚で読むと落ちます。必ず次の順番で判定します。途中でコンパイルエラーが確定したら、実行結果は追いません。

1構文

括弧、セミコロン、予約語、宣言位置、到達不能コード。

2型

代入、戻り値、引数、演算結果、キャスト、ジェネリクス。

3アクセス

private、default、protected、public、staticから非static参照。

4実行時

null、配列範囲外、キャスト失敗、0除算、文字列→数値変換。

5出力

評価順、参照先、オーバーライド、catch/finally、文字列連結。

例題：どこで止まるか

public class Main {
    public static void main(String[] args) {
        String s = null;
        System.out.println(s.length());
    }
}

判定：構文は正しい。型も合っている。コンパイルは通る。実行時に s.length() で NullPointerException。

public class Main {
    public static void main(String[] args) {
        int a;
        System.out.println(a);
    }
}

判定：ローカル変数 a が初期化されていないためコンパイルエラー。実行時例外ではない。

試験での基本姿勢

  • 「実行したら例外になりそう」でも、先にコンパイル可否を見る。
  • 選択肢に「コンパイルエラー」がある問題は、型・構文・初期化・チェック例外を最優先で疑う。
  • 選択肢に「実行時例外」がある問題は、null、配列、キャスト、parse、0除算を疑う。
  • 出力問題は最後。コンパイルも実行もできることが確定してから追う。
