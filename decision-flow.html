判定フロー

コード問題は、必ず「コンパイル可否 → 実行時例外 → 出力結果」の順に判定します。

基本フロー

1構文・宣言

クラス名、ファイル名、main、修飾子、戻り値、引数、セミコロンを確認。

2型・スコープ

代入可能性、初期化、var、ローカル変数、アクセス制御を確認。

3呼び出し解決

static/instance、overload、override、宣言型で見えるメンバを確認。

4実行時例外

null、配列範囲、キャスト、parse、0除算、switch(null)を確認。

5出力

副作用、参照先、文字列連結、finally、close順を最後まで追う。

コンパイルエラーを先に拾う理由

実行結果を聞かれていても、コンパイルできなければ実行されません。出力候補がもっともらしくても、型・修飾子・到達不能コード・チェック例外未処理があれば、その時点でコンパイルエラーです。

見る場所典型例判定
宣言int num = num;右辺のローカル変数numはまだ初期化されていない。
型byte b = b + 1;b + 1 はint。byteへ代入不可。
呼び出しA a = new B(); a.bye();宣言型Aにbyeがなければ呼べない。
例外throw new IOException();チェック例外はthrowsまたはcatchが必要。
到達不能return; System.out.println();return後に通常到達できる文は不可。

実行時例外の見分け方

実行時例外は、コンパイルは通るが実行中の値や参照先で失敗するものです。参照型がnullか、添字が範囲外か、キャスト先の実体型が合うかを確認します。

String s = null;
System.out.println(s.length()); // NullPointerException

Object obj = "text";
Integer i = (Integer) obj;      // ClassCastException

int[] a = new int[0];
a[0] = 10;                      // ArrayIndexOutOfBoundsException
