正解：E

varを使った変数宣言に関する問題です。
varは、ローカル変数を宣言する際のデータ型を推論する機能です。型推論を
行うためには、次のようにデータ型の代わりにvarを使って変数を宣言しま
す。

例: varを使った変数宣言

```java
var a = 10;
```

varを使った型推論は、コンパイル時にコンパイラが代入演算子「=」の右辺
から変数のデータ型を推論し、そのデータ型に置き換える機能です。そのた
め、変数が初期化されていなかったり、nullで初期化したりするなど、デー
タ型を特定できない場合はコンパイルエラーとなります。よって、選択肢A
とBは誤りです。

ローカル変数の型推論は、リテラルだけでなく、次のようにメソッドの戻り
値を受け取るのにも使えます。これは、メソッドの定義から戻り値の型が判
別できるためです。

例: メソッドの戻り値からローカル変数の型を推論する

```java
var a = sample();
```

一方、ラムダ式だけではその式を扱う変数の型を推論できないため、ラムダ式
を受け取るための変数宣言に型推論は使えません。よって、選択肢Cも誤りです。

例: ラムダ式からは変数の型を推論できない（コンパイルエラー）

```java
var a = () -> {};
```

ほかにも、具体的な型を変数の型から推論している例として、配列の初期化
式があります。配列の初期化式は、配列型変数の宣言と同時に記述すること
で、変数の型で宣言されている配列インスタンスを生成し、要素を初期化し
ます。そのため、配列の初期化式を代入する変数をvarで宣言すると、何型の
配列インスタンスを生成すればよいのかを判断できず、コンパイルエラーと
なります。よって、選択肢Dも誤りです。

例: 配列の初期化式を使うと型推論できない

```java
int[] array = {1, 2, 3};  ← 変数の型からint配列型のインスタンスを生成している
var a = {1, 2, 3};        ← 型を特定できない
```

なお、ジェネリクスのダイヤモンド演算子「<>」による型推論では、変数宣
言時のジェネリクスで指定された型パラメータを参照します。もし、参照でき
る型情報がなければObject型が型パラメータとして与えられます。そのため、
選択肢Eは以下のコードと同じ意味を持つことになり、右辺を参照することで
変数の型推論を行うことができます。以上のことから、選択肢Eが正解です。

例: ダイヤモンド演算子で型が推論できない場合にはObject型が使われる

```java
var e = new ArrayList<Object>();
```
